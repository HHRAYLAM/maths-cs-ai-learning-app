## 🧀 故事引入：Tom 的奶酪快递网络

Tom 今天开了一家奶酪快递公司！

他需要在一个城市里建立配送网络。这个城市有：
- 🏠 房子（客户）
- 🛣️ 道路（连接房子）
- 🧀 奶酪配送站（起点）

这就是一个图（Graph）！

```
         🏠---🏠---🏠
         |    |    |
    🏠---🏠---🏠---🏠---🏠
    |    |    |    |
    🏠---🏠---🧀---🏠---🏠
         |    |    |
         🏠---🏠---🏠
```

- 每个🏠（房子）= 节点（Vertex/Node）
- 每条路线 = 边（Edge）
- 路线长度 = 权重（Weight）

---

## 📚 第一部分：图的基础

### 核心概念

| 术语 | Tom 的比喻 | 技术定义 |
|------|-----------|---------|
| 顶点 (Vertex) | 🏠 房子/配送站 | 图中的基本单元 |
| 边 (Edge) | 🛣️ 连接房子的路 | 顶点之间的连接 |
| 权重 (Weight) | 📏 路的长度/时间 | 边上的数值 |
| 度 (Degree) | 🔗 一个房子连几条路 | 与顶点相连的边数 |
| 路径 (Path) | 🚚 从 A 到 B 的路线 | 顶点的序列 |

### 图的类型

```
┌─────────────────────────────────────────────────────────┐
│                    图的分类                             │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1️⃣ 无向图 vs 有向图                                    │
│                                                         │
│     无向图：🏠 --- 🏠  (路可以双向走)                   │
│     有向图：🏠 --> 🏠  (单行道)                         │
│                                                         │

│     无权：🏠 ------- 🏠  (只表示连接)                   │
│                                                         │
│  3️⃣ 连通图 vs 非连通图                                  │
│                                                         │
│     连通：所有房子都能互相到达                          │
│     非连通：有些房子是孤立的                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 🔑 第二部分：图的表示方法

### 1. 邻接矩阵（Adjacency Matrix）

Tom 的距离表：

Tom 做了一个表格，记录每个房子之间的距离：

```
        A    B    C    D
     ┌────────────────────┐
  A  │ 0    5    ∞    3   │
  B  │ 5    0    2    ∞   │
  C  │ ∞    2    0    4   │
  D  │ 3    ∞    4    0   │
     └────────────────────┘

0 = 自己到自己
数字 = 距离
∞ = 没有直接的路
```

优点：查询两点之间是否有边 O(1)
缺点：稀疏图浪费空间

### 2. 邻接表（Adjacency List）

Tom 的邻居清单：

```
A: [B(5km), D(3km)]
B: [A(5km), C(2km)]
C: [B(2km), D(4km)]
D: [A(3km), C(4km)]

括号里是距离
```

优点：节省空间，遍历高效
缺点：查询两点是否有边较慢

## 🚚 第三部分：图遍历算法

### 1. 广度优先搜索（BFS）

Tom 的"地毯式搜索"：

Tom 要从🧀出发，访问所有房子。BFS 策略：
- 先访问所有相邻的房子
- 再访问相邻的相邻

```
步骤演示：

初始：     🧀(起点)

第 1 步：   🧀 → 🏠₁ 🏠₂ 🏠₃ (所有相邻)

第 2 步：   🏠₁ 🏠₂ 🏠₃ → 🏠₄ 🏠₅ 🏠₆ (下一层)

特点：一层一层向外扩展，像水波纹！
```

代码实现：

```python
def bfs(graph, start):
    visited = set()
    queue = [start]
    result = []
    
    while queue:
        node = queue.pop(0)
        if node not in visited:
            visited.add(node)
            result.append(node)
            # 添加所有未访问的邻居
            queue.extend(neighbor for neighbor in graph[node] 
                        if neighbor not in visited)
    
    return result
```

应用场景：
- 寻找最短路径（无权图）
- 社交网络"六度分隔"
- 网络爬虫

### 2. 深度优先搜索（DFS）

Tom 的"一条路走到黑"：

DFS 策略：
- 选择一条路一直走
- 走不通了再回退（回溯）

```
步骤演示：

🧀 → 🏠₁ → 🏠₂ → 🏠₃ (走到头)
            ↓
         (回溯) 
            ↓
         🏠₄ → 🏠₅

特点：一条路走到底，撞墙再回头！
```

代码实现：

```python
def dfs(graph, start, visited=None):
    if visited is None:
        visited = set()
    
    visited.add(start)
    result = [start]
    
    for neighbor in graph[start]:
        if neighbor not in visited:
            result.extend(dfs(graph, neighbor, visited))
    
    return result
```

应用场景：
- 拓扑排序
- 检测环
- 迷宫求解
- 连通分量

## 🗺️ 第四部分：最短路径算法

### 1. Dijkstra 算法

Tom 找最短配送路线：

问题：从🧀到🏠ₓ，哪条路最短？

```
示例图：

        2         3
    A ----- B ----- C
    |       |       |
   1|       |4      |2
    |       |       |
    🧀 ----- D ----- E
            5
```

Dijkstra 步骤：

```
1. 从起点开始，初始化所有距离为∞
   🧀=0, A=∞, B=∞, C=∞, D=∞, E=∞

2. 更新相邻节点距离
   🧀→A=1, 🧀→D=5

3. 选择距离最小的未访问节点 (A=1)
   A→B=1+2=3

4. 继续选择最小节点 (B=3)
   B→C=3+3=6, B→D=min(5, 3+4)=5

5. 继续... 直到访问所有节点

最终：🧀到 C 的最短路径 = 6 (🧀→A→B→C)
```

代码实现：

```python
import heapq

def dijkstra(graph, start):
    # 初始化
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)]  # (距离，节点)
    
    while pq:
        current_dist, current_node = heapq.heappop(pq)
        
        # 如果当前距离大于已记录距离，跳过
        if current_dist > distances[current_node]:
            continue
        
        # 遍历邻居
        for neighbor, weight in graph[current_node].items():
            distance = current_dist + weight
            
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))
    
    return distances
```

### 2. BFS vs Dijkstra

| 特性 | BFS | Dijkstra |
|------|-----|----------|
| 适用图 | 无权图 | 加权图 |
| 数据结构 | 队列 | 优先队列 |
| 时间复杂度 | O(V+E) | O((V+E)logV) |
| Tom 比喻 | 数经过几个房子 | 计算总路程多少 km |

## 🌲 第五部分：最小生成树（MST）

### 问题背景

Tom 要铺设奶酪配送管道：

- 需要连接所有房子
- 管道成本与距离成正比
- 目标：用最短的管道连接所有房子

这就是最小生成树问题！

### Prim 算法

"从一个点开始生长"：

```
1. 从任意节点开始（比如🧀）

2. 找到与当前树相连的最短边
   🧀---1km---A  ← 最短！选它

3. 把新节点加入树
   现在树有：{🧀, A}

4. 重复步骤 2-3，直到所有节点加入
```

### Kruskal 算法

"从边开始合并"：

```
1. 把所有边按权重排序
   1km, 2km, 3km, 4km, 5km...

2. 从最短边开始，依次添加
   如果添加后不形成环，就加入

3. 直到有 V-1 条边（V=节点数）
```

---

## 📊 图算法复杂度对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| BFS | O(V+E) | O(V) | 无权图最短路径 |
| DFS | O(V+E) | O(V) | 拓扑排序、检测环 |
| Dijkstra | O((V+E)logV) | O(V) | 加权图最短路径 |
| Prim | O(ElogV) | O(V) | 稠密图 MST |
| Kruskal | O(ElogE) | O(V) | 稀疏图 MST |
