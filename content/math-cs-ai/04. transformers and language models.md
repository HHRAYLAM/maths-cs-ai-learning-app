## 🏗️ Transformer 架构回顾

**核心操作**：缩放点积注意力

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 多头注意力

```
输入 → 线性投影 → Q, K, V
          ↓
   分割成 h 个头
          ↓
   每个头独立注意力
          ↓
     拼接所有头
          ↓
      输出投影
```

**为什么多头**？
- 不同头关注不同方面
- 头 1：语法关系
- 头 2：语义关系
- 头 3：指代关系

### 前馈网络（FFN）

$$\text{FFN}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$

- 内部维度 = 4×模型维度
- 占每层 2/3 参数
- 独立应用于每个位置

</div>
  <div class="lesson-visual-column">

```
Transformer 块结构：

输入 x
   ↓
┌─────────────────┐
│ LayerNorm       │  ← Pre-norm（更稳定）
└─────────────────┘
   ↓
┌─────────────────┐
│ 多头自注意力     │
└─────────────────┘
   ↓
+ x（残差连接）
   ↓
┌─────────────────┐
│ LayerNorm       │
└─────────────────┘
   ↓
┌─────────────────┐
│ FFN（两层 MLP）  │
└─────────────────┘
   ↓
+ x（残差连接）
   ↓
输出

Pre-norm：先归一化再处理，梯度更好！
```

## 📍 位置编码 - 给序列顺序信息

注意力本身是**排列等变**的 —— 不知道 token 顺序！需要位置编码：

| 类型 | 方法 | 模型使用 |
|------|------|---------|
| **正弦编码** | 固定正弦/余弦函数 | 原始 Transformer |
| **学习嵌入** | 可训练向量 | BERT、GPT-2 |
| **RoPE** | 旋转向量 | LLaMA、PaLM |
| **ALiBi** | 线性偏置 | MPT、BLOOM |

---

### RoPE（旋转位置编码）

**RoPE** 通过旋转向量编码位置：

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的理解</div>
  <div class="story-card-content">
    <strong>核心思想</strong>：在 2D 平面旋转向量
    <br><br>
    <strong>美妙性质</strong>：
    - 注意力分数只依赖<strong>相对位置</strong>
    - 可以泛化到未见过的序列长度
    <br><br>
    <strong>为什么</strong>：$R_m^T R_n = R_{n-m}$
    <br>
    向后旋转 m + 向前旋转 n = 旋转 n-m
  </div>
</div>

---

### RoPE 数学原理

$$\begin{bmatrix} q'_{2i} \\ q'_{2i+1} \end{bmatrix} = \begin{bmatrix} \cos m\theta_i & -\sin m\theta_i \\ \sin m\theta_i & \cos m\theta_i \end{bmatrix} \begin{bmatrix} q_{2i} \\ q_{2i+1} \end{bmatrix}$$

$$\text{注意力分数} = q'^T k' = q^T R_{n-m} k$$

**关键**：只依赖相对位置 $n-m$，而非绝对位置！

---

### ALiBi（线性偏置注意力）

**ALiBi** 更简单：直接给注意力分数加距离惩罚

$$\text{score}_{ij} = q_i^T k_j - m \cdot |i - j|$$

- $m$ = 头特定的斜率
- 不同头关注不同距离
- 无需学习位置参数
- 泛化到更长序列

```
ALiBi 偏置矩阵（4 个 token）：

    1   2   3   4
1   0  -1  -2  -3
2  -1   0  -1  -2
3  -2  -1   0  -1
4  -3  -2  -1   0

距离越远，偏置越负，注意力越弱！
```

## 🎭 三大 Transformer 范式

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 1. 仅编码器（BERT）

**双向注意力**：每个 token 可关注所有其他 token

**预训练目标**：
- **MLM**：掩码语言建模（预测 15% 掩码词）
- **NSP**：下一句预测（判断两句是否连续）

**用途**：分类、NER、问答

```
输入：[CLS] The cat sat [MASK] the mat [SEP]
              ↓ 双向注意力
输出：[CLS] 表示用于分类
      [MASK] 位置预测"on"
```

### 2. 仅解码器（GPT）

**因果注意力**：每个 token 只能关注前面的 token

**预训练目标**：自回归语言建模（预测下一个词）

**用途**：文本生成、少样本学习

```
输入：The cat sat on
        ↓ 因果掩码
输出：the mat...（自回归生成）
```

</div>
  <div class="lesson-visual-column">

### 3. 编码器 - 解码器（T5/BART）

```
编码器：双向注意力处理输入
          ↓
     交叉注意力
          ↓
解码器：因果注意力 + 关注编码器
          ↓
       生成输出
```

**预训练目标**：
- **T5**：跨度损坏（替换连续跨度为哨兵 token）
- **BART**：多种损坏（掩码、删除、置换、旋转）

**用途**：翻译、摘要、文本到文本任务

### 范式对比

| 范式 | 注意力 | 用途 | 代表 |
|------|--------|------|------|
| **仅编码器** | 双向 | 理解 | BERT、RoBERTa |
| **仅解码器** | 因果 | 生成 | GPT、LLaMA |
| **编解码器** | 两者 | 转换 | T5、BART |

## 🔧 参数高效微调（PEFT）

**问题**：大模型全参数微调太贵（175B 参数需要数百 GB）！

**解决**：只训练一小部分参数

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的 PEFT</div>
  <div class="story-card-content">
    <strong>适配器</strong>：插入小瓶颈层（<5% 参数）
    <br><br>
    <strong>LoRA</strong>：低秩分解，训练 BA 而非 W
    <br><br>
    <strong>前缀微调</strong>：添加可学习虚拟 tokens
    <br><br>
    <strong>提示工程</strong>：无需训练，设计输入文本
  </div>
</div>

---

### LoRA（低秩适应）

**LoRA** 不更新完整权重 $W$，而是学习低秩分解：

$$W' = W + BA$$

- $W$：冻结的原始权重（$d \times d$）
- $B$：$d \times r$（向下投影）
- $A$：$r \times d$（向上投影）
- $r \ll d$：通常 $r = 4$ 到 $64$

<div class="lesson-two-column">
  <div class="lesson-text-column">

### LoRA 优势

| 优势 | 说明 |
|------|------|
| **参数少** | 减少 10-32 倍可训练参数 |
| **无延迟** | 推理时可合并回原权重 |
| **效果好** | 匹配全参数微调 |
| **易切换** | 不同任务只需换 LoRA 权重 |

### 典型设置

```
原模型：7B 参数（冻结）
LoRA 配置：
- r = 16
- α = 32
- dropout = 0.1

可训练参数：~10M（仅 0.1%）
效果：匹配全微调！
```

</div>
  <div class="lesson-visual-column">

### LoRA 结构

```
输入 x
   ↓
┌─────────────┐
│ 原始权重 W   │ ← 冻结
│ (d×d)       │
└─────────────┘
   ↓
   +        ← 加 LoRA 路径
   ↑
┌─────────────┐
│   B × A     │ ← 可训练
│ (d×r)×(r×d) │
└─────────────┘
   ↓
输出

训练时：分别计算，然后相加
推理时：合并为 W' = W + BA
```

## 💡 提示工程与上下文学习

### 提示技术

| **少样本** | 提供几个输入 - 输出示例 |

---

### 思维链（Chain-of-Thought）

```
普通提示：
Q: "小明有 5 个苹果，吃了 2 个，又买了 3 个，现在有几个？"
A: 直接回答 → 容易错

思维链提示：
Q: "小明有 5 个苹果，吃了 2 个，又买了 3 个，现在有几个？"
A: "让我们逐步思考：
   - 开始有 5 个
   - 吃了 2 个 → 5-2=3 个
   - 买了 3 个 → 3+3=6 个
   答案是 6 个"

效果好得多！模型学会分解问题
```

---

### 上下文学习（In-Context Learning）

**现象**：模型从提示中的示例学习，无需梯度更新！

```
提示（少样本）：
"英语→法语：
cat → chat
dog → chien
house → maison
car → ?"

模型输出："voiture"

模型没有更新权重，
但用示例作为隐式规范！
```

**研究假设**：注意力层在前向传递中实现类似梯度下降的操作！

## 📈 缩放定律 - 规模即能力

**缩放定律**描述性能如何随规模提升：

$$L(N) \propto N^{-\alpha_N}, \quad L(D) \propto D^{-\alpha_D}$$

<div class="lesson-two-column">
  <div class="lesson-text-column">

### Kaplan 定律（2020）

损失随参数量、数据量、计算量呈幂律下降：

| 变量 | 含义 |
|------|------|
| $N$ | 参数量 |
| $D$ | 数据集大小 |
| $C$ | 计算预算 |

**结论**：越大越好，可预测改进！

### Chinchilla 定律（2022）

**重大发现**：大多数模型训练不足！

**最优比例**：
- $N_{opt} \propto C^{0.5}$
- $D_{opt} \propto C^{0.5}$

同等扩展模型大小和数据！

</div>
  <div class="lesson-visual-column">

### Chinchilla vs Kaplan

```
固定计算预算：

Kaplan 建议：
大模型 + 少数据
  ↓
训练不足！

Chinchilla 建议：
适中模型 + 多数据
  ↓
性能更好！

经验法则：20 tokens/参数
7B 模型应看 140B tokens
```

## 🧠 MoE（专家混合）- 更多参数，相同计算

**MoE** 扩展模型容量而不增加计算：

<div class="story-card">
  <div class="story-card-title">🐱 Tom 的 MoE</div>
  <div class="story-card-content">
    <strong>结构</strong>：多个专家 FFN + 路由器
    <br><br>
    <strong>路由</strong>：每 token 选 top-K 专家（K=1 或 2）
    <br><br>
    <strong>优势</strong>：
    - 8 专家 + 选 2 = 4 倍参数，2 倍计算
    <br><br>
    <strong>挑战</strong>：
    - 负载平衡（防止热门专家）
    <br>
    - All-to-All 通信
  </div>
</div>

---

### MoE 路由

$$G(x) = \text{TopK}(\text{softmax}(W_g x))$$

**负载平衡损失**：

$$\mathcal{L}_{\text{balance}} = E \cdot \sum_{i=1}^{E} f_i \cdot p_i$$

- $f_i$ = 分配给专家 $i$ 的 tokens 比例
- $p_i$ = 专家 $i$ 的平均路由概率

鼓励均匀分布！

## 📊 NLP 评估指标

| 指标 | 说明 | 用途 |
|------|------|------|
| **完全匹配（EM）** | 输出是否完全匹配答案 | SQuAD 问答 |
| **精度** | TP / (TP + FP) | NER、分类 |
| **召回率** | TP / (TP + FN) | 信息检索 |
| **F1** | 精度和召回率的调和平均 | 综合评估 |
| **BLEU** | n-gram 重叠 | 翻译 |
| **ROUGE** | 召回率为主的 n-gram | 摘要 |
