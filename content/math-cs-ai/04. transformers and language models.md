# Transformers 和语言模型

*Transformers 用自注意力替换了循环，成为语言理解和生成的主导架构。本章涵盖 BERT、GPT、T5、位置编码（正弦、RoPE）、预训练目标（MLM、CLM）、微调、提示工程和缩放定律——现代 LLM 背后的蓝图。*

- 在第 06 章中，我们介绍了 Transformer 架构：自注意力、多头注意力、位置编码和编码器 - 解码器结构。这里我们专注于如何调整 transformers 以适应特定 NLP 范式、定义现代 NLP 的模型（BERT、GPT、T5）以及在大规模上使它们实用的技术。

- 回忆核心操作：**缩放点积注意力** 计算 $\text{softmax}(QK^T / \sqrt{d_k}) V$，其中查询、键和值是输入的线性投影。**多头注意力** 并行运行 $h$ 个注意力头，每个都有不同的学习投影，并拼接结果。Transformer 块用残差连接、层归一化和逐位置前馈网络包装这个（第 06 章）。

- 微妙但重要的架构选择是**层归一化** 的位置。原始 Transformer 使用 **post-norm**：残差和归一化在子层之后，如 $\text{LayerNorm}(x + \text{Sublayer}(x))$。

- 大多数现代模型使用 **pre-norm**：在子层之前归一化，如 $x + \text{Sublayer}(\text{LayerNorm}(x))$。Pre-norm 在训练期间更稳定，因为残差连接通过恒等路径直接传递梯度，而不受归一化影响。这使训练非常深的模型更容易，无需仔细的学习率预热。

- 每个 Transformer 块中的**前馈子层** 是独立应用于每个 token 位置的两层 MLP：

$$\text{FFN}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$

- 内部维度通常是模型维度的 4 倍（例如 $d_{\text{model}} = 768$，$d_{\text{ff}} = 3072$）。这个 FFN 占每个块中大约三分之二的参数，被认为用作键值记忆，存储训练期间学习的事实知识。

- **位置编码** 为模型提供关于 token 顺序的信息，因为注意力本身是排列等变的。原始**正弦编码**（第 06 章）使用不同频率的固定正弦和余弦函数。**学习位置嵌入** 简单地为每个位置添加可训练向量（用于 BERT 和 GPT-2）。两者都是绝对编码：位置 5 获得相同的向量，无论上下文如何。

- **旋转位置嵌入（RoPE）** 通过在 2D 子空间中旋转查询和键向量来编码位置。对于一对维度 $(q_{2i}, q_{2i+1})$，角度 $m\theta_i$ 的旋转（其中 $m$ 是位置，$\theta_i = 10000^{-2i/d}$）应用：

```math
\begin{bmatrix} q'_{2i} \\ q'_{2i+1} \end{bmatrix} = \begin{bmatrix} \cos m\theta_i & -\sin m\theta_i \\ \sin m\theta_i & \cos m\theta_i \end{bmatrix} \begin{bmatrix} q_{2i} \\ q_{2i+1} \end{bmatrix}
```

![RoPE：每个位置在 2D 子空间中以不同角度旋转查询和键向量，因此注意力分数仅依赖于相对位置](../images/rope_rotation.svg)

- RoPE 的美妙之处在于旋转向量 $q'$ 和 $k'$ 之间的点积 $q'^T k'$ 仅依赖于相对位置 $m - n$，而非绝对位置。

- 要理解为什么，将旋转写为 $q' = R_m q$ 和 $k' = R_n k$，其中 $R_m$ 是块对角旋转矩阵。注意力分数变为：

$$q'^T k' = (R_m q)^T (R_n k) = q^T R_m^T R_n \, k = q^T R_{n-m} \, k$$

- 最后一步遵循旋转群属性：$R_m^T R_n = R_{n-m}$（向后旋转 $m$ 然后向前旋转 $n$ 等于旋转 $n - m$）。

- 这意味着注意力分数仅依赖于相对距离 $n - m$，而非单独依赖于绝对位置 $m$ 和 $n$。

- 模型获得自然的距离概念，无需任何学习的位置参数，并可以泛化到训练期间未见过的序列长度。

- **ALiBi**（带线性偏置的注意力）采用甚至更简单的方法：根据距离向注意力分数添加固定线性惩罚，如 $\text{score}_{ij} = q_i^T k_j - m \cdot |i - j|$，其中 $m$ 是头特定的斜率。不同的头使用不同的斜率，允许一些头局部关注，其他头全局关注。ALiBi 无需学习位置参数，并能很好地泛化到比训练期间所见更长的序列。

- Transformer 语言模型的三个主导范式是**仅编码器**、**仅解码器** 和**编码器 - 解码器**。它们的不同在于模型能看到什么（注意力掩码）以及如何训练。

![三个 Transformer 范式：仅编码器（BERT）具有双向注意力用于分类，仅解码器（GPT）具有因果注意力用于生成，编码器 - 解码器（T5）结合两者用于 seq2seq 任务](../images/transformer_paradigms.svg)

- **BERT**（来自 Transformers 的双向编码器表示，Devlin 等人，2019）是典型的仅编码器模型。它使用完全双向注意力处理文本：每个 token 可以关注所有其他 token，左右皆可。这给 BERT 丰富的上下文表示，但意味着它不能自回归生成文本。

- BERT 用两个目标预训练。**掩码语言建模（MLM）** 随机掩码 15% 的输入 tokens 并训练模型预测它们。在选定的 tokens 中，80% 被替换为 [MASK] token，10% 替换为随机词，10% 保持不变（防止模型学习仅在看到 [MASK] 时预测）。训练目标是：

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(w_i \mid w_{\backslash \mathcal{M}})$$

- 其中 $\mathcal{M}$ 是掩码位置集，$w_{\backslash \mathcal{M}}$ 是这些位置被掩码的句子。这是**去噪** 目标：模型学习重建损坏的输入。

![BERT 掩码语言建模：15% 的 tokens 在输入中被掩码，双向 Transformer 在掩码位置预测原始 tokens](../images/bert_mlm.svg)

- **下一句预测（NSP）** 训练 BERT 预测两个句子在原文中是否连续。输入开头的特殊 [CLS] token 用于这个二元分类。NSP 被包含以帮助需要理解句子关系的任务（如问答），尽管后来的工作（RoBERTa）表明它贡献很小，可以省略。

- BERT 的预训练表示通过在顶部添加特定于任务的头（简单线性层）并微调整个模型来适应下游任务。对于分类任务，使用 [CLS] token 表示。对于 token 级任务（NER、POS 标记），使用每个 token 的表示。这种**微调** 方法将在预训练期间学到的语言知识转移到具有相对较少标记数据的新任务。

- **GPT**（生成式预训练 Transformer，Radford 等人，2018）是典型的仅解码器模型。它使用**因果（自回归）注意力**：每个 token 只能关注早期位置（及其自身）的 tokens。这通过在注意力矩阵中掩码未来位置（将它们的分数设为 $-\infty$，在 softmax 之前）来强制执行。训练目标是简单**因果语言建模**：给定所有前面的 tokens 预测下一个 token。

$$\mathcal{L}_{\text{CLM}} = -\sum_{i=1}^{n} \log P(w_i \mid w_1, \ldots, w_{i-1})$$

- 这与文件 02 中的 n-gram 语言模型目标相同，但具有 Transformer 参数化，可以基于整个前面的上下文而非仅最后 $k-1$ 个 tokens。

- **GPT-2** 将其扩展到 15 亿参数，并展示了强大的零样本性能：无需任何微调，它可以通过自然语言提示执行任务（"将英语翻译成法语：..."）。

- **GPT-3**（1750 亿参数）表明仅凭规模就能实现**上下文学习**：通过在提示中提供几个输入 - 输出示例，模型可以执行新任务，无需任何梯度更新。

- **编码器 - 解码器模型** 如 **T5**（文本到文本迁移 Transformer，Raffel 等人，2020）将每个 NLP 任务框架为文本到文本：输入是文本字符串（可能带有任务前缀，如"将英语翻译成德语："），输出是文本字符串。编码器使用双向注意力处理输入，解码器使用对编码器的交叉注意力自回归生成输出。

- T5 用**跨度损坏** 预训练：随机连续跨度 tokens 被替换为哨兵 tokens，模型必须生成原始 tokens。例如，"The cat sat on the mat" 可能变成 "The [X] on [Y]" 作为输入，目标是 "[X] cat sat [Y] the mat"。这是 BERT MLM 泛化到跨度而非单个 tokens。

- **BART**（Lewis 等人，2020）是另一个编码器 - 解码器模型，用去噪目标预训练，但它应用更广泛的损坏策略集：token 掩码、token 删除、跨度掩码、句子置换和文档旋转。损坏的多样性迫使模型学习更稳健的表示。

- 随着语言模型变大，**完全微调**（更新所有参数）变得不切实际：1750 亿参数模型需要数百 GB 仅存储优化器状态。**参数高效微调（PEFT）** 方法仅适应一小部分参数。

- **适配器** 插入小瓶颈层（通常两个线性层带非线性：向下投影到小维度，然后向上投影回）在现有 Transformer 层之间。仅训练适配器权重；原始模型权重被冻结。这增加少于 5% 的新参数，同时在大多数任务上匹配完全微调性能。

- **LoRA**（低秩适应）修改权重矩阵本身，而无需添加新层。LoRA 不更新完整权重矩阵 $W$，而是学习更新的低秩分解：$W' = W + BA$，其中 $B$ 是 $d \times r$，$A$ 是 $r \times d$，且 $r \ll d$（通常 $r = 4$ 到 $r = 64$）。原始 $W$ 被冻结；仅训练 $A$ 和 $B$。在推理时，更新可以合并到原始权重中，无额外延迟：

$$W' = W + BA$$

![LoRA：冻结的权重矩阵 W 被通过小矩阵 A 和 B 的低秩路径绕过，减少 32 倍可训练参数，同时匹配完全微调](../images/lora_decomposition.svg)

- **前缀微调** 在每个注意力层的键和值矩阵前添加一系列可学习的"虚拟 tokens"。模型关注这些前缀向量，就像它们是真实 tokens，仅训练前缀参数。这类似于提示微调，但在激活空间而非嵌入空间操作。

- **提示工程** 是设计输入文本的艺术，从预训练模型中引出期望行为，无需任何参数更新。

    - **零样本提示** 用自然语言描述任务（"分类以下评论的情感："）。

    - **少样本提示** 在实际查询前提供输入 - 输出示例。

    - **思维链（CoT）** 提示添加"让我们逐步思考"或在示例中包含推理轨迹，通过引导模型分解问题显著改善算术和逻辑推理任务的性能。

- **上下文学习（ICL）** 是大型语言模型可以从提示中提供的示例学习执行任务的现象，无需任何梯度更新。模型的权重不改变；它使用示例作为某种隐式规范。

- ICL 如何在机械上工作仍然是一个活跃的研究问题；一个假设是注意力层在前向传递中实现某种形式的梯度下降，有效地在上下文示例上"训练"。

- **缩放定律** 描述模型大小、数据大小、计算预算和性能（通过损失测量）之间的可预测关系。Kaplan 等人（2020）发现损失遵循每个变量的幂律：

$$L(N) \propto N^{-\alpha_N}, \quad L(D) \propto D^{-\alpha_D}, \quad L(C) \propto C^{-\alpha_C}$$

- 其中 $N$ 是参数数量，$D$ 是数据集大小，$C$ 是计算预算。这些幂律在多个数量级上成立，表明简单扩展产生可预测的改进。

![缩放定律：损失在对数 - 对数轴上作为幂律减少，Kaplan 和 Chinchilla 发现显示随规模可预测改进](../images/scaling_laws.svg)

- **Chinchilla 缩放定律**（Hoffmann 等人，2022）通过表明大多数大型模型训练不足来修订这个。对于固定计算预算 $C$，最优分配同等扩展模型大小和训练数据：

$$N_{\text{opt}} \propto C^{0.5}, \quad D_{\text{opt}} \propto C^{0.5}$$

- 这意味着如果你将计算预算加倍，你应该将模型大小和数据集大小都增加 $\sqrt{2}$ 倍，而不仅是使模型更大。

- Kaplan 等人建议扩展 $N$ 快于 $D$，这导致非常大但训练不足的模型。Chinchilla（700 亿参数，1.4 万亿 tokens）用相同计算预算匹配 Gopher（2800 亿参数，3000 亿 tokens）的性能，表明早期模型严重缺乏数据。

- 实用经验法则：每个参数大约训练 20 个 tokens。

- **专家混合（MoE）** 是一种架构，扩展模型容量而无需按比例扩展计算。MoE 不使用一个大型前馈层，而是使用多个**专家** FFN 层和**门控网络**（路由器），为每个 token 选择激活哪些专家。

- 门控函数为每个专家计算路由分数并选择前 $k$ 个（通常 $k = 1$ 或 $k = 2$）：

$$G(x) = \text{TopK}(\text{softmax}(W_g x))$$

- 仅选定的专家处理 token，因此计算成本按 $k$（活动专家数量）而非专家总数 $E$ 扩展。具有 8 个专家和前 2 路由的模型具有密集模型的 4 倍参数，但仅 2 倍计算。

![MoE 层：输入 token 通过路由器计算每个专家的分数，前 2 个专家被选中，它们的输出通过门控分数加权并求和](../images/moe_layer.svg)

- MoE 的关键挑战是**负载平衡**：如果路由器将大多数 tokens 发送到少数热门专家，其他专家被浪费。训练添加辅助**负载平衡损失**，鼓励均匀专家利用率：

$$\mathcal{L}_{\text{balance}} = E \cdot \sum_{i=1}^{E} f_i \cdot p_i$$

- 其中 $f_i$ 是分配给专家 $i$ 的 tokens 比例，$p_i$ 是专家 $i$ 的平均路由概率。当 tokens 比例和概率都均匀（每个等于 $1/E$）时，这个乘积最小化。

- **专家并行** 在不同加速器上分布不同专家。在前向传递期间，all-to-all 通信步骤将 tokens 路由到托管其分配专家的设备，然后将结果路由回。这个通信成本是大规模 MoE 的主要工程挑战。像 Switch Transformer、Mixtral 和 GShard 这样的模型使用 MoE 以实际推理成本实现强大性能。

- 构建模型是一半工作；测量它们是否工作是另一半。NLP 评估独特地困难，因为语言是模糊的、主观的和开放式的。

- 翻译可以用许多不同方式正确。摘要可以很好，即使它与参考没有完全相同的词。

- 聊天机器人响应可以有帮助、无害和诚实，但合理的人类会有不同意见。

- **完全匹配（EM）** 是最简单的指标：模型的输出是否完全匹配黄金答案？它用于具有简短、明确答案的任务，如提取式问答（SQuAD）或封闭式数学。

- EM 是严苛的；"New York City"和"new york city"无法匹配，除非应用归一化——但它的简单性使其明确。

- **Token 级指标** 将 NLP 视为 token 级的分类问题，使用第 06 章的精度、召回率和 F1。

- **精度** 测量模型预测的 tokens 中有多少比例是正确的：$P = \text{TP} / (\text{TP} + \text{FP})$。预测很少实体但全部正确的模型具有高精度。

- **召回率** 测量模型找到多少比例的黄金 tokens：$R = \text{TP} / (\text{TP} + \text{FN})$。将每个 token 预测为实体的模型具有完美召回率但糟糕精度。

- **F1** 是精度和召回率的调和平均：

$$F_1 = \frac{2PR}{P + R}$$

- 调和平均（而非算术）惩罚不平衡：如果 $P$ 或 $R$ 低，$F_1$ 低。对于 NER（文件 02），F1 按实体类型计算，然后在类型上宏观平均。对于 POS 标记，token 级准确率更常见，因为每个 token 获得一个标记。

- **跨度级 F1**（用于 SQuAD）比较预测跨度中的 token 集与黄金跨度中的 token 集。这比完全匹配更宽容：如果黄金答案是"the Eiffel Tower"，模型预测"Eiffel Tower"，跨度 F1 很高（5 个中有 4 个重叠 tokens），即使 EM 为零。

- **BLEU**（双语评估替补，Papineni 等人，2002）是机器翻译的经典指标。它测量候选翻译与一个或多个参考翻译之间的 n-gram 重叠。分数结合多个 n-gram 级别（从 unigram 到 4-gram）的精度与简短惩罚：

$$\text{BLEU} = \text{BP} \cdot \exp\!\left(\sum_{n=1}^{N} w_n \log p_n\right)$$

- 其中 $p_n$ 是**修正的 n-gram 精度**：候选中每个 n-gram 的计数被剪切到任何参考中的最大计数，防止退化候选如"the the the the"得分高。权重 $w_n$ 通常均匀（$w_n = 1/N$，$N = 4$）。

- **简短惩罚**$\text{BP} = \min(1, \exp(1 - r/c))$ 惩罚比参考短的候选（$c$ 是候选长度，$r$ 是参考长度）。没有这个，模型可以通过输出很少、很安全的词获得高精度。

- BLEU 在语料库级别（平均许多句子）与人类判断合理相关，但在句子级别较差。

- 它奖励完全 n-gram 匹配，错过有效释义："the cat is on the mat"和"a feline sits atop the rug"具有零 bigram 重叠，尽管意思相同。

- BLEU 也完全忽略召回率——仅产生最常见词的候选在精度上得分好。

- **ROUGE**（摘要评估的召回导向替补，Lin，2004）是摘要的标准指标。与强调精度的 BLEU 不同，ROUGE 强调召回率：参考的多少 n-gram 出现在候选中？

- **ROUGE-N** 计算 n-gram 的召回率：$\text{ROUGE-N} = \frac{|\text{n-grams}_{\text{ref}} \cap \text{n-grams}_{\text{cand}}|}{|\text{n-grams}_{\text{ref}}|}$。ROUGE-1（unigram）和 ROUGE-2（bigram）最常见。

- ROUGE-L 使用候选和参考之间的**最长公共子序列（LCS）**，捕捉句子级词排序，无需连续匹配。

- LCS 长度除以参考长度给出召回率，除以候选长度给出精度，F 度量结合它们。

- LCS 通过动态规划在 $O(mn)$ 时间内计算（类似于文件 02 中的编辑距离）：

$$R_{\text{LCS}} = \frac{\text{LCS}(X, Y)}{m}, \quad P_{\text{LCS}} = \frac{\text{LCS}(X, Y)}{n}, \quad F_{\text{LCS}} = \frac{(1 + \beta^2) R_{\text{LCS}} P_{\text{LCS}}}{R_{\text{LCS}} + \beta^2 P_{\text{LCS}}}$$

- 其中 $m$ 和 $n$ 是参考和候选的长度，$\beta$ 通常设置为 favor 召回率（$\beta \to \infty$ 给出纯召回率）。

- **METEOR**（带显式排序的翻译评估指标，Banerjee 和 Lavie，2005）通过结合同义词、词干和词序来解决 BLEU 的弱点。

- 它首先使用完全匹配、词干匹配（通过文件 02 的 Porter 词干提取）和同义词匹配（通过文件 01 的 WordNet）对齐候选和参考之间的词。

- 然后计算 unigram 精度和召回率的调和平均，偏向召回率，并应用碎片惩罚，惩罚匹配词以与参考不同顺序出现的候选。

- **ChrF**（字符 n-gram F 分数）在字符 n-gram 而非词 n-gram 上计算 F 分数。这使其对形态变化稳健（对文件 01 的黏着语关键），并部分处理 token 化差异。ChrF++ 添加词 bigrams 到字符 n-grams。

- 它已成为机器翻译与 BLEU 一起推荐的指标，特别是对形态丰富语言。

- **困惑度**（文件 02）测量语言模型预测保留测试集的好坏。它是语言模型的标准内在指标：$\text{PPL} = \exp(-\frac{1}{N} \sum_{i} \log P(w_i \mid w_{<i}))$。越低越好。

- 困惑度仅在使用相同 token 化的模型之间可比，因为不同 token 化器对相同文本产生不同序列长度 $N$。

- 具有较大词汇表的模型倾向于每 token 更低困惑度，但每句子处理更少 tokens。

- **每字节比特数**（BPB）通过文本中 UTF-8 字节数而非 tokens 数归一化，使其与 token 化无关：

```math
\text{BPB} = \frac{-\sum_{i} \log_2 P(w_i \mid w_{<i})}{\text{number of UTF-8 bytes}}
```

- **BERTScore**（Zhang 等人，2020）通过在嵌入空间中计算相似度超越表面级 n-gram 匹配。候选中的每个 token 使用上下文嵌入的余弦相似度（通常来自预训练 BERT 模型）匹配到参考中最相似的 token。分数聚合为精度、召回率和 F1：

$$R_{\text{BERT}} = \frac{1}{|r|} \sum_{r_i \in r} \max_{c_j \in c} \cos(r_i, c_j), \quad P_{\text{BERT}} = \frac{1}{|c|} \sum_{c_j \in c} \max_{r_i \in r} \cos(c_j, r_i)$$

- 其中 $r_i$ 和 $c_j$ 是参考和候选 tokens 的上下文嵌入。这捕捉 n-gram 指标错过的语义相似度："automobile"和"car"得分高，因为它们的 BERT 嵌入相似，即使它们不共享任何字符。

- **BLEURT**（Sellam 等人，2020）通过直接在人类质量判断上微调 BERT 模型更进一步。给定参考和候选对，它输出标量质量分数。BLEURT 在合成数据（通过 BLEU 和 METEOR 等指标评级的参考翻译的随机扰动）上训练，然后在人类评级上微调。它比任何表面级指标与人类判断更好相关。

- **COMET**（翻译评估的跨语言优化指标，Rei 等人，2020）是机器翻译的学习指标，条件于源句、参考和候选——不仅参考和候选。它使用多语言编码器（XLM-R）嵌入所有三个并预测质量分数。通过看到源，COMET 可以检测仅参考指标错过的意义错误（例如，流畅但事实错误的翻译）。

- **LLM 作为裁判** 是大规模评估的现代方法。LLM 裁判不针对参考计算指标，而是被提示评估模型输出的质量。裁判接收输入、模型的响应和可选参考答案，并产生评级（如 1-5）或成对偏好（响应 A 比响应 B 好）。

- **成对比较**（用于 Chatbot Arena）是最可靠的 LLM 裁判格式。裁判看到两个响应并选择更好的一个，而非分配绝对分数。这避免校准问题（不同裁判可能对"3 分制中的 3 分"有不同基线）。结果聚合为 **Elo 评级**（来自国际象棋），其中每个模型从基础评级开始，根据与其他模型的输赢获得或失去积分。模型 $A$ 对模型 $B$ 的预期获胜概率是：

$$P(A \succ B) = \frac{1}{1 + 10^{(R_B - R_A) / 400}}$$

- 其中 $R_A, R_B$ 是 Elo 评级。每次比较后，评级更新：$R_A' = R_A + K(S - P(A \succ B))$，其中 $S \in \{0, 1\}$ 是实际结果，$K$ 控制更新幅度。持续击败强对手的模型迅速上升；输给弱对手的模型下降。

- **位置偏差** 是 LLM 裁判的已知问题：他们倾向于偏好首先呈现的响应（或在某些模型中，第二呈现的响应）。**交换**（以两种顺序评估每对两次）并平均结果缓解这个。

- **冗长偏差** 是另一个：裁判倾向于偏好更长、更详细的响应，即使简洁答案更好。

- **自洽性** 检查裁判在多次评估相同输入时是否给出相同评级。高方差表明评估信号有噪声。

- **标注者间协议**（Cohen's kappa 或 Krippendorff's alpha）测量多个裁判是否同意，提供评估可靠性的上限。

- **污染** 是关键问题：如果评估数据出现在模型的训练集中，基准分数是膨胀和无意义的。

- 这对在网页抓取数据上训练的 LLM 特别有问题，流行基准可能存在。缓解策略包括：使用未公开发布的保留测试集、创建定期重新生成问题的动态基准、**canary 字符串**（嵌入基准数据中的唯一标识符，用于检测泄漏）以及比较污染与清洁子集的性能。

- **标准 NLU 基准** 评估多样化任务的语言理解。

- **GLUE**（通用语言理解评估）和 **SuperGLUE** 是多任务基准，涵盖情感（SST-2）、文本相似度（STS-B）、自然语言推理（MNLI、RTE）、指代消解（WSC）和问答（BoolQ）。

- 模型在每个任务上单独评估，并通过聚合指标评分。GLUE 现在被认为已饱和（模型在大多数任务上超过人类性能）；SuperGLUE 仍然更具挑战性。

- **MMLU**（大规模多任务语言理解）在 57 个学术科目（数学、历史、法律、医学、计算机科学等）上评估知识和推理，使用多项选择题。

- 它测试模型是否在预训练期间吸收了广泛知识。分数按科目报告和作为宏观平均。

- **MMLU-Pro** 添加更难、多步推理问题，有 10 个答案选择而非 4 个。

- **HellaSwag** 通过要求模型选择场景最合理的延续来测试常识推理。错误答案是 adversarially 生成的（使用模型），表面合理但语义错误。

- **WinoGrande** 用相差一个词的最小对测试常识指代消解。

- **ARC**（AI2 推理挑战）使用小学科学问题，简单和挑战集，测试事实和推理能力。

- **推理和数学基准** 评估区分强 LLM 和弱 LLM 的问题解决能力。

- **GSM8K**（小学数学 8K）包含 8,500 道 elementary 数学应用题，需要多步算术推理。它是基本数学推理和评估思维链提示（文件 04）的标准基准。

- **MATH** 是更难的比赛级数学问题数据集，涵盖代数、数论、几何、计数和概率。问题需要多步符号推理，MATH-500 是常报告的 500 题子集。

- **AIME**（美国邀请数学考试）问题是比赛级：正确解决它们需要多步深度数学推理。DeepSeek-R1 在 AIME 2024 上得分 79.8%，表明 RL 训练的推理模型（文件 05）可以接近强人类竞争者。

- **HumanEval** 和 **MBPP**（大多数基础编程问题）通过检查模型代码是否通过单元测试来评估代码生成。HumanEval 包含 164 个 Python 问题，具有函数签名和 docstrings；模型必须生成函数体。

- 指标是**pass@k**：$k$ 个生成解决方案中至少一个通过所有测试的概率。对于单个样本：

$$\text{pass@}k = 1 - \frac{\binom{n-c}{k}}{\binom{n}{k}}$$

- 其中 $n$ 是生成样本总数，$c$ 是通过的数量。这个公式修正简单地取 $k$ 个样本中最好的偏差。

- **SWE-bench** 更进一步，评估模型是否可以通过修改现有代码库解决真实 GitHub 问题——对实用软件工程能力的更难测试。

- **GPQA**（研究生级 Google 免疫 QA）包含生物学、物理学和化学的专家级问题，对领域专家也很难。它测试模型是否有真正理解而非模式匹配。"Diamond"子集是最难的。

- **安全和对齐基准** 评估模型是否有帮助、无害和诚实。

- **TruthfulQA** 测试模型是否复制常见误解。问题设计为使最常见的互联网答案错误（例如，"如果你吞下口香糖会发生什么？"，常见神话是它停留 7 年，但真实答案是它正常通过）。记忆流行但不正确声明的模型得分低。

- **BBQ**（问答偏见基准）测试年龄、性别、种族和宗教等类别的社会偏见。问题结构化使偏见模型系统地选择刻板答案。**Toxigen** 评估模型对特定人口群体生成有毒内容的倾向。

- **MT-Bench** 使用 80 个精心设计的跨写作、角色扮演、推理、数学、编码、提取、STEM 和人文学科的问题评估多轮对话能力。LLM 裁判（GPT-4）在 1-10 分制上对响应评分。多轮格式测试模型是否可以跟进、维护上下文和处理澄清请求。

- **Chatbot Arena**（LMSYS）使用真实用户进行匿名模型之间的盲成对比较。用户提交提示并投票选择更好的响应，而不知道哪个模型生成它。产生的 Elo 排行榜被认为是通用 LLM 质量最生态有效的评估，因为它反映真实用户在不同、未策划提示上的偏好。

- **AlpacaEval** 通过在固定指令集上将模型输出与参考模型（GPT-4）比较来自动化成对评估。裁判模型确定胜率。

- **AlpacaEval 2.0** 使用长度控制胜率来校正冗长偏差。

- **特定领域评估** 需要为专业领域定制指标。

- **词错误率（WER）** 用于语音识别：$\text{WER} = (S + D + I) / N$，其中 $S$、$D$、$I$ 是替换、删除和插入错误，$N$ 是参考词数。这是在词级应用的编辑距离（文件 02）除以参考长度归一化。

- **槽位 F1** 用于任务导向对话系统，测量模型是否从用户话语中正确提取结构化信息（例如，从"明天给我订去巴黎的航班"中提取"目的地：巴黎"和"日期：明天"）。

- **引用准确性** 用于 RAG 系统（文件 05），检查模型生成的引用是否实际支持所做的声明。针对检索段落验证声明，指标计算完全、部分或不被支持的声明比例。

- **评估陷阱** 是常见的，可以使整个基准比较无效。

- **为测试而教**：优化基准性能而非真正能力。在 MMLU 风格多项选择上微调的模型在 MMLU 上得分好，但在以开放式提出的相同问题上可能失败。

- **指标博弈**：模型可以优化为在自动指标上产生得分好的输出（高 BLEU、低困惑度），而无需真正好。BLEU 最优翻译通常是安全、通用的释义，而非自然、流畅的翻译。

- **基准饱和**：当模型接近或超过基准上的人类性能时，基准停止提供信息。GLUE、SQuAD 1.1 和其他几个现在已饱和。

- 该领域不断创建更难的基准，但创建、饱和和替换的循环使纵向比较困难。

- **人类评估** 仍然是黄金标准，但昂贵、缓慢且难以重现。不同标注者池（众包与领域专家、不同文化、不同语言）产生不同判断。报告标注者间协议和标注者人口统计对可重现性至关重要。

1. 从头实现完整的 Transformer 编码器块（多头注意力、前馈、残差连接、层归一化）。将其应用于简单序列分类任务。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def layer_norm(x, gamma, beta, eps=1e-5):
    mean = x.mean(axis=-1, keepdims=True)
    var = x.var(axis=-1, keepdims=True)
    return gamma * (x - mean) / jnp.sqrt(var + eps) + beta

def multi_head_attention(Q, K, V, W_q, W_k, W_v, W_o, n_heads):
    B, T, D = Q.shape
    head_dim = D // n_heads

    q = Q @ W_q  # (B, T, D)
    k = K @ W_k
    v = V @ W_v

    # Reshape to (B, n_heads, T, head_dim)
    q = q.reshape(B, T, n_heads, head_dim).transpose(0, 2, 1, 3)
    k = k.reshape(B, T, n_heads, head_dim).transpose(0, 2, 1, 3)
    v = v.reshape(B, T, n_heads, head_dim).transpose(0, 2, 1, 3)

    scores = q @ k.transpose(0, 1, 3, 2) / jnp.sqrt(head_dim)
    weights = jax.nn.softmax(scores, axis=-1)
    out = (weights @ v).transpose(0, 2, 1, 3).reshape(B, T, D)
    return out @ W_o, weights

def transformer_block(x, params):
    # Pre-norm multi-head self-attention
    normed = layer_norm(x, params['ln1_g'], params['ln1_b'])
    attn_out, weights = multi_head_attention(
        normed, normed, normed,
        params['W_q'], params['W_k'], params['W_v'], params['W_o'],
        n_heads=4
    )
    x = x + attn_out

    # Pre-norm feed-forward
    normed = layer_norm(x, params['ln2_g'], params['ln2_b'])
    ff = jax.nn.gelu(normed @ params['W1'] + params['b1'])
    ff = ff @ params['W2'] + params['b2']
    x = x + ff
    return x, weights

# Initialise parameters
d_model, d_ff, n_heads = 32, 128, 4
key = jax.random.PRNGKey(42)
keys = jax.random.split(key, 10)

params = {
    'W_q': jax.random.normal(keys[0], (d_model, d_model)) * 0.05,
    'W_k': jax.random.normal(keys[1], (d_model, d_model)) * 0.05,
    'W_v': jax.random.normal(keys[2], (d_model, d_model)) * 0.05,
    'W_o': jax.random.normal(keys[3], (d_model, d_model)) * 0.05,
    'ln1_g': jnp.ones(d_model), 'ln1_b': jnp.zeros(d_model),
    'ln2_g': jnp.ones(d_model), 'ln2_b': jnp.zeros(d_model),
    'W1': jax.random.normal(keys[4], (d_model, d_ff)) * 0.05,
    'b1': jnp.zeros(d_ff),
    'W2': jax.random.normal(keys[5], (d_ff, d_model)) * 0.05,
    'b2': jnp.zeros(d_model),
}

# Test with random input
x = jax.random.normal(keys[6], (2, 8, d_model))  # batch=2, seq_len=8
out, attn_weights = transformer_block(x, params)
print(f"Input shape:  {x.shape}")
print(f"Output shape: {out.shape}")
print(f"Attention weights shape: {attn_weights.shape}")  # (B, n_heads, T, T)

# Visualise attention patterns for each head
fig, axes = plt.subplots(1, 4, figsize=(16, 3.5))
for h in range(4):
    im = axes[h].imshow(attn_weights[0, h], cmap='Blues', vmin=0)
    axes[h].set_title(f"Head {h}")
    axes[h].set_xlabel("Key pos"); axes[h].set_ylabel("Query pos")
plt.suptitle("Multi-Head Attention Patterns")
plt.tight_layout(); plt.show()
```

2. 实现因果（自回归）注意力掩码并与双向注意力比较。展示掩码如何防止信息从未来 tokens 流向过去 tokens。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.T / jnp.sqrt(d_k)
    if mask is not None:
        scores = jnp.where(mask, scores, -1e9)
    weights = jax.nn.softmax(scores, axis=-1)
    return weights @ V, weights

seq_len, d_model = 6, 8
key = jax.random.PRNGKey(0)
k1, k2, k3 = jax.random.split(key, 3)
Q = jax.random.normal(k1, (seq_len, d_model))
K = jax.random.normal(k2, (seq_len, d_model))
V = jax.random.normal(k3, (seq_len, d_model))

# Bidirectional (encoder-style): all positions visible
bidir_mask = jnp.ones((seq_len, seq_len), dtype=bool)
bidir_out, bidir_weights = attention(Q, K, V, bidir_mask)

# Causal (decoder-style): only past and current positions visible
causal_mask = jnp.tril(jnp.ones((seq_len, seq_len), dtype=bool))
causal_out, causal_weights = attention(Q, K, V, causal_mask)

fig, axes = plt.subplots(1, 3, figsize=(14, 4))
tokens = [f"t{i}" for i in range(seq_len)]

axes[0].imshow(bidir_weights, cmap='Blues', vmin=0, vmax=0.5)
axes[0].set_title("Bidirectional Attention\n(BERT-style)")
axes[0].set_xticks(range(seq_len)); axes[0].set_xticklabels(tokens)
axes[0].set_yticks(range(seq_len)); axes[0].set_yticklabels(tokens)

axes[1].imshow(causal_mask.astype(float), cmap='Greys', vmin=0, vmax=1)
axes[1].set_title("Causal Mask\n(1 = allowed, 0 = blocked)")
axes[1].set_xticks(range(seq_len)); axes[1].set_xticklabels(tokens)
axes[1].set_yticks(range(seq_len)); axes[1].set_yticklabels(tokens)

axes[2].imshow(causal_weights, cmap='Blues', vmin=0, vmax=0.5)
axes[2].set_title("Causal Attention\n(GPT-style)")
axes[2].set_xticks(range(seq_len)); axes[2].set_xticklabels(tokens)
axes[2].set_yticks(range(seq_len)); axes[2].set_yticklabels(tokens)

for ax in axes:
    ax.set_xlabel("Key"); ax.set_ylabel("Query")
plt.tight_layout(); plt.show()

# Verify: in causal attention, output at position i depends only on positions <= i
print("Causal attention weight at position 2 (should only attend to 0, 1, 2):")
```

3. 实现 LoRA（低秩适应）并展示它如何通过比完全微调少得多的可训练参数修改权重矩阵。
```python
import jax
import jax.numpy as jnp

d_model = 256
rank = 4  # LoRA rank (much smaller than d_model)

key = jax.random.PRNGKey(42)
k1, k2, k3 = jax.random.split(key, 3)

# Original frozen weight matrix
W_frozen = jax.random.normal(k1, (d_model, d_model)) * 0.02

# LoRA matrices (only these are trainable)
B = jnp.zeros((d_model, rank))       # initialised to zero
A = jax.random.normal(k2, (rank, d_model)) * 0.01  # random init

# Forward pass: W_effective = W_frozen + B @ A
x = jax.random.normal(k3, (8, d_model))

# Without LoRA
y_original = x @ W_frozen.T

# With LoRA
W_effective = W_frozen + B @ A
y_lora = x @ W_effective.T

# Parameter counts
full_params = d_model * d_model
lora_params = d_model * rank + rank * d_model  # B + A

print(f"Model dimension: {d_model}")
print(f"LoRA rank: {rank}")
print(f"Full fine-tuning parameters: {full_params:,}")
print(f"LoRA parameters: {lora_params:,}")
print(f"Parameter reduction: {full_params / lora_params:.1f}x")
print(f"\nSince B is initialised to zeros, initial LoRA output matches original:")
print(f"  Max difference: {jnp.abs(y_original - y_lora).max():.2e}")

# Simulate training: only update A and B
def lora_forward(A, B, W_frozen, x):
    return x @ (W_frozen + B @ A).T

def dummy_loss(A, B, W_frozen, x, target):
    pred = lora_forward(A, B, W_frozen, x)
    return jnp.mean((pred - target) ** 2)

# Target: some transformation of x
target = x @ jax.random.normal(jax.random.PRNGKey(99), (d_model, d_model)).T * 0.02

grad_fn = jax.jit(jax.grad(dummy_loss, argnums=(0, 1)))
lr = 0.01

for step in range(200):
    gA, gB = grad_fn(A, B, W_frozen, x, target)
    A = A - lr * gA
    B = B - lr * gB

loss_before = dummy_loss(jnp.zeros_like(A), jnp.zeros_like(B), W_frozen, x, target)
loss_after = dummy_loss(A, B, W_frozen, x, target)
print(f"\nLoss before LoRA: {loss_before:.6f}")
print(f"Loss after LoRA:  {loss_after:.6f}")
print(f"Effective weight change rank: {jnp.linalg.matrix_rank(B @ A)}")
```
