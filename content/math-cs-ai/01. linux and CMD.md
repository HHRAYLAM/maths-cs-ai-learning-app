# Linux 和命令行

*命令行是 ML 工程的主要界面：训练任务、服务器管理、数据管道和集群管理都通过终端进行。本文件涵盖 shell、文件系统、权限、进程管理、包管理器、环境变量、SSH 以及每位 ML 工程师每天使用的基本命令。*

- GUI 在浏览网页时很方便。但它们在凌晨 2 点在远程 GPU 集群上运行训练任务时很糟糕。命令行（或终端、或 shell）是可扩展的工具：它适用于任何机器、可以编写脚本、可组合，并且在你的笔记本电脑、云 VM 和 HPC 集群上都相同。

- 如果你是一位只使用 Jupyter notebook 和 VS Code 按钮的 ML 工程师，你正在放弃巨大的生产力。每个生产 ML 系统都通过命令行部署、监控和调试。

## Shell

- Shell 是从你这里读取命令并执行它们的程序。它是你和操作系统之间的中介（第 13 章）。最常见的 shell 是bash（大多数 Linux 系统的默认）和zsh（macOS 的默认）。

- 命令的形式为：`command [options] [arguments]`

```bash
ls -la /home/user    # command=ls, options=-la, argument=/home/user
```

- 选项修改行为（通常 `-` 前缀表示短选项，`--` 前缀表示长选项）。`ls -l` 以长格式列出，`ls --all` 显示隐藏文件。许多选项可以组合：`ls -la` 意味着 `-l` 和 `-a` 一起。

### 基本导航

```bash
pwd                 # 打印工作目录（我在哪里？）
ls                  # 列出当前目录中的文件
ls -la              # 列出所有文件（包括隐藏文件）及详细信息
cd /path/to/dir     # 更改目录
cd ..               # 向上一级
cd ~                # 前往家目录
cd -                # 返回上一个目录
```

### 文件操作

```bash
cp source dest      # 复制文件
cp -r dir1 dir2     # 递归复制目录
mv old new          # 移动/重命名文件
rm file             # 删除文件（无回收站 —— 永久消失）
rm -rf dir          # 递归删除目录（危险 —— 无需确认）
mkdir -p a/b/c      # 创建嵌套目录
touch file.txt      # 创建空文件（或更新时间戳）
cat file.txt        # 打印文件内容
head -n 20 file     # 前 20 行
tail -f logfile     # 实时跟踪日志文件（对监控训练非常宝贵）
```

- 陷阱：`rm -rf` 是计算中最危险的命令。没有撤销。在按回车之前再三检查路径。永远不要运行 `rm -rf /` 或 `rm -rf ~`。

### 管道和重定向

- Shell 的杀手级功能是可组合性：将小命令连接在一起做复杂的事情。

- 管道（`|`）：将一个命令的输出作为输入发送到下一个命令。

```bash
cat training.log | grep "loss" | tail -5    # 包含"loss"的最后 5 行
ps aux | grep python                        # 查找运行中的 Python 进程
history | grep "docker"                     # 查找之前的 docker 命令
```

- 重定向：将输出发送到文件而不是屏幕。

```bash
python train.py > output.log 2>&1    # stdout 和 stderr 都到文件
python train.py >> output.log        # 追加（不要覆盖）
echo "data" > file.txt               # 覆盖文件
echo "more" >> file.txt              # 追加到文件
```

- `2>&1` 将 stderr（文件描述符 2）重定向到 stdout（文件描述符 1）。没有它，错误消息仍然出现在屏幕上，而只有正常输出进入文件。

### 文本处理

```bash
grep "error" logfile.txt             # 查找包含"error"的行
grep -r "import torch" src/          # 在目录中递归搜索
grep -i "warning" log.txt            # 不区分大小写搜索
grep -c "epoch" train.log            # 计数匹配的行

wc -l file.txt                       # 计数行数
wc -w file.txt                       # 计数单词

sort data.txt                        # 按字母顺序排序行
sort -n numbers.txt                  # 按数字排序
sort -u data.txt                     # 排序并去重
uniq -c sorted.txt                   # 计数连续重复

cut -d',' -f2,3 data.csv            # 从 CSV 中提取第 2 和第 3 列
awk '{print $1, $3}' data.txt       # 打印第 1 和第 3 个空白分隔字段
sed 's/old/new/g' file.txt          # 将所有"old"替换为"new"
```

- 这些可以优美地组合：

```bash
# 在日志文件中查找 10 个最常见的错误类型
grep "ERROR" app.log | awk -F': ' '{print $2}' | sort | uniq -c | sort -rn | head -10
```

### 查找文件

```bash
find . -name "*.py"                  # 查找所有 Python 文件
find . -name "*.pyc" -delete         # 查找并删除编译的 Python 文件
find /data -size +100M               # 大于 100 MB 的文件
find . -mtime -1                     # 过去 24 小时内修改的文件

which python                        # python 可执行文件在哪里？
locate filename                      # 快速文件搜索（使用预建索引）
```

## 文件系统层次结构

- Linux 将所有内容组织在以 `/` 为根的单个树中：

| 目录 | 用途 |
|-----------|---------|
| `/` | 整个文件系统的根 |
| `/home/user` | 你的个人文件、配置、项目 |
| `/etc` | 系统范围的配置文件 |
| `/usr` | 用户程序、库、文档 |
| `/usr/local` | 本地安装的软件（非来自包管理器） |
| `/var` | 可变数据：日志（`/var/log`）、数据库、缓存 |
| `/tmp` | 临时文件（重启时清除） |
| `/opt` | 可选的第三方软件 |
| `/proc` | 虚拟文件系统，暴露内核和进程信息 |
| `/dev` | 设备文件（磁盘、GPU 显示在这里） |

- 对于 ML：你的训练数据通常在 `/data` 或 `/home/user/data`，模型在 `/home/user/models`，CUDA 在 `/usr/local/cuda`。GPU 设备显示为 `/dev/nvidia0`、`/dev/nvidia1` 等。

## 文件权限

- 每个文件和目录对三类用户有三种权限类型：

| 权限 | 文件 | 目录 |
|------------|------|-----------|
| r (读) | 查看内容 | 列出内容 |
| w (写) | 修改内容 | 在内部创建/删除文件 |
| x (执行) | 作为程序运行 | 进入（cd 进入）目录 |

- 三类用户：所有者（u）、组（g）、其他（o）。

```bash
ls -l script.py
# -rwxr-xr-- 1 henry ml_team 2048 Mar 28 script.py
#  ^^^         所有者权限：rwx（读、写、执行）
#     ^^^      组权限：r-x（读、执行、无写）
#        ^^^   其他权限：r--（只读）
```

```bash
chmod 755 script.py       # 所有者=rwx, 组=rx, 其他=rx
chmod +x script.py        # 为每个人添加执行权限
chmod u+w,g-w file.txt    # 为所有者添加写权限，为组移除写权限
chown henry:ml_team file  # 更改所有者和组
```

- 陷阱：顶部有 `#!/usr/bin/env python3` 的 Python 脚本需要执行权限（`chmod +x`）才能作为 `./script.py` 运行。没有它，你必须使用 `python3 script.py`。

## 进程管理

- 进程是正在运行的程序（第 13 章）。Shell 为你提供管理它们的工具：

```bash
ps aux                    # 列出所有运行中的进程
ps aux | grep python      # 查找 Python 进程
top                       # 实时进程监控（CPU、内存）
htop                      # 更好的 top 版本（单独安装）
nvidia-smi                # GPU 使用情况（ML 必备）
watch -n 1 nvidia-smi     # 每秒刷新 nvidia-smi

kill PID                  # 优雅地终止进程
kill -9 PID               # 强制杀死（在优雅失败时使用）
killall python            # 杀死所有 Python 进程

# 后台运行
python train.py &                    # 后台运行
nohup python train.py > log.txt &    # 后台运行、注销后存活
```

- `nohup` 对 ML 训练至关重要：没有它，关闭 SSH 连接会杀死训练任务。`nohup` 将进程从终端分离。

- `screen` 和`tmux` 是终端复用器，创建持久会话。你可以在 tmux 会话中启动训练任务，断开 SSH 连接，稍后重新连接，会话（和训练）仍在运行。

```bash
tmux new -s training          # 创建命名会话
# ... 开始训练 ...
# Ctrl+B, 然后 D              # 从会话分离
tmux attach -t training       # 稍后重新连接（即使在 SSH 重新连接后）
tmux ls                       # 列出会话
```

## 包管理器

- 系统包（操作系统级软件）：

```bash
# Debian/Ubuntu
sudo apt update               # 刷新包列表
sudo apt install htop         # 安装包
sudo apt upgrade              # 升级所有包

# macOS
brew install wget             # 通过 Homebrew 安装
```

- Python 包：

```bash
pip install torch             # 从 PyPI 安装
pip install -e .              # 以可编辑模式安装当前项目
pip install -r requirements.txt  # 从需求文件安装
pip freeze > requirements.txt    # 导出已安装的包

# Conda（用于复杂依赖如 CUDA）
conda create -n myenv python=3.11
conda activate myenv
conda install pytorch torchvision cudatoolkit=12.1 -c pytorch
```

- 陷阱：永远不要 `pip install` 到系统 Python。始终使用虚拟环境（`python -m venv env`、`conda create` 或 `uv venv`）。系统 Python 由 OS 工具共享；破坏它可以破坏你的系统。

## 环境变量

- 环境变量是所有程序可访问的键值对。它们配置行为而无需更改代码。

```bash
export CUDA_VISIBLE_DEVICES=0,1    # 仅使用 GPU 0 和 1
export PYTHONPATH=/home/user/src   # 添加到 Python 的导入路径
export WANDB_API_KEY=abc123        # Weights & Biases 的 API 密钥

echo $PATH                         # 查看当前 PATH
export PATH=$PATH:/usr/local/cuda/bin  # 添加 CUDA 到 PATH
```

- `.bashrc`（或 `.zshrc`）：每次打开 shell 时运行的命令。将你的 `export` 语句放在这里以便持久化。

- `.env` 文件：由 `python-dotenv` 等工具加载的项目特定变量。将秘密（API 密钥、数据库密码）保存在 `.env` 中并将 `.env` 添加到 `.gitignore`。永远不要将秘密提交到 git。

## SSH（安全 Shell）

- SSH 通过加密通道将你连接到远程机器。这是你访问云 VM、GPU 服务器和 HPC 集群的方式。

```bash
ssh user@hostname              # 连接到远程机器
ssh -i ~/.ssh/key.pem user@ip  # 使用特定密钥连接
ssh -L 8888:localhost:8888 user@server  # 端口转发（远程上的 Jupyter）
```

- SSH 密钥（公钥/私钥对）替代密码：

```bash
ssh-keygen -t ed25519          # 生成密钥对
ssh-copy-id user@server        # 复制公钥到服务器
# 现在你可以 SSH 而无需输入密码
```

- SSH 配置（`~/.ssh/config`）保存连接详细信息：

```
Host gpu-server
    HostName 10.0.1.42
    User henry
    IdentityFile ~/.ssh/gpu_key
    LocalForward 8888 localhost:8888
```

- 现在 `ssh gpu-server` 自动使用所有这些设置连接。

- `scp` 和`rsync` 在机器之间传输文件：

```bash
scp model.pt user@server:/data/models/     # 复制文件到远程
scp -r user@server:/data/results/ ./       # 从远程复制目录
rsync -avz --progress data/ user@server:/data/  # 同步并显示进度（比 scp 更智能）
```

## 基本 ML 命令速查表

```bash
# GPU 监控
nvidia-smi                                   # GPU 使用情况快照
watch -n 1 nvidia-smi                        # 实时监控
gpustat                                      # 更清晰的 GPU 概览（pip install gpustat）

# 训练管理
nohup python train.py > train.log 2>&1 &     # 后台训练，注销后存活
tail -f train.log                            # 监控训练输出
kill %1                                      # 杀死最后一个后台作业

# 磁盘使用（数据集很大）
df -h                                        # 所有挂载点的磁盘空间
du -sh /data/*                               # /data 中每个项目的大小
du -sh --max-depth=1 .                       # 子目录大小

# 内存
free -h                                      # RAM 使用情况
cat /proc/meminfo                            # 详细内存信息

# 网络
curl -O https://example.com/dataset.tar.gz   # 下载文件
wget https://example.com/model.bin           # 替代下载器
curl -X POST http://localhost:8080/predict \
    -H "Content-Type: application/json" \
    -d '{"text": "hello"}'                   # 测试模型服务端点

# 归档
tar -czf archive.tar.gz directory/           # 压缩
tar -xzf archive.tar.gz                      # 解压
zip -r archive.zip directory/                # zip
unzip archive.zip                            # unzip

# 快速数据检查
head -5 data.csv                             # CSV 前 5 行
wc -l data.csv                               # 计数行
cut -d',' -f1 data.csv | sort -u | wc -l    # 计数第 1 列的唯一值
```
