# 为什么是 C++ 以及 ML 框架如何工作

*本书中的每个 `jnp.matmul`、每个 `torch.nn.Linear`、每个 `np.dot` 调用都在底层执行 C++ 和 CUDA 代码。本文件揭开这层面纱：为什么 ML 框架这样构建、Python 工程师的快速 C++ 基础、何时编写自定义 C++ 内核，以及如何将它们绑定到 Python —— 这是你编写的代码与运行它的硬件之间的桥梁。*

- 你已经花了 15 章编写 Python。你导入 JAX，调用 `jax.grad`，运行训练循环，构建模型。这一切感觉都像 Python。但真相是：**几乎没有实际计算发生在 Python 中。**

- 当你在 PyTorch 中编写 `output = model(input)` 或在 JAX 中编写 `output = jnp.matmul(W, x)` 时，Python 几乎什么都不做。它构建计算的描述（操作图），然后将其交给执行实际工作的 C++/CUDA 后端。Python 是方向盘，C++ 是引擎。

## 为什么是 Python 前端、C++ 后端

- 这种双语言架构存在是因为 Python 和 C++ 擅长相反的事情：

| | Python | C++ |
|--|--------|-----|
| 开发速度 | 快（动态类型、REPL、无需编译） | 慢（静态类型、头文件、编译时间） |
| 执行速度 | 比 C 慢约 100 倍（解释型、GIL） | 近硬件速度（编译型、无开销） |
| 内存控制 | 自动（垃圾回收），无法控制布局 | 手动，精确控制每个字节 |
| 硬件访问 | 无（无 SIMD、无 GPU、无自定义内存） | 完整（内在函数、CUDA、内联汇编） |
| 生态系统 | ML 丰富（笔记本、可视化、数据） | 系统丰富（操作系统、驱动、引擎） |

- 核心思想：**让每种语言做它擅长的事**。Python 处理人类生产力重要的部分（实验设计、超参数调优、数据探索）。C++ 处理机器性能重要的部分（矩阵乘法、卷积、注意力内核）。

- 单个矩阵乘法 `jnp.matmul(A, B)`（其中 $A$ 是 $4096 \times 4096$）执行约 1370 亿次浮点运算。在纯 Python 中（嵌套循环），这需要约 30 分钟。在优化后的 C++ 中（使用 AVX-512 SIMD 和多线程），只需约 10 毫秒。这是**180,000 倍**的差异。再多的 Python 技巧也无法弥合这个差距。

## ML 框架的结构

- 每个主要 ML 框架都遵循相同的架构：

```
用户代码（Python）
    ↓
Python API 层（torch.nn、jax.numpy、numpy）
    ↓
分发/JIT 编译器（torch.compile、XLA、NumPy 分发）
    ↓
C++ 内核库（ATen/PyTorch、XLA、BLAS/LAPACK）
    ↓
特定硬件后端（CUDA、cuDNN、MKL、oneDNN、Metal）
    ↓
硬件（CPU SIMD 单元、GPU 核心、TPU MXU）
```

### NumPy

- NumPy 的核心用 C 编写。当你调用 `np.dot(A, B)` 时，Python 调用一个 C 函数，该函数调用 BLAS（基础线性代数子程序），通常是 Intel MKL 或 OpenBLAS。BLAS 是手工优化的 C 和 Fortran 代码，使用 SIMD 指令、缓存感知内存访问模式和多线程。数十年的优化使矩阵乘法变得快速。

- NumPy 仅限 CPU。它不使用 GPU。但在 CPU 上，它非常快，因为它委托给最佳的 BLAS 实现。

### PyTorch

- PyTorch 的计算引擎是**ATen**（A Tensor Library），用 C++ 编写。ATen 实现了约 2000 个张量操作（add、matmul、conv2d、softmax 等），每个都有 CPU 和 CUDA 后端。

- 当你调用 `torch.matmul(A, B)` 时：
    1. Python 分发到 ATen C++ 函数。
    2. ATen 检查设备（CPU 或 CUDA）和数据类型。
    3. 在 CPU 上：调用 MKL/OpenBLAS。在 GPU 上：调用 cuBLAS（NVIDIA 的 GPU 优化 BLAS）。
    4. 结果被包装在 Python 张量对象中并返回。

- **torch.compile**（PyTorch 2.0+）更进一步：它跟踪你的 Python 代码，构建计算图，并使用**Triton**（GPU）或**C++/OpenMP**（CPU）编译它。编译后的代码融合操作，消除 Python 开销，可以比 eager 模式快 2-5 倍。

### JAX

- JAX 将 Python 函数编译为**XLA**（加速线性代数），这是 Google 针对 ML 工作负载的编译器。当你 `jax.jit` 一个函数时：
    1. JAX 跟踪函数，将操作捕获为 XLA 计算图（HLO — 高级操作）。
    2. XLA 优化图：融合操作、消除冗余计算、优化内存布局。
    3. XLA 编译到目标后端：CPU（通过 LLVM）、GPU（通过 CUDA/PTX）或 TPU（通过 TPU 专用指令）。
    4. 编译后的代码直接在硬件上运行，无需 Python 参与。

- 这就是为什么 `jax.jit` 如此重要：没有它，每个操作都是单独的 Python→C++ 往返。有了它，整个函数是单个编译内核。

## Python 工程师的快速 C++ 基础

- 你不需要成为 C++ 专家。你需要理解足够的知识来阅读内核代码、编写简单的扩展并理解性能讨论。以下是要点。

### 类型和变量

```cpp
// C++ 需要显式类型（与 Python 不同）
int count = 0;           // 32 位整数
float loss = 0.5f;       // 32 位浮点数
double lr = 3e-4;        // 64 位浮点数
bool training = true;    // 布尔值

// 数组（固定大小、栈分配）
float weights[1024];     // 1024 个浮点数，内存中连续

// 指针：保存内存地址的变量
float* ptr = weights;    // ptr 指向 weights 的第一个元素
float val = ptr[42];     // 通过指针算术访问第 42 个元素
// ptr[42] 等价于 *(ptr + 42)
```

- **指针**是与 Python 的最大概念差异。在 Python 中，一切都是引用，你从不需要考虑内存地址。在 C++ 中，指针给你直接访问内存的能力 —— 强大但危险（悬垂指针、缓冲区溢出）。

### 函数

```cpp
// 函数声明：返回类型 名称 (参数类型 参数名)
float relu(float x) {
    return x > 0.0f ? x : 0.0f;
}

// 通过引用传递（避免复制大对象）
void scale_vector(std::vector<float>& vec, float factor) {
    for (size_t i = 0; i < vec.size(); i++) {
        vec[i] *= factor;
    }
}

// const 引用：只读、无复制
float sum(const std::vector<float>& vec) {
    float total = 0.0f;
    for (float x : vec) {  // 基于范围的 for 循环（类似 Python 的 for x in vec）
        total += x;
    }
    return total;
}
```

### 内存：栈 vs 堆

```cpp
// 栈分配：快速、自动生命周期（函数返回时释放）
float buffer[256];   // 栈上 256 个浮点数

// 堆分配：手动、在函数之外存活
float* data = new float[n];   // 在堆上分配 n 个浮点数
// ... 使用 data ...
delete[] data;                 // 你必须释放它（无垃圾回收）

// 现代 C++：智能指针（自动清理，类似 Python 引用）
#include <memory>
auto data = std::make_unique<float[]>(n);  // 超出作用域时自动释放
```

- **关键规则**：栈快速但有限（通常 1-8 MB）。大数组（张量、特征图）必须放在堆上。在 Python 中，一切都在堆上，垃圾回收器处理清理。在 C++ 中，你自己管理（或使用智能指针）。

### 模板（泛型）

```cpp
// 适用于任何数值类型的函数
template <typename T>
T add(T a, T b) {
    return a + b;
}

add<float>(1.5f, 2.5f);   // 返回 4.0f
add<int>(3, 4);             // 返回 7
```

- 模板是 C++ 库（如 ATen）编写适用于 float16、float32、float64 等的代码而无需重复实现的方式。

### 标准库要点

```cpp
#include <vector>      // 动态数组（类似 Python 列表）
#include <string>      // 字符串类型
#include <unordered_map>  // 哈希映射（类似 Python 字典）
#include <algorithm>   // sort、find、transform 等

vec.push_back(4.0f);            // 追加
float first = vec[0];           // 索引
size_t len = vec.size();        // 长度

std::unordered_map<std::string, int> counts;
counts["hello"] = 5;            // 插入
if (counts.count("hello")) { }  // 检查存在性
```

## 何时编写自定义 C++ 内核

- 大多数 ML 工程师从不需要编写 C++。框架的内置操作覆盖 99% 的用例。你只应在以下情况考虑自定义 C++：

1. **你的操作在框架中不存在**：新颖的激活函数、自定义注意力模式、无法表示为现有操作组合的专用损失函数。

2. **融合操作以提高性能**：你的模型执行 `relu(layernorm(matmul(x, W) + b))`。每个操作启动单独的内核，读取和写入内存，并同步。融合内核在一次通过中完成所有操作，避免内存往返。这可以快 2-5 倍。

3. **减少内存使用**：自定义内核可以在不存储所有中间激活的情况下计算梯度（内核级别的梯度检查点）。

4. **针对新型硬件**：新加速器（如 Cerebras、Groq）可能没有框架支持。你直接编写内核。

- 对于情况 1-2，**Triton**（第 16 章，文件 05）通常足够且比直接编写 CUDA C++ 容易得多。只有当 Triton 无法表达你的需求时才使用 CUDA C++。

## 如何将 C++ 绑定到 Python

- 编写 C++ 是一半的工作。你还需要从 Python 调用它。

### pybind11（通用）

- pybind11 以最少的样板代码为 C++ 函数创建 Python 绑定：

```cpp
// my_ops.cpp
#include <pybind11/pybind11.h>
#include <pybind11/numpy.h>
namespace py = pybind11;

// 简单的自定义操作
py::array_t<float> custom_relu(py::array_t<float> input) {
    auto buf = input.request();
    float* ptr = static_cast<float*>(buf.ptr);
    size_t n = buf.size;

    auto result = py::array_t<float>(n);
    float* out = static_cast<float*>(result.request().ptr);

    for (size_t i = 0; i < n; i++) {
        out[i] = ptr[i] > 0 ? ptr[i] : 0;
    }
    return result;
}

PYBIND11_MODULE(my_ops, m) {
    m.def("custom_relu", &custom_relu, "自定义 ReLU 操作");
}
```

```bash
# 编译
```

### PyTorch C++ 扩展

- PyTorch 提供了一种简化的方式来添加自定义操作：

```cpp
// custom_op.cpp
#include <torch/extension.h>

torch::Tensor custom_gelu(torch::Tensor x) {
    return x * 0.5 * (1.0 + torch::erf(x / std::sqrt(2.0)));
}

PYBIND11_MODULE(TORCH_EXTENSION_NAME, m) {
    m.def("custom_gelu", &custom_gelu, "自定义 GELU 激活");
}
```

```python
# 即时加载和编译
from torch.utils.cpp_extension import load

custom_ops = load(
    name="custom_ops",
    sources=["custom_op.cpp"],
    extra_cflags=["-O3"],
)

x = torch.randn(1000)
y = custom_ops.custom_gelu(x)
```

- `torch.utils.cpp_extension.load` 编译 C++ 代码，创建共享库，并将其作为 Python 模块加载，全部在一次调用中完成。这是在 PyTorch 中实验自定义 C++ 操作的最简单方法。

### JAX 自定义调用

- JAX 使用 XLA 自定义调用。这个过程更复杂（你向 XLA 注册 C 函数），但概念相同：编写 C/C++，绑定它，从 Python 调用它。

- 对于大多数 JAX 用户，**Pallas**（文件 05 中介绍）是更好的选择：它让你用类似 Python 的语法编写 GPU 内核，由 XLA 编译，无需离开 JAX 生态系统。

## 全局视角

- 本文件解释了 Python 和硬件之间的层次。本章剩余文件更深入：
    - **文件 01**：硬件本身（CPU 架构、GPU 架构、内存系统）
    - **文件 02-03**：CPU 上的 SIMD 编程（ARM NEON、x86 AVX）—— 你编写使用 CPU 向量单元的 C++
    - **文件 04**：使用 CUDA 进行 GPU 编程 —— 你编写在上千个 GPU 核心上运行的 C++
    - **文件 05**：Triton、Pallas 和更高级的 GPU 编程 —— 你编写编译为 GPU 内核的 Python

- 这个递进反映了抽象阶梯：C++ 内在函数（最低、最多控制）→ CUDA（GPU 特定）→ Triton/Pallas（Python 风格、编译）→ JAX/PyTorch（最高、自动）。每一层用控制换取便利。理解底层使你更好地使用高层。

## 编程任务（使用 g++ 或 clang++ 编译）

1. 编写你的第一个 C++ 程序。分配数组、填充它、计算总和并测量时间。这介绍了编译、数组、指针和计时。
```cpp
// task1_basics.cpp
// 编译：g++ -O3 -o task1 task1_basics.cpp
// 运行：./task1

#include <iostream>
#include <chrono>
#include <vector>

int main() {
    const int N = 10'000'000;  // C++ 允许 ' 作为数字分隔符
    std::vector<float> data(N);

    // 填充数组
    for (int i = 0; i < N; i++) {
        data[i] = static_cast<float>(i) * 0.001f;
    }

    // 计算总和
    auto start = std::chrono::high_resolution_clock::now();
    float sum = 0.0f;
    for (int i = 0; i < N; i++) {
        sum += data[i];
    }
    auto end = std::chrono::high_resolution_clock::now();
    double elapsed = std::chrono::duration<double, std::milli>(end - start).count();

    std::cout << "总和：" << sum << std::endl;
    std::cout << "时间：" << elapsed << " 毫秒" << std::endl;
    std::cout << "元素：" << N << std::endl;
    std::cout << "吞吐量：" << (N * sizeof(float)) / elapsed / 1e6 << " GB/s" << std::endl;

    return 0;
}
```

```cpp
PYBIND11_MODULE(my_relu, m) {
```

3. 编写一个 C++ 程序来演示为什么内存布局很重要。比较行主序和列主序访问模式并测量性能差异。
```cpp
// task3_layout.cpp
// 编译：g++ -O3 -o task3 task3_layout.cpp

#include <iostream>
#include <chrono>
#include <vector>

int main() {
    const int N = 4096;
    std::vector<float> matrix(N * N, 1.0f);

    // 行主序访问：顺序内存地址（缓存友好）
    auto start = std::chrono::high_resolution_clock::now();
    float sum_row = 0.0f;
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            sum_row += matrix[i * N + j];  // 步长为 1 的访问
        }
    }
    auto end = std::chrono::high_resolution_clock::now();
    double row_ms = std::chrono::duration<double, std::milli>(end - start).count();

    // 列主序访问：步长为 N 的访问（缓存不友好）
    start = std::chrono::high_resolution_clock::now();
    float sum_col = 0.0f;
    for (int j = 0; j < N; j++) {
        for (int i = 0; i < N; i++) {
            sum_col += matrix[i * N + j];  // 步长为 N 的访问（缓存未命中！）
        }
    }
    end = std::chrono::high_resolution_clock::now();
    double col_ms = std::chrono::duration<double, std::milli>(end - start).count();

    std::cout << "行主序（缓存友好）: " << row_ms << " 毫秒" << std::endl;
    std::cout << "列主序（缓存不友好）:  " << col_ms << " 毫秒" << std::endl;
    std::cout << "减速：" << col_ms / row_ms << "x" << std::endl;
    std::cout << "(两种总和：" << sum_row << ", " << sum_col << ")" << std::endl;

    return 0;
}
```
