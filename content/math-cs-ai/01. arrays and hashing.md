# 数组和哈希

*数组和哈希表是编程中两个最基础的数据结构。本章介绍它们在底层的运作方式，然后构建关键的问题解决模式：双指针、滑动窗口、前缀和以及基于哈希的查找，通过渐进式难度的问题和每个步骤的常见陷阱。*

- 如果你深入理解数组和哈希映射，你可以解决约 40% 的所有编码面试问题。这两个结构无处不在，因为它们提供了算法最需要的两件事：**快速索引访问**（数组）和**按键快速查找**（哈希映射）。

- 本章教授的是模式，而非解决方案。目标是当你遇到新问题时，你能识别出适用哪种模式以及为什么，而不是试图回忆死记硬背的解决方案。

## 数组

- **数组**是一块连续的内存块，元素存储在固定偏移处。访问元素 $i$ 的成本是 $O(1)$，因为地址只需计算 `base + i * element_size`。这是可能的最快数据访问，也是为什么数组是默认选择。

- **动态数组**（Python 的 `list`、Java 的 `ArrayList`、C++ 的 `vector`）在满时自动增长。策略是**倍增摊销**：当数组满时，分配一个双倍大小的新数组并复制所有内容。复制成本是 $O(n)$，但它发生的频率很低（每 $n$ 次插入才发生一次），因此每次插入的摊销成本是 $O(1)$。

- **缓存局部性**是为什么数组在实践中而不仅是在理论上很快的原因。因为元素连续存储，访问一个元素会将附近元素加载到 CPU 缓存中（第 13 章）。遍历数组对缓存友好；在链表中跟随指针则不是。这个常数因子差异在实践中可达 10-100 倍。

| 操作 | 数组 | 动态数组 |
|-----------|-------|---------------|
| 按索引访问 | $O(1)$ | $O(1)$ |
| 追加 | n/a | $O(1)$ 摊销 |
| 在位置 $i$ 插入 | $O(n)$ | $O(n)$ |
| 在位置 $i$ 删除 | $O(n)$ | $O(n)$ |
| 搜索（未排序） | $O(n)$ | $O(n)$ |

- **陷阱**：在数组中间插入或删除是 $O(n)$，因为所有后续元素必须移位。如果你需要频繁在中间插入，考虑链表或完全不同的方法。

## 字符串

- **字符串**是字符数组。在 Python 中，字符串是不可变的：每次连接都会创建一个新字符串。在循环中逐个字符构建字符串是 $O(n^2)$，因为每次连接都会复制到目前为止的整个字符串。

```python
# 错误：O(n^2) 字符串连接
s = ""
for c in characters:
    s += c  # 每次都复制整个字符串

# 正确：O(n) 使用列表然后连接
parts = []
for c in characters:
    parts.append(c)
s = "".join(parts)
```

- **陷阱**：在 Python 中，循环内的 `s += c` 是最常见的性能 bug 之一。始终收集到列表中并使用 `.join()`。

- **编码**：ASCII 使用 7 位（128 个字符）。**UTF-8** 是变长的：ASCII 字符使用 1 字节，带重音的字符使用 2 字节，中文/日文字符使用 3 字节，表情符号使用 4 字节。当问题说"小写英文字母"时，字母表大小是 26，这意味着你可以使用固定大小数组而非哈希映射。

## 哈希表

- **哈希表**将键映射到值，具有 $O(1)$ 平均情况查找、插入和删除。它通过计算**哈希函数** $h(key)$ 将键转换为数组索引来工作。

- 哈希函数必须是：**确定性的**（相同键总是给出相同哈希）、**均匀的**（将键均匀分布到桶中）和**快速**计算。

- **冲突**发生在两个不同键哈希到相同索引时。两种主要策略：

    - **链地址法**：每个桶存储键值对的链表。冲突时，追加到列表。最坏情况（所有键哈希到同一桶）：$O(n)$。具有好哈希函数的平均情况：$O(1)$。

    - **开放寻址**：冲突时，探测下一个空槽。**线性探测**检查下一个槽，然后下一个，等等。它对缓存友好但遭受**聚集**（长串被占用的槽）。**Robin Hood 哈希**通过位移"更靠近家"的条目来减少方差。

- **负载因子** $\alpha = n / m$（项目数/桶数）决定性能。当 $\alpha$ 超过阈值（通常为 0.75）时，表**重新哈希**：分配更大的表并重新插入所有元素。这成本 $O(n)$ 但不常发生。

- **哈希映射**（Python 中的 `dict`，Java 中的 `HashMap`）存储键值对。**哈希集合**（Python 中的 `set`，Java 中的 `HashSet`）只存储键（用于快速成员测试）。

| 操作 | 平均 | 最坏情况 |
|-----------|---------|------------|
| 查找 | $O(1)$ | $O(n)$ |
| 插入 | $O(1)$ | $O(n)$ |
| 删除 | $O(1)$ | $O(n)$ |

- **布隆过滤器**是节省空间的概率集合。它们可以告诉你"肯定不在集合中"或"可能在集合中"（具有可调节的假阳性率）。它们使用 $k$ 个哈希函数和位数组。用于数据库（避免对不存在的键进行磁盘读取）、Web 缓存和拼写检查器。

- **何时使用哈希映射**：每当你需要在 $O(1)$ 时间内回答"我之前见过这个吗？"或"与这个键关联的计数/索引/值是什么？"。如果你正在进行重复的线性扫描来查找某些东西，哈希映射几乎肯定会让它更快。

## 模式：哈希映射查找

- 最基本的模式：使用哈希映射将 $O(n)$ 扫描替换为 $O(1)$ 查找。

### 简单：两数之和

- **问题**：给定一个整数数组和一个目标值，返回两个数的索引，使它们相加等于目标值。

- **暴力** $O(n^2)$：检查每一对。

- **模式洞察**：对于每个数字 `num`，你需要 `target - num` 存在于数组中的某个地方。与其扫描数组查找它，不如将之前见过的数字存储在哈希映射中。

```python
def two_sum(nums, target):
    seen = {}  # 值 -> 索引
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

- **为什么有效**：遍历数组一次。对于每个元素，哈希映射查找是 $O(1)$。总计：$O(n)$ 时间，$O(n)$ 空间。

- **陷阱**：不要在检查补数之前将当前数字添加到哈希映射中，否则你可能会将元素与自身匹配。上面代码中的顺序是正确的：先检查，然后插入。

### 中等：字母异位词分组

- **问题**：给定一个字符串列表，将字母异位词分组在一起。（"eat"、"tea"、"ate"）是一组。

- **模式洞察**：字母异位词包含相同的字符但顺序不同。如果你对每个字符串排序，字母异位词会产生相同的排序键。使用该排序键作为哈希映射键。

```python
from collections import defaultdict

def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))  # 或使用字符计数元组
        groups[key].append(s)
    return list(groups.values())
```

- **优化**：对每个字符串排序的成本是 $O(k \log k)$，其中 $k$ 是字符串长度。对于更快的键，计算字符频率并使用计数元组作为键：

```python
def group_anagrams_fast(strs):
    groups = defaultdict(list)
    for s in strs:
        count = [0] * 26
        for c in s:
            count[ord(c) - ord('a')] += 1
        groups[tuple(count)].append(s)
    return list(groups.values())
```

- 这是每个字符串 $O(k)$ 而不是 $O(k \log k)$。字符计数元组是**规范形式**：对于组的所有成员都相同的表示。

- **陷阱**：在 Python 中，列表不可哈希（不能作为字典键）。你必须转换为元组。当人们尝试 `groups[count].append(s)` 时会被绊倒。

### 困难：最长连续序列

- **问题**：给定一个未排序的数组，找到最长连续序列的长度（例如，[100, 4, 200, 1, 3, 2] → 4，因为 [1, 2, 3, 4]）。

- **暴力** $O(n \log n)$：对数组排序，然后扫描连续序列。

- **模式洞察**：将所有数字放入哈希集合以实现 $O(1)$ 查找。对于每个数字，检查它是否是序列的**开始**（即，`num - 1` 不在集合中）。如果是，计算序列延伸多远。

```python
def longest_consecutive(nums):
    num_set = set(nums)
    best = 0

    for num in num_set:
        # 仅从序列开头开始计数
        if num - 1 not in num_set:
            length = 1
            while num + length in num_set:
                length += 1
            best = max(best, length)

    return best
```

- **为什么是 $O(n)$**：内层 `while` 循环在所有迭代中最多运行 $n$ 次（每个数字最多被访问两次：外层循环一次，`while` 扩展一次）。`if num - 1 not in num_set` 守卫确保我们只从序列开头开始计数。

- **陷阱**：没有 `if num - 1 not in num_set` 检查，你会从每个元素开始计数，在最坏情况下变成 $O(n^2)$（例如，[1, 2, 3, ..., n] 会从每个起点扫描整个序列）。

## 模式：双指针

- **双指针**模式使用两个在数组中移动的索引，通常从两端或以不同速度从同一端移动。当数组已排序或你需要比较对时它有效。

- **何时使用**：问题涉及对、子数组或分区，且数组已排序（或可以在不丢失必要信息的情况下排序）。

### 简单：有效回文串

- **问题**：确定字符串是否是回文串，只考虑字母数字字符并忽略大小写。

- **模式**：一个指针在开头，一个在结尾。将它们向内移动，比较字符。

```python
def is_palindrome(s):
    left, right = 0, len(s) - 1

    while left < right:
        # 跳过非字母数字字符
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1

        if s[left].lower() != s[right].lower():
            return False

        left += 1
        right -= 1

    return True
```

- **陷阱**：忘记在内层 while 循环中检查 `left < right`。没有它，指针在像"!!!"（全非字母数字）这样的字符串上可能会越界。

### 中等：三数之和

- **问题**：找到数组中所有和为零的唯一三元组。

- **模式**：对数组排序。固定一个元素，然后在剩余部分使用双指针查找和为固定元素相反数的对。

```python
def three_sum(nums):
    nums.sort()
    result = []

    for i in range(len(nums) - 2):
        # 跳过重复的固定元素
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        left, right = i + 1, len(nums) - 1
        target = -nums[i]

        while left < right:
            total = nums[left] + nums[right]
            if total < target:
                left += 1
            elif total > target:
                right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                # 跳过重复
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1

    return result
```

- **为什么有效**：排序是 $O(n \log n)$。对于每个固定元素，双指针扫描是 $O(n)$。总计：$O(n^2)$，这对于这个问题是最优的（你必须考虑所有对）。

- **陷阱**：处理重复是最难的部分。没有重复跳过逻辑（对于固定元素和双指针结果两者），你会返回重复的三元组。`if i > 0 and nums[i] == nums[i-1]: continue` 行是关键的。

### 困难：接雨水

- **问题**：给定一个海拔图（非负整数数组），计算下雨后它能捕获多少水。

- **模式洞察**：对于每个位置，水位由其左侧最大高度和右侧最大高度的最小值减去当前高度决定。从两端开始的双指针跟踪这些运行最大值。

```python
def trap(height):
    left, right = 0, len(height) - 1
    left_max, right_max = 0, 0
    water = 0

    while left < right:
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1

    return water
```

- **为什么有效**：关键洞察是如果 `height[left] < height[right]`，位置 `left` 的水被 `left_max` 限制（我们知道右侧有更高的条，所以右侧不能是瓶颈）。我们处理较短的一侧，保证另一侧有更高的条。

- **陷阱**：许多人尝试先预计算 `left_max[i]` 和 `right_max[i]` 数组（这有效但使用 $O(n)$ 空间）。双指针方法实现 $O(1)$ 空间。另外，在最大值更新中混淆 `>=` 与 `>` 可能导致差一的水量计算。

## 模式：滑动窗口

- **滑动窗口**模式维护一个窗口（连续子数组），随着迭代扩展和收缩。它适用于要求满足某些条件的子数组或子字符串的问题。

- **何时使用**：问题要求满足约束的最长/最短子数组或子字符串，且扩展/收缩窗口是单调的（添加元素只能使约束更难/更容易满足，而不是两者）。

- **模板**：

```python
def sliding_window(arr):
    left = 0
    state = ...  # 窗口状态（计数、和等）
    best = ...

    for right in range(len(arr)):
        # 扩展：将 arr[right] 添加到窗口状态
        update_state(state, arr[right])

        # 收缩：当违反约束时从左侧缩小
        while constraint_violated(state):
            remove_from_state(state, arr[left])
            left += 1

        # 更新答案
        best = max(best, right - left + 1)  # 或 min，取决于问题

    return best
```

### 简单：买卖股票的最佳时机

- **问题**：给定每日价格，找到从一次买入和一次卖出（先买后卖）获得的最大利润。

- **模式**：跟踪到目前为止看到的最小价格（窗口的左边界）并计算每天的价格。

```python
def max_profit(prices):
    min_price = float('inf')
    max_profit = 0

    for price in prices:
        min_price = min(min_price, price)
        max_profit = max(max_profit, price - min_price)

    return max_profit
```

- 这是退化的滑动窗口：左指针（最小价格）仅在找到新最小值时向前移动。$O(n)$ 时间，$O(1)$ 空间。

### 中等：无重复字符的最长子串

- **问题**：找到不含任何重复字符的最长子串的长度。

- **模式**：通过移动 `right` 扩展窗口。当发现重复时，从左侧收缩直到移除重复。

```python
def length_of_longest_substring(s):
    char_index = {}  # 字符 -> 其最近索引
    left = 0
    best = 0

    for right, char in enumerate(s):
        if char in char_index and char_index[char] >= left:
            left = char_index[char] + 1  # 跳过重复

        char_index[char] = right
        best = max(best, right - left + 1)

    return best
```

- **为什么 `char_index[char] >= left`**：字符可能来自当前窗口开始之前在映射中。没有这个检查，你会错误地缩小对于实际上不在当前窗口中的字符的窗口。

- **陷阱**：使用集合并从左侧逐个移除字符是正确的但较慢。哈希映射方法直接跳到正确位置。

### 困难：最小覆盖子串

- **问题**：给定字符串 `s` 和 `t`，在 `s` 中找到包含 `t` 所有字符的最小窗口。

- **模式**：扩展窗口以包含所有必需字符，然后从左侧收缩以找到最小有效窗口。

```python
from collections import Counter

def min_window(s, t):
    if not t or not s:
        return ""

    need = Counter(t)       # 我们需要的字符及其计数
    have = 0                # 我们有足够数量的唯一字符数
    required = len(need)    # 我们需要的唯一字符数

    left = 0
    best = (float('inf'), 0, 0)  # (长度，左，右)

    window_counts = {}

    for right in range(len(s)):
        char = s[right]
        window_counts[char] = window_counts.get(char, 0) + 1

        # 检查该字符的计数现在是否满足要求
        if char in need and window_counts[char] == need[char]:
            have += 1

        # 当窗口有效时从左侧收缩
        while have == required:
            # 更新最佳
            if (right - left + 1) < best[0]:
                best = (right - left + 1, left, right)

            # 移除最左侧字符
            left_char = s[left]
            window_counts[left_char] -= 1
            if left_char in need and window_counts[left_char] < need[left_char]:
                have -= 1
            left += 1

    length, start, end = best
    return s[start:end + 1] if length != float('inf') else ""
```

- **陷阱**：`have` 计数器是关键的优化。没有它，你需要在每一步比较整个 `window_counts` 字典与 `need`，这是每步 $O(|\text{唯一字符}|)$。`have` 计数器使有效性检查为 $O(1)$。

- **陷阱**：检查 `window_counts[char] == need[char]`（不是 `>=`）确保我们每个字符只递增 `have` 一次。如果我们使用 `>=`，我们会过度计数。

## 模式：前缀和

- **前缀和**数组存储累积和：`prefix[i] = sum(arr[0:i])`。一旦在 $O(n)$ 内构建，任何子数组和都可以在 $O(1)$ 内计算：`sum(arr[l:r]) = prefix[r] - prefix[l]`。

```python
def build_prefix(arr):
    prefix = [0] * (len(arr) + 1)
    for i in range(len(arr)):
        prefix[i + 1] = prefix[i] + arr[i]
    return prefix

# arr[l:r] 的和（包含 l，不包含 r）
def range_sum(prefix, l, r):
    return prefix[r] - prefix[l]
```

- **何时使用**：问题涉及多个子数组和查询，或查找具有特定和的子数组。

### 简单：范围和查询

- **问题**：给定一个数组，回答多个"从索引 $l$ 到 $r$ 的和是多少？"的查询。

- 没有前缀和：每个查询是 $O(n)$。有了前缀和：$O(n)$ 预计算，然后每个查询 $O(1)$。

### 中等：子数组和等于 K

- **问题**：计算和为 $k$ 的连续子数组的数量。

- **模式洞察**：从索引 $l$ 到 $r$ 的子数组和等于 `prefix[r+1] - prefix[l]`。我们希望这等于 $k$，所以 `prefix[l] = prefix[r+1] - k`。对于每个位置，使用哈希映射计算有多少更早的前缀和等于 `current_prefix - k`。

```python
def subarray_sum(nums, k):
    count = 0
    prefix = 0
    prefix_counts = {0: 1}  # 空前缀和

    for num in nums:
        prefix += num
        # 有多少更早的前缀和等于 prefix - k？
        count += prefix_counts.get(prefix - k, 0)
        prefix_counts[prefix] = prefix_counts.get(prefix, 0) + 1

    return count
```

- 这结合了前缀和与哈希映射查找：$O(n)$ 时间，$O(n)$ 空间。

- **陷阱**：忘记初始化 `prefix_counts = {0: 1}`。空前缀（在任何元素之前）的和为 0。没有这个，你会错过从索引 0 开始的子数组。

### 困难：除自身以外数组的乘积

- **问题**：给定一个数组，返回一个数组，其中每个元素是所有其他元素的乘积。你不能使用除法。

- **模式**：从左侧构建前缀乘积，从右侧构建后缀乘积。每个位置的答案是 `left_product * right_product`。

```python
def product_except_self(nums):
    n = len(nums)
    result = [1] * n

    # 左遍：result[i] = nums[0..i-1] 的乘积
    prefix = 1
    for i in range(n):
        result[i] = prefix
        prefix *= nums[i]

    # 右遍：乘以 nums[i+1..n-1] 的乘积
    suffix = 1
    for i in range(n - 1, -1, -1):
        result[i] *= suffix
        suffix *= nums[i]

    return result
```

- $O(n)$ 时间，$O(1)$ 额外空间（输出数组不计入）。这使用输出数组本身来存储中间前缀乘积，然后在第二遍中乘以后缀乘积。

- **陷阱**：如果数组包含零，基于除法的方法会失败。这种前缀/后缀方法正确处理零，因为它从不除法。

## 常见陷阱总结

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 窗口大小差一错误 | `right - left` vs `right - left + 1` | 画一个 2 元素示例 |
| Python 中的可变默认值 | `def f(seen={})` 在调用之间共享状态 | 使用 `def f(seen=None)` |
| 循环中的字符串连接 | `s += c` 在 Python 中是 $O(n^2)$ | 使用 `list.append` + `"".join"` |
| 前缀和中忘记 `{0: 1}` | 错过从索引 0 开始的子数组 | 始终用空前缀初始化 |
| 检查前使用哈希映射 | 两数之和：在检查补数之前添加 `num` | 先检查，然后插入 |
| 不处理重复 | 三数之和返回重复三元组 | 跳过连续相等的值 |
| 整数溢出 | C++/Java 中大数组的和 | 使用 `long` 或检查边界 |
