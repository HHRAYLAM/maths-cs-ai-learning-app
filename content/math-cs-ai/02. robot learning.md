# 机器人学习

*机器人学习弥合算法和物理行动之间的差距。本文件涵盖运动学、动力学、经典控制、模仿学习、模拟到真实迁移、操作、移动和安全，赋予机器人移动、抓取、行走和与现实世界交互能力的技术。*

- 在前面的章节中，我们学习了如何感知世界（第 8 章、第 11 章文件 1）和如何从数据学习（第 6 章）。但感知和学习不够。机器人必须**行动**：移动手臂抓取杯子、走过不平坦地形、或导航仓库。这是机器人学习发挥作用的地方。

- 中心挑战是物理世界是连续、高维、接触丰富且无情的。图像识别中的分类错误是错误标签。机器人学中的控制错误是损坏的机器人或掉落的物体。风险不同。

## 机器人运动学

- **运动学**描述运动的几何而不考虑力。机器人手臂是刚性连杆链通过关节连接。每个关节有一个自由度 (DoF)：它要么旋转（旋转关节）要么滑动（棱柱关节）。

- 机器人的**配置**是所有关节角度（或位移）的集 $\mathbf{q} = [q_1, q_2, \ldots, q_n]^T$。这个向量存在于**关节空间**（或配置空间），$n$ 维空间其中每轴对应一个关节。6-DoF 机器人手臂有 6D 配置空间。

![2 连杆机器人手臂：关节角度 q1 和 q2 通过正向运动学确定末端执行器位置](../images/robot_arm_fk.svg)

- **正向运动学** (FK) 计算给定关节角度的末端执行器（"手"）的位置和方向。这是函数 $\mathbf{x} = f(\mathbf{q})$ 从关节空间映射到**任务空间**（末端执行器的 3D 位置和方向，也称为笛卡尔空间）。

- 每个关节用 $4 \times 4$ 齐次变换矩阵描述（回忆第 2 章的仿射变换）。**Denavit-Hartenberg** (DH) 约定用四个数参数化每个关节：连杆长度 $a$、连杆扭转 $\alpha$、连杆偏移 $d$ 和关节角度 $\theta$。关节 $i$ 的变换是：

$$T_i = \begin{bmatrix} \cos\theta_i & -\sin\theta_i \cos\alpha_i & \sin\theta_i \sin\alpha_i & a_i \cos\theta_i \\ \sin\theta_i & \cos\theta_i \cos\alpha_i & -\cos\theta_i \sin\alpha_i & a_i \sin\theta_i \\ 0 & \sin\alpha_i & \cos\alpha_i & d_i \\ 0 & 0 & 0 & 1 \end{bmatrix}$$

- 完整正向运动学是所有关节变换的乘积：$T_{0 \to n} = T_1 T_2 \cdots T_n$。这是矩阵乘法链接变换（第 2 章）：每个关节的变换按顺序应用，从基到末端执行器旋转和平移帧。

- **逆向运动学** (IK) 是反向问题：给定期望末端执行器位姿 $\mathbf{x}^*$，找关节角度 $\mathbf{q}$ 使得 $f(\mathbf{q}) = \mathbf{x}^*$。这难得多因为：

    - 映射是非线性的（涉及正弦和余弦）。
    - 可能有多个解（不同手臂配置到达同一点）。
    - 可能无解（目标超出 reach）。

- 解析解仅对特定机器人几何存在。对通用机器人，IK 用**雅可比**迭代求解。雅可比 $J(\mathbf{q})$ 关联关节角度的小变化到末端执行器位置的小变化（回忆第 3 章的雅可比）：

$$\dot{\mathbf{x}} = J(\mathbf{q}) \dot{\mathbf{q}}$$

- 移动末端执行器小量 $\Delta \mathbf{x}$，我们需要 $\Delta \mathbf{q} = J^{-1} \Delta \mathbf{x}$（或 $J^+ \Delta \mathbf{x}$ 使用伪逆当 $J$ 不平方）。这迭代直到末端执行器到达目标，本质是牛顿法（第 3 章）应用于运动学方程。

- 在**奇点**附近，雅可比失去秩（一些列变得线性相关，如第 2 章研究）。物理上，这意味着机器人失去自由度：无论关节多快移动，末端执行器无法在某些方向移动。伪逆在奇点附近爆炸，所以用阻尼最小二乘（添加正则化项 $\lambda^2 I$）代替：

$$\Delta \mathbf{q} = J^T(JJ^T + \lambda^2 I)^{-1} \Delta \mathbf{x}$$

## 动力学和控制

- **动力学**添加力到图中。机器人手臂的运动方程遵循**机械臂方程**：

$$M(\mathbf{q})\ddot{\mathbf{q}} + C(\mathbf{q}, \dot{\mathbf{q}})\dot{\mathbf{q}} + \mathbf{g}(\mathbf{q}) = \boldsymbol{\tau}$$

- 其中 $M(\mathbf{q})$ 是质量（惯性）矩阵，$C(\mathbf{q}, \dot{\mathbf{q}})$ 捕捉 Coriolis 和离心效应，$\mathbf{g}(\mathbf{q})$ 是重力向量，$\boldsymbol{\tau}$ 是关节扭矩向量（控制输入）。这是二阶微分方程组，每关节一个。

- 质量矩阵 $M$ 总是对称和正定（回忆第 2 章正定矩阵保证唯一最小值，这里它确保系统对应用扭矩可预测响应）。

- **PID 控制**是机器人学中最广泛使用的控制器。对每个关节，它基于误差 $e(t) = q_{\text{desired}}(t) - q_{\text{actual}}(t)$ 计算扭矩：

$$\tau(t) = K_p e(t) + K_i \int_0^t e(s) \, ds + K_d \dot{e}(t)$$

- 三个项有直观角色：
    - **比例** ($K_p$）：与当前误差成比例纠正。更大误差 → 更大纠正。像弹簧将关节拉向目标。
    - **积分** ($K_i$）：累积过去误差消除稳态偏移。如果关节一致欠冲，积分项建立并提供额外推动。
    - **微分** ($K_d$）：对误差变化率反应，提供阻尼。它减慢响应当误差减少，防止过冲和振荡。

![PID 控制器调优：高 Kp 振荡，高 Kd 迟缓，调优良好的 PID 快速到达目标](../images/pid_response.svg)

- 调优 $K_p, K_i, K_d$ 是平衡：太多 $K_p$ 引起振荡，太多 $K_d$ 使系统迟缓，太多 $K_i$ 引起 wind-up（积分在持续误差期间无界增长）。

- **模型预测控制** (MPC) 向前看。每时间步，它求解优化问题：找最小化成本函数（例如，跟踪误差 + 控制努力）在有限范围上的未来控制序列，服从动力学模型和约束。仅第一个控制被应用，然后在下一时间步过程重复。

$$\min_{\mathbf{u}_{0:T}} \sum_{t=0}^{T} \left[ \|\mathbf{x}_t - \mathbf{x}_t^*\|_Q^2 + \|\mathbf{u}_t\|_R^2 \right] \quad \text{subject to} \quad \mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t)$$

- 这里 $\|\mathbf{x}\|_Q^2 = \mathbf{x}^T Q \mathbf{x}$ 是用正定矩阵 $Q$ 的加权范数（第 2 章），它让你不同地惩罚不同状态误差。MPC 自然处理约束（关节限制、扭矩限制、障碍避免）因为它们显式包含在优化中。

- **阻抗控制**调节力和运动之间的关系而非跟踪刚性轨迹。不是命令"去位置 $x$"，它命令"表现像以 $x$ 为中心的弹簧 - 阻尼系统"：

$$F = K_s(\mathbf{x}^* - \mathbf{x}) + D(\dot{\mathbf{x}}^* - \dot{\mathbf{x}})$$

- 其中 $K_s$ 是刚度矩阵，$D$ 是阻尼矩阵。这使机器人柔顺：如果它接触障碍，它屈服而非强制通过。阻抗控制对接触丰富任务如将销插入孔或递物体给人 essential。

## 模仿学习

- 不是手工设计控制器，我们可以从演示学习控制策略。人执行任务，机器人观察，学习算法提取策略。这是**模仿学习**（或从演示学习）。

- **行为克隆** (BC) 是最简单方法：将演示视为监督学习数据集。给定来自专家的观察 - 行动对 $\{(\mathbf{o}_t, \mathbf{a}_t)\}$，训练策略 $\pi_\theta(\mathbf{a} \mid \mathbf{o})$ 从观察预测专家的行动。这是标准监督学习（第 6 章）：最小化损失：

$$\mathcal{L}(\theta) = \mathbb{E}_{(\mathbf{o}, \mathbf{a}) \sim \mathcal{D}} \left[ \| \pi_\theta(\mathbf{o}) - \mathbf{a} \|^2 \right]$$

![行为克隆中的分布偏移：小误差复合，导致学习策略漂移远离专家轨迹](../images/distribution_shift_bc.svg)

- 问题是**分布偏移**（也称为**复合误差问题**）。训练期间，策略看到专家的状态。部署期间，策略自己的小误差将它推入专家从未访问的状态。这些不熟悉状态导致更差行动，导致甚至更不熟悉状态，误差快速复合。

- 想象通过观看完美司机学习驾驶。你从未看到小摇摆后发生什么因为专家从未摇摆。第一次你漂移一点，你不知道如何恢复。

- **DAgger**（数据集聚合）通过迭代解决这个问题：
    1. 在当前数据上训练策略。
    2. 在环境中运行策略，收集新状态。
    3. 请专家标记这些新状态带正确行动。
    4. 添加新数据到数据集并重新训练。

- 在迭代上，数据集覆盖学习策略实际访问的状态，非仅专家的轨迹。策略改进因为它已看到并学习从自己错误恢复。

- **带 Transformers 的行动分块** (ACT) 是现代方法其中策略预测未来行动序列（"块"）而非一次一个行动。这用条件 VAE 带 transformer 主干实现。预测行动块更鲁棒因为它捕捉时间相关：伸手运动的平滑性编码在块中而非依赖可能漂移的自回归单步预测。

- **扩散策略**应用扩散模型（第 8 章）到行动生成。不是预测单个行动，它建模可能行动的完整分布条件于观察。从噪声开始，它迭代去噪产生行动序列。这自然处理**多模态**：当有多个有效方式完成任务（从左或右伸手），扩散模型可以表示两个模式，而回归策略将平均它们（并伸手中间某处，可能都无效）。

## 模拟到真实迁移

- 在现实世界训练机器人昂贵、慢且危险。机器人通过试错学习抓取可能需要数千次尝试，沿途破坏物体和它自己。**模拟**提供无限、安全、快速体验。但模拟器不完美：物理近似、视觉合成、接触简化。

- **模拟到真实差距**是模拟和真实性能之间的差异。在模拟中完美工作的策略可能在真实机器人上完全失败因为它过拟合模拟器特定细节。

![模拟到真实通过域随机化：在许多随机化模拟上训练所以真实世界只是另一个变体](../images/sim_to_real.svg)

- **域随机化**通过在广泛模拟器设置上训练对抗这个。不是一个模拟，用数千随机化：
    - 物理：摩擦系数、质量、阻尼
    - 视觉：照明、纹理、颜色、相机位置
    - 动力学：电机延迟、噪声级别

- 思想是如果策略在所有这些变体上工作，真实世界只是分布内"另一个变体"。策略学习对随机化属性不变的特征，这些不变特征迁移。

- **系统识别**采取相反方法：不是随机化一切，仔细测量真实系统的物理参数并调整模拟器匹配。这给出更准确模拟但脆弱（任何未建模效应引起差距）。

- 在实践中，最好结果组合两者：系统识别让模拟器合理接近，然后域随机化覆盖剩余不确定性。

- **通过微调的模拟到真实**主要在模拟训练，然后做少量现实世界微调。模拟提供好初始化，现实世界数据校正模拟器特定偏置。这比从头训练需要少得多的现实世界数据。

## 机器人世界模型

- 上面所有 RL 和模仿学习方法都是**无模型**：策略通过直接交互（或演示）学习行动而无需显式建模世界如何工作。替代是**基于模型**学习：先学习环境的动力学模型，然后用该模型规划或生成合成经验。

- **世界模型**学习转移函数 $p(s_{t+1} \mid s_t, a_t)$：给定当前状态和行动，预测下一状态（如第 10 章介绍）。在机器人学中，这意味着预测如果机器人采取特定行动将发生什么："如果我推这个块向左，它将滑动 3cm 且它后面的杯子将 topple。"

- 吸引力是**样本效率**。现实世界机器人交互昂贵。如果机器人可以从适度现实数据学习世界模型，它可以通过展开模型在其"头脑"中"想象"数千轨迹，规划和精炼其策略而无需接触物理世界。这类似国际象棋棋手通过心理上模拟移动向前思考。

- **DreamerV3** 是通用基于模型 RL 代理。它联合学习三个组件：
    - **表示模型**将观察编码为紧凑潜在状态。
    - **转移模型**（世界模型）预测给定当前状态和行动的下一潜在状态。
    - **奖励模型**从潜在状态预测奖励。

- 代理然后通过在潜在空间展开转移模型"梦想"数千步，在这些想象轨迹上训练策略，并转移策略到真实环境。关键创新是所有想象发生在潜在空间（紧凑学习表示），非像素空间，使计算可行。

$$\hat{s}_{t+1} = f_\theta(s_t, a_t), \quad \hat{r}_t = g_\theta(s_t)$$

- 转移模型 $f_\theta$ 和奖励模型 $g_\theta$ 在真实经验上训练，策略在想象展开上训练。这解耦数据收集和策略优化。

- 对机器人操作，世界模型启用**心理排练**。在尝试抓取前，机器人可以在其学习模型中模拟几种方法并选择最可能成功的。这对接触丰富任务特别有价值其中现实世界试错慢且风险大。

- 世界模型也自然连接到**模拟到真实**：在真实数据上训练的世界模型有效是学习模拟器，自动捕获真实世界物理，完全绕过模拟到真实差距。模型可能比手工构建模拟器对良好理解场景不太准确，但它捕获效应（摩擦、变形、接触动力学）手工构建模拟器经常弄错。

- **JEPA**（联合嵌入预测架构，第 10 章介绍）提供像素级预测的替代。JEPA 不是预测精确未来观察，而是在嵌入空间预测："下一状态的潜在表示将接近这个向量。"这避免预测像素完美未来的困难（这既不必要也计算浪费）并聚焦预测对决策重要的未来方面。

- 世界模型的限制是**复合预测误差**。转移模型中的小不精确在长展开上累积，导致想象轨迹偏离现实。缓解包括短想象范围、集成模型（用不确定性检测预测何时变得不可靠）和定期用新鲜现实世界数据基础模型。

## 操作

- **操作**是使用机器人末端执行器与物体交互的艺术：拾取、放置、推、插入、组装。

- **抓取**是基础操作技能。目标是找稳定抓取位姿：将安全握住物体的夹爪位置和方向。

- **分析抓取规划**使用物理。抓取是稳定的如果接触力可以抵抗外部扳手（力和扭矩）。对平行夹爪夹，最简单标准是**力封闭**条件：接触法线必须跨越所有力方向，所以抓取可以抵抗任何干扰。这涉及检查抓取扳手矩阵的秩，第 2 章秩概念的直接应用。

- **数据驱动抓取**学习从感觉输入预测抓取成功。给定桌上物体的深度图像，网络预测每个候选夹爪位姿的抓取质量分数。**GraspNet** 和类似架构用点云编码器（PointNet 风格，第 8 章）预测 6-DoF 抓取位姿（位置 + 方向）带置信分数。

- **灵巧操作**超越简单拾取和放置。多指手有 20+ DoF 可以执行任务如手中旋转（手指间转笔）、工具使用和精细组装。状态空间巨大且接触复杂，使这成为机器人学中最难问题之一。

- 学习灵巧操作经常用强化学习（第 6 章）在模拟中带重域随机化。OpenAI 在 Rubik 魔方求解上用 Shadow 手训练 PPO 策略在模拟中带随机化物理，实现转移到真实机器人手。

- **接触丰富任务**如销入孔插入或擦拭表面需要机器人维持与环境的受控接触。这些任务需要力传感和柔顺控制（阻抗控制），它们难以准确模拟因为接触物理 notoriously 难建模。

## 移动

- 移动是移动机器人身体通过世界：行走、跑步、攀爬、游泳。与操作的关键区别是机器人必须维持平衡同时移动，与地面的接触点随时间变化。

- **腿式移动**挑战因为它固有不稳定。双足机器人（人形）在步期间单腿站立像倒立摆。质心必须保持在支撑多边形（与地面接触的脚的凸包）上，或机器人跌倒。

- **零力矩点** (ZMP) 是地面上净扭矩为零的点从重力和惯性力。如果 ZMP 保持在支撑多边形内，机器人不会翻倒。传统人形控制器（如本田 ASIMO）规划保持 ZMP 在界限内的轨迹。

- **中枢模式发生器** (CPG) 是基于振荡器的控制器灵感来自生物学。动物用脊髓中的神经回路生成节律移动模式（行走、小跑、飞奔），无需持续大脑参与。CPG 模型用耦合微分方程：

$$\dot{\phi}_i = \omega_i + \sum_j w_{ij} \sin(\phi_j - \phi_i - \psi_{ij})$$

- 其中 $\phi_i$ 是振荡器 $i$ 的相位，$\omega_i$ 是自然频率，$w_{ij}$ 是耦合强度，$\psi_{ij}$ 是期望相位偏移。不同相位关系产生不同步态：所有腿同步（跳跃）、交替对（小跑）、顺序（行走）。正弦耦合自然同步振荡器，类似傅里叶级数（第 3 章）将运动分解为频率分量。

- **移动强化学习**已成为敏捷四足和人形机器人的主导方法。机器人通过模拟中的试错学习策略 $\pi(\mathbf{a} \mid \mathbf{o})$（第 6 章），带前向速度、稳定性和能源效率的奖励，和跌倒、关节限制违反和急动运动的惩罚。

- 最近工作（如 Agility Robotics、Boston Dynamics 和学术实验室）的关键洞察是 RL 训练移动策略比手工设计控制器鲁棒得多。它们自然学习从推挤恢复、适应地形变化、处理工程师未预料的情况。训练通常用 PPO（第 6 章）带域随机化。

- **四足机器人**（如 Boston Dynamics Spot 或 Unitree Go2）已成为腿式机器人的主力。四条腿提供固有稳定性（三脚架三腿总可以支持身体同时一条腿移动）。四足 RL 策略实现令人印象深刻结果：3+ m/s 奔跑、爬楼梯、导航岩石地形、从踢恢复。

- **人形移动**更难因为双足有更小支撑多边形和更高质心。最近进展（Tesla Optimus、Figure、Unitree H1）用 RL 在模拟中训练带仔细奖励塑造。人形必须学习不仅行走还协调手臂摆动平衡、导航不平表面、从扰动恢复。

## 机器人学习中的安全

- 随机探索学习的机器人（如在 RL 中）可能损坏自己、其环境或附近人类。**安全机器人学习**约束探索避免灾难结果。

- **约束 RL** 添加安全约束到 MDP（第 6 章）。目标变成：最大化奖励服从 $J_c(\pi) \leq d$，其中 $J_c$ 是期望累积成本（例如，碰撞事件），$d$ 是最大允许成本。算法如约束策略优化 (CPO) 扩展 PPO 处理这些约束。

- **安全包络**定义机器人必须永不跨越的硬边界，无论学习策略说什么。安全控制器监控机器人状态并在约束即将违反时覆盖学习策略（例如，接近关节限制、在人类附近移动太快、超过力阈值）。这是分层架构：学习算法处理性能，安全层处理约束。

- **风险感知规划**显式建模环境和机器人自己状态估计的不确定性。不是为最可能结果规划，它为置信边界内最坏情况规划。这连接到条件数概念（第 2 章）：良好条件系统对扰动鲁棒，风险感知规划寻求在扰动下保持安全的控制策略。

1. 为简单 2 连杆平面机器人手臂实现正向运动学。计算并可视化不同关节角度的末端执行器位置。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

def forward_kinematics(q1, q2, l1=1.0, l2=0.8):
    """计算 2 连杆手臂的关节和末端执行器位置。"""
    x1 = l1 * jnp.cos(q1)
    y1 = l1 * jnp.sin(q1)
    x2 = x1 + l2 * jnp.cos(q1 + q2)
    y2 = y1 + l2 * jnp.sin(q1 + q2)
    return jnp.array([0, x1, x2]), jnp.array([0, y1, y2])

fig, ax = plt.subplots(figsize=(6, 6))
configs = [(0.5, 0.3), (1.0, -0.5), (1.5, 1.0), (2.0, -1.5)]
colors = ["#e74c3c", "#3498db", "#27ae60", "#9b59b6"]

for (q1, q2), c in zip(configs, colors):
    xs, ys = forward_kinematics(q1, q2)
    ax.plot(xs, ys, "o-", color=c, linewidth=2, markersize=6,
            label=f"q=({q1:.1f}, {q2:.1f})")

ax.set_xlim(-2, 2); ax.set_ylim(-2, 2)
ax.set_aspect("equal"); ax.grid(True); ax.legend()
ax.set_title("2 连杆机器人手臂：正向运动学")
plt.show()
```

2. 用雅可比伪逆实现逆向运动学。从随机配置开始并迭代移动末端执行器到目标。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

l1, l2 = 1.0, 0.8

def end_effector(q):
    x = l1 * jnp.cos(q[0]) + l2 * jnp.cos(q[0] + q[1])
    y = l1 * jnp.sin(q[0]) + l2 * jnp.sin(q[0] + q[1])
    return jnp.array([x, y])

jacobian_fn = jax.jacobian(end_effector)

target = jnp.array([0.5, 1.2])
q = jnp.array([0.1, 0.1])
trajectory = [end_effector(q)]

for _ in range(50):
    pos = end_effector(q)
    error = target - pos
    if jnp.linalg.norm(error) < 1e-4:
        break
    J = jacobian_fn(q)
    # 阻尼伪逆处理接近奇点
    dq = J.T @ jnp.linalg.solve(J @ J.T + 0.01 * jnp.eye(2), error)
    q = q + dq
    trajectory.append(end_effector(q))

traj = jnp.stack(trajectory)
plt.plot(traj[:, 0], traj[:, 1], "b.-", label="末端执行器路径")
plt.plot(*target, "r*", markersize=15, label="目标")
plt.gca().set_aspect("equal"); plt.grid(True); plt.legend()
plt.title(f"IK 在 {len(trajectory)-1} 步收敛")
plt.show()
```

3. 模拟简单 PID 控制器跟踪期望关节轨迹。观察调优增益的效果。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

# 期望轨迹：平滑正弦运动
dt = 0.01
t = jnp.arange(0, 5, dt)
q_desired = jnp.sin(2 * t)

# 模拟二阶动力学：m * q_ddot + b * q_dot = tau
m, b_damp = 1.0, 0.5

for Kp, Kd, Ki, label in [(10, 5, 0, "PD 仅"), (10, 5, 2, "PID"), (50, 10, 2, "激进 PID")]:
    q, q_dot, integral = 0.0, 0.0, 0.0
    qs = []
    for i in range(len(t)):
        error = q_desired[i] - q
        integral += error * dt
        d_error = -q_dot  # 误差导数（期望速度已知但简化）
        tau = Kp * error + Kd * d_error + Ki * integral
        q_ddot = (tau - b_damp * q_dot) / m
        q_dot += q_ddot * dt
        q += q_dot * dt
        qs.append(float(q))

    plt.plot(t, qs, label=label)

plt.plot(t, q_desired, "k--", label="期望", linewidth=2)
plt.xlabel("时间 (s)"); plt.ylabel("关节角度")
plt.legend(); plt.title("PID 控制器跟踪")
plt.show()
```
