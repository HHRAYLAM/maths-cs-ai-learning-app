### 2. 对比学习（Contrastive Learning）

**CLIP 的核心思想**：

想象 Tom 在玩**配对游戏**：

```
正确配对 ✅（奖励奶酪🧀）:
  [奶酪图片] ←→ "这是一块瑞士奶酪"
  [猫的图片] ←→ "这是一只可爱的猫"
  [Tom 追 Jerry] ←→ "猫在追逐老鼠"

错误配对 ❌（没有奶酪）:
  [奶酪图片] ←→ "这是一辆汽车"
  [猫的图片] ←→ "这是一个香蕉"
  [Tom 追 Jerry] ←→ "太阳在天上"
```

**技术原理**：
- 拉近匹配的图像 - 文本对
- 推远不匹配的图像 - 文本对
- 学习一个共享的嵌入空间

```python
# CLIP 风格的对比损失
def contrastive_loss(image_features, text_features, temperature=0.07):
    # 计算相似度矩阵
    logits = (image_features @ text_features.T) / temperature
    
    # 对称交叉熵损失
    loss_i = cross_entropy(logits, range(N))  # 图像→文本
    loss_t = cross_entropy(logits.T, range(N))  # 文本→图像
    
    return (loss_i + loss_t) / 2
```

## 🧪 多模态融合策略

### 1. 早期融合（Early Fusion）

```
[图像] → 编码器 ↓
                    [融合] → 联合模型 → 输出
[文本] → 编码器 ↓
```

**Tom 比喻**：边看边听边做决定

### 2. 晚期融合（Late Fusion）

```
[图像] → 编码器 → 决策 ↘
                         [投票] → 最终输出
[文本] → 编码器 → 决策 ↗
```

**Tom 比喻**：先用眼睛判断，再用鼻子判断，最后综合结论

### 3. 混合融合（Hybrid Fusion）

```
[图像] → 编码器 → 早期融合 → 联合表示 ↘
                                          [决策] → 输出
[文本] → 编码器 → 晚期融合 ← 单独表示 ↗
```

**Tom 比喻**：灵活使用多种策略

## 🎯 核心概念总结

### 多模态表示的核心挑战

1. **表示对齐（Representation Alignment）**
   - 问题：如何让图像和文本在同一个"语言空间"对话？
   - 解决：对比学习、共享嵌入空间

2. **模态缺失处理（Modality Dropout）**
   - 问题：如果只有图像没有文本描述怎么办？
   - 解决：训练时使用模态 dropout，增强鲁棒性

3. **融合时机（Fusion Timing）**
   - 问题：早期融合 vs 晚期融合哪个更好？
   - 解决：取决于任务，实验验证
