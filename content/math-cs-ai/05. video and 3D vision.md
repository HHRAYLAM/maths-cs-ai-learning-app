# 视频和 3D 视觉

*视频和 3D 视觉将图像理解扩展到时间和空间域。本文件涵盖光流、视频分类（3D CNN、TimeSformer）、物体跟踪（SORT、DeepSORT）、动作识别、深度估计（单目和立体）、点云、NeRF 和 3D 高斯溅射。*

- 文件 01-04 将图像视为孤立快照。但视觉世界是连续的：物体移动、场景变化、深度存在。本文件将计算机视觉扩展到时间域（视频）和空间域（3D），涵盖模型如何理解运动、跟踪物体、估计深度和重建场景。

- 视频是随时间捕获的图像（帧）序列。每秒 30 帧，10 秒片段包含 300 帧。关键挑战是建模时间维度：物体如何移动、场景如何演变，以及我们如何关联跨帧的信息？

- 光流估计两连续帧之间像素的表观运动。对于帧 $t$ 中的每个像素，光流产生 2D 位移向量 $(u, v)$ 指向该像素在帧 $t+1$ 中移动的位置。结果是与图像大小相同的稠密运动场。

![两连续视频帧和它们之间的光流场，可视化为彩色箭头显示像素运动方向和幅度](../images/optical_flow.svg)

- 光流在亮度恒度假设下计算：像素强度在移动时不变化。如果帧 $t$ 中位置 $(x, y)$ 的像素有强度 $I(x, y, t)$ 并在小时间间隔 $\delta t$ 内移动 $(u, v)$：

$$I(x + u\delta t, \, y + v\delta t, \, t + \delta t) = I(x, y, t)$$

- 取一阶泰勒展开（第 03 章）并除以 $\delta t$：

$$I_x u + I_y v + I_t = 0$$

- 其中 $I_x, I_y$ 是空间梯度（Sobel，文件 01），$I_t$ 是时间梯度（连续帧之间的差异）。这是光流约束方程。一个方程，两个未知数 ($u, v$)：我们需要额外约束。

- Lucas-Kanade 假设流动在小窗口内恒定（例如 5x5 像素）。这给出超定系统（25 个方程，2 个未知数），用最小二乘法求解（第 06 章的法方程）：

```math
\begin{bmatrix} u \\ v \end{bmatrix} = \begin{bmatrix} \sum I_x^2 & \sum I_x I_y \\ \sum I_x I_y & \sum I_y^2 \end{bmatrix}^{-1} \begin{bmatrix} -\sum I_x I_t \\ -\sum I_y I_t \end{bmatrix}
```

- 2x2 矩阵是文件 01 中的结构张量（与 Harris 角点检测使用的矩阵相同）。Lucas-Kanade 对小运动效果良好，但当物体在帧间移动超过几个像素时失败。

- Farneback 方法将多项式展开拟合到每个像素的邻域，并估计最能解释帧间变化的位移场。它产生稠密流动（每个像素一个向量）并处理比 Lucas-Kanade 更大的运动。

- 现代深度学习光流方法（FlowNet、RAFT）学习从帧对端到端预测光流。RAFT（循环全对场变换，Teed 和 Deng，2020）计算两帧中所有像素对之间的 4D 相关体积，并使用基于 GRU 的更新算子迭代细化流动估计。RAFT 实现最先进的精度，已成为标准流动骨干。

- 双流网络（Simonyan 和 Zisserman，2014）是视频理解的早期方法。一个流处理单个 RGB 帧（外观），另一个处理光流帧栈（运动）。两个流在最后融合（通过平均或连接）。这个架构显式分离"事物看起来如何"和"它们如何移动"。

- 3D 卷积网络将 2D 卷积扩展到时间维度。3D 卷积应用大小为 $k \times k \times k_t$ 的滤波器，跨越空间和时间维度，直接学习 spatiotemporal 特征。

- C3D（Tran 等人，2015）堆叠 3x3x3 滤波器的 3D 卷积，表明时间卷积可以学习运动特征，无需显式光流。成本很高：3D 卷积比 2D 对应物多 $k_t$ 倍参数和计算。

- I3D（膨胀 3D，Carreira 和 Zisserman，2017）采用更实用方法：从预训练 2D CNN（如 Inception 或 ResNet）开始，通过沿时间维度重复权重并除以 $k_t$ 将所有 2D 滤波器"膨胀"到 3D。这将 ImageNet 预训练迁移到视频，同时添加时间建模。2D $k \times k$ 滤波器变成 $k \times k \times k_t$ 滤波器，初始化为 $W_{\text{3D}}[:,:,j] = W_{\text{2D}} / k_t$ 对于所有时间位置 $j$。

- SlowFast 网络（Feichtenhofer 等人，2019）使用两个以不同时间分辨率操作的并行通路：
    - Slow 通路以低帧率（例如每 16 帧）处理帧，具有高空间分辨率和许多通道，捕捉精细空间细节。
    - Fast 通路以高帧率（每 2 帧）处理帧，具有降低的空间分辨率和更少通道（通常是 Slow 通路的 $1/8$），捕捉快速时间变化。
    - 侧向连接通过步幅卷积将信息从 Fast 融合到 Slow。

- 洞察是空间和时间信息有不同带宽需求：物体外观变化缓慢，但运动可以快速。SlowFast 通过设计匹配这种不对称性。

- TimeSformer（Bertasius 等人，2021）将 Vision Transformer 应用于视频。它将完整 spatiotemporal 注意力（这将非常昂贵：$T$ 帧和每帧 $N$ 块的 $O((T \times N)^2)$）分解为分割注意力：每个块在时间注意力（每个块在同一空间位置跨时间关注）和空间注意力（每个块在同一帧内跨空间关注）之间交替。这将成本从 $O(T^2 N^2)$ 降低到 $O(T^2 + N^2)$。

- VideoMAE（Tong 等人，2022）将掩码自编码器思想（文件 04）扩展到视频。使用极高掩码率（90-95%），因为视频有高时间冗余：邻近帧看起来几乎相同，所以掩码大多数块仍留下足够信息用于重建。VideoMAE 在无标签视频上预训练 ViT 骨干并迁移到下游任务。

- 动作识别将视频片段分类到许多动作类别之一（例如"跑步"、"烹饪"、"弹吉他"）。它是图像分类的视频类比。标准基准包括 Kinetics-400（400 个动作类别，约 30 万片段）、Something-Something（174 个需要时间推理的细粒度动作）和 ActivityNet（200 个类别，长未剪辑视频）。

- 时间动作检测超越分类：给定长未剪辑视频，找到每个动作的开始时间、结束时间和类别。这是物体检测的时间类比。像 ActionFormer 这样的方法使用 Transformer 处理时间特征并预测动作边界。

- 视频物体跟踪在第一帧识别后跨帧跟踪特定物体。

- SORT（简单在线实时跟踪，Bewley 等人，2016）结合检测模型（独立检测每帧中的物体）与卡尔曼滤波用于运动预测和匈牙利算法用于分配。

- 卡尔曼滤波维护每个跟踪物体的状态估计（位置、速度、大小），并使用线性运动模型预测它将在下一帧的位置。当新检测到达时，卡尔曼滤波通过组合预测和观察来更新其估计，根据它们各自的不确定性加权。这是应用于跟踪的贝叶斯更新（第 05 章）。

- 匈牙利算法解决双线性分配问题：给定 $M$ 个跟踪物体和 $N$ 个新检测，找到最小化总成本的最优一对一匹配（使用来自文件 03 的 IoU 距离）。不匹配的检测开始新轨道；不匹配的轨道在宽限期后终止。

- DeepSORT 通过添加深度外观特征扩展 SORT：每个检测到的物体通过小型 CNN 产生外观嵌入（描述符向量）。匹配成本结合嵌入空间中的 IoU 距离和余弦距离（第 01 章）。这处理遮挡和重识别：即使物体在其他物体后面消失几帧，其外观嵌入允许在重新出现时重新匹配。

- ByteTrack（Zhang 等人，2022）通过使用每个检测（包括低置信度检测）改进跟踪。大多数跟踪器丢弃低于置信度阈值的检测。ByteTrack 首先将高置信度检测匹配到现有轨道，然后将剩余低置信度检测匹配到不匹配轨道。这恢复暂时遮挡或模糊（因此检测置信度低）的物体。

- 3D 视觉恢复 2D 图像投影中丢失的第三空间维度（文件 01）。

- 深度估计预测从相机到场景中每个点的距离。

- 立体深度使用两个相机，由已知基线 $b$ 分隔。相同点在左右图像中出现在不同水平位置（这个偏移称为视差 $d$）。深度与视差成反比：

$$Z = \frac{f \cdot b}{d}$$

- 其中 $f$ 是焦距，$b$ 是基线距离。计算视差需要在两图像之间找到对应点（立体匹配），这是沿水平扫描线的 1D 搜索（因为相机水平对齐，3D 中相同高度的点在两图像中投影到相同行）。

- 单目深度估计从单个图像预测深度，这根本上是不适定的（无限多 3D 场景可以产生相同 2D 图像）。但人类使用相对大小、纹理梯度、遮挡和大气雾等线索轻松做到。深度网络从训练数据学习这些线索。

- 像MiDaS和Depth Anything这样的模型从单个图像预测相对深度图（排名哪个物体更近）。它们在多样化数据集上用尺度不变损失训练，尽管理论歧义但产生惊人准确的结果。

- 点云是 3D 点 $(x, y, z)$ 集合，可选带颜色或其他属性，由 LiDAR 传感器或立体重建捕获。与图像不同，点云是无序且不规则间隔的。

- PointNet（Qi 等人，2017）通过独立对每个点应用共享 MLP，然后用最大池化聚合（这是置换不变的，解决排序问题）直接处理点云。PointNet++ 添加分层分组以在多个尺度捕捉局部结构。

- 神经辐射场（NeRF）（Mildenhall 等人，2020）将 3D 场景表示为连续函数，将 3D 位置 $(x, y, z)$ 和观察方向 $(\theta, \phi)$ 映射到颜色 $(r, g, b)$ 和密度 $\sigma$。这个函数由 MLP 参数化：

$$F_\theta: (x, y, z, \theta, \phi) \to (r, g, b, \sigma)$$

- 渲染像素时，光线从相机穿过该像素投射到场景中。沿光线采样点，MLP 预测每个点的颜色和密度。像素颜色通过体积渲染计算：沿光线积分颜色，按密度加权：

$$C(\mathbf{r}) = \int_{t_n}^{t_f} T(t) \cdot \sigma(\mathbf{r}(t)) \cdot \mathbf{c}(\mathbf{r}(t), \mathbf{d}) \, dt$$

- 其中 $T(t) = \exp(-\int_{t_n}^{t} \sigma(\mathbf{r}(s)) \, ds)$ 是累积透射率（多少光已被吸收）。实践中，这个积分通过沿光线采样 $N$ 个点并求和来近似：

$$\hat{C} = \sum_{i=1}^{N} T_i \cdot (1 - \exp(-\sigma_i \delta_i)) \cdot c_i$$

- NeRF 通过最小化渲染像素和来自一组定位照片的真实像素之间的 MSE 来训练。训练后，NeRF 可以从任何相机位置渲染真实感新视角。限制是速度：渲染需要评估 MLP 数百万次（每像素每采样点一次），使实时渲染困难。

- 3D 高斯溅射（Kerbl 等人，2023）通过将场景表示为 3D 高斯原语集合而非连续体积函数来解决 NeRF 的速度限制。每个高斯有 3D 位置（均值）、3D 协方差矩阵（控制形状和方向）、不透明度和颜色（表示为球谐函数用于视角依赖效果）。

- 渲染将每个 3D 高斯投影到图像平面（产生 2D 高斯"溅射"），按深度排序，并使用 alpha 混合从前到后合成。这是在 GPU 上实时运行（100+ FPS）的光栅化过程，比 NeRF 的光线追踪快几个数量级。高斯溅射匹配或超过 NeRF 质量，同时实现实时渲染。

- SLAM（同时定位与建图）是在同时跟踪相机在环境内位置时构建未知环境地图的问题。它是机器人、自动驾驶和 AR 的基础。

- 视觉里程计通过跨图像跟踪特征估计逐帧相机运动。特征点（SIFT、ORB，来自文件 01）在连续帧之间匹配，相机的旋转和平移使用本质矩阵（编码两视图之间的几何关系，来自文件 01 的内参和外参导出）从对应关系估计。

- 基于特征的 SLAM 通过维护持久地图扩展视觉里程计。ORB-SLAM（Mur-Artal 等人，2015）是最广泛使用的基于特征的 SLAM 系统。它有三个并行线程：
    1. 跟踪：将新帧中的 ORB 特征匹配到地图，使用 PnP（透视 -n- 点）和 RANSAC 估计相机姿态
    2. 局部建图：从匹配特征三角化新地图点，使用光束平差（最小化看到每个点的所有视图的重投影误差）优化它们的位置
    3. 闭环检测：检测相机何时重新访问先前建图区域（使用视觉词袋），然后通过全局优化地图校正累积漂移

- LiDAR SLAM 使用 LiDAR 传感器的 3D 点云，而非（或除了）相机图像。LiDAR 提供直接深度测量，使几何估计更鲁棒，但硬件成本更高。像 LOAM（LiDAR 里程计和建图）这样的方法使用迭代最近点（ICP）对齐注册连续扫描之间的点云。

- 视觉 - 惯性 SLAM 融合相机数据与 IMU（加速度计 + 陀螺仪）的测量。IMU 提供高频旋转和加速度估计，桥接相机帧之间的间隙并处理快速运动或暂时视觉特征丢失。

- VR/AR 应用是计算机视觉最苛刻的消费者之一。

- 姿态估计从图像确定人体（或脸、或手）的位置和方向。身体姿态通常表示为一组 2D 或 3D 关键点位置（关节：肩膀、肘部、手腕、臀部、膝盖、脚踝）。像OpenPose和MediaPipe这样的模型使用热图回归预测这些关键点：对于每个关节，模型输出热图，峰值指示关节的位置。

- 自上而下方法首先用边界框检测器检测人（文件 03），然后估计每个框内的姿态。自下而上方法首先检测图像中的所有关键点，然后使用部分亲和字段（编码连接关节之间关联的向量场）将它们分组到个体。

- 场景重建从传感器数据构建环境的 3D 模型。在 AR 中，这支持将虚拟物体放置在真实表面上，在真实物体后面遮挡虚拟物体，并投射虚拟阴影。实时场景重建方法（如 ARKit 和 ARCore 中的深度传感器系统）构建环境的稀疏网格，随用户移动更新。

- VR 中的实时渲染约束极其严格：双眼需要 90+ FPS 的单独渲染（避免晕动症），从头部移动 to 显示更新的延迟低于 20 毫秒。像注视点渲染（仅用高分辨率渲染用户看的地方，使用眼动追踪）和重投影（根据新头部姿态扭曲前一帧以在下一帧渲染时填补空白）等技术对满足这些约束至关重要。

- 实时神经渲染（3D 高斯溅射）、鲁棒跟踪（视觉 - 惯性 SLAM）和高效姿态估计的融合使真实感、交互式 AR/VR 体验越来越可行。

1. 从头实现 Lucas-Kanade 光流算法。在正方形向右移动的两个合成帧之间计算流动。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

def lucas_kanade(frame1, frame2, window_size=5):
    """Lucas-Kanade optical flow."""
    # Compute gradients
    Ix = jnp.zeros_like(frame1)
    Iy = jnp.zeros_like(frame1)
    It = frame2 - frame1

    # Sobel-like gradients
    Ix = Ix.at[1:-1, :].set((frame1[2:, :] - frame1[:-2, :]) / 2)
    Iy = Iy.at[:, 1:-1].set((frame1[:, 2:] - frame1[:, :-2]) / 2)

    H, W = frame1.shape
    half_w = window_size // 2
    u = jnp.zeros_like(frame1)
    v = jnp.zeros_like(frame1)

    for i in range(half_w, H - half_w):
        for j in range(half_w, W - half_w):
            Ix_win = Ix[i-half_w:i+half_w+1, j-half_w:j+half_w+1].ravel()
            Iy_win = Iy[i-half_w:i+half_w+1, j-half_w:j+half_w+1].ravel()
            It_win = It[i-half_w:i+half_w+1, j-half_w:j+half_w+1].ravel()

            A = jnp.stack([Ix_win, Iy_win], axis=1)
            ATA = A.T @ A
            ATb = -A.T @ It_win

            # Check if the system is well-conditioned
            det = ATA[0,0] * ATA[1,1] - ATA[0,1] * ATA[1,0]
            if jnp.abs(det) > 1e-6:
                flow = jnp.linalg.solve(ATA, ATb)
                u = u.at[i, j].set(flow[0])
                v = v.at[i, j].set(flow[1])

    return u, v

# Create two frames: a white square that moves right
frame1 = jnp.zeros((64, 64))
frame1 = frame1.at[20:40, 15:35].set(1.0)

frame2 = jnp.zeros((64, 64))
frame2 = frame2.at[20:40, 20:40].set(1.0)  # shifted 5 pixels right

u, v = lucas_kanade(frame1, frame2, window_size=7)

# Visualise
fig, axes = plt.subplots(1, 3, figsize=(14, 4))
axes[0].imshow(frame1, cmap='gray'); axes[0].set_title('Frame 1'); axes[0].axis('off')
axes[1].imshow(frame2, cmap='gray'); axes[1].set_title('Frame 2'); axes[1].axis('off')

# Quiver plot of flow (subsample for clarity)
step = 4
Y, X = jnp.mgrid[0:64:step, 0:64:step]
axes[2].imshow(frame1, cmap='gray', alpha=0.5)
axes[2].quiver(X, Y, u[::step, ::step], v[::step, ::step],
               color='#e74c3c', scale=50, width=0.005)
axes[2].set_title('Optical Flow'); axes[2].axis('off')

plt.tight_layout(); plt.show()

# Check average flow in the moving region
region_u = u[20:40, 15:35]
print(f"Average horizontal flow in object region: {region_u[region_u != 0].mean():.2f} pixels")
```

2. 实现简单卡尔曼滤波用于 2D 物体跟踪。模拟噪声轨迹并显示卡尔曼滤波如何平滑估计。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def kalman_predict(x, P, F, Q):
    """Kalman filter prediction step."""
    x_pred = F @ x
    P_pred = F @ P @ F.T + Q
    return x_pred, P_pred

def kalman_update(x_pred, P_pred, z, H, R):
    """Kalman filter update step."""
    y = z - H @ x_pred                        # innovation
    S = H @ P_pred @ H.T + R                  # innovation covariance
    K = P_pred @ H.T @ jnp.linalg.inv(S)      # Kalman gain
    x_updated = x_pred + K @ y
    P_updated = (jnp.eye(len(x_pred)) - K @ H) @ P_pred
    return x_updated, P_updated

# State: [x, y, vx, vy]
dt = 1.0
F = jnp.array([[1, 0, dt, 0],    # state transition
                [0, 1, 0, dt],
                [0, 0, 1, 0],
                [0, 0, 0, 1]])
H = jnp.array([[1, 0, 0, 0],     # observation: we measure x, y
                [0, 1, 0, 0]])
Q = jnp.eye(4) * 0.01            # process noise
R = jnp.eye(2) * 4.0             # measurement noise (noisy detector)

# Simulate ground truth: circular motion
n_steps = 50
t = jnp.linspace(0, 2 * jnp.pi, n_steps)
true_x = 10 * jnp.cos(t) + 20
true_y = 10 * jnp.sin(t) + 20

# Noisy observations
key = jax.random.PRNGKey(42)
noise = jax.random.normal(key, (n_steps, 2)) * 2.0
obs_x = true_x + noise[:, 0]
obs_y = true_y + noise[:, 1]

# Run Kalman filter
x = jnp.array([obs_x[0], obs_y[0], 0.0, 0.0])  # initial state
P = jnp.eye(4) * 10.0                             # initial uncertainty

kalman_x, kalman_y = [], []
for i in range(n_steps):
    x, P = kalman_predict(x, P, F, Q)
    z = jnp.array([obs_x[i], obs_y[i]])
    x, P = kalman_update(x, P, z, H, R)
    kalman_x.append(x[0])
    kalman_y.append(x[1])

kalman_x = jnp.array(kalman_x)
kalman_y = jnp.array(kalman_y)

# Visualise
plt.figure(figsize=(8, 8))
plt.plot(true_x, true_y, 'k-', linewidth=2, label='Ground Truth')
plt.scatter(obs_x, obs_y, c='#e74c3c', s=20, alpha=0.5, label='Noisy Observations')
plt.plot(kalman_x, kalman_y, '#3498db', linewidth=2, label='Kalman Filter')
plt.legend(); plt.grid(alpha=0.3)
plt.title('Kalman Filter Tracking')
plt.xlabel('x'); plt.ylabel('y')
plt.axis('equal'); plt.show()

obs_error = jnp.mean(jnp.sqrt((obs_x - true_x)2 + (obs_y - true_y)2))
kalman_error = jnp.mean(jnp.sqrt((kalman_x - true_x)2 + (kalman_y - true_y)2))
print(f"Observation RMSE: {obs_error:.2f}")
print(f"Kalman filter RMSE: {kalman_error:.2f}")
print(f"Error reduction: {(1 - kalman_error/obs_error) * 100:.1f}%")
```

3. 实现简化 NeRF 风格体积渲染管道。通过简单 3D 场景（已知颜色和密度的球体）投射光线，并通过沿每条光线积分渲染图像。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def render_ray(origin, direction, spheres, n_samples=64, t_near=1.0, t_far=6.0):
    """Volume render a single ray through a scene of spheres."""
    t_vals = jnp.linspace(t_near, t_far, n_samples)
    deltas = jnp.concatenate([jnp.diff(t_vals), jnp.array([1e-3])])

    colour = jnp.zeros(3)
    transmittance = 1.0

    for i in range(n_samples):
        point = origin + t_vals[i] * direction

        # Compute density and colour at this point
        density = 0.0
        point_colour = jnp.zeros(3)

        for center, radius, col, sigma in spheres:
            dist = jnp.linalg.norm(point - center)
            # Soft sphere: density falls off with distance from surface
            d = jnp.exp(-jnp.maximum(0, dist - radius) * sigma) * sigma
            density += d
            point_colour += d * jnp.array(col)

        # Normalise colour by total density
        point_colour = jnp.where(density > 1e-6, point_colour / density, point_colour)

        # Volume rendering equation
        alpha = 1.0 - jnp.exp(-density * deltas[i])
        colour += transmittance * alpha * point_colour
        transmittance *= (1.0 - alpha)

    return colour

# Scene: three coloured spheres
spheres = [
    (jnp.array([0.0, 0.0, 4.0]), 0.8, [1.0, 0.2, 0.2], 5.0),   # red
    (jnp.array([1.5, 0.5, 5.0]), 0.6, [0.2, 1.0, 0.2], 5.0),   # green
    (jnp.array([-1.0, -0.5, 3.5]), 0.5, [0.2, 0.2, 1.0], 5.0), # blue
]

# Camera setup
img_h, img_w = 64, 64
focal = 60.0
origin = jnp.array([0.0, 0.0, 0.0])

image = jnp.zeros((img_h, img_w, 3))
for i in range(img_h):
    for j in range(img_w):
        # Compute ray direction
        px = (j - img_w / 2) / focal
        py = -(i - img_h / 2) / focal
        direction = jnp.array([px, py, 1.0])
        direction = direction / jnp.linalg.norm(direction)

        colour = render_ray(origin, direction, spheres)
        image = image.at[i, j].set(jnp.clip(colour, 0, 1))

plt.figure(figsize=(6, 6))
plt.imshow(image)
plt.title('NeRF-style Volume Rendering\n(3 spheres)')
plt.axis('off')
plt.tight_layout(); plt.show()
print(f"Image shape: {image.shape}")
print(f"Rendered {img_h * img_w} rays with 64 samples each")
```
