## 📐 梯度 - 多变量函数的导数

**梯度**是标量函数关于所有输入的偏导数组成的向量：

$$\nabla f(x) = \left[ \frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \ldots, \frac{\partial f}{\partial x_n} \right]$$

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 直观理解

| 概念 | 含义 |
|------|------|
| **梯度方向** | 函数增长最快的方向 |
| **负梯度方向** | 函数下降最快的方向 |
| **梯度为零** | 可能到达极值点 |

### 几何意义

```
等高线图：

         ╱───╲
        ╱     ╲
       ╱   ●   ╲  ← 梯度向量（垂直等高线）
      ╱   ╱│    ╲
     ╱───╱ │     ╲
        ╱  ╲
       
梯度总是指向海拔增加最快的方向！
```

</div>
  <div class="lesson-visual-column">

```
梯度的可视化：

         山顶
          ●
         ╱│╲
        ╱ │ ╲  梯度线
       ╱  ↓  ╲
      ╱───────╲
     
梯度 = 最陡上升方向
负梯度 = 最陡下降方向

Tom 要下山 → 跟着负梯度走！
```

## 🚶 梯度下降算法

**梯度下降**通过迭代方式最小化损失函数：

$$\theta_{t+1} = \theta_t - \eta \cdot \nabla_\theta J(\theta_t)$$

- $\theta$ = 模型参数（权重和偏置）
- $\eta$ = **学习率**（步长）
- $\nabla_\theta J$ = 损失函数关于参数的梯度

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的理解</div>
  <div class="story-card-content">
    Tom 下山步骤：
    <br><br>
    1. 感受脚下坡度（计算梯度）
    <br>
    2. 向最陡的方向迈一步（参数更新）
    <br>
    3. 重复直到到达谷底（收敛）
    <br><br>
    <strong>学习率太大</strong>：会错过最低点！
    <br>
    <strong>学习率太小</strong>：要走很久！
  </div>
</div>

---

### 梯度下降的三种变体

| 变体 | 每次更新用多少数据 | 优点 | 缺点 |
|------|------------------|------|------|
| **批量梯度下降** (BGD) | 全部训练集 | 稳定，收敛平滑 | 慢，内存占用大 |
| **随机梯度下降** (SGD) | 1 个样本 | 快，可跳出局部最优 | 波动大，不够稳定 |
| **小批量梯度下降** (Mini-batch) | 一小批样本（如 32、64） | 平衡速度和稳定性 | 需要选择批量大小 |

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 批量梯度下降

```
每次更新：
  计算所有样本的梯度
      ↓
  平均梯度
      ↓
  更新参数
```

**优点**：稳定、可预测

**缺点**：大数据集很慢

</div>
  <div class="lesson-visual-column">

### 随机梯度下降

```
每次更新：
  随机选 1 个样本
      ↓
  计算梯度
      ↓
  更新参数
  
路径：●~●~●~●（波动）
```

</div>
</div>

### 学习率的选择

**学习率 $\eta$** 控制每步走多大：

```
学习率太大 (η=0.5)：
起点 ●
      ↓ 太大步
        ● 冲过头！
      ╲
       ╲________谷底

学习率合适 (η=0.1)：
起点 ●
  ↓
   ●
    ↓
     ● 到达谷底

学习率太小 (η=0.01)：
起点 ●
↓
●
 ↓
●  （走太久！）
```

<div class="key-point">
  <div class="key-point-title">🎯 学习率选择策略</div>
  <ul class="key-point-list">
    <li><strong>常见值</strong>：0.1, 0.01, 0.001, 0.0001</li>
    <li><strong>学习率衰减</strong>：训练过程中逐渐减小</li>
    <li><strong>学习率预热</strong>：开始时小，然后增大，再减小</li>
    <li><strong>自适应学习率</strong>：Adam 等算法自动调整</li>
  </ul>

## 🔙 反向传播 - 神经网络的学习算法

**反向传播**是梯度下降在神经网络中的高效实现！

<div class="story-card">
  <div class="story-card-title">🐱 Tom 的神经网络工厂</div>
  <div class="story-card-content">
    <strong>前向传播</strong>：输入 → 隐藏层 → 输出
    <br><br>
    <strong>反向传播</strong>：计算误差 → 反向传递梯度 → 更新权重
    <br><br>
    核心：<strong>链式法则</strong>！
  </div>
</div>

---

### 反向传播的步骤

```
1. 前向传播：计算每一层的输出
       ↓
2. 计算输出层误差：δ^L = ∂J/∂a^L
       ↓
3. 反向传播误差：δ^l = (W^(l+1))^T δ^(l+1) ⊙ σ'(z^l)
       ↓
4. 计算梯度：∂J/∂W^l = δ^l (a^(l-1))^T
       ↓
5. 更新权重：W^l = W^l - η · ∂J/∂W^l
```

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 链式法则回顾

$$\frac{\partial J}{\partial w} = \frac{\partial J}{\partial a} \cdot \frac{\partial a}{\partial z} \cdot \frac{\partial z}{\partial w}$$

- $J$ = 损失
- $a$ = 激活输出
- $z$ = 加权输入
- $w$ = 权重

### 直观理解

误差像水流一样从输出层**反向流**回输入层，每一层承担相应的责任！

</div>
  <div class="lesson-visual-column">

```
反向传播可视化：

输入层 → 隐藏层 → 输出层
        ↑      ↑
        │      │
    梯度流 ← 误差计算
    
误差从右向左传播，
每层计算自己的梯度！
```

## ⚡ 优化算法 - 更聪明的梯度下降

### 1. Momentum（动量法）

**思想**：像球滚下山，积累动量！

$$v_t = \gamma v_{t-1} + \eta \nabla_\theta J(\theta)$$
$$\theta = \theta - v_t$$

- $\gamma$ = 动量系数（通常 0.9）

```
普通梯度下降：  ● → ● → ● → ● （之字形）
动量法：        ● ━━●━━●━━● （更直更快）
```

**优点**：
- 加速收敛
- 减少震荡

---

### 2. RMSprop

**思想**：对频繁更新的参数使用小学习率，对稀疏更新的参数使用大学习率！

$$E[g^2]_t = 0.9 E[g^2]_{t-1} + 0.1 g_t^2$$
$$\theta = \theta - \frac{\eta}{\sqrt{E[g^2]_t + \epsilon}} g_t$$

**优点**：
- 适合处理稀疏梯度
- 自动调整学习率

---

### 3. Adam（自适应矩估计）

**思想**：动量 + RMSprop 的结合！

| 算法 | 特点 |
|------|------|
| **Momentum** | 加速收敛，减少震荡 |
| **RMSprop** | 自适应学习率 |
| **Adam** | 两者优点结合！ |

**默认参数**：
- $\beta_1 = 0.9$（一阶矩衰减）
- $\beta_2 = 0.999$（二阶矩衰减）
- $\epsilon = 10^{-8}$（数值稳定性）

<div class="key-point">
  <div class="key-point-title">🎯 Adam 是默认选择</div>
  <ul class="key-point-list">
    <li>大多数情况下 Adam 表现最好</li>
    <li>对超参数不敏感，容易调参</li>
    <li>但如果需要更好的泛化，SGD+ 动量可能更好</li>
  </ul>
</div>

## 🛡️ 正则化 - 防止过拟合

**过拟合**：模型记住了训练数据，但无法泛化到新数据！

```
训练准确率：99% ✓
测试准确率：70% ✗  ← 过拟合！
```

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的考试比喻</div>
  <div class="story-card-content">
    <strong>过拟合</strong>：死记硬背答案，换个题就不会
    <br><br>
    <strong>欠拟合</strong>：根本没学会
    <br><br>
    <strong>刚好</strong>：真正理解了概念！
  </div>
</div>

---

### L1 正则化（Lasso）

在损失函数中添加权重的绝对值：

$$J_{\text{new}}(\theta) = J(\theta) + \lambda \sum_{i} |\theta_i|$$

**效果**：
- 倾向于产生**稀疏权重**（很多权重恰好为零）
- 可用于**特征选择**

---

### L2 正则化（Ridge）

在损失函数中添加权重的平方：

$$J_{\text{new}}(\theta) = J(\theta) + \lambda \sum_{i} \theta_i^2$$

**效果**：
- 权重趋向于**小但不为零**
- 更常用，通常效果更好

<div class="lesson-two-column">
  <div class="lesson-text-column">

### L1 vs L2 可视化

```
L1 正则化：
      ●
     ╱│╲
    ╱ │ ╲
   ╱  │  ╲
  ●───●───●  ← 很多权重=0

L2 正则化：
      ●
     ╱│╲
    ╱ │ ╲
   ●──●──●   ← 权重都小但非零
```

### 选择建议

| 场景 | 推荐 |
|------|------|
| 特征选择 | L1 |
| 一般情况 | L2 |
| 不确定 | L2（更安全） |

</div>
  <div class="lesson-visual-column">

```
正则化效果：

无正则化：  ━━━━━━━━ 过拟合
L2 正则化： ━━━━ 刚好
L1 正则化： ━━━ 稀疏解

正则化强度 λ：
太小 → 没效果
太大 → 欠拟合
```

### Dropout - 神经网络的正则化

**思想**：训练时随机"丢弃"一部分神经元！

```
训练时：
层 1: ● ● ● ● ●
       │ │ │ │ │  ← 随机关闭 50%
层 2: ● ○ ● ○ ●

测试时：
层 1: ● ● ● ● ●
       │ │ │ │ │  ← 全部使用
层 2: ● ● ● ● ●  ← 权重乘以 0.5
```

**效果**：
- 防止神经元共适应
- 相当于训练多个网络的集成
- 最常用的深度学习正则化技术

## 📊 学习曲线 - 诊断模型

**学习曲线**显示训练集和验证集的性能随训练的变化：

```
过拟合：
准确率
  ↑    训练集 ━━━━
  │          ╱
  │         ╱
  │        ╱ 验证集 ─ ─ ─
  │       ╱
  └────────────→ 训练轮数

欠拟合：
准确率
  ↑    训练集 ─ ─ ─
  │    验证集 ━━━━
  │    （都很低）
  └────────────→ 训练轮数

刚好：
准确率
  ↑    训练集 ━━━━
  │    验证集 ─ ─ ─
  │    （接近且高）
  └────────────→ 训练轮数
```

<div class="key-point">
  <div class="key-point-title">🎯 诊断与解决方案</div>
  <ul class="key-point-list">
    <li><strong>高偏差（欠拟合）</strong>：增加模型复杂度、减少正则化、训练更久</li>
    <li><strong>高方差（过拟合）</strong>：增加数据、增加正则化、减少模型复杂度</li>
    <li><strong>刚好</strong>：训练集和验证集性能接近且都高</li>
  </ul>
</div>
