# 空间与极端机器人

*空间和极端环境机器人技术将自主性推向极限，通信延迟、辐射和非结构化地形要求机器人能够独立思考。本章涵盖行星探测车、轨道服务、通信受限自主、抗辐射计算、水下机器人、搜救机器人、群体机器人和人机交互。*

- 在本章之前，我们研究了在相对良性环境中运行的自主系统：有车道的道路、有平坦地面的仓库、有已知物体类别的厨房。但机器人技术一些最具影响力的应用是在人类无法进入的地方，或人类存在成本极高的地方：火星表面、深海海底、核灾难现场和燃烧的建筑物。

- 这些极端环境有共同的挑战：通信受限或中断、地形非结构化且不可预测、硬件必须能在恶劣条件下生存、出现问题时没有人类在旁边修理。机器人必须是真正的自主，而不仅仅是"有人类盯着屏幕的自主"。

## 空间机器人

- 太空是终极极端环境。没有空气，温度在 -170°C 到 +120°C 之间波动，辐射轰击电子设备，救援远在数百万公里之外。空间机器人必须非常可靠、节能且自主。

- 行星探测车是探索其他世界表面的移动机器人。NASA 的火星探测车（勇气号、机遇号、好奇号、毅力号）是最著名的例子。每一代都比前一代更自主。

![地球 - 火星通信延迟：单程 4-24 分钟，往返 8-48 分钟，使实时控制成为不可能](../images/earth_mars_delay.svg)

- 根本限制是通信延迟。火星通过无线电距离地球 4-24 分钟（取决于轨道位置），所以往返通信需要 8-48 分钟。探测车不能像摇杆一样实时控制。如果遇到岩石，不能向地球求助并等待回应。它必须自己决定。

- 早期探测车（勇气号、机遇号）严重依赖地面参与规划：人类会研究图像、规划路径、上传指令，探测车执行这些指令。单个驾驶周期需要整整一个火星日（sol）。探测车每个火星日只能行驶 50-100 米。

- 好奇号和毅力号上的 AutoNav（自主导航）大幅提高了自主性。探测车使用立体相机建立局部 3D 地图（回顾第 8 章的立体深度），评估地形可通行性（坡度、粗糙度、岩石大小），使用基于网格的规划器和可通行性成本地图规划安全路径。探测车在人类团队睡觉时自主行驶，将每日行驶距离增加到 100 米以上。

- 火星探测车上的感知流程受到抗辐射处理器的限制，这些处理器比消费级硬件慢几个数量级（下面讨论）。算法必须在计算上节约：使用经典立体匹配而非深度神经网络，使用简单成本地图规划器而非学习策略。

- 轨道服务涉及在轨道上检查、维修、加油或使卫星离轨的机器人。随着太空变得更加拥挤，这是一个不断发展的领域。OSAM-1（NASA）和商业公司（Astroscale、Northrop Grumman MEV）等任务使用机械臂和对接机构为卫星提供服务。

- 挑战是近距离操作：服务航天器必须接近目标卫星（可能正在翻滚、不合作且缺乏对接接口），并在微重力下执行精确操作。基于视觉的位姿估计（从相机图像确定目标的 3D 位置和方向）至关重要。这使用了第 8 章的技术：特征检测、PnP（透视 -n- 点）求解，以及最近的基于深度学习的位姿估计器。

- 卫星检查使用小型航天器目视检查其他卫星是否有损坏或异常。检查器必须自主导航绕目标运行，避免碰撞，并从最佳视角拍摄高分辨率图像。这是一个规划问题：找到覆盖所有检查点的轨迹，同时遵守燃料限制、光照条件和避障。

## 通信限制

- 在太空中，通信受到光速、可用带宽和轨道几何的限制（火星背面的探测车如果没有中继卫星根本无法与地球通信）。

- 这些限制从根本上改变了自主架构。在地球上，机器人可以将高清视频流传输到云服务器，在 GPU 集群上运行推理，并在几毫秒内接收指令。在太空中，机器人必须在机上完成所有事情。

- 高延迟意味着机器人必须在没有实时人类指导的情况下计划和行动。自主软件必须处理正常操作、检测异常并响应危险，而无需等待人类输入。这需要强大的机上状态估计、故障检测和应急规划。

- 有限带宽意味着机器人不能传输原始传感器数据。单张高分辨率图像可能有几兆字节，但火星到地球的数据速率通过直接对地链路只有几千比特每秒（通过轨道中继更高，但仍然有限）。机器人必须积极压缩数据，优先发送哪些数据，并在本地做出大多数决策。

- 通信窗口是间歇性的。火星探测车只能在特定轨道几何期间与地球通信，通常每个火星日通过中继卫星只有几小时。在这些窗口之外，探测车完全靠自己。

- 对 AI 的启示是机上自主必须高度可靠。系统需要检测是否出了问题（轮子卡住、传感器故障、前方地形不可通行），决定安全响应，并继续运行直到下一个通信窗口，届时可以报告并接收更新的指令。

## 抗辐射计算

- 太空充满电离辐射：宇宙射线、太阳粒子事件和行星磁场中的捕获辐射。高能粒子可以翻转内存中的位（单粒子翻转，SEU）、永久损坏晶体管（总电离剂量，TID）或在电路中引起破坏性闩锁。

- 抗辐射处理器设计用于承受这种环境。它们使用更大的晶体管几何、冗余逻辑（三重模块冗余：每个电路的三个副本对输出投票）和专用制造工艺。代价是性能：最先进的抗辐射处理器可能提供 200 MIPS，而消费级 GPU 每秒可执行数十亿次操作。

- RAD750（BAE Systems）为好奇号和许多其他航天器提供动力。它运行在 200 MHz，约 400 MIPS 处理能力，相当于 1990 年代中期的台式电脑。毅力号使用类似级别的处理器。在这样的硬件上运行现代神经网络（数百万参数、数十亿次乘加运算）是不可行的。

- 模型压缩变得至关重要。第 6 章的技术（量化、剪枝、知识蒸馏）用于缩小神经网络以适应极端计算预算。在笔记本电脑 GPU 上几毫秒运行的模型在抗辐射处理器上可能需要几分钟，或者可能根本不适合内存。

- 另一种方法使用商用现成（COTS）处理器，配合软件中的辐射缓解：纠错码、看门狗定时器、定期内存清理和优雅降级策略。一些现代任务使用这种方法来获得更强大的计算能力，但代价是软件复杂性和风险增加。

- 未来的行星任务正在探索 FPGA 和专用 AI 加速器，这些可以耐辐射，同时提供比传统抗辐射 CPU 显著更多的计算能力，可能首次实现机载深度学习。

## 非结构化地形自主导航

- 在地球上，道路是平坦的、标记清晰的、有地图的。在火星、月球或灾难现场，没有道路。地形是非结构化的：岩石、斜坡、沙子、裂缝，以及可能无法支撑机器人重量的表面。

- 地形分类评估每块地面是否安全通行。特征包括来自 3D 重建的坡度、粗糙度（表面法向量的方差）、岩石密度和土壤类型。经典方法从立体深度地图计算这些特征；现代方法在视觉和几何特征上使用学习分类器。

- 视觉 - 惯性里程计（VIO）通过在相机帧之间跟踪视觉特征并与 IMU 测量融合来估计机器人的运动。这是核心 SLAM 组件（第 8 章）适应极端条件。在火星上，VIO 必须处理：无特征的沙地地形（很少可跟踪的视觉特征）、恶劣光照（极端阴影）和有限计算。

- 估计使用扩展卡尔曼滤波（EKF）或因子图优化融合视觉和惯性数据。状态向量包括位置、速度、方向和 IMU 偏差。预测步骤使用 IMU 积分：

$$\mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t)$$

- 其中 $\mathbf{u}_t$ 是 IMU 测量（加速度和角速度）。更新步骤使用视觉特征观测来纠正预测。这是贝叶斯估计（第 5 章）：IMU 提供先验，视觉观测更新信念。

- 危险规避在行星着陆期间至关重要。当航天器向表面下降时，它必须使用机载相机或激光雷达实时识别安全着陆区。NASA 在毅力号上的地形相对导航（TRN）系统将机载相机图像与预载的轨道地图进行比较，以确定下降过程中的位置，然后转向避开危险地形。这使得能够在耶泽罗陨石坑着陆，这是一个科学丰富但地形危险的地点，对于以前的任务来说太冒险了。

## 水下机器人

- 深海与太空一样陌生：在深海底部有极高的压力（1000+ 大气压）、接近零能见度、没有 GPS、通信受限。水下机器人对海洋科学、海上基础设施检查、深海采矿和搜索行动至关重要。

- AUV（自主水下航行器）无缆操作，携带自己的电力和计算。它们遵循预设的测量模式或使用机载智能适应发现。AUV 用于海底测绘、管道检查和环境监测。

- ROV（遥控水下航行器）通过电缆连接到水面船只，电缆提供电力和通信。它们用于需要实时人类控制的任务：深海操作、建造和维修。电缆消除了通信限制，但限制了范围并增加了操作复杂性。

- 声学通信是主要的水下通信方法（无线电波在水中迅速衰减）。声学调制解调器在几公里范围内实现 1-10 kbps 的数据速率，而陆地上的无线电为每秒千兆比特。这甚至比火星通信更受限，迫使 AUV 高度自主。

- 水下 SLAM 特别具有挑战性。声纳提供范围测量，但角分辨率差且噪声显著（海底和海表面的多径反射）。相机只能在很短的距离内工作（清水中几米，浑浊条件下更短）。基于特征的视觉 SLAM（第 8 章）必须适应水下场景的独特视觉特征：颜色衰减（红光首先被吸收）、后向散射和造成亮点和深阴影的人工照明。

- 无 GPS 导航使用航位推算（从多普勒测速仪 DVL 积分速度，DVL 使用声学多普勒频移测量相对于海底的速度），偶尔浮出水面进行 GPS 定位或通过表面换能器进行声学定位。这与仅 IMU 导航相同的漂移问题：小的速度误差在长任务中累积。

## 搜救机器人

- 地震、建筑物倒塌或工业事故后，机器人可以进入对人类救援人员太危险的空间：结构不稳定的建筑物、有毒环境、火灾或狭窄空间。

- 要求是：快速部署（几分钟而不是几小时）、在无 GPS 环境中运行（建筑物内、地下）、通过墙壁和瓦砾的稳健通信，以及在高度杂乱、部分倒塌的空间中导航的能力，这些空间有瓦砾、灰尘和照明不良。

- 多机器人协调在搜救中很有价值，因为一群机器人可以比单个机器人更快地覆盖大片区域。挑战是协调：机器人必须划分搜索区域、避免重复工作并分享发现。

- 基于前沿的探索将机器人分配到已探索和未探索空间之间的边界（"前沿"）。每个机器人导航到最近的未探索前沿、测绘它、然后继续。集中或分布式规划器将前沿分配给机器人以最小化总探索时间。这是一个覆盖优化问题。

- 穿过瓦砾的通信不可靠。机器人可能与操作员和彼此失去联系。系统必须能够应对间歇性通信：每个机器人应该能够独立运行，构建自己的局部地图并做出自己的决定，然后在通信恢复时合并信息。

## 群体机器人

- 群体机器人使用大量简单、低成本的机器人，通过局部交互实现复杂的集体行为。没有一个机器人单独有能力，但群体整体可以执行任何个体都无法完成的任务。

- 灵感来自生物群体：蚂蚁用身体搭建桥梁、蜜蜂对巢址进行集体决策、鱼群通过协调运动躲避捕食者。在每种情况下，简单的局部规则（跟随邻居、避免碰撞、向食物移动）产生复杂的全局行为。

- 去中心化控制意味着没有中央指挥官。每个机器人遵循相同的局部规则，只对其邻居和直接环境做出反应。全局行为从这些局部交互中涌现。这使得群体固有地稳健：如果一个机器人故障，群体继续运行。没有单点故障。

- 共识算法使群体能够通过仅局部通信就集体决定达成一致（例如，向哪个方向移动、优先处理哪个任务）。一个简单的共识协议让每个机器人与其邻居平均值：

$$x_i(t+1) = \frac{1}{|N_i| + 1} \left( x_i(t) + \sum_{j \in N_i} x_j(t) \right)$$

![群体共识：机器人开始时分散，与邻居迭代平均，收敛到共享位置](../images/swarm_consensus.svg)

- 其中 $N_i$ 是机器人 $i$ 的邻居集。迭代直到所有机器人收敛到相同值（全局平均值）。收敛速率取决于通信图的拓扑，特别是其代数连通性（图拉普拉斯的第二小特征值，连接到第 2 章的特征值）。

![雷诺兹的三 flocking 规则：分离避免碰撞、对齐匹配航向、凝聚留在群体中](../images/reynolds_flocking.svg)

- Flocking 算法（雷诺兹规则）用每个机器人的三个简单规则产生协调的群体运动：
    - 分离：远离太近的邻居（避免碰撞）。
    - 对齐：向邻居的平均航向转向（同方向移动）。
    - 凝聚：向邻居的平均位置转向（留在群体中）。

- 每个规则是机器人速度的向量贡献。这些向量的加权和产生自然的 flocking 行为。这是向量的线性组合（第 1 章），其中权重控制每个行为的相对重要性。

- 群体机器人的应用包括环境监测（在大面积上分布传感器）、精准农业（协调无人机进行作物喷洒）、建造（机器人集体组装结构）和搜索行动（高效覆盖大面积）。

## 人机交互

- 大多数现实世界的自主系统与人类一起运行，而不是孤立运行。人类和机器人之间的交互、它们如何通信、共享控制和建立信任，与机器人的技术能力一样重要。

![共享自主谱：从完全人类遥操作（alpha=1）通过混合控制到完全机器人自主（alpha=0）](../images/shared_autonomy_spectrum.svg)

- 共享自主混合人类和机器人控制。共享自主不是完全遥操作（人类控制一切）或完全自主（机器人控制一切），而是让人类提供高级意图，而机器人处理低级执行。例如，人类可能指向一个物体并说"捡起那个"，机器人自主规划抓握和手臂运动。

- 在数学上，共享自主可以建模为人类输入 $\mathbf{u}_h$ 和机器人自主动作 $\mathbf{u}_r$ 的混合：

$$\mathbf{u} = \alpha \mathbf{u}_h + (1 - \alpha) \mathbf{u}_r$$

- 其中 $\alpha \in [0, 1]$ 是混合参数。当 $\alpha = 1$ 时，人类有完全控制（遥操作）。当 $\alpha = 0$ 时，机器人完全自主。自适应共享自主根据情况调整 $\alpha$：机器人有信心时承担更多控制，不确定或情况新颖时让出控制。

- 遥操作对于超出当前自主能力的任务仍然重要。人类操作员远程控制机器人，通过机器人的相机查看场景。挑战是延迟：即使 100 毫秒的延迟也使遥操作困难，太空中的多秒延迟使精细操作几乎不可能。预测显示（显示机器人预测的未来状态）和虚拟夹具（防止操作员命令危险动作的软件引导）有助于补偿。

- 信任校准是确保人类对机器人有适当信任的问题：不能太多（过度信任导致自满和在需要时未能干预），不能太少（信任不足导致不必要的干预和利用不足）。信任应该校准到机器人的实际能力：在它处理好的情况下信任它，在接近其能力边缘的情况下保持怀疑。

- 研究表明，信任受以下因素影响：机器人的透明度（是否解释其决策？）、可靠性（是否可预测地故障还是随机故障？）和通信（是否表达不确定性？）。说"我有 40% 的信心这是安全路径，我应该继续吗？"的机器人比默默向前的机器人能实现更好的人类决策。

- 机器人运动中的可读性意味着机器人以向附近人类传达其意图的方式移动。如果机器人伸手拿物体，其路径应该使在它到达之前就清楚它瞄准哪个物体。这涉及规划最大化观察者早期推断目标能力的轨迹，可以公式化为最大化给定观测部分轨迹的真实目标的后验概率：

$$\pi^* = \arg\max_\pi P(G \mid \xi_{0:t})$$

- 其中 $G$ 是目标，$\xi_{0:t}$ 是迄今为止观测到的轨迹。这使用贝叶斯推断（第 5 章）：观察者对可能的目标有先验，机器人的轨迹提供更新这个信念的证据。

```python
def get_neighbours(positions, k=3):
    dists = jnp.linalg.norm(positions[:, None] - positions[None, :], axis=-1)
    # 对于每个机器人，找到 k 个最近的（排除自己）
    neighbours = jnp.argsort(dists, axis=1)[:, 1:k+1]
    return neighbours

history = [positions.copy()]

for step in range(30):
    neighbours = get_neighbours(positions)
    new_positions = jnp.zeros_like(positions)
    for i in range(n_robots):
        nbr_pos = positions[neighbours[i]]
        new_positions = new_positions.at[i].set(
            (positions[i] + nbr_pos.sum(axis=0)) / (len(neighbours[i]) + 1)
        )
    positions = new_positions
    history.append(positions.copy())

# 绘制收敛
fig, axes = plt.subplots(1, 3, figsize=(15, 4))
for ax, step_idx, title in zip(axes, [0, 10, 29], ["初始", "第 10 步", "最终"]):
    h = history[step_idx]
    ax.scatter(h[:, 0], h[:, 1], s=50)
    ax.set_xlim(-6, 6); ax.set_ylim(-6, 6)
    ax.set_aspect("equal"); ax.grid(True); ax.set_title(title)
plt.suptitle("群体共识：机器人收敛到一致")
plt.tight_layout()
plt.show()
```

2. 实现雷诺兹 flocking 规则（分离、对齐、凝聚）并模拟群体一起移动。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

n = 30
rng = jax.random.PRNGKey(1)
k1, k2 = jax.random.split(rng)
pos = jax.random.uniform(k1, (n, 2), minval=-5, maxval=5)
vel = jax.random.uniform(k2, (n, 2), minval=-0.5, maxval=0.5)

dt = 0.1
separation_radius = 1.0
neighbour_radius = 3.0

trajectories = [pos.copy()]

for _ in range(200):
    new_vel = jnp.zeros_like(vel)
    for i in range(n):
        diffs = pos - pos[i]
        dists = jnp.linalg.norm(diffs, axis=1)

        # 半径内的邻居（排除自己）
        nbr_mask = (dists < neighbour_radius) & (dists > 0)
        sep_mask = (dists < separation_radius) & (dists > 0)

        # 分离：远离非常近的邻居
        if sep_mask.any():
            sep = -diffs[sep_mask].sum(axis=0)
        else:
            sep = jnp.zeros(2)

        # 对齐：匹配邻居的平均速度
        if nbr_mask.any():
            align = vel[nbr_mask].mean(axis=0) - vel[i]
        else:
            align = jnp.zeros(2)

        # 凝聚：向邻居的平均位置转向
        if nbr_mask.any():
            cohesion = pos[nbr_mask].mean(axis=0) - pos[i]
        else:
            cohesion = jnp.zeros(2)

        new_vel = new_vel.at[i].set(vel[i] + 1.5 * sep + 0.5 * align + 0.3 * cohesion)

    # 限制速度
    speeds = jnp.linalg.norm(new_vel, axis=1, keepdims=True)
    vel = jnp.where(speeds > 2.0, new_vel / speeds * 2.0, new_vel)
    pos = pos + vel * dt
    trajectories.append(pos.copy())

# 绘制快照
fig, axes = plt.subplots(1, 3, figsize=(15, 4))
for ax, idx, title in zip(axes, [0, 50, 199], ["开始", "第 50 步", "第 200 步"]):
    p = trajectories[idx]
    v = vel if idx == 199 else jnp.zeros_like(vel)
    ax.scatter(p[:, 0], p[:, 1], s=20, c="blue")
    ax.set_aspect("equal"); ax.grid(True); ax.set_title(title)
    lim = max(abs(p).max() + 1, 6)
    ax.set_xlim(-lim, lim); ax.set_ylim(-lim, lim)
plt.suptitle("雷诺兹 Flocking：分离 + 对齐 + 凝聚")
plt.tight_layout()
plt.show()
```

3. 模拟共享自主混合：人类提供有噪声的方向输入，机器人的自主系统提供平滑路径到目标。用不同的 alpha 值混合它们。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

goal = jnp.array([10.0, 5.0])
pos = jnp.array([0.0, 0.0])
dt = 0.1

rng = jax.random.PRNGKey(3)

fig, axes = plt.subplots(1, 3, figsize=(15, 4))
for ax, alpha in zip(axes, [1.0, 0.5, 0.0]):
    pos = jnp.array([0.0, 0.0])
    path = [pos.copy()]

    for step in range(150):
        # 机器人自主：平滑路径到目标
        direction = goal - pos
        u_robot = direction / (jnp.linalg.norm(direction) + 1e-6) * 1.0

        # 人类输入：大致正确的方向但有噪声
        noise = jax.random.normal(jax.random.fold_in(rng, step), (2,)) * 0.5
        u_human = u_robot + noise

        # 混合
        u = alpha * u_human + (1 - alpha) * u_robot
        pos = pos + u * dt
        path.append(pos.copy())

        if jnp.linalg.norm(pos - goal) < 0.3:
            break

    path = jnp.stack(path)
    ax.plot(path[:, 0], path[:, 1], "b-", alpha=0.7)
    ax.plot(*goal, "r*", markersize=15)
    ax.plot(0, 0, "go", markersize=10)
    ax.set_title(f"α={alpha:.1f} ({'人类' if alpha==1 else '机器人' if alpha==0 else '共享'})")
    ax.set_xlim(-1, 12); ax.set_ylim(-3, 8)
    ax.set_aspect("equal"); ax.grid(True)

plt.suptitle("共享自主：混合人类和机器人控制")
plt.tight_layout()
plt.show()
```
