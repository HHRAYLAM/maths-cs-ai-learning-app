# 视觉语言模型

*视觉语言模型联合理解图像和文本，实现视觉问答、图像描述和视觉推理。本文件涵盖 VQA、图像描述、视觉定位以及像 VisualBERT、BLIP、LLaVA、Flamingo、PaLI 和 Qwen-VL 这样的架构，它们融合视觉编码器与大型语言模型。*

- 想象一位博物馆向导可以看着一幅画并阐述关于它的一切：存在什么物体、讲述什么故事、传达什么情感，以及回答访客可能提出的任何问题。**视觉语言模型 (VLM)** 是计算等价物——一个联合理解图像和文本的系统，使它能够描述视觉场景、回答关于它们的问题、遵循视觉指令，甚至根据自然语言查询定位图像内的特定物体。

- VLM 位于你在第 8 章遇到的视觉编码器和第 7 章的语言模型的交叉点。核心工程挑战是桥接两个非常不同的表示世界：视觉骨干的空间、连续特征图和语言模型的序列、离散令牌嵌入。本文件中的每个架构本质上都是对这个问题有不同的答案：你如何融合视觉和语言？

## 视觉问答

- 想象有人向你展示一张照片并问"公园里有多少只狗？"你毫不费力地解析图像、定位狗、计数它们并给出答案。**视觉问答 (VQA)** 形式化这个：给定图像 $I$ 和自然语言问题 $q$，预测答案 $a$。

- 该任务可以用几种方式框架。最常见的是**开放端分类**：模型从固定词汇表中最频繁的答案中选择（例如，VQA v2 中前 3,129 个答案）。或者，它可以作为**生成式回答**处理，模型生成自由形式文本串——这是现代 VLM 使用的方法。

- 形式上，你想学习函数 $f(I, q) \to a$，最大化正确答案的似然。在分类设置中，这变为：

$$p(a \mid I, q) = \text{softmax}(W \cdot g(v, h))$$

- 其中 $v$ 是视觉特征向量（来自 CNN 或 ViT），$h$ 是问题编码（来自 LSTM 或 Transformer），$g$ 是组合它们的融合函数。$g$ 的设计是真正架构创造力所在。

- **VQA v1** (Antol et al., 2015) 引入基准，包含 MS COCO 204,000 张图像上的 614,000 个问题。研究人员很快发现，模型可以通过利用**语言先验**实现令人惊讶的高准确率——对"多少"问题回答"2"或对"是否有"问题回答"yes"，甚至不看图像。

- **VQA v2** (Goyal et al., 2017) 通过为每个问题配对两张相似但产生不同答案的图像来解决这个问题。这迫使模型实际将推理建立在视觉内容中。平衡对设置大致使数据集翻倍，使仅语言的捷径效果大大降低。

- 其他重要 VQA 数据集包括**GQA** (Hudson & Manning, 2019)，包含需要多步推理的组合问题，**OK-VQA** (Marino et al., 2019) 需要图像之外的外部知识，以及**TextVQA** (Singh et al., 2019)，其中答案依赖于读取图像内的文本。

![VQA 流水线：图像通过视觉编码器，问题通过文本编码器，它们的表示被融合，融合向量被分类为答案](../images/vqa_pipeline.svg)

- 早期 VQA 模型使用简单策略：从预训练 CNN（通常是 ResNet 或 VGGNet 的倒数第二层，第 8 章）提取图像特征，用 LSTM（第 6 章）编码问题，然后组合它们。组合函数 $g$ 快速发展：从简单的逐元素乘法，到双线性池化，到多模态 Tucker 分解。**双线性注意力**计算 $v^T W h$，其中 $W$ 是可学习交互矩阵，但完整双线性形式有 $O(d_v \times d_h)$ 参数，这过大。**MLB**（多模态低秩双线性池化）将其分解为两个低秩投影，使其可行。

- VQA 的突破是注意力。**堆叠注意力网络** (Yang et al., 2016) 使用问题编码来注意力空间图像区域，迭代细化关注图像的哪些部分。这个想法——让问题"看向"相关图像区域——成为标准。

## 图像描述

- 想象一位朋友看着你的度假照片并描述他们看到的内容："一只金毛猎犬在阳光明媚的海滩上接飞盘。"**图像描述**是生成图像的自然语言描述的任务。与 VQA 不同，没有问题——模型必须自行决定什么值得描述。

- **Show and Tell** (Vinyals et al., 2015) 为描述建立了典型的编码器 - 解码器架构。CNN 编码器（例如，Inception 或 ResNet）生成单个图像特征向量 $v$。该向量用作 LSTM 解码器的初始隐藏状态，然后自回归地逐词生成描述：

$$p(w_t \mid w_{1:t-1}, I) = \text{LSTM}(w_{t-1}, h_{t-1})$$

- 整个模型通过最大化真实描述的对数似然端到端训练。在推理时，使用束搜索（第 7 章）找到高概率描述。

- Show and Tell 的问题是整个图像压缩为单个向量。对复杂场景，单个向量无法捕获所有相关细节。你丢失空间信息——模型无法在生成不同词时"回看"图像的特定部分。

- **Show, Attend and Tell** (Xu et al., 2015) 通过引入**对图像区域的注意力**解决这个问题。不是将图像编码为一个向量，CNN 生成空间特征网格（例如，VGGNet 最后一层卷积的 $14 \times 14 \times 512$）。在每个解码步，模型计算这些空间位置的注意力权重，产生突出显示当前词最相关区域的上下文向量。

- 回忆第 6 章的注意力机制：解码器隐藏状态充当查询，空间特征充当键和值，注意力权重告诉模型看哪里。作者提出两种变体：**软注意力**（可微分，所有区域的加权平均）和**硬注意力**（随机采样单个区域，用 REINFORCE 训练）。

![基于注意力的描述：在每个解码步，模型注意力图像的不同空间区域，聚焦相关区域如生成"dog"时的狗](../images/attention_captioning.svg)

- 这些模型产生的注意力图非常可解释：生成"dog"时，注意力峰值在狗区域；生成"beach"时，它转移到沙和水。这是注意力提供内置可解释性的最早令人信服的演示之一。

- **CIDEr** (Vedantam et al., 2015)、**METEOR**、**BLEU** 和**SPICE** 是标准描述评估指标。CIDEr 计算生成和参考描述间的 TF-IDF 加权 n-gram 相似度，专门为描述评估设计。现代 VLM 通常在 CIDEr 上评估 MS COCO Captions 和 NoCaps 等描述基准。

- 后来的描述模型结合**自下而上的注意力** (Anderson et al., 2018)，其中目标检测器（Faster R-CNN，第 8 章）首先提出显著图像区域，描述模型注意力这些区域特征而非均匀网格。这是在基于 ViT 的编码器接管之前的主导方法。

## 架构模式

- 每个 VLM 必须回答一个基本设计问题：视觉和语言在何时交互？答案定义了模型的架构家族。有三种主要模式，每种模式有不同的权衡。

### 双编码器

- 想象两位翻译独立工作——一位阅读法语文档，另一位阅读英文文档——他们各自用"通用语言"生成摘要。他们翻译期间从不交流，但他们的摘要可直接比较。这是**双编码器**模式。

- 视觉编码器 $f_v$ 和文本编码器 $f_t$ 独立将各自输入映射到维度 $d$ 的共享嵌入空间。图像嵌入是 $v = f_v(I) \in \mathbb{R}^d$，文本嵌入是 $t = f_t(q) \in \mathbb{R}^d$。相似度通过点积或余弦相似度计算：$\text{sim}(I, q) = v^T t / (\|v\| \|t\|)$。

- CLIP (Radford et al., 2021) 在前一文件多模态表示中涵盖，是典型的双编码器。它用对比目标（InfoNCE）在互联网上抓取的 4 亿图像 - 文本对上训练。因为编码器独立，你可以预计算和缓存所有图像嵌入，使检索极其高效——你只需在搜索时编码查询文本。

- 双编码器的弱点是视觉和语言从不在特征级别交互。模型无法执行细粒度跨模态推理：例如，它无法确定描述中的特定词是否对应图像中的特定区域。这限制了它对 VQA 或定位描述等任务的有用性。

### 融合编码器

- 现在想象两位翻译在同一房间，积极讨论两份文档。他们可以指向特定段落、互相提问，并建立联合理解。这是**融合编码器**模式。

- 两种模态都编码然后通过**交叉注意力层**融合，其中一种模态的令牌注意力另一种模态的令牌。图像首先由视觉编码器处理为补丁或区域令牌序列 $V = [v_1, \ldots, v_N]$。文本分词为 $T = [t_1, \ldots, t_M]$。在融合层中，文本令牌通过交叉注意力注意力图像令牌：

$$\text{CrossAttn}(T, V) = \text{softmax}\!\left(\frac{(TW_Q)(VW_K)^T}{\sqrt{d_k}}\right)(VW_V)$$

- 这实现细粒度交互：每个文本令牌可以注意力它需要的特定图像区域。**VisualBERT**、**VilBERT** 和**UNITER** 等模型使用这种模式。代价是你无法为检索预计算单独的嵌入——每个图像 - 文本对需要通过融合层的完整前向传递。

![双编码器与融合编码器：双编码器计算单独的嵌入和相似度分数，而融合编码器通过交叉注意力层合并模态](../images/dual_vs_fusion_encoder.svg)

### 编码器 - 解码器

- **编码器 - 解码器**模式将视觉编码器与自回归生成输出令牌的文本解码器结合，类似于第 7 章的 seq2seq 模型。视觉编码器生成上下文图像表示，文本解码器在生成输出文本时交叉注意力它们。

- 这种模式自然支持生成任务：描述、VQA 带自由形式答案和视觉对话。**GIT**（Generative Image-to-text Transformer, Wang et al., 2022）、**CoCa** (Contrastive Captioner, Yu et al., 2022) 和**PaLI** 等模型使用这种架构。CoCa 巧妙结合双编码器和编码器 - 解码器模式：文本解码器的前半部分作为单模态文本编码器（用于对比学习），而后半部分交叉注意力图像特征（用于生成描述），获得两者的最佳。

- 这三种模式之间的选择取决于目标任务。双编码器对大规模检索最优。融合编码器对细粒度理解任务最佳。编码器 - 解码器对生成任务最通用。现代最先进的 VLM 越来越多采用编码器 - 解码器或仅解码器范式，将每个视觉 - 语言任务视为文本生成。

## Flamingo：少样本多模态学习

- 想象一位资深专家，在研究艺术和文学多年后，可以看到全新的绘画风格，并在仅看一两个例子后雄辩地描述它。**Flamingo** (Alonso et al., 2022, DeepMind) 基于相同原则构建：它利用强大的预训练语言模型和预训练视觉编码器，用轻量架构组件连接它们，实现多模态任务的少样本学习。

- Flamingo 的设计理念是保守有效的：保持预训练视觉编码器（NFNet）和语言模型（Chinchilla）冻结，仅学习连接它们的"胶水"。这种胶由两个组件组成：**Perceiver 重采样器**和**门控交叉注意力层**。

- **Perceiver 重采样器**获取视觉编码器的可变长度输出（取决于图像分辨率）并将其压缩为固定集 $N$ 个视觉令牌（通常 $N = 64$）。它的工作原理是初始化一组 $N$ 个可学习查询向量，并使用交叉注意力让这些查询注意力视觉编码器输出的完整集。这本质上是 Perceiver 架构 (Jaegle et al., 2021) 作为瓶颈应用——它产生紧凑、固定大小的视觉表示，无论输入图像大小如何。

$$z = \text{CrossAttn}(Q_{\text{learned}}, V_{\text{image}}) \in \mathbb{R}^{N \times d}$$

- **门控交叉注意力层**交错在冻结语言模型层之间。在每个这样的层，语言模型的文本令牌交叉注意力 Perceiver 重采样器产生的视觉令牌。关键的是，每个门控交叉注意力层包括可学习标量门 $\alpha$，初始化为零，在添加到残差流之前乘以交叉注意力输出：

$$\hat{x} = x + \alpha \cdot \text{CrossAttn}(x, z)$$

- 初始化 $\alpha = 0$ 意味着在训练开始时，交叉注意力没有贡献，模型表现完全像原始冻结语言模型。门在训练期间逐渐打开，平滑整合视觉信息而不破坏语言模型的预训练表示。

![Flamingo 架构：冻结视觉编码器馈送到 Perceiver 重采样器，产生固定长度视觉令牌，通过 LM 块之间交错的门控交叉注意力层注入到冻结 LM](../images/flamingo_architecture.svg)

- Flamingo 原生处理**交错图像 - 文本序列**。你可以向它提供包含多个图像穿插文本的提示，例如："[图像 1] 这是一只猫。[图像 2] 这是一只狗。[图像 3] 这是一个___。"模型通过视觉编码器和 Perceiver 重采样器处理每个图像，生成的视觉令牌插入文本序列中的对应位置。语言模型的因果注意力掩码确保每个文本令牌只能注意力当前和先前图像的视觉令牌。

- 这种交错支持强大的**少样本多模态学习**。通过在上下文中提供几个图像 - 文本示例，Flamingo 可以执行新任务而无需任何梯度更新。在 VQAv2、OK-VQA 和描述等基准上，80B 参数的 Flamingo 实现最先进的少样本性能，通常仅用 4 或 32 个示例匹配或超过微调的专用模型。

## LLaVA 和视觉指令微调

- 想象你有一位杰出的语言专家（LLM）和一位杰出的艺术评论家（视觉编码器）。如果你能教艺术评论家"说语言专家的语言"，他们可以无缝协作。**LLaVA**（Large Language and Vision Assistant, Liu et al., 2023）正是这样做的：它用简单线性层将视觉特征投影到 LLM 的令牌嵌入空间，然后在指令跟随数据上微调整个系统。

- LLaVA 的架构非常简单。图像由预训练 CLIP ViT-L/14 视觉编码器编码为补丁特征网格 $V \in \mathbb{R}^{N \times d_v}$，其中 $N = 256$ 补丁（对 336px 图像带 14px 补丁）。**投影层** $W$ 将这些视觉特征映射到 LLM 的嵌入维度：

$$H_v = VW, \quad W \in \mathbb{R}^{d_v \times d_{\text{LLM}}}$$

- 投影视觉令牌 $H_v$ 简单地与文本令牌嵌入连接，并作为单个序列馈送到 LLM（Vicuna，微调 LLaMA）。LLM 用其标准因果自注意力处理它们——没有特殊交叉注意力层、没有 perceiver、只是连接。视觉令牌被视为恰好编码视觉信息的文本令牌。

![LLaVA 架构：CLIP ViT 将图像编码为补丁特征，线性投影将它们映射到 LLM 嵌入空间，投影视觉令牌 prepended 到文本令牌并馈送到 LLM](../images/llava_architecture.svg)

- **视觉指令微调**是 LLaVA 的关键训练创新。作者用 GPT-4 从 COCO 图像生成 158,000 个多模态指令跟随示例。每个示例由图像配对对话指令组成（例如，"详细描述这张图像"、"这张图像有什么不寻常？"、"如果我是游客访问这个地方，我应该知道什么？"）。模型训练为给定图像和指令生成 GPT-4 撰写的响应。

- 训练分两阶段进行。**阶段 1（预训练）**：仅投影层 $W$ 在图像 - 描述对（来自 CC3M 的 595K）上训练，而视觉编码器和 LLM 都冻结。这教 $W$ 将视觉特征与 LLM 的嵌入空间对齐。**阶段 2（微调）**：投影层和 LLM 在指令跟随数据上联合微调，而视觉编码器保持冻结。这教模型遵循复杂视觉指令。

- **LLaVA-1.5** 用三个关键改进改进原始版本：将单一线性投影替换为两层 MLP（更具表达力的映射）、使用更高分辨率图像（336px 而非 224px，产生更多补丁令牌），并在训练混合中添加学术 VQA 数据集。这些看似微小的修改产生大幅基准性能提升。

- LLaVA 方法表明你不需要像 Flamingo 的 Perceiver 重采样器或门控交叉注意力这样的复杂架构创新。简单线性投影与高质量指令微调数据结合，足以有效地将视觉编码器连接到 LLM。这种简单性使 LLaVA 极其有影响力——大多数后续开源 VLM 遵循类似配方。

## 缩放视觉语言模型

- 该领域从概念验证 VLM 迅速发展到在数十亿图像 - 文本对上训练的工业规模系统。三个模型家族说明不同的缩放方法。

### PaLI

- **PaLI**（Pathways Language and Image model, Chen et al., 2022, Google）同时缩放视觉编码器和语言模型。PaLI 使用 ViT-e（4B 参数）作为视觉编码器，mT5（13B 参数）作为语言模型，总共 17B 参数。图像编码为补丁令牌序列，prepended 到文本令牌并馈送到编码器 - 解码器 mT5。

- PaLI 的关键洞察是**缩放视觉编码器与缩放语言模型一样重要**。先前的工作通常使用固定、中等大小的视觉骨干（例如，ViT-B 或 ViT-L）并将所有参数预算投入 LLM。PaLI 表明 4B 参数 ViT-e 在 JFT-4B（40 亿标注图像）上预训练，显著提高 OCR 和空间推理等细粒度视觉任务的性能。

- PaLI 在 WebLI 上训练，这是 109 种语言的 100 亿图像 - 文本对数据集，使其天生多语言。模型用任务混合预训练：图像描述、VQA 和图像 - 文本匹配，都作为文本到文本生成（遵循第 7 章的 T5 范式）。**PaLI-X**（55B 参数）和**PaLI-3**（5B，使用 SigLIP 作为视觉编码器）是后续迭代。

### Qwen-VL

- **Qwen-VL** (Bai et al., 2023, Alibaba) 基于 Qwen LLM 构建，添加 ViT 视觉编码器和单层交叉注意力模块（类似于 Flamingo 的 Perceiver 重采样器），将视觉编码器输出压缩为固定集 256 个视觉令牌。视觉令牌与文本令牌连接并由 Qwen LLM 处理。

- Qwen-VL 的训练使用三阶段配方。阶段 1：在 14 亿弱监督图像 - 文本对上预训练，仅视觉编码器解冻。阶段 2：在更高质量数据上多任务预训练，包括 VQA、描述、定位和 OCR 数据集，完整模型解冻。阶段 3：在指令跟随和对话数据上监督微调。这种从噪声网络数据到策划指令数据的渐进细化，是大多数现代 VLM 共享的模式。

- **Qwen2-VL** (2024) 引入**动态分辨率**支持：不是将所有图像调整为固定大小，它通过动态调整视觉令牌数量以原生分辨率处理图像。更高分辨率图像产生更多令牌，更低分辨率图像产生更少。这提高文档理解和细粒度识别等细节敏感任务的性能，而不在低分辨率输入上浪费计算。

### InternVL

- **InternVL** (Chen et al., 2024, Shanghai AI Lab) 激进缩放视觉编码器，使用 InternViT-6B——60 亿参数视觉 transformer——配对语言模型。关键架构贡献是**动态高分辨率处理**：图像划分为 448x448 像素的瓦片，每个由视觉编码器独立处理，生成的瓦片特征与完整图像的缩略图特征连接。这允许模型处理任意纵横比和分辨率的图像。

- InternVL-2 进一步引入**渐进对齐训练**：首先用对比目标（如 CLIP）对齐视觉编码器，然后通过轻量 MLP 连接器将其连接到 LLM，最后在指令数据上端到端微调。渐进策略防止视觉编码器预训练表示的灾难性遗忘。

![缩放 VLM：PaLI、Qwen-VL 和 InternVL 的比较，显示连接视觉编码器和语言模型的不同方法，包括它们的训练阶段](../images/scaling_vlms_comparison.svg)

- 所有三个家族的共同主题是**训练数据策划**的重要性。原始网络抓取的图像 - 文本对嘈杂且经常不一致。连续训练阶段渐进过滤和细化数据，从数十亿噪声对到数百万高质量指令示例。最终微调数据的质量通常比模型的原始参数计数更重要。

## 定位和指代

- 想象指着人群中的人说"戴红帽子的女士"。你正在使用语言指代特定空间区域。**视觉定位**是相反的：给定图像和自然语言表达，模型必须识别（定位）所指物体。**指代表达理解**产生边界框；**指代表达分割**产生像素掩码。

- 形式上，给定图像 $I$ 和指代表达 $r$（例如，"左边的大棕色狗"），模型预测边界框 $b = (x, y, w, h)$ 或定位所指的一组坐标。数据集包括**RefCOCO**、**RefCOCO+** 和**RefCOCOg**，每个包含带多个对象的图像和每个对象的明确指代表达。

- 早期定位模型使用两阶段方法：首先生成区域提议（来自 Faster R-CNN 或类似），然后用融合模型针对语言查询评分每个提议。最高分区域是预测。这计算昂贵且受提议质量限制。

- 现代 VLM 将定位直接整合到生成框架中。关键想法是将边界框坐标表示为**文本令牌**。你将连续坐标空间离散化为箱（例如，$x, y, w, h$ 每个 1000 箱）并将特殊位置令牌如 `<loc_342>` 添加到词汇表。模型然后通过输出位置令牌序列生成边界框：

$$\text{输出：} \texttt{<loc\_102><loc\_215><loc\_487><loc\_398>}$$

- 这个分词技巧允许任何自回归语言模型执行定位而无需任何架构更改——它只是学习"说坐标"。**Pix2Seq** (Chen et al., 2022) 为对象检测开创这种方法，Qwen-VL、Ferret 和 Kosmos-2 等模型将其扩展到指代表达理解和短语定位。

- **Kosmos-2** (Peng et al., 2023, Microsoft) 通过表示空间位置为嵌入生成文本中的特殊令牌，为多模态 LLM 添加定位能力。例如，它可以生成："<phrase>金毛猎犬</phrase> <box> <loc_102> <loc_215> <loc_487> <loc_398> </box> 正在接飞盘。"这种文本和空间令牌的交错实现同时描述和定位。

![通过坐标分词的定位：模型生成穿插离散边界框坐标令牌的文本令牌，定位描述中提到的物体](../images/grounding_coordinate_tokens.svg)

- **指向**进一步采用定位：模型预测单个点（通常所指物体的中心）。这对交互应用有用，用户问"最近的出口在哪里？"，模型响应叠加在图像上的坐标。像**Shikra**和**Ferret**这样的模型支持基于点的指代以及基于框的定位。

## 无 OCR 文档理解

- 传统文档理解流水线复杂：首先运行 OCR 引擎提取文本和布局，然后将提取的文本馈送到语言模型。这种多阶段方法脆弱——OCR 错误传播到下游，空间布局信息经常丢失或表示不佳。如果模型可以直接从像素读取，就像你一样，会怎样？

- **Donut**（Document Understanding Transformer, Kim et al., 2022）完全消除 OCR。它使用 Swin Transformer（第 8 章）作为视觉编码器处理文档图像，BART 风格 Transformer 解码器直接从视觉特征生成结构化文本输出。解码器可以生成 JSON、键值对或纯文本，取决于任务。

- Donut 的训练是两阶段。**预训练**：模型通过执行合成 OCR 学习读取——给定文档图像，它生成完整文本内容。这在数百万从文本语料渲染的合成文档图像上训练，教视觉编码器识别字符、字体和布局。**微调**：模型适应特定下游任务如收据解析、表单理解或文档分类，通过训练它生成任务特定结构化输出。

- Donut 解码器使用特殊提示方案：任务由提示令牌指定（例如，分类用 `<doc_class>` 或收据解析用 `<parse_receipt>`），模型生成以该提示为条件的输出。这个统一接口允许单个模型处理多个文档理解任务。

- **Pix2Struct** (Lee et al., 2023, Google) 采用无 OCR 想法并将其应用于网页理解和图表/图形理解。关键预训练目标是**截图解析**：给定网页的掩码截图，模型生成产生可见区域的底层 HTML。这教模型理解视觉渲染和结构化标记之间的关系。

- Pix2Struct 引入**可变分辨率输入处理**：不是将所有图像调整为固定大小（这会扭曲纵横比并破坏精细文本），它将图像打包到固定数量的补丁中，同时保持原始纵横比。高而窄的文档产生高而窄的补丁网格。这对文档理解至关重要，其中纵横比携带语义信息（收据窄而高；电子表格宽而短）。

![无 OCR 文档理解：Donut 和 Pix2Struct 直接通过视觉编码器处理文档图像，无需任何 OCR 预处理生成结构化文本输出](../images/ocr_free_document_understanding.svg)

- **Nougat** (Blecher et al., 2023, Meta) 专门将 Donut 架构应用于学术论文，直接从 PDF 页面图像生成完整 LaTeX 标记。它可以处理复杂数学方程、表格和图形——传统 OCR 流水线严重挣扎的任务。模型在 PDF 页面图像及其对应 LaTeX 源代码对上训练。

- 无 OCR 模型的成功证明了深度学习中的更广泛原则：直接从原始输入（像素）学习的端到端模型通常优于复杂多阶段流水线，因为它们可以联合优化所有组件并学习专门针对最终任务的表示。中间 OCR 步骤是限制模型学习内容的瓶颈。

## 视觉令牌流水线

- 无论架构家族如何，每个 VLM 必须将图像转换为语言模型可以处理的令牌序列。理解这个流水线至关重要。过程因模型而异，但一般流程是：

- **步骤 1：补丁提取。** 图像（高 $H$，宽 $W$）划分为不重叠的 $P \times P$ 补丁，产生 $N = HW / P^2$ 补丁。对 336x336 图像带 14x14 补丁，$N = 576$。

- **步骤 2：视觉编码。** 每个补丁线性投影并通过视觉编码器（通常是 ViT）。输出是上下文补丁嵌入序列 $V = [v_1, \ldots, v_N] \in \mathbb{R}^{N \times d_v}$。这些嵌入携带局部外观信息和全局上下文（来自自注意力）。

- **步骤 3：令牌压缩（可选）。** 一些模型将 $N$ 个视觉令牌压缩为更小集 $M \ll N$ 令牌，减少语言模型的计算负担。Flamingo 使用 Perceiver 重采样器（$M = 64$）；Qwen-VL 使用交叉注意力（$M = 256$）；**Q-Former**（用于 BLIP-2, Li et al., 2023）使用 $M = 32$ 个可学习查询令牌集，交叉注意力视觉编码器的输出。

- **步骤 4：投影。** 视觉令牌（完整集或压缩集）通过线性层或 MLP 投影到语言模型的嵌入空间。投影后，视觉令牌与文本令牌嵌入具有相同维度，可以与它们连接。

- **步骤 5：注入 LLM。** 投影视觉令牌插入特殊 `<image>` 占位符令牌位置的令牌序列，组合序列由语言模型处理。LLM 的自注意力允许文本令牌注意力视觉令牌，反之亦然。

![视觉令牌流水线：图像补丁提取，由 ViT 编码，可选由 Perceiver 或 Q-Former 压缩，投影到 LLM 维度，与文本令牌连接](../images/visual_token_pipeline.svg)

- 视觉令牌数量直接影响计算成本。每个视觉令牌参与 LLM 的自注意力，这是序列长度的二次方。高分辨率图像带许多补丁可以产生数百或数千个视觉令牌，主导 LLM 的上下文窗口。这就是为什么令牌压缩重要：将 576 个视觉令牌减少到 64 将视觉对注意力的贡献减少约 9 倍。

- **BLIP-2** (Li et al., 2023) 因其高效桥接策略而引人注目。它引入轻量**Q-Former**（带可学习查询的小型 Transformer），位于冻结视觉编码器和冻结 LLM 之间。Q-Former 是唯一可训练组件——视觉编码器和 LLM 都保持冻结。它分两阶段预训练：首先用图像 - 文本对比学习、匹配和描述目标（连接到视觉编码器），然后用语言生成目标（连接到 LLM）。这种模块化设计允许 BLIP-2 将任何视觉编码器插入任何 LLM。

## 训练目标

- VLM 用目标组合训练，取决于架构模式：

- **图像 - 文本对比损失 (ITC)**：在共享嵌入空间中对齐图像和文本表示，如 CLIP 中。这是双编码器的主要目标，经常用作融合模型的预训练目标。损失是前一文件的 InfoNCE 损失。

- **图像 - 文本匹配 (ITM)**：二元分类目标——给定图像和文本，预测它们是否匹配。硬负例（相似但与不同图像配对的文本）使这个任务具有挑战性，迫使模型学习细粒度对齐。

- **语言建模 (LM)**：标准自回归语言建模目标——给定所有先前令牌预测下一个令牌。对 VLM，"先前令牌"包括视觉令牌，所以模型学习生成以视觉输入为条件的文本。这是编码器 - 解码器和仅解码器 VLM 的主要目标。

$$\mathcal{L}_{\text{LM}} = -\sum_{t=1}^{T} \log p(w_t \mid w_{<t}, V)$$

- **前缀语言建模**：变体，其中图像和文本前缀作为上下文提供（不训练），模型训练仅生成延续。这在 PaLI 和 SimVLM 等模型中使用。

- 大多数现代 VLM 在预训练期间结合多个目标（例如，BLIP 中的 ITC + ITM + LM，CoCa 中的 ITC + LM），然后在指令数据上用纯 LM 目标微调。

## 编码任务（使用 CoLab 或 notebook）

1. 实现简单基于注意力的图像描述解码器。使用随机"图像特征"作为编码器输出，训练解码器生成固定描述，观察注意力权重在每个解码步如何跨空间位置移动。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# 模拟 4x4 空间网格图像特征（16 个区域，dim=32）
key = jax.random.PRNGKey(42)
k1, k2, k3 = jax.random.split(key, 3)
img_features = jax.random.normal(k1, (16, 32))  # 16 个空间区域，32-dim

# 词汇表：0=<start>, 1="a", 2="red", 3="car", 4=<end>
vocab_size, embed_dim, hidden_dim = 5, 16, 32
W_embed = jax.random.normal(k2, (vocab_size, embed_dim)) * 0.1
W_attn_q = jax.random.normal(k3, (hidden_dim, 32)) * 0.1  # 查询投影

def attend(h, img_feats, W_q):
    """给定解码器状态 h，计算图像特征的软注意力。"""
    query = h @ W_q  # (32,)
    scores = img_feats @ query  # (16,)
    weights = jax.nn.softmax(scores)  # (16,)
    context = weights @ img_feats  # (32,)
    return context, weights

# 简单 GRU 风格步（为说明，仅线性 + tanh）
W_h = jax.random.normal(jax.random.PRNGKey(0), (embed_dim + 32, hidden_dim)) * 0.1

def decode_step(h, word_idx, img_feats):
    context, attn_weights = attend(h, img_feats, W_attn_q)
    word_emb = W_embed[word_idx]  # (16,)
    inp = jnp.concatenate([word_emb, context])  # (48,)
    h_new = jnp.tanh(inp @ W_h)  # (32,)
    return h_new, attn_weights

# 运行解码序列：<start> -> "a" -> "red" -> "car" -> <end>
target_seq = [0, 1, 2, 3, 4]
h = jnp.zeros(hidden_dim)
all_attn = []
for word_idx in target_seq[:-1]:
    h, attn_w = decode_step(h, word_idx, img_features)
    all_attn.append(attn_w)

# 可视化每个步的注意力图（重塑为 4x4 网格）
words = ["<start>", "a", "red", "car"]
fig, axes = plt.subplots(1, 4, figsize=(14, 3))
for i, (ax, w) in enumerate(zip(axes, words)):
    ax.imshow(all_attn[i].reshape(4, 4), cmap='viridis')
    ax.set_title(f'生成"{w}"后注意力')
    ax.axis('off')
plt.suptitle('每个解码步对图像区域的注意力')
plt.tight_layout(); plt.show()
# 尝试更改 img_features 看注意力模式如何移动！
```

2. 模拟视觉令牌流水线：将图像分块为补丁，投影补丁到嵌入空间，与文本令牌嵌入连接，在组合序列上运行单个自注意力层。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

key = jax.random.PRNGKey(7)

# 创建合成 8x8"图像"带 3 通道
k1, k2, k3, k4 = jax.random.split(key, 4)
image = jax.random.uniform(k1, (8, 8, 3))

# 步骤 1：分块为 4x4 补丁 -> 4 个补丁
patch_size = 4
patches = image.reshape(2, patch_size, 2, patch_size, 3)
patches = patches.transpose(0, 2, 1, 3, 4).reshape(4, patch_size * patch_size * 3)  # (4, 48)
print(f"补丁数：{patches.shape[0]}, 补丁维度：{patches.shape[1]}")

# 步骤 2：投影补丁到嵌入维度 (d=16)
d_model = 16
W_patch = jax.random.normal(k2, (patches.shape[1], d_model)) * 0.1
visual_tokens = patches @ W_patch  # (4, 16)

# 步骤 3：创建文本令牌嵌入（模拟 3 个文本令牌）
text_tokens = jax.random.normal(k3, (3, d_model)) * 0.1

# 步骤 4：连接视觉 + 文本令牌
combined = jnp.concatenate([visual_tokens, text_tokens], axis=0)  # (7, 16)
print(f"组合序列长度：{combined.shape[0]} (4 个视觉 + 3 个文本)")

# 步骤 5：组合序列上的单头自注意力
W_Q = jax.random.normal(k4, (d_model, d_model)) * 0.1
k5, k6 = jax.random.split(k4)
W_K = jax.random.normal(k5, (d_model, d_model)) * 0.1
W_V = jax.random.normal(k6, (d_model, d_model)) * 0.1

Q = combined @ W_Q
K = combined @ W_K
V = combined @ W_V
attn_scores = (Q @ K.T) / jnp.sqrt(d_model)
attn_weights = jax.nn.softmax(attn_scores, axis=-1)  # (7, 7)

output = attn_weights @ V  # (7, 16)

# 可视化跨模态注意力模式
labels = ['V1', 'V2', 'V3', 'V4', 'T1', 'T2', 'T3']
fig, ax = plt.subplots(figsize=(6, 5))
im = ax.imshow(attn_weights, cmap='Blues')
ax.set_xticks(range(7)); ax.set_xticklabels(labels)
ax.set_yticks(range(7)); ax.set_yticklabels(labels)
ax.set_xlabel('Key'); ax.set_ylabel('Query')
ax.set_title('自注意力：视觉 (V) 和文本 (T) 令牌')
plt.colorbar(im, ax=ax); plt.tight_layout(); plt.show()
# 观察：文本令牌注意力视觉令牌（跨模态注意力）！
```

3. 为视觉定位实现坐标分词。给定边界框，转换为离散令牌；给定离散令牌，重建边界框。可视化不同箱分辨率的量化误差。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

def encode_bbox(bbox, num_bins=1000):
    """将连续 bbox (x, y, w, h) 在 [0,1] 中转换为离散令牌。"""
    tokens = jnp.round(jnp.array(bbox) * (num_bins - 1)).astype(jnp.int32)
    return tokens

def decode_bbox(tokens, num_bins=1000):
    """将离散令牌转回连续 bbox。"""
    return tokens.astype(jnp.float32) / (num_bins - 1)

# 真实边界框（归一化到 [0, 1]）
gt_bbox = jnp.array([0.123, 0.456, 0.333, 0.222])

# 测试不同箱分辨率的量化
bin_sizes = [10, 50, 100, 500, 1000]
errors = []
for n_bins in bin_sizes:
    tokens = encode_bbox(gt_bbox, n_bins)
    reconstructed = decode_bbox(tokens, n_bins)
    error = jnp.max(jnp.abs(gt_bbox - reconstructed))
    errors.append(float(error))
    print(f"箱={n_bins:>5d} | 令牌={tokens} | "
          f"重建={reconstructed} | 最大误差={error:.6f}")

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(bin_sizes, errors, 'o-', color='#e74c3c', linewidth=2, markersize=8)
ax.set_xlabel('箱数'); ax.set_ylabel('最大量化误差')
ax.set_title('边界框量化误差与箱分辨率')
ax.set_xscale('log'); ax.set_yscale('log')
ax.grid(True, alpha=0.3); plt.tight_layout(); plt.show()
# 尝试：很少箱时发生什么（例如，5）？误差何时可接受？
```
