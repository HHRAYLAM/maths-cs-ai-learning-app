# 计算机架构

*计算机架构是我们如何构建执行指令的机器。本文件涵盖数制、逻辑门、CPU 设计、指令集架构、流水线、内存层次结构和虚拟内存，这是每个程序、框架和 AI 模型最终运行的硬件基础。*

- 每个神经网络、每个训练循环、每个推理调用最终都成为流过晶体管的电信号序列。理解硬件对于严肃的 ML 从业者来说不是可选的：它解释了为什么矩阵乘法很快，为什么内存是瓶颈，为什么 GPU 主导 AI 训练，为什么缓存友好的代码可以比朴素代码快 100 倍。

## 数制

- 计算机将所有内容表示为二进制（基 2）：0 和 1 的序列。每位是一个比特。8 位一组是一个字节。二进制数 $b_{n-1} b_{n-2} \ldots b_1 b_0$ 的值是 $\sum_{i=0}^{n-1} b_i \cdot 2^i$。

- 例如，$1011_2 = 1 \cdot 8 + 0 \cdot 4 + 1 \cdot 2 + 1 \cdot 1 = 11_{10}$。

- 十六进制（基 16）是二进制的紧凑表示法。每个十六进制数字代表 4 位：$0\text{-}9$ 映射到 $0000\text{-}1001$，$A\text{-}F$ 映射到 $1010\text{-}1111$。所以 $\text{0xFF} = 1111\,1111_2 = 255_{10}$。内存地址和颜色代码通常用十六进制书写。

- 二进制补码表示有符号整数。对于 $n$ 位数，最高有效位的权重是 $-2^{n-1}$ 而不是 $+2^{n-1}$。8 位二进制补码范围从 $-128$ 到 $+127$。要对数字取反：翻转所有位并加 1。这种表示使加法和减法使用相同的硬件电路，这就是它普遍使用的原因。

- IEEE 754 浮点数将实数表示为 $(-1)^s \times 1.m \times 2^{e-\text{bias}}$，其中 $s$ 是符号位，$m$ 是尾数（小数部分），$e$ 是偏置指数。

![IEEE 754 float32 布局：1 位符号、8 位指数、23 位尾数](../images/ieee754_float.svg)

    - float32（单精度）：1 位符号 + 8 位指数 + 23 位尾数 = 32 位。范围：$\approx \pm 3.4 \times 10^{38}$，精度：$\approx 7$ 位十进制数字。
    - float64（双精度）：1 位符号 + 11 位指数 + 52 位尾数 = 64 位。范围：$\approx \pm 1.8 \times 10^{308}$，精度：$\approx 15$ 位十进制数字。
    - float16（半精度）：1 + 5 + 10 = 16 位。范围和精度有限，但使用一半的内存和带宽。广泛用于 ML 训练（混合精度，第 6 章）。
    - bfloat16：1 + 8 + 7 = 16 位。与 float32 相同的指数范围但精度更低。由 Google 专门为 ML 设计：完整指数范围防止训练期间溢出，降低的精度对梯度更新可接受。

- 浮点算术不精确。$0.1 + 0.2 \neq 0.3$ 在 float64 中（它等于 $0.30000000000000004$）。这是因为 $0.1$ 没有精确的二进制表示，就像 $1/3$ 没有精确的十进制表示一样。在数百万次操作（如梯度下降）中累积这些误差可能导致数值不稳定，这就是为什么存在损失缩放（第 6 章）和 Kahan 求和等技术。

## 逻辑门

- 所有计算简化为逻辑门：实现布尔操作的物理电路（第 1 章中的命题逻辑）。

- 基本门：
    - AND：仅当两个输入都为 1 时输出为 1。
    - OR：如果至少一个输入为 1 则输出为 1。
    - NOT（非门）：翻转输入。
    - NAND（非与）：通用门。任何其他门可以仅从 NAND 门构建。这就是为什么 NAND 是数字电路的基本构建块。
    - XOR（异或）：如果输入不同则输出为 1。对加法（二进制加法的和位是 XOR）和加密至关重要。

- 半加器使用 XOR（和）和 AND（进位）添加两个单个比特。全加器添加两个比特加进位输入，链接在一起创建 $n$ 位加法器。这就是 CPU 执行整数加法的方式：简单逻辑门的级联。

- 多路复用器 (MUX) 根据控制信号选择多个输入之一。带 $n$ 个控制位，它从 $2^n$ 个输入中选择。多路复用器是 if-else 链的硬件等价物，广泛用于 CPU 数据路径路由数据。

- 现代处理器包含数十亿个晶体管，每个晶体管充当微小开关。晶体管要么导通（导通，表示 1）要么截止（不导通，表示 0）。门由晶体管构建，加法器由门构建，ALU 由加法器构建，CPU 由 ALU 构建。整个计算层次结构建立在这个基础上。

## CPU 架构

- 中央处理单元 (CPU) 执行指令。其核心组件：

    - ALU（算术逻辑单元）：执行整数算术（加、减、乘）和逻辑操作（AND、OR、XOR、移位）。这是实际计算发生的地方，由上述逻辑门构建。

    - 寄存器：CPU 内部的超快存储位置。现代 CPU 有数十个通用寄存器，每个保存一个字（64 位 CPU 上 64 位）。寄存器是系统中最快的内存：访问耗时约 0.3 纳秒。

    - 程序计数器 (PC)：保存要执行的下一条指令的内存地址。

    - 控制单元：解码指令并协调数据路径，告诉 ALU 执行什么操作以及使用哪些寄存器。

- 指令周期（取指 - 解码 - 执行）每秒重复数十亿次：

    1. 取指：从 PC 地址的内存读取指令。
    2. 解码：确定指令做什么（加法？从内存加载？分支？）以及使用哪些操作数。
    3. 执行：执行操作（ALU 计算、内存访问或分支）。
    4. 递增 PC（除非指令是分支/跳转）。

- 4 GHz 运行的 CPU 每秒执行 40 亿个周期。每个周期耗时 0.25 纳秒。在该时间内，光传播约 7.5 厘米，这就是为什么物理芯片尺寸很重要：信号无法在一个周期内穿过大芯片。

## 指令集架构

- 指令集架构 (ISA) 是硬件和软件之间的契约：它定义 CPU 理解的指令、寄存器集、内存模型和编码格式。

- CISC（复杂指令集计算机）：指令可能复杂、变长，并可能直接访问内存。单个指令可能相乘两个内存值并存储结果。x86（Intel/AMD）是主导的 CISC ISA，为大多数台式机和服务器提供动力。其向后兼容性（现代 x86 CPU 仍能运行 1980 年代的代码）既是其优势也是其负担。

- RISC（精简指令集计算机）：指令简单、固定长度，仅对寄存器操作。内存访问需要单独的加载/存储指令。更简单的指令实现更快的时钟速度和更容易的流水线。

    - ARM：移动设备的主导 RISC ISA，越来越用于服务器和笔记本电脑（Apple M 系列芯片是 ARM）。ARM 的功耗效率使其 ideal 用于电池供电和热受限设备。
    - RISC-V：开源 RISC ISA。任何人都可以设计 RISC-V 芯片而无需许可费。在嵌入式系统、研究和 AI 加速器中采用增长。

- CISC 与 RISC 的区别已模糊：现代 x86 CPU 在内部将复杂 CISC 指令解码为更简单的微操作（本质上是内部 RISC），获得两方面的好处。

## 流水线

- 没有流水线，CPU 在开始下一条之前完全完成一条指令。这浪费硬件：当 ALU 执行时，取指和解码单元空闲。

![CPU 流水线：指令在取指、解码、执行、内存和写回阶段重叠](../images/cpu_pipeline.svg)

- 流水线重叠指令执行，像装配线。当指令 1 执行时，指令 2 解码，指令 3 被取指。5 级流水线（取指、解码、执行、内存访问、写回）可以同时有 5 条指令在执行中。

- 吞吐量接近每周期一条指令（即使每条指令需要 5 个周期完成）。这与 ML 中的流水线原理相同：数据并行性重叠计算和通信（第 6 章）。

- 冒险是流水线中断的情况：

    - 数据冒险：指令 2 需要指令 1 尚未产生的结果。"Add R1, R2, R3"后跟"Sub R4, R1, R5"——第二条指令需要 R1，第一条仍在计算。转发（旁路）通过直接将结果从一个流水线阶段路由到另一个阶段而无需等待写回阶段来解决这个问题。

    - 控制冒险：分支指令（if-else）意味着 CPU 在分支解决之前不知道要取哪条指令。分支预测猜测分支将走哪条路并沿预测路径推测性地取指令。现代预测器准确率 >95%，使用历史表和类似神经网络的模式匹配。错误预测成本约 15 个周期（流水线必须清空并重新启动）。

    - 结构冒险：两条指令同时需要相同的硬件资源（例如，都需要内存端口）。通过复制资源或插入停顿解决。

## 内存层次结构

- 计算机内存的基本矛盾：快速内存昂贵且小，便宜内存慢且大。内存层次结构通过利用局部性弥合这个差距：程序倾向于重复访问相同数据（时间局部性）并访问附近数据（空间局部性）。

![内存层次结构金字塔：顶部寄存器（快、小）到底部 HDD（慢、大）](../images/memory_hierarchy.svg)

- 层次结构从最快到最慢：

    - 寄存器：约 0.3 ns 访问，约 KB 总计。在 CPU 内部。
    - L1 缓存：约 1 ns，每核 32-64 KB。分为指令缓存和数据缓存。
    - L2 缓存：约 4 ns，每核 256 KB-1 MB。
    - L3 缓存：约 10 ns，多核共享 8-64 MB。
    - RAM (DRAM)：约 50-100 ns，8-512 GB。主内存。
    - SSD：约 10-100 μs，256 GB-8 TB。持久存储。
    - HDD：约 5-10 ms，1-20 TB。机械式，随机访问非常慢。

- 寄存器和 RAM 之间的速度差距约 300 倍。寄存器和磁盘之间，约 30,000,000 倍。缓存层次结构隐藏这个差距：如果 CPU 需要的数据在 L1 缓存中（缓存命中），访问快。如果不在（缓存未命中），CPU 停顿同时从较慢级别获取数据。

- 缓存相联性确定内存地址可以存储在缓存中的位置：
    - 直接映射：每个地址映射到恰好一个缓存行。简单但引起冲突。
    - 全相联：任何地址可以去任何地方。灵活但搜索昂贵。
    - 组相联（$k$ 路）：每个地址映射到 $k$ 个位置的组。实际 CPU 中使用的实际折衷（通常 4 路或 8 路）。

- 缓存一致性确保所有 CPU 核看到一致的内存视图。当核 1 写入核 2 缓存的地址时，一致性协议（如 MESI）使核 2 的副本无效或更新。这对并发编程（文件 4）至关重要，也是共享内存并行困难的原因之一。

- 对于 ML 从业者，内存层次结构解释了为什么：
    - 矩阵操作应顺序访问内存（行主与列主布局很重要）。
    - 批大小影响性能：大批量摊销内存延迟。
    - 混合精度（float16/bfloat16）使有效内存带宽翻倍，这通常是瓶颈。

## 虚拟内存

- 虚拟内存给每个进程拥有自己大型连续内存空间的错觉，即使物理 RAM 有限并在进程间共享。

- 地址空间分成固定大小的页（通常 4 KB）。页表将虚拟页号映射到物理帧号。当程序访问虚拟地址 0x1234 时，CPU 通过查找页表将其转换为物理地址。

- Translation Lookaside Buffer (TLB) 是页表项的缓存。由于页表驻留在 RAM（慢）中，TLB 将最近使用的转换存储在快速硬件中。TLB 未命中需要在内存中遍历页表，成本数百个周期。

- 缺页中断发生在程序访问不在物理 RAM 中的页时。OS 从磁盘加载页（交换），耗时数百万个周期。过度缺页中断（抖动）严重破坏性能。这就是为什么 ML 训练需要足够 RAM 来保存模型、优化器状态和合理的数据批次。

- 页替换算法在 RAM 满时决定驱逐哪个页：
    - LRU（最近最少使用）：驱逐最长未访问的页。对大多数工作负载在实践中最优。用时钟算法（带引用位的循环列表）在硬件中近似。
    - FIFO：驱逐最旧的页。简单但可能驱逐频繁使用的页。
    - 最优 (Bélády)：驱逐最长不会使用的页。无法实现（需要未来知识）但作为理论基准有用。

- 虚拟内存还提供隔离：每个进程有自己的虚拟地址空间。一个进程中的 bug 无法破坏另一个进程的内存，因为它们的虚拟地址映射到不同的物理帧。这是 OS 安全和稳定性的基础。

## I/O、中断和 DMA

- CPU 需要与外部世界通信：磁盘、网卡、键盘、GPU。这是I/O 子系统。

- 程序化 I/O（轮询）：CPU 在循环中反复检查设备的状态寄存器，等待数据准备好。简单但浪费 CPU 周期自旋而非做有用工作。

- 中断驱动 I/O：设备在数据准备好时发送硬件中断。CPU 继续正常执行直到中断到达，然后运行中断处理程序（内核函数）处理数据。这比轮询高效得多，因为 CPU 在等待时不空闲。

- 中断机制：
    1. 设备通过硬件线信号中断。
    2. CPU 完成当前指令，保存当前状态（寄存器、程序计数器）到栈。
    3. CPU 在中断向量表（函数指针表，每个中断类型一个）中查找中断处理程序地址。
    4. 处理程序在内核态运行，处理 I/O 并返回。
    5. CPU 恢复保存的状态并恢复被中断的程序。

- 这与上下文切换（文件 3）的保存/恢复模式相同，但由硬件触发而非定时器。

- DMA（直接内存访问）：对于大数据传输（磁盘读取、网络包、GPU 内存复制），让 CPU 逐字节复制数据是浪费的。DMA 控制器直接在设备和 RAM 之间传输数据而无需 CPU 参与。CPU 设置传输（源、目标、大小），DMA 控制器处理它，完成后 CPU 获得中断。

- DMA 对 ML 至关重要：当你调用 `model.to('cuda')` 时，数据通过 PCIe 总线上的 DMA 从系统 RAM 传输到 GPU 内存。训练期间，跨 GPU 梯度同步使用基于 DMA 的 RDMA（远程 DMA）进行高带宽、低延迟传输（第 6 章）。

- 总线将 CPU 连接到内存和 I/O 设备。现代系统使用PCIe（外设组件互连快速）用于高速设备（GPU、NVMe SSD、网卡）。PCIe 4.0 每 x16 插槽提供约 32 GB/s；PCIe 5.0 翻倍。总线带宽通常是 GPU 训练的瓶颈：GPU 计算快于数据可以喂给它的速度。

- MMIO（内存映射 I/O）：设备寄存器映射到内存地址。CPU 使用普通加载/存储指令读写这些地址，硬件将访问路由到设备而非 RAM。这将内存和 I/O 访问统一到单个机制，简化硬件和软件。

1. 探索 IEEE 754 浮点表示。将浮点数转换为其二进制表示并观察符号、指数和尾数字段。
```python
import struct

def float_to_bits(f):
    """显示 float32 的 IEEE 754 二进制表示。"""
    packed = struct.pack('>f', f)
    bits = ''.join(f'{byte:08b}' for byte in packed)
    sign = bits[0]
    exponent = bits[1:9]
    mantissa = bits[9:]
    return sign, exponent, mantissa

for val in [1.0, -1.0, 0.1, 0.5, 3.14, float('inf'), float('nan')]:
    s, e, m = float_to_bits(val)
    print(f"{val:>10}  sign={s}  exp={e} ({int(e, 2) - 127:>4d})  mantissa={m[:10]}...")
```

2. 模拟直接映射缓存。跟踪一系列内存访问的命中和未命中。
```python
def simulate_cache(accesses, cache_size=8, block_size=1):
    """模拟直接映射缓存。"""
    cache = [None] * cache_size
    hits, misses = 0, 0

    for addr in accesses:
        cache_line = addr % cache_size
        if cache[cache_line] == addr:
            hits += 1
            status = "HIT "
        else:
            misses += 1
            cache[cache_line] = addr
            status = "MISS"
        print(f"  Access {addr:3d} → line {cache_line}: {status}")

    print(f"\nHits: {hits}, Misses: {misses}, Hit rate: {hits/(hits+misses):.1%}")

# 顺序访问（好局部性）
print("顺序访问：")
simulate_cache([0, 1, 2, 3, 4, 5, 6, 7, 0, 1, 2, 3])

# 跨步访问（冲突未命中）
print("\n跨步访问（步长 = 缓存大小）：")
simulate_cache([0, 8, 0, 8, 0, 8])
```

import jax.numpy as jnp

a = jnp.float32(1e8)
