# ARM 和 NEON

*ARM 处理器为每部智能手机、大多数平板电脑、Apple 的笔记本电脑以及越来越多份额的数据中心服务器提供动力。本文件涵盖 ARM 架构、使用 C++ 内在函数的 NEON SIMD 编程、SVE/SVE2 用于可扩展向量处理、Apple Silicon  specifics，以及实用的向量化内核示例*

- 如果你拥有 iPhone、MacBook 或使用 AWS Graviton 实例，你正在运行 ARM。ARM 的能效使其在移动和嵌入式领域占据主导地位，并在服务器和 ML 推理中越来越有竞争力。理解 ARM SIMD 让你编写的代码在大多数人实际使用的硬件上快速运行。

- 对于生产中的 ARM SIMD 内核的真实示例，请参阅Cactus —— 用于移动设备和可穿戴设备的低延迟 AI 引擎：[github.com/cactus-compute/cactus](https://github.com/cactus-compute/cactus)。Cactus 实现了自定义 ARM NEON 和 NPU 加速的注意力、KV-cache 量化和分块预填充内核，在 ARM CPU 上实现最快的推理，RAM 比其他引擎低 10 倍。它的三层架构（引擎 → 图 → 内核）是本文件中 SIMD 概念如何用于构建生产 ML 基础设施的具体示例。

## ARM 架构基础

- ARM 是RISC（精简指令集计算机）架构（第 13 章）。关键特征：

    - 加载 - 存储架构：算术指令仅在寄存器上操作，从不在内存上直接操作。要从内存添加两个数字，你必须：（1）将它们加载到寄存器中，（2）添加寄存器，（3）将结果存储回内存。这比 x86（可以在一个指令中添加寄存器和内存位置）更简单，但启用更清晰的流水线。

    - 固定宽度指令：每个 ARMv8（AArch64）指令正好 32 位。这使解码快速可预测（不像 x86 的 1-15 字节可变长度指令）。

    - 32 个通用寄存器（x0-x30，每个 64 位）加上栈指针（sp）和零寄存器（xzr）。相比 x86 的 16 个通用寄存器。更多寄存器 = 更少内存访问 = 更快代码。

    - 32 个 SIMD/浮点寄存器（v0-v31，每个 128 位）用于 NEON 和浮点操作。

```cpp
// ARM 汇编（只是为了看看风格 —— 你将使用内在函数，而不是汇编）
// 添加两个寄存器
add x0, x1, x2    // x0 = x1 + x2

// 从内存加载
ldr x0, [x1]      // x0 = *x1（从 x1 中的地址加载 64 位）

// NEON：添加四个浮点数
fadd v0.4s, v1.4s, v2.4s  // v0 = v1 + v2（四个 32 位浮点数）
```

- 你不会编写汇编。你将使用内在函数：C/C++ 函数，1:1 映射到特定指令。编译器处理寄存器分配、调度和其他低级细节。

## NEON：128 位 SIMD

- NEON 是 ARM 的 SIMD 扩展。每个 NEON 寄存器 128 位宽，可以保存：

| 数据类型 | 每寄存器元素数 | 符号 |
|-----------|-----------------------|----------|
| float32 | 4 | `float32x4_t` |
| float16 | 8 | `float16x8_t` |
| int32 | 4 | `int32x4_t` |
| int16 | 8 | `int16x8_t` |
| int8 | 16 | `int8x16_t` |

- 128 位比 x86 的 AVX（256 位）或 AVX-512（512 位）窄。但 ARM 以出色的能效和广泛可用性补偿。

### NEON 内在函数：基础

- NEON 内在函数遵循命名约定：`v[operation][qualifier]_[type]`

```cpp
#include <arm_neon.h>

// 从内存加载 4 个浮点数到 NEON 寄存器
float32x4_t a = vld1q_f32(ptr);        // vld1q = 向量加载 1，q = 128 位（四倍）

// 从 NEON 寄存器存储 4 个浮点数到内存
vst1q_f32(out_ptr, a);                   // vst1q = 向量存储 1，q = 128 位

// 算术
float32x4_t c = vaddq_f32(a, b);        // c = a + b（4 个浮点数）
float32x4_t d = vmulq_f32(a, b);        // d = a * b（4 个浮点数）
float32x4_t e = vfmaq_f32(c, a, b);     // e = c + a * b（融合乘加，4 个浮点数）

// 比较（返回掩码：真为全 1，假为全 0）
uint32x4_t mask = vcgtq_f32(a, b);      // mask[i] = (a[i] > b[i]) ? 0xFFFFFFFF : 0

// 基于掩码选择元素（如 numpy.where）
float32x4_t result = vbslq_f32(mask, a, b);  // result[i] = mask[i] ? a[i] : b[i]

// 归约：将 4 个元素求和为标量
float total = vaddvq_f32(a);             // total = a[0] + a[1] + a[2] + a[3]
```

- `vfmaq_f32`（融合乘加）是 ML 最重要的 SIMD 指令。它在一条指令中计算 $c = c + a \times b$，单次舍入（比单独乘法然后加法更准确）。点积、矩阵乘法和卷积由 FMA 构建。

### 实用示例：向量化点积

- 点积是矩阵乘法的内循环。让我们用标量 C++ 编写它，然后用 NEON 向量化。

```cpp
#include <arm_neon.h>

// 标量点积
float dot_scalar(const float* a, const float* b, int n) {
    float sum = 0.0f;
    for (int i = 0; i < n; i++) {
        sum += a[i] * b[i];
    }
    return sum;
}

// NEON 向量化点积
float dot_neon(const float* a, const float* b, int n) {
    float32x4_t sum_vec = vdupq_n_f32(0.0f);  // 初始化 4 个累加器为 0

    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t va = vld1q_f32(a + i);     // 从 a 加载 4 个元素
        float32x4_t vb = vld1q_f32(b + i);     // 从 b 加载 4 个元素
        sum_vec = vfmaq_f32(sum_vec, va, vb);   // sum_vec += va * vb
    }

    // 将 4 个累加器归约为单个标量
    float sum = vaddvq_f32(sum_vec);

    // 处理剩余元素（如果 n 不是 4 的倍数）
    for (; i < n; i++) {
        sum += a[i] * b[i];
    }

    return sum;
}
```

- 关键 C++ 概念：
    - `const float*`：指向只读浮点数据的指针。`const` 承诺我们不会通过这个指针修改数据。
    - `a + i`：指针算术。`a + i` 指向数组的第 $i$ 个元素（与 `&a[i]` 相同）。
    - 末尾的"清理循环"处理 $n$ 不是 4 的倍数的情况。这是 SIMD 代码中的通用模式：在向量化块中处理主体，然后在标量代码中处理剩余部分。

- 为什么 `sum_vec` 中有 4 个累加器：我们使用 4 个独立累加器（每个 SIMD 车道一个），而不是单个标量累加器。这避免数据依赖：每次迭代的 FMA 依赖于 `sum_vec`，但具有 4 个独立车道，CPU 可以流水线化 FMA。最后，我们将 4 个部分和归约为一个。

### 实用示例：向量化 ReLU

```cpp
#include <arm_neon.h>

void relu_neon(const float* input, float* output, int n) {
    float32x4_t zero = vdupq_n_f32(0.0f);

    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t x = vld1q_f32(input + i);
        float32x4_t result = vmaxq_f32(x, zero);  // max(x, 0) = ReLU
        vst1q_f32(output + i, result);
    }

    // 标量清理
    for (; i < n; i++) {
        output[i] = input[i] > 0 ? input[i] : 0;
    }
}
```

- `vmaxq_f32` 计算两个向量的逐元素最大值。由于一个向量全为零，这正好是 ReLU。无分支、无比较 —— 只有一条指令。

## I8MM：整数矩阵乘法

- I8MM（Int8 矩阵乘法）是 ARMv8.6 扩展，添加 INT8 矩阵乘法与 INT32 累加的专用指令 —— 正是量化 ML 推理需要的。

- 关键指令是`SMMLA`（有符号矩阵乘加）：它采用两个 8×2 INT8 值块，并将结果累加到 2×2 INT32 块中：

```cpp
#include <arm_neon.h>

// I8MM：将两个 8 元素 INT8 向量相乘，累加到 4 个 INT32 结果
// 这从 2x8 × 8x2 输入块计算输出矩阵的 2x2 块
void matmul_i8mm_tile(const int8_t* A, const int8_t* B, int32_t* C) {
    // 从 A 加载 8 字节（2 行 4 元素，打包）
    int8x16_t va = vld1q_s8(A);   // 16 字节 = 2 行 × 8 元素
    int8x16_t vb = vld1q_s8(B);   // 16 字节 = 2 行 × 8 元素

    // 加载现有累加器（2x2 = 4 个 int32 值）
    int32x4_t acc = vld1q_s32(C);

    // I8MM 指令：acc += A_tile × B_tile^T
    // 从 2×8 × 8×2 输入计算 2×2 输出
    acc = vmmlaq_s32(acc, va, vb);  // I8MM 指令

    vst1q_s32(C, acc);
}
```

- 为什么 I8MM 重要：没有 I8MM，NEON 上的 INT8 matmul 需要展宽乘法（`vmull`），然后成对添加 —— 每个输出元素多条指令。有了 I8MM，硬件在单条指令中完成 8 元素点积（2×8 × 8×2 = 2×2）。对于 INT8 推理工作负载，这比纯 NEON 快 4-8 倍。

- 可用性：Apple M1+（所有 Apple Silicon）、ARM Cortex-A510/A710/X2+（ARMv9）、AWS Graviton3+。使用 `#ifdef __ARM_FEATURE_MATMUL_INT8` 检查。

- 对于 ML 推理：在 ARM 服务器（Graviton）或 Apple Silicon 上运行的 INT8 量化模型（第 18 章）从 I8MM 受益巨大。ONNX Runtime 和 llama.cpp 等框架在运行时检测 I8MM 并自动使用优化内核。

## SME 和 SME2：可扩展矩阵扩展

- SME（可扩展矩阵扩展）是 ARM 对 Intel AMX 和 NVIDIA Tensor Core 的回答：用于矩阵运算的专用硬件。SME2（ARMv9.2）进一步扩展它。

- SME 引入ZA 图块寄存器：2D 矩阵存储在硬件中，高达 SVL×SVL 字节（其中 SVL 是流式向量长度，通常每维 128-512 位）。与 NEON（1D 向量）或甚至 SVE（1D 可扩展向量）不同，SME 本机操作2D 图块。

- 编程模型有两种模式：
    - 普通模式：标准 ARM 执行（NEON、SVE 照常工作）。
    - 流式 SVE 模式：通过 `smstart` 进入，启用 SME 指令。SVE 指令在此模式下也工作，但可能使用不同的寄存器宽度。

```cpp
#include <arm_sme.h>

// SME2：外积累加用于矩阵乘法
// 将 A_col × B_row 累加到 ZA 图块寄存器
void sme2_matmul_outer(const float* A_col, const float* B_row, int K) {
    // 进入流式模式
    // smstart;  // （通过编译器内在函数或内联汇编完成）

    // 清零 ZA 图块累加器
    svzero_za();

    for (int k = 0; k < K; k++) {
        // 将 A 的列和 B 的行加载到 SVE 寄存器
        svfloat32_t a = svld1_f32(svptrue_b32(), &A_col[k * SVL]);
        svfloat32_t b = svld1_f32(svptrue_b32(), &B_row[k * SVL]);

        // 外积：ZA += a × b^T
        // 这在一条指令中累加 SVL×SVL 图块
        svmopa_za32_f32_m(0, svptrue_b32(), svptrue_b32(), a, b);
    }

    // 将 ZA 图块存储到内存
    // svst1_za(...);

    // 退出流式模式
    // smstop;
}
```

- 关键概念：
    - `svmopa`（外积累加）：核心 SME 指令。它计算两个向量的完整外积并累加到 ZA 图块。对于 SVL=512 位（16 个浮点数），这是 16×16 外积 —— 一条指令 256 个 FMA 操作。
    - ZA 图块：在流式模式内在指令间持久。你将多个外积（每个 K 迭代一个）累加到同一图块中，构建完整矩阵乘法图块。
    - 流式模式：SME 指令仅在流式模式下工作。进入/退出流式模式的开销意味着 SME 最适合持续矩阵计算，而不是短脉冲。

- SME2 补充：多向量操作（同时处理 2 个或 4 个 SVE 向量）、额外图块操作，以及与普通模式的改进集成。

- 可用性：ARM Neoverse V2（AWS Graviton4）、一些即将推出的移动芯片。Apple Silicon 上还没有（截至 2026 年）。SME 仍处于早期阶段 —— 大多数 ML 框架还没有 SME 优化内核。

- 进展：NEON（128 位向量、逐元素）→ I8MM（INT8 矩阵图块）→ SVE（可扩展向量）→ SME（可扩展 2D 矩阵图块）。每一代在硬件中更接近本机矩阵运算。

## SVE 和 SVE2：可扩展向量扩展

- NEON 具有固定 128 位宽度。SVE（可扩展向量扩展）引入向量长度不可知（VLA）编程：你编写代码一次，它在任何向量宽度（128 到 2048 位）的硬件上运行。硬件在运行时确定宽度。

```cpp
#include <arm_sve.h>

void add_sve(const float* a, const float* b, float* c, int n) {
    int i = 0;
    svbool_t pred = svwhilelt_b32(i, n);  // 谓词：哪些车道活动

    while (svptest_any(svptrue_b32(), pred)) {
        svfloat32_t va = svld1(pred, a + i);
        svfloat32_t vb = svld1(pred, b + i);
        svst1(pred, c + i, svadd_x(pred, va, vb));

        i += svcntw();  // 按硬件向量宽度前进（以 32 位元素为单位）
        pred = svwhilelt_b32(i, n);
    }
}
```

- 谓词寄存器（`svbool_t`）替换标量清理循环。每个车道有一个谓词位：活动车道参与，非活动车道被掩码关闭。`svwhilelt_b32(i, n)` 指令创建谓词，其中对应于 `i, i+1, ..., n-1` 的车道活动。这自动处理尾部。

- `svcntw()` 在运行时返回每个向量寄存器的 32 位元素数。在具有 256 位 SVE 的 CPU 上，这返回 8。在 512 位 SVE 上，它返回 16。你的代码自动适应。

- SVE 在 ARM Neoverse V1/V2（AWS Graviton3/4、一些服务器芯片）上可用。Apple Silicon 上还没有。

## Apple Silicon  specifics

- Apple 的 M 系列芯片（M1、M2、M3、M4）是基于 ARM 的，具有自定义微架构：

- 性能和能效核心：P 核心（Firestorm/Avalanche 等）用于重型计算，E 核心（Icestorm/Blizzard 等）用于后台任务。调度器将线程分配给适当的核心类型。

- AMX（Apple 矩阵扩展）：专用矩阵乘法单元，与 NEON 分开。AMX 未记录（Apple 不发布 ISA），但 Accelerate 框架在内部使用它进行 BLAS 操作。当你在 Mac 上调用 `np.dot` 时，它通过 Accelerate，它使用 AMX。你不能直接编程 AMX（无需逆向工程）。

- 统一内存：CPU 和 GPU 共享相同的物理 RAM。在其他系统上，数据必须从 CPU 内存复制到 GPU 内存（通过 PCIe，约 32 GB/s）。在 Apple Silicon 上，无复制 —— GPU 读取 CPU 写入的同一内存。这消除了 ML 工作负载的主要瓶颈。

- 神经引擎：16 核专用 ML 加速器。为 INT8 推理执行约 30 TOPS（每秒万亿次操作）。Core ML 用于设备推理。

- 对于 Apple Silicon 上的 ML：使用 MLX（Apple 的 ML 框架），专为统一内存架构设计。PyTorch 也有 MPS（Metal Performance Shaders）后端支持，尽管它不如 CUDA 成熟。

## 自动向量化

- 编写 SIMD 内在函数很繁琐。编译器可以自动向量化你的代码吗？

- 可以，但有注意事项。现代编译器（GCC、Clang）可以自动向量化简单循环：

```cpp
// 编译器可以自动向量化这个（使用 -O3 -march=native）
void add_auto(const float* a, const float* b, float* c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}
```

- 帮助自动向量化的模式：
    - 具有已知遍历次数的简单循环。
    - 迭代之间无数据依赖（`c[i]` 不依赖于 `c[i-1]`）。
    - 连续内存访问（无分散/聚集）。
    - `const` 和 `restrict` 指针（告诉编译器数组不重叠）。

```cpp
// restrict 告诉编译器：a、b、c 指向不重叠的内存
void add_restrict(const float* __restrict__ a,
                  const float* __restrict__ b,
                  float* __restrict__ c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}
```

- 没有 `restrict`，编译器必须假设 `c` 可能与 `a` 或 `b` 重叠（写入 `c[i]` 可能改变 `a[i+1]`），阻止向量化。

- 阻止自动向量化的模式：
    - 数据依赖：`a[i] = a[i-1] + b[i]`（每次迭代依赖于前一次）。
    - 复杂控制流：循环内的 `if` 语句（除非编译器可以转换为谓词）。
    - 循环内的函数调用（除非函数内联）。
    - 指针别名（数组可能重叠，无 `restrict`）。

- 检查自动向量化：使用编译器标志查看什么被向量化：

```bash
# GCC：显示向量化决策
g++ -O3 -march=native -fopt-info-vec-optimized code.cpp

# Clang：显示向量化报告
clang++ -O3 -march=native -Rpass=loop-vectorize code.cpp
```

- 何时使用内在函数 vs 自动向量化：从干净的 C++ 和编译器优化开始。如果编译器向量化你的循环，很好。如果性能仍然不足，检查编译器的向量化报告以了解原因，然后只为关键内循环编写内在函数。过早的内在函数使代码不可读，没有保证的好处。

## 编程任务（在 ARM 上用 g++ 或 clang++ 编译 —— Mac M 系列或 Linux aarch64）

1. 编写标量点积和 NEON 向量化点积。基准测试两者并测量加速。
```cpp
// task1_neon_dot.cpp
// 编译（Mac/ARM Linux）：clang++ -O3 -o task1 task1_neon_dot.cpp
// 注意：NEON 在 AArch64 上默认启用，无需特殊标志

#include <iostream>
#include <chrono>
#include <vector>
#include <arm_neon.h>

float dot_scalar(const float* a, const float* b, int n) {
    float sum = 0.0f;
    for (int i = 0; i < n; i++) {
        sum += a[i] * b[i];
    }
    return sum;
}

float dot_neon(const float* a, const float* b, int n) {
    float32x4_t sum_vec = vdupq_n_f32(0.0f);
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t va = vld1q_f32(a + i);
        float32x4_t vb = vld1q_f32(b + i);
        sum_vec = vfmaq_f32(sum_vec, va, vb);
    }
    float sum = vaddvq_f32(sum_vec);
    for (; i < n; i++) sum += a[i] * b[i];
    return sum;
}

int main() {
    const int N = 10'000'000;
    std::vector<float> a(N, 1.0f), b(N, 2.0f);

    // 预热
    volatile float s1 = dot_scalar(a.data(), b.data(), N);
    volatile float s2 = dot_neon(a.data(), b.data(), N);

    // 基准测试标量
    auto start = std::chrono::high_resolution_clock::now();
    for (int t = 0; t < 100; t++) {
        s1 = dot_scalar(a.data(), b.data(), N);
    }
    auto end = std::chrono::high_resolution_clock::now();
    double scalar_ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;

    // 基准测试 NEON
    start = std::chrono::high_resolution_clock::now();
    for (int t = 0; t < 100; t++) {
        s2 = dot_neon(a.data(), b.data(), N);
    }
    end = std::chrono::high_resolution_clock::now();
    double neon_ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;

    std::cout << "标量：" << scalar_ms << " 毫秒 (结果：" << s1 << ")\n";
    std::cout << "NEON:   " << neon_ms << " 毫秒 (结果：" << s2 << ")\n";
    std::cout << "加速：" << scalar_ms / neon_ms << "x\n";
    return 0;
}
```

2. 实现 NEON ReLU 和 softmax 最大值查找。练习不同操作的加载→计算→存储模式。
```cpp
// task2_neon_ops.cpp
// 编译：clang++ -O3 -o task2 task2_neon_ops.cpp

#include <iostream>
#include <vector>
#include <cmath>
#include <arm_neon.h>

void relu_neon(const float* in, float* out, int n) {
    float32x4_t zero = vdupq_n_f32(0.0f);
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t x = vld1q_f32(in + i);
        vst1q_f32(out + i, vmaxq_f32(x, zero));
    }
    for (; i < n; i++) out[i] = in[i] > 0 ? in[i] : 0;
}

float max_neon(const float* data, int n) {
    float32x4_t max_vec = vdupq_n_f32(-INFINITY);
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        max_vec = vmaxq_f32(max_vec, vld1q_f32(data + i));
    }
    float result = vmaxvq_f32(max_vec);
    for (; i < n; i++) result = result > data[i] ? result : data[i];
    return result;
}

int main() {
    std::vector<float> data = {-3, 1, -1, 4, 2, -5, 0, 7, -2, 3};
    std::vector<float> out(data.size());

    relu_neon(data.data(), out.data(), data.size());
    std::cout << "ReLU: ";
    for (float x : out) std::cout << x << " ";
    std::cout << "\n";

    float mx = max_neon(data.data(), data.size());
    std::cout << "最大值：" << mx << " (预期：7)\n";
    return 0;
}
```

3. 比较自动向量化代码与手写 NEON 内在函数。使用 `-fopt-info-vec`（GCC）或 `-Rpass=loop-vectorize`（Clang）编译以查看编译器的操作。
```cpp
// task3_auto_vs_manual.cpp
// 编译：clang++ -O3 -Rpass=loop-vectorize -o task3 task3_auto_vs_manual.cpp
//    或：g++ -O3 -fopt-info-vec-optimized -o task3 task3_auto_vs_manual.cpp

#include <iostream>
#include <chrono>
#include <vector>
#include <arm_neon.h>

// 让编译器自动向量化
void add_auto(const float* __restrict__ a, const float* __restrict__ b,
              float* __restrict__ c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}

// 手写 NEON
void add_neon(const float* a, const float* b, float* c, int n) {
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        vst1q_f32(c + i, vaddq_f32(vld1q_f32(a + i), vld1q_f32(b + i)));
    }
    for (; i < n; i++) c[i] = a[i] + b[i];
}

int main() {
    const int N = 10'000'000;
    std::vector<float> a(N, 1.0f), b(N, 2.0f), c(N);

    auto bench = [&](auto fn, const char* name) {
        fn(a.data(), b.data(), c.data(), N);  // 预热
        auto start = std::chrono::high_resolution_clock::now();
        for (int t = 0; t < 100; t++) fn(a.data(), b.data(), c.data(), N);
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        std::cout << name << ": " << ms << " 毫秒\n";
    };

    bench(add_auto, "自动向量化");
    bench(add_neon, "手写 NEON");
    // 它们应该非常接近 —— 编译器很好地自动向量化这个简单循环
    return 0;
}
```
