# GPU 架构和 CUDA

*GPU 通过提供数千个核心实现大规模并行性， transformed AI。本文件涵盖 GPU 与 CPU 设计哲学、GPU 内存层次结构、C++ 中的 CUDA 编程、SIMT 执行模型、内存访问模式、同步、流、分析和 NVIDIA GPU 代际，编写和理解 GPU 内核所需的知识。*

- 对于带完整工作示例的实战 CUDA 教程，请参阅 companion 仓库：[github.com/HenryNdubuaku/cuda-tutorials](https://github.com/HenryNdubuaku/cuda-tutorials)。

- 现代 NVIDIA GPU 有超过 10,000 个 CUDA 核心。CPU 有 4-128 个核心。这 100-1000 倍核心优势是 GPU 主导 ML 的原因：训练变压器需要数万亿乘加运算，GPU 以 CPU 无法匹配的规模并行处理它们。

- 即使你从不编写 CUDA 内核，理解 GPU 架构解释：为什么批量大小重要（需要足够工作饱和 GPU）、为什么内存通常是瓶颈（不是计算）、为什么某些运算（scatter、条件分支）在 GPU 上慢。

## GPU 与 CPU：根本不同设计

- CPU 设计用于**延迟**：最小化完成一个任务的时间。它将大部分晶体管预算用于缓存、分支预测器和乱序执行——都是使一个线程快的技巧。

- GPU 设计用于**吞吐量**：最大化每秒完成的任务数。它将大部分晶体管用于执行单元（ALU）。单个线程慢，但有数千个。

| | CPU | GPU |
|--|-----|-----|
| 核心 | 4-128（复杂、快） | 1,000-20,000（简单、慢） |
| 时钟速度 | 3-5 GHz | 1-2.5 GHz |
| 缓存 | 大（32 MB+ L3） | 小（每 SM 共享内存） |
| 分支预测 | 复杂 | 无（所有线程遵循相同路径） |
| 最适合 | 低延迟、复杂控制流 | 高吞吐量、数据并行工作 |
| 典型 FLOPS (FP32) | 1-5 TFLOPS | 30-80 TFLOPS |
| 内存带宽 | 50-100 GB/s | 1-3 TB/s |

- GPU 的内存带宽优势（10-30 倍）通常比其计算优势更重要。许多 ML 运算是内存受限（元素级运算、归一化、注意力），GPU 的带宽让它足够快地将数据馈送到核心。

## GPU 内存层次结构

- 理解 GPU 内存至关重要，因为**内存访问是主要瓶颈**，不是计算。

| 内存 | 大小 | 延迟 | 带宽 | 范围 |
|--------|------|---------|-----------|-------|
| 寄存器 | ~256 KB 每 SM | 0 周期 | 最高 | 每线程 |
| 共享内存 | 48-228 KB 每 SM | ~5 周期 | ~20 TB/s | 每线程块 |
| L1 缓存 | 128-256 KB 每 SM | ~30 周期 | | 每 SM |
| L2 缓存 | 4-96 MB | ~200 周期 | ~6 TB/s | 全局 |
| 全局内存 (HBM) | 24-192 GB | ~400 周期 | 1-3.3 TB/s | 全局 |

- **寄存器**最快但最有限。每个线程有私有寄存器集（通常最多 255 个）。每线程使用太多寄存器减少**占用率**（更少线程可以同时运行）。

- **共享内存**是程序员管理的缓存，由块内所有线程共享。它是编写快速 CUDA 内核的关键：从慢全局内存加载数据图块到快共享内存，然后在其上计算。这是主导 GPU 编程的**分块**模式。

- **全局内存（HBM）**：主要 GPU 内存（VRAM）。大但慢（400 周期延迟）。所有数据从这里开始和结束。内核优化的目标是最小化全局内存访问。

## CUDA 编程模型

- CUDA（Compute Unified Device Architecture）是 NVIDIA 的 GPU 编程模型。你编写**内核**：在 GPU 上运行的函数，由数千线程同时执行。

### 层次结构：网格、块、线程

```
网格（整个启动）
├── 块 (0,0)
│   ├── 线程 (0,0)
│   ├── 线程 (1,0)
│   ├── 线程 (2,0)
│   └── ...（每块最多 1024 线程）
├── 块 (1,0)
│   ├── 线程 (0,0)
│   └── ...
└── ...（可能数百万块）
```

- **线程**：最小单元。每个线程在块内有唯一 ID（`threadIdx.x`）。
- **块**：线程组，可以共享内存和同步。块 ID：`blockIdx.x`。块大小：`blockDim.x`（最多 1024 线程）。
- **网格**：单个内核启动的所有块。可以是 1D、2D 或 3D。

- 每个线程计算其全局索引：`int idx = blockIdx.x * blockDim.x + threadIdx.x;`

### 你的第一个 CUDA 内核

```cpp
// vector_add.cu — CUDA 源文件（.cu 扩展名）

#include <stdio.h>

// __global__ 标记此为 GPU 内核（从 CPU 调用，在 GPU 上运行）
__global__ void vector_add(const float* a, const float* b, float* c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {           // 边界检查（网格可能大于数据）
        c[idx] = a[idx] + b[idx];
    }
}

int main() {
    int n = 1 << 20;  // 约 100 万元素
    size_t bytes = n * sizeof(float);

    // 分配主机（CPU）内存
    float *h_a = new float[n];
    float *h_b = new float[n];
    float *h_c = new float[n];

    // 初始化
    for (int i = 0; i < n; i++) {
        h_a[i] = 1.0f;
        h_b[i] = 2.0f;
    }

    // 分配设备（GPU）内存
    float *d_a, *d_b, *d_c;
    cudaMalloc(&d_a, bytes);
    cudaMalloc(&d_b, bytes);
    cudaMalloc(&d_c, bytes);

    // 从 CPU 复制数据到 GPU
    cudaMemcpy(d_a, h_a, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(d_b, h_b, bytes, cudaMemcpyHostToDevice);

    // 启动内核：每块 256 线程，足够块覆盖 n 元素
    int block_size = 256;
    int grid_size = (n + block_size - 1) / block_size;  // 向上取整除法
    vector_add<<<grid_size, block_size>>>(d_a, d_b, d_c, n);

    // 从 GPU 复制结果到 CPU
    cudaMemcpy(h_c, d_a, bytes, cudaMemcpyDeviceToHost);

    // 验证
    printf("c[0] = %f (expected 3.0)\n", h_c[0]);

    // 释放内存
    cudaFree(d_a); cudaFree(d_b); cudaFree(d_c);
    delete[] h_a; delete[] h_b; delete[] h_c;

    return 0;
}
```

```bash
# 用 NVIDIA 编译器编译
nvcc -O3 -o vector_add vector_add.cu
./vector_add
```

- **CUDA 中的关键 C++ 概念**：
    - `__global__`：CUDA 关键字，标记内核函数。从 CPU（`host`）调用，在 GPU（`device`）上运行。
    - `<<<grid_size, block_size>>>`：内核启动语法。指定使用多少块和线程。
    - `cudaMalloc` / `cudaFree`：分配/释放 GPU 内存（如 `new`/`delete` 但用于 GPU）。
    - `cudaMemcpy`：在 CPU 和 GPU 之间复制数据。这通常是最大瓶颈（PCIe 带宽约 32 GB/s，而 GPU 内存带宽约 3 TB/s）。

### Warps 和 SIMT

- GPU 以 32 个线程组执行称为**warps**。warp 中所有 32 个线程同时执行**相同指令**（单指令多线程 — SIMT）。这相当于 GPU 的 SIMD，但在线程级别。

- **Warp 分歧**发生在同一 warp 中的线程采用 `if` 语句的不同分支时。GPU 不能在一个 warp 中同时执行两个不同指令，所以它顺序执行两个分支，掩蔽出不应该参与的线程。这减半性能（或更糟）。

```cpp
// 坏：warp 分歧（同一 warp 中的线程采用不同路径）
if (threadIdx.x % 2 == 0) {
    c[idx] = a[idx] + b[idx];    // 偶数线程做这个
} else {
    c[idx] = a[idx] - b[idx];    // 奇数线程做这个（同一 warp，序列化）
}

// 更好：无分支（无分歧）
float sign = (threadIdx.x % 2 == 0) ? 1.0f : -1.0f;
c[idx] = a[idx] + sign * b[idx];  // 所有线程执行相同指令
```

### 内存合并

- **合并访问**：当连续线程访问连续内存地址时，GPU 将它们组合成单个内存事务。这对性能至关重要。

```cpp
// 好：合并 — 线程 0 读 a[0]，线程 1 读 a[1]，...
c[idx] = a[idx] + b[idx];

// 坏：跨步 — 线程 0 读 a[0]，线程 1 读 a[stride]，...
c[idx] = a[idx * stride] + b[idx * stride];  // stride > 1 浪费带宽
```

- 对于 32 线程的 warp，合并访问在一个事务中加载 128 字节（32 × 4 字节用于 float32）。跨步访问需要多个事务，每个加载 128 字节但只使用一小部分。32 的 stride 是最坏情况：每个事务加载 128 字节但只有一个线程使用 4 字节（3% 利用率）。

### 共享内存和分块

- **分块模式** 是最重要的 GPU 优化技术。思想：从慢全局内存加载数据块到快共享内存，计算，然后写回结果。

```cpp
// 带共享内存分块的矩阵乘法（简化）
__global__ void matmul_tiled(const float* A, const float* B, float* C,
                              int M, int N, int K) {
    // 共享内存用于 A 的一个图块和 B 的一个图块
    __shared__ float tile_A[TILE_SIZE][TILE_SIZE];
    __shared__ float tile_B[TILE_SIZE][TILE_SIZE];

    int row = blockIdx.y * TILE_SIZE + threadIdx.y;
    int col = blockIdx.x * TILE_SIZE + threadIdx.x;
    float sum = 0.0f;

    // 循环超过图块
    for (int t = 0; t < (K + TILE_SIZE - 1) / TILE_SIZE; t++) {
        // 加载 A 和 B 的一个图块到共享内存
        if (row < M && t * TILE_SIZE + threadIdx.x < K)
            tile_A[threadIdx.y][threadIdx.x] = A[row * K + t * TILE_SIZE + threadIdx.x];
        else
            tile_A[threadIdx.y][threadIdx.x] = 0.0f;

        if (col < N && t * TILE_SIZE + threadIdx.y < K)
            tile_B[threadIdx.y][threadIdx.x] = B[(t * TILE_SIZE + threadIdx.y) * N + col];
        else
            tile_B[threadIdx.y][threadIdx.x] = 0.0f;

        __syncthreads();  // 等待所有线程完成加载

        // 从这个图块计算部分点积
        for (int k = 0; k < TILE_SIZE; k++) {
            sum += tile_A[threadIdx.y][k] * tile_B[k][threadIdx.x];
        }

        __syncthreads();  // 等待再加载下一个图块
    }

    if (row < M && col < N)
        C[row * N + col] = sum;
}
```

- **`__shared__`**：声明由块内所有线程共享的内存（快、片上）。
- **`__syncthreads()`**：屏障，等待块内所有线程到达此点。在写入共享内存和从它读取之间需要（否则一些线程读取陈旧数据）。
- **为什么分块有效**：没有它，每个线程为每个乘法从全局内存加载。有分块，TILE_SIZE × TILE_SIZE 数据块加载一次到共享内存并被块内所有线程重用。重用因子是 TILE_SIZE，减少全局内存流量这个因子。

## 流和并发

- 默认情况下，CUDA 运算是顺序的：CPU 启动内核，等待完成，然后启动下一个。**流**启用重叠：

```cpp
cudaStream_t stream1, stream2;
cudaStreamCreate(&stream1);
cudaStreamCreate(&stream2);

// 这些运算可以重叠：不同流并发执行
cudaMemcpyAsync(d_a, h_a, bytes, cudaMemcpyHostToDevice, stream1);
cudaMemcpyAsync(d_b, h_b, bytes, cudaMemcpyHostToDevice, stream2);

kernel1<<<grid, block, 0, stream1>>>(d_a, d_c);
kernel2<<<grid, block, 0, stream2>>>(d_b, d_d);
```

- 流重叠数据传输与计算：当一个流的内核运行时，另一个流复制数据。这隐藏 PCIe 传输延迟并保持 GPU 忙碌。

## 分析 CUDA 代码

```bash
# NVIDIA Nsight Compute：内核级分析
ncu --set full ./my_program

# NVIDIA Nsight Systems：系统级时间线
nsys profile ./my_program

# 快速指标
ncu --metrics sm__throughput,dram__throughput ./my_program
```

- **寻找什么**：
    - **占用率**：SM 容量的使用分数。低占用率（< 50%）意味着太少线程隐藏内存延迟。原因：每线程太多寄存器、每块太多共享内存。
    - **内存吞吐量**：与峰值带宽比较。如果实现 < 50% 峰值，内存访问模式效率低（非合并、银行冲突）。
    - **计算吞吐量**：与峰值 FLOPS 比较。如果内存和计算吞吐量都低，内核是延迟受限（不够并行）。

## 高级优化技术

- 超越合并和共享内存分块的基础，高性能 GPU（和 CPU）代码使用几种高级技术：

### 数据布局：AoS 与 SoA

- **结构数组（AoS）**：每个元素存储其所有字段在一起。`[{x,y,z}, {x,y,z}, {x,y,z}]`。
- **数组结构（SoA）**：每个字段存储在自己的连续数组中。`{[x,x,x], [y,y,y], [z,z,z]}`。

```cpp
// AoS：对 SIMD/GPU 坏（访问所有 x 值接触非连续内存）
struct Particle { float x, y, z, mass; };
Particle particles[N];
// particles[0].x, particles[1].x 相距 16 字节

// SoA：对 SIMD/GPU 好（所有 x 值连续）
struct Particles {
    float x[N], y[N], z[N], mass[N];
};
// x[0], x[1] 相距 4 字节 — 完美用于合并访问和 SIMD
```

- SoA 几乎总是对数据并行工作负载更快（SIMD、GPU）。AoS 在总是一起访问一个元素的所有字段时更好（数值代码中罕见）。PyTorch 张量本质上是 SoA：每个特征是连续维度。

### 软件预取

- CPU 可以告诉在需要前开始加载数据，隐藏内存延迟：

```cpp
#include <xmmintrin.h>  // 用于 _mm_prefetch

for (int i = 0; i < n; i += 4) {
    _mm_prefetch((char*)(a + i + 64), _MM_HINT_T0);  // 预取 64 元素超前
    // 用 SIMD 处理 a[i:i+4]
    __m128 va = _mm_load_ps(a + i);
    // ...
}
```

- 预取指令是提示：如果数据已经在缓存中，它是无操作。如果不在，CPU 在执行其他指令时在后台获取它。预取距离（此示例中 64 元素超前）应调整以匹配内存延迟和循环迭代时间。

### 内核融合

- **内核融合**组合多个运算到单个内核以避免写入中间结果到内存。这是对 ML 影响最大的 GPU 优化：

```
// 未融合：3 个内核启动，3 个全局内存往返
y = matmul(x, W)     // 写 y 到全局内存
z = y + bias          // 读 y，写 z
out = relu(z)         // 读 z，写 out

// 融合：1 个内核启动，1 个全局内存写
out = fused_matmul_bias_relu(x, W, bias)  // y 和 z 从不离开 SRAM
```

- 对于内存受限运算（偏置加、ReLU、层归一化），内存流量主导执行时间。融合完全消除流量。PyTorch 的 `torch.compile` 和 Triton 自动或以最小努力启用融合。

### 混合精度内核

- 使用更低精度（FP16、BF16、INT8）用于计算，更高精度（FP32）用于累加，给出两全其美：

```cpp
// Tensor Core：乘法 FP16 矩阵，累加在 FP32
// 每个 Tensor Core 指令：D (FP32) = A (FP16) × B (FP16) + C (FP32)
nvcuda::wmma::mma_sync(c_frag, a_frag, b_frag, c_frag);
```

- FP16 是 FP32 的 2 倍小，所以它加倍内存带宽（通常瓶颈）并适合 2 倍更多数据在缓存中。Tensor Core 以 8-16 倍于 FP32 CUDA 核心的速率处理 FP16。这就是为什么混合精度训练（第 6 章）提供 2-3 倍加速，最小准确性损失。

### 内存池分配器

- `cudaMalloc` 慢（每调用约 1 毫秒）因为它与 GPU 同步。在每次迭代分配临时缓冲区的训练循环中，这累积。

- **内存池**（PyTorch 的缓存分配器、CUDA 内存池）预分配大块 GPU 内存并从中子分配，无系统调用：

```python
# PyTorch 自动做这个 — 但理解为什么重要
# 每个 torch.empty() 重用池中的内存，无 cudaMalloc
temp = torch.empty(1024, 1024, device='cuda')  # 微秒，不是毫秒
```

- 这就是为什么 PyTorch 的 `torch.cuda.memory_allocated()` 和 `torch.cuda.max_memory_allocated()` 不同：allocated 是当前使用的，max 是峰值（池可能持有比当前使用更多）。

### 分析引导优化

- 不要盲目优化。**先分析**，识别瓶颈，优化，再分析。屋顶线模型（文件 01）告诉你瓶颈是内存还是计算：

    - **内存受限**（低算术强度）：优化数据布局（SoA）、融合内核、使用更低精度、预取。
    - **计算受限**（高算术强度）：使用 Tensor Core、增加并行、使用更快指令（FMA）。
    - **延迟受限**（不足并行）：增加占用率、减少寄存器使用、启动更多线程。

- 大多数 ML 工作负载是**内存受限**。令人惊讶的含义：更快 GPU（更多 FLOPS）通常无帮助。更快内存（HBM3 vs HBM2e）帮助更多。这就是为什么 A100→H100 升级不仅是关于 FLOPS — H100 还有 2 倍内存带宽。

## NVIDIA GPU 代际

| 代 | 年份 | 关键创新 | AI 相关性 |
|------------|------|----------------|--------------|
| Pascal (P100) | 2016 | HBM2, NVLink | 第一个严肃深度学习 GPU |
| Volta (V100) | 2017 | **Tensor Core**（混合精度矩阵乘法） | 启用 FP16 训练，125 TFLOPS TF32 |
| Ampere (A100) | 2020 | TF32, 稀疏性，第 3 代 Tensor Core | 312 TFLOPS TF32，结构稀疏性 2:4 |
| Hopper (H100) | 2022 | **Transformer Engine** (FP8), HBM3 | 989 TFLOPS FP8，动态精度切换 |
| Blackwell (B200) | 2024 | 第 2 代 Transformer Engine, NVLink 5 | 2.5 PFLOPS FP4，多芯片设计 |

- **Tensor Core** 是专用矩阵乘法单元。单个 Tensor Core 指令在一个周期中计算 4×4 矩阵乘法（D = A×B + C）。常规 CUDA 核心需要 64 个 FMA 运算。Tensor Core 是混合精度训练（float16 计算、float32 累加）快的原因。

- **Transformer Engine**（Hopper+）在单层内动态切换 FP8 和 FP16 精度，仅在需要的地方选择更高精度。这最大化吞吐量而不牺牲模型质量。它专为变压器架构（注意力 + MLP）设计，主导现代 AI。

## 编码任务（用 nvcc 编译）

1. 编写 CUDA 内核对数组应用 ReLU。测量包括内存传输的时间。这教授内核编写、cudaMalloc/cudaMemcpy 和主机↔设备传输瓶颈。
```cpp
// task1_relu.cu
// 编译：nvcc -O3 -o task1_relu task1_relu.cu

#include <stdio.h>
#include <stdlib.h>
#include <cuda_runtime.h>

__global__ void relu_kernel(const float* input, float* output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        output[idx] = input[idx] > 0.0f ? input[idx] : 0.0f;
    }
}

int main() {
    const int N = 1 << 24;  // 约 16M 元素
    size_t bytes = N * sizeof(float);

    // 分配主机内存
    float* h_input  = (float*)malloc(bytes);
    float* h_output = (float*)malloc(bytes);
    for (int i = 0; i < N; i++) {
        h_input[i] = (float)(i % 100) - 50.0f;  // 混合正和负
    }

    // 分配设备内存
    float *d_input, *d_output;
    cudaMalloc(&d_input, bytes);
    cudaMalloc(&d_output, bytes);

    // 计时完整管道：复制到 GPU、计算、复制回
    cudaEvent_t start, stop;
    cudaEventCreate(&start);
    cudaEventCreate(&stop);

    cudaEventRecord(start);
    cudaMemcpy(d_input, h_input, bytes, cudaMemcpyHostToDevice);

    int block_size = 256;
    int grid_size = (N + block_size - 1) / block_size;
    relu_kernel<<<grid_size, block_size>>>(d_input, d_output, N);

    cudaMemcpy(h_output, d_output, bytes, cudaMemcpyDeviceToHost);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);

    float ms = 0;
    cudaEventElapsedTime(&ms, start, stop);

    // 验证
    int errors = 0;
    for (int i = 0; i < N; i++) {
        float expected = h_input[i] > 0.0f ? h_input[i] : 0.0f;
        if (h_output[i] != expected) errors++;
    }

    printf("Time (including transfers): %.2f ms\n", ms);
    printf("Bandwidth: %.1f GB/s\n", 2.0 * bytes / ms / 1e6);  // 读 + 写
    printf("Errors: %d / %d\n", errors, N);

    cudaFree(d_input); cudaFree(d_output);
    free(h_input); free(h_output);
    return 0;
}
```

2. 用 CUDA 编写分块矩阵乘法使用共享内存。比较性能与朴素（非分块）版本。这教授共享内存、`__syncthreads` 和为什么分块重要。
```cpp
// task2_matmul.cu
// 编译：nvcc -O3 -o task2_matmul task2_matmul.cu

#include <stdio.h>
#include <cuda_runtime.h>

#define TILE 16

// 朴素 matmul：每个线程计算 C 的一个元素
__global__ void matmul_naive(const float* A, const float* B, float* C, int N) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    if (row < N && col < N) {
        float sum = 0.0f;
        for (int k = 0; k < N; k++) {
            sum += A[row * N + k] * B[k * N + col];
        }
        C[row * N + col] = sum;
    }
}

// 分块 matmul：使用共享内存减少全局内存访问
__global__ void matmul_tiled(const float* A, const float* B, float* C, int N) {
    __shared__ float sA[TILE][TILE];
    __shared__ float sB[TILE][TILE];

    int row = blockIdx.y * TILE + threadIdx.y;
    int col = blockIdx.x * TILE + threadIdx.x;
    float sum = 0.0f;

    for (int t = 0; t < (N + TILE - 1) / TILE; t++) {
        sA[threadIdx.y][threadIdx.x] = (row < N && t*TILE+threadIdx.x < N)
            ? A[row * N + t*TILE + threadIdx.x] : 0.0f;
        sB[threadIdx.y][threadIdx.x] = (t*TILE+threadIdx.y < N && col < N)
            ? B[(t*TILE + threadIdx.y) * N + col] : 0.0f;

        __syncthreads();
        for (int k = 0; k < TILE; k++)
            sum += sA[threadIdx.y][k] * sB[k][threadIdx.x];
        __syncthreads();
    }

    if (row < N && col < N)
        C[row * N + col] = sum;
}

int main() {
    const int N = 1024;
    size_t bytes = N * N * sizeof(float);

    float *d_A, *d_B, *d_C;
    cudaMalloc(&d_A, bytes); cudaMalloc(&d_B, bytes); cudaMalloc(&d_C, bytes);

    // 用 1 初始化（容易验证：C 应该全 N）
    float* h_A = new float[N*N];
    for (int i = 0; i < N*N; i++) h_A[i] = 1.0f;
    cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(d_B, h_A, bytes, cudaMemcpyHostToDevice);

    dim3 block(TILE, TILE);
    dim3 grid((N+TILE-1)/TILE, (N+TILE-1)/TILE);

    // 基准朴素
    cudaEvent_t start, stop;
    cudaEventCreate(&start); cudaEventCreate(&stop);

    cudaEventRecord(start);
    for (int i = 0; i < 10; i++)
        matmul_naive<<<grid, block>>>(d_A, d_B, d_C, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float naive_ms; cudaEventElapsedTime(&naive_ms, start, stop);

    // 基准分块
    cudaEventRecord(start);
    for (int i = 0; i < 10; i++)
        matmul_tiled<<<grid, block>>>(d_A, d_B, d_C, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float tiled_ms; cudaEventElapsedTime(&tiled_ms, start, stop);

    double gflops_naive = 2.0 * N * N * N * 10 / naive_ms / 1e6;
    double gflops_tiled = 2.0 * N * N * N * 10 / tiled_ms / 1e6;

    printf("Naive:  %.2f ms, %.1f GFLOPS\n", naive_ms/10, gflops_naive);
    printf("Tiled:  %.2f ms, %.1f GFLOPS\n", tiled_ms/10, gflops_tiled);
    printf("Speedup: %.1fx\n", naive_ms / tiled_ms);

    cudaFree(d_A); cudaFree(d_B); cudaFree(d_C);
    delete[] h_A;
    return 0;
}
```

3. 演示 warp 分歧。编写内核其中同一 warp 中的线程采用不同分支，并与无分支版本比较。
```cpp
// task3_divergence.cu
// 编译：nvcc -O3 -o task3_diverge task3_divergence.cu

#include <stdio.h>
#include <cuda_runtime.h>

// 坏：warp 分歧 — 偶/奇线程采用不同路径
__global__ void divergent_kernel(float* data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        if (idx % 2 == 0) {
            data[idx] = data[idx] * 2.0f + 1.0f;
        } else {
            data[idx] = data[idx] * 0.5f - 1.0f;
        }
    }
}

// 好：无分支 — 所有线程执行相同指令
__global__ void branchless_kernel(float* data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        float scale = (idx % 2 == 0) ? 2.0f : 0.5f;
        float offset = (idx % 2 == 0) ? 1.0f : -1.0f;
        data[idx] = data[idx] * scale + offset;
    }
}

int main() {
    const int N = 1 << 24;
    float* d_data;
    cudaMalloc(&d_data, N * sizeof(float));
    cudaMemset(d_data, 0, N * sizeof(float));

    int block = 256, grid = (N + block - 1) / block;

    cudaEvent_t start, stop;
    cudaEventCreate(&start); cudaEventCreate(&stop);

    // 分歧
    cudaEventRecord(start);
    for (int i = 0; i < 100; i++)
        divergent_kernel<<<grid, block>>>(d_data, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float div_ms; cudaEventElapsedTime(&div_ms, start, stop);

    // 无分支
    cudaEventRecord(start);
    for (int i = 0; i < 100; i++)
        branchless_kernel<<<grid, block>>>(d_data, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float nodiv_ms; cudaEventElapsedTime(&nodiv_ms, start, stop);

    printf("Divergent:  %.2f ms\n", div_ms / 100);
    printf("Branchless: %.2f ms\n", nodiv_ms / 100);
    printf("Speedup:    %.2fx\n", div_ms / nodiv_ms);

    cudaFree(d_data);
    return 0;
}
```
