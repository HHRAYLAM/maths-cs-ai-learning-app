# 视觉 - 语言 - 动作模型

*视觉 - 语言 - 动作模型（VLA）将视觉、理解语言和行动统一到一个神经网络中。本文件涵盖 VLA 架构、动作标记化、RT-2、Octo、OpenVLA、预训练策略、泛化、体现无关模型和基准测试*

- 在前面的文件中，我们涵盖了感知（感知世界）和机器人学习（控制身体）。传统上，这些是独立的管道：感知模块检测对象，语言模块解释指令，控制模块生成动作。每个模块都是独立设计、训练和调试的。

- 视觉 - 语言 - 动作模型（VLA） 将这个管道折叠成单个神经网络。模型接收图像（视觉）、自然语言指令（语言），并输出电机命令（动作）。一个模型，端到端。

- 这遵循我们在第 10 章中看到的相同统一趋势：正如多模态模型将视觉和语言理解合并到一个架构中，VLA 将其扩展到物理动作。洞察是语言为指定任务提供了自然、灵活的接口（"拿起红色的杯子并将其放在架子上"），大型预训练视觉 - 语言模型已经理解图像和指令。

## 从视觉 - 语言到动作

- 回想第 10 章，视觉 - 语言模型（VLM） 如 LLaVA 和 Flamingo 接收图像和文本作为输入并产生文本作为输出。它们理解场景、回答问题并遵循指令，都使用语言。

- VLA 提出：如果输出不是文本而是机器人动作会怎样？模型不是生成"红色杯子在桌子左侧"，而是生成一系列电机命令，移动手臂抓握那个杯子。

- 关键架构洞察是动作可以表示为标记，就像单词一样。如果 VLM 使用下一个标记预测逐标记生成语言，VLA 以相同方式生成动作标记。变压器从根本上不关心输出标记是指"杯子"还是"向前移动夹爪 2 厘米"。

- 这将机器人控制重新构建为序列建模问题，变压器擅长于此（第 7 章）。模型学习映射：（图像观察，语言指令）$\to$（动作标记序列）。

## VLA 架构

![VLA 架构：相机图像和语言指令被编码为标记，由 LLM 主干处理，并解码为机器人动作](../images/vla_architecture.svg)

- 典型的 VLA 有三个组件：

    - 视觉编码器：将相机图像处理为视觉标记。通常是预训练的 ViT（第 8 章）或 SigLIP 编码器（第 10 章）。图像被分割为图块，每个嵌入为标记，完全像标准视觉变压器。

    - 语言模型主干：预训练的 LLM（例如 LLaMA、PaLM），处理视觉标记和语言标记的交错序列。这是推理发生的地方：模型通过关注指令和视觉特征来理解"拿起红色杯子"。

    - 动作头：将 LLM 的输出映射到机器人动作。这可以是简单的 MLP，将最后一个隐藏状态映射到连续动作值，或者是标记化方案，将动作转换为离散标记，由 LLM 现有的词汇表预测。

- 架构看起来像：

$$\text{图像} \xrightarrow{\text{ViT}} \text{视觉标记} \quad + \quad \text{指令} \xrightarrow{\text{标记器}} \text{语言标记} \quad \xrightarrow{\text{LLM}} \quad \text{动作标记}$$

- 视觉标记和语言标记被连接（或交错）并馈送到变压器主干，自回归地产生动作标记。这与 VLM 架构相同（第 10 章），但输出模态是动作而不是文本。

## 动作标记化

- 机器人动作是连续的：关节速度、末端执行器位置、夹爪宽度。这些必须转换为离散标记供 LLM 生成。

![动作标记化：连续动作值被分箱为离散索引，LLM 作为标记生成](../images/action_tokenisation.svg)

- 最简单的方法是均匀离散化。每个动作维度被划分为 $N$ 个箱，跨越有效值范围。例如，如果 x 速度范围为 -0.1 到 0.1 m/s，我们使用 256 个箱，每个箱表示$\frac{0.2}{256} \approx 0.8$ mm/s。动作值被映射到其最近的箱索引，成为标记。

- 对于 7 个动作维度（6 DoF + 夹爪）和每个 256 个箱，动作词汇表有$7 \times 256 = 1792$个标记。这些添加到 LLM 现有的文本文汇表中。模型自回归地为每个维度生成一个动作标记，就像生成单词一样。

- 动作分块一次性预测多个未来时间步长，而不是单个动作。如果分块大小是$H$，模型输出$H \times d$个标记（其中$d$是动作维度）。这对平滑、时间相干的运动至关重要。一次预测一个步骤可能产生生硬行为，因为每个预测是独立的。分块迫使模型规划短轨迹，捕获时间结构。

- 更复杂的方法使用 VQ-VAE（第 10 章）的学习标记化。VQ-VAE 编码器将连续动作序列映射到离散码本索引序列，解码器从这些索引重建连续动作。LLM 然后生成码本索引，而不是均匀分箱值。这类似于图像标记化器（第 10 章）如何将视觉信息压缩为紧凑离散代码。

## 关键 VLA 模型

- RT-2（Robotic Transformer 2，Google DeepMind）是第一个大规模 VLA。它采用预训练的 VLM（PaLM-E 或 PaLI-X，高达 55B 参数），并在机器人演示数据上微调它。动作表示为文本字符串：标记序列"1 128 91 241 5 101 127"编码 7 维动作（每个数字是箱索引）。

- RT-2 展示了卓越的特性：来自 VLM 主干的涌现能力转移到机器人。模型可以遵循涉及它在机器人数据中从未见过的概念的指令（例如，"将香蕉移到以 A 开头的国家"需要视觉对象识别 + 世界知识 + 动作）。VLM 的语言理解和视觉推理"免费获得"。

- RT-2 的局限性是它使用来自单个机器人体现（特定手臂和特定夹爪）的数据训练。它不能泛化到不同的机器人。

- Octo（UC Berkeley）是开源的、体现无关的VLA，设计为跨不同机器人平台工作。关键创新是：

    - 扩散动作头而不是自回归标记预测。动作头采用变压器的输出，通过去噪扩散过程产生动作（第 8 章）。这自然处理多模态动作分布（见下图），其中有多种有效方式完成任务。

![多模态动作分布：回归将两个有效路径平均为通过障碍物的无效路径](../images/multimodal_action_distribution.svg)

    - 灵活的观察和动作空间：Octo 为不同机器人配置使用特定任务的标记器。它在 Open X-Embodiment 数据集上预训练，包含来自 22 个不同机器人体现的演示。

    - 高效微调：Octo 可以用少至 100 个演示微调到新机器人，使数据有限的实验室实用。

- OpenVLA（斯坦福，UC Berkeley）采用微调现有开源 VLM（基于 Llama）用于机器人。它使用 7B 参数主干、均匀动作标记化（每个维度 256 个箱），并在 Open X-Embodiment 数据上训练。它的优势是简单：架构是标准 VLM，动作标记附加到词汇表，使使用现有 LLM 基础设施训练和部署变得容易。

- $\pi_0$（Physical Intelligence）代表最先进水平。它使用预训练的 VLM 主干，带流匹配动作头（第 8 章）。流匹配通过学习速度场生成动作，将噪声传输到动作分布，产生平滑、时间相干的动作轨迹。$\pi_0$ 展示了卓越的通用性，跨多个机器人体现执行任务，包括双手操作和灵巧手控制。

## 预训练配方

- VLA 从预训练的 VLM 主干中受益匪浅，后者已经理解视觉场景和语言。训练管道通常遵循阶段：

    1. VLM 预训练：在来自互联网的数十亿图像 - 文本对上训练（或使用现成的）视觉 - 语言模型（CLIP、SigLIP、LLaVA 风格训练，如第 10 章所述）。

    2. 机器人数据共同训练：在互联网数据和机器人演示数据的混合上微调 VLM。互联网数据防止视觉和语言理解的灾难性遗忘，而机器人数据教授动作生成。混合比例很重要：太多机器人数据降低语言理解，太少无法学习动作。

    3. 特定任务微调：可选地在特定任务或机器人的演示上微调，通常使用 LoRA（第 10 章）保持可训练参数数量小。

- 机器人数据的数量比互联网数据小几个数量级。VLM 可能在数十亿图像上预训练，但最大的机器人数据集（Open X-Embodiment）在所有体现中只有数百万帧。这种数据稀缺性解释了为什么从预训练 VLM 开始至关重要：视觉和语言表示转移，只有动作映射需要从有限的机器人数据中学习。

## 泛化

- VLA 的承诺是泛化：在训练期间未见过任务、未见过的对象、未见过的环境、遵循未见过的指令。

- VLA 沿几个轴泛化：

    - 新对象：VLM 主干从互联网预训练识别对象。如果模型从网络图像中知道"螺丝刀"是什么样子，即使机器人演示从未包括螺丝刀，它也可以操作一个。

    - 新指令：组合语言理解允许模型遵循已知概念的新组合。"将蓝色方块堆叠在绿色方块上"有效，即使训练只显示堆叠红色方块，因为模型从语言预训练中理解颜色形容词。

    - 新环境：在某种程度上，VLA 跨视觉域迁移（不同的桌子、照明、背景），因为视觉编码器在不同网络图像上预训练。但这有限制：在实验室训练的机器人可能在杂乱厨房中挣扎。

    - 新体现：这是最难的轴。不同的机器人有不同的动作空间（关节角度与末端执行器速度）、不同的传感器（手腕相机与头顶相机）和不同的物理能力。体现无关模型如 Octo 和$\pi_0$通过灵活的标记化和跨多种机器人类型预训练解决这个问题。

- 泛化在保留任务上评估：要求机器人执行它从未训练过的任务。新任务 50-80% 的成功率被认为是强结果，而分布内任务>90%。随着模型扩展和机器人数据集增长，差距正在缩小。

## 体现无关模型

- 该领域正朝着一个模型，多个机器人发展。VLA 不是为每个机器人训练单独的策略，单个 VLA 处理多个体现。

- 这需要解决动作空间不匹配问题。7-DoF 臂带平行夹爪有 7 个动作维度。双手装置有 14 个。四足动物有 12 个。人形动物有 30+ 个。动作标记化必须足够灵活以处理所有这些。

- 解决方案包括：
    - 填充动作向量：使用最大的动作空间并用零填充较小的。
    - 每个体现动作头：共享变压器主干，每个机器人类型有单独的小 MLP。
    - 标准化动作表示：在公共帧中表达所有动作（例如，世界帧中的末端执行器速度），以便产生相似末端执行器运动的不同机器人共享相同的动作标记。

- 共享主干学习通用视觉和语言理解，加上常见操作策略（从上方接近、与对象对齐、关闭夹爪）。体现特定组件只需要将这些高级策略转换为特定电机命令。

## 基准测试和评估

- 评估 VLA 具有独特的挑战性，因为它需要物理机器人实验（或高保真模拟）。

- SIMPLER（Simulated Manipulation Policy Evaluation for Robot learning）提供标准化模拟环境，用于比较 VLA 性能，无需物理硬件。它与现实世界成功率密切相关，实现可重复的基准测试。

- 现实世界评估仍然是黄金标准。典型协议：
    1. 定义具有明确成功标准的任务（对象到达目标位置、选择正确对象、在时间限制内完成任务）。
    2. 每个任务运行$N$次试验（通常 10-50 次）。
    3. 带置信区间的报告成功率。
    4. 包括保留（从未训练过）任务以测量泛化。

- Open X-Embodiment数据集和基准汇总来自 22 个机构跨多个机器人平台的机器人数据。它为共享演示提供标准化格式，为跨体现迁移提供通用评估套件。

1. 实现动作标记化：将连续动作离散化为箱并重建它们。观察量化误差作为箱数量的函数。
```python
import jax.numpy as jnp

# 连续动作：7 维度（6 DoF + 夹爪）
action_true = jnp.array([0.023, -0.051, 0.012, 0.1, -0.03, 0.005, 0.8])
action_min = jnp.array([-0.1, -0.1, -0.1, -0.5, -0.5, -0.5, 0.0])
action_max = jnp.array([ 0.1,  0.1,  0.1,  0.5,  0.5,  0.5, 1.0])

for n_bins in [16, 64, 256, 1024]:
    # 标记化：将连续值映射到箱索引
    normalised = (action_true - action_min) / (action_max - action_min)
    tokens = jnp.clip((normalised * n_bins).astype(int), 0, n_bins - 1)

    # 逆标记化：将箱索引映射回连续值
    reconstructed = (tokens + 0.5) / n_bins * (action_max - action_min) + action_min

    error = jnp.linalg.norm(action_true - reconstructed)
    print(f"bins={n_bins:4d}  tokens={tokens}  error={error:.6f}")
```

2. 模拟动作分块与单步预测。生成平滑轨迹，给单步预测添加噪声，并与基于分块的预测比较。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# 真实平滑轨迹（例如，到达运动）
t = jnp.linspace(0, 2 * jnp.pi, 100)
gt_x = jnp.sin(t)
gt_y = 1 - jnp.cos(t)

# 单步：每个预测有独立噪声
rng = jax.random.PRNGKey(42)
noise_ss = jax.random.normal(rng, (100, 2)) * 0.05
single_step = jnp.stack([gt_x, gt_y], axis=1) + noise_ss
# 来自单步误差的累积漂移
single_step_cumulative = jnp.cumsum(noise_ss, axis=0) * 0.3 + jnp.stack([gt_x, gt_y], axis=1)

# 分块（chunk_size=10）：噪声在分块内相关，更平滑
chunk_size = 10
rng2 = jax.random.PRNGKey(7)
chunks = []
for i in range(0, 100, chunk_size):
    chunk_noise = jax.random.normal(jax.random.fold_in(rng2, i), (2,)) * 0.05
    chunk = jnp.stack([gt_x[i:i+chunk_size], gt_y[i:i+chunk_size]], axis=1)
    chunks.append(chunk + chunk_noise)
chunked = jnp.concatenate(chunks, axis=0)

plt.figure(figsize=(8, 4))
plt.plot(gt_x, gt_y, "k-", linewidth=2, label="Ground truth")
plt.plot(single_step_cumulative[:, 0], single_step_cumulative[:, 1],
         "r-", alpha=0.7, label="Single-step (drifts)")
plt.plot(chunked[:, 0], chunked[:, 1], "b-", alpha=0.7, label="Chunked (stable)")
plt.legend(); plt.axis("equal"); plt.grid(True)
plt.title("Action Chunking vs Single-Step Prediction")
plt.show()
```

3. 可视化 VLA 的动作分布如何是多模态的。使用简单的 2D 高斯混合来展示为什么扩散/流匹配动作头优于回归。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# 两种有效方式绕过障碍物：左或右
rng = jax.random.PRNGKey(0)
k1, k2 = jax.random.split(rng)

mode1 = jax.random.normal(k1, (200, 2)) * 0.15 + jnp.array([-1.0, 0.5])
mode2 = jax.random.normal(k2, (200, 2)) * 0.15 + jnp.array([ 1.0, 0.5])
samples = jnp.concatenate([mode1, mode2])

# 回归预测均值 = 模式的平均值（无效！）
mean_pred = samples.mean(axis=0)

plt.figure(figsize=(6, 5))
plt.scatter(samples[:, 0], samples[:, 1], s=5, alpha=0.5, label="True action distribution")
plt.plot(*mean_pred, "rx", markersize=15, markeredgewidth=3, label="Regression mean (invalid!)")
plt.plot(-1, 0.5, "g^", markersize=12, label="Mode 1 (go left)")
plt.plot(1, 0.5, "b^", markersize=12, label="Mode 2 (go right)")
plt.legend(); plt.grid(True)
plt.title("Multimodal Actions: Why Regression Fails")
plt.xlabel("Action dim 1"); plt.ylabel("Action dim 2")
plt.show()
```
