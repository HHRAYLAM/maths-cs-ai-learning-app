# 量化

*量化降低模型权重和激活的精度，使模型更小、更快、更便宜运行。本文件涵盖数字格式、训练后量化、量化感知训练、仅权重方法（GPTQ、AWQ）、激活量化、混合精度和 KV-cache 量化*

- 70B 参数模型在 float16 中需要 140 GB 内存，超过任何单个 GPU。量化到 INT4 后，它适合 35 GB（一个 A100）或甚至 20 GB（消费级 RTX 4090 带卸载）。量化不是优化装饰；它是使大模型部署在经济上可行的关键。

- 根本权衡：更低精度意味着更少内存、更高吞吐量和更低功耗，但引入量化误差，可能降低模型质量。量化的艺术是最小化这种退化。

## 为什么量化

- 内存减少：INT8 比 FP16 小 2 倍，INT4 小 4 倍。对于 LLM，模型权重主导内存。精度减半使内存需求减半。

- 吞吐量增益：更低精度意味着每秒更多操作。NVIDIA Tensor Core（第 16 章）实现 FP16 相对 FP32 的 2 倍吞吐量，INT8 相对 FP16 再 2 倍，INT4 相对 INT8 再 2 倍。H100 在 FP8 中实现 989 TFLOPS，而在 FP32 中为 67 TFLOPS —— 15 倍差异。

- 带宽节省：LLM 推理通常是内存带宽绑定（第 16 章，rooftop 模型）。瓶颈是从 GPU 内存加载权重，而不是用它们计算。更小的权重意味着更少的字节传输，直接增加每秒 token 数。这就是为什么量化通常为 LLM 推理带来近乎线性的加速。

- 节能：更低精度每次操作使用更少能源。在数据中心规模（数千 GPU），这转化为显著的电费成本降低。

## 数字格式

- 我们在第 13 章（计算机架构）中介绍了 IEEE 754 浮点数。这里是 ML 的完整精度格局：

| 格式 | 位 | 指数 | 尾数 | 范围 | 用例 |
|--------|------|----------|----------|-------|----------|
| FP32 | 32 | 8 | 23 | ±3.4×10³⁸ | 训练（黄金标准） |
| TF32 | 19 | 8 | 10 | ±3.4×10³⁸ | Tensor Core 训练（A100+） |
| FP16 | 16 | 5 | 10 | ±65504 | 混合精度训练 |
| BF16 | 16 | 8 | 7 | ±3.4×10³⁸ | 训练（与 FP32 相同范围） |
| FP8 E4M3 | 8 | 4 | 3 | ±448 | 前向传播（Hopper+） |
| FP8 E5M2 | 8 | 5 | 2 | ±57344 | 梯度（更宽范围） |
| INT8 | 8 | — | — | -128 到 127 | PTQ 推理 |
| INT4 | 4 | — | — | -8 到 7 | 仅权重量化 |
| INT2/三进制 | 2 | — | — | {-1, 0, 1} | 极端压缩 |

- FP8 有两种变体：E4M3（4 位指数、3 位尾数、更窄范围但更多精度）用于前向传播，E5M2（5 位指数、2 位尾数、更宽范围但更少精度）用于梯度。Transformer Engine（第 16 章）在每个张量之间自动切换它们。

- BF16 vs FP16：BF16 具有与 FP32 相同的指数范围（无溢出风险）但尾数精度更低。FP16 具有更多精度但范围窄（最大 65504），训练期间需要损失缩放。对于推理，两者都有效；对于训练，BF16 更安全。

- 整数格式没有指数 —— 它们表示定点值。要在 float 和 int 之间转换，你需要缩放因子和可选的零点：$x_{\text{float}} = \text{scale} \times (x_{\text{int}} - \text{zero\_point})$。

## 量化方程

- 所有量化方法将浮点值映射到整数并返回：

$$x_q = \text{clamp}\left(\text{round}\left(\frac{x}{\text{scale}}\right) + \text{zero\_point}, \; q_{\min}, \; q_{\max}\right)$$

$$\hat{x} = \text{scale} \times (x_q - \text{zero\_point})$$

- 缩放决定分辨率：$\text{scale} = \frac{x_{\max} - x_{\min}}{q_{\max} - q_{\min}}$。对于 INT8：$q_{\min} = -128$，$q_{\max} = 127$。

- 对称量化设置 $\text{zero\_point} = 0$，所以 $\text{scale} = \frac{\max(|x|)}{127}$。更简单更快（推理期间无需零点减法）。

- 非对称量化使用非零 $\text{zero\_point}$ 处理非对称分布（例如，ReLU 输出都是非负的）。将 $[x_{\min}, x_{\max}]$ 映射到 $[0, 255]$ 用于无符号 INT8。

![量化粒度：每张量对整个矩阵使用一个缩放，每通道对每列使用一个，每组对小块使用一个](../images/quantisation_granularity.svg)

- 量化粒度：多少值共享相同的缩放因子：
    - 每张量：整个张量一个缩放。最简单但精度最低（一个异常值扭曲整个张量的缩放）。
    - 每通道：每个输出通道一个缩放（对于卷积）或每行一个（对于线性层）。更好的精度，开销最小。
    - 每组：每 $g$ 个元素组一个缩放（如 $g = 128$）。最佳精度，用于现代仅权重量化（GPTQ、AWQ）。
    - 每 token：激活每个 token 一个缩放。处理不同 token 具有非常不同激活幅度的事实。

## 训练后量化（PTQ）

- PTQ 量化预训练模型，无需任何重新训练。你传递校准集（小的代表性数据集，通常 128-512 样本）通过模型来收集激活统计，然后计算最优缩放因子。

### 校准方法

- 最小 - 最大：基于观察到的最小值和最大值设置缩放。简单但对异常值敏感（一个极端值将大部分量化范围浪费在很少使用的值上）。

- 百分位数：使用第 99.99 百分位数而不是绝对最大值。裁剪极端异常值，为大多数值提供更好的分辨率。裁剪值饱和到 $q_{\min}$ 或 $q_{\max}$。

- MSE 最优：找到最小化原始和量化张量之间均方误差的缩放。这是一维优化（在可能的裁剪值上搜索），通常给出最佳 PTQ 精度。

- 基于熵（KL 散度）：找到最小化原始和量化值分布之间 KL 散度的缩放。用于 TensorRT 的 INT8 校准。

### PTQ 实践

```python
# 简化的 PTQ 与 PyTorch（概念性）
import torch

def quantise_tensor_symmetric(tensor, bits=8):
    qmax = 2  (bits - 1) - 1  # INT8 为 127
    scale = tensor.abs().max() / qmax
    quantised = torch.clamp(torch.round(tensor / scale), -qmax, qmax).to(torch.int8)
    return quantised, scale

def dequantise(quantised, scale):
    return quantised.float() * scale

# 量化权重矩阵
weight = torch.randn(512, 512)  # 预训练权重
weight_q, scale = quantise_tensor_symmetric(weight, bits=8)
weight_reconstructed = dequantise(weight_q, scale)

# 量化误差
error = (weight - weight_reconstructed).abs().mean()
print(f"平均绝对误差：{error:.6f}")
print(f"压缩：{weight.numel() * 4 / (weight_q.numel() * 1 + 4):.1f}x")  # +4 字节用于缩放
```

- PTQ 对于 INT8 在大多数模型上效果良好，精度降低 <1%。对于 INT4，PTQ 质量显著下降 —— 仅权重方法（下面）更好地处理 INT4。

## 量化感知训练（QAT）

- QAT 在训练图中插入假量化操作：权重和激活在前向传播期间被量化和去量化，但梯度像没有量化一样流动（直通估计器）。

$$\text{前向：} \hat{W} = \text{dequant}(\text{quant}(W))$$
$$\text{反向：} \frac{\partial L}{\partial W} \approx \frac{\partial L}{\partial \hat{W}}$$

- 模型在训练期间学习对量化噪声具有鲁棒性。QAT 通常恢复 PTQ 丢失的大部分或全部精度，特别是在低位宽（INT4、INT2）。

- 成本：QAT 需要重新训练（或微调）模型，这对于大模型很昂贵。对于 70B 参数模型，QAT 可能花费 10,000-100,000 美元计算。PTQ 成本基本为零（只是校准）。

- 何时使用 QAT：当 PTQ 质量不可接受时（通常 INT4 或更低）、当你部署到具有严格延迟预算的边缘设备时，或者当模型将被量化数百万次时（一次性 QAT 成本被摊销）。

## 仅权重量化

- 对于 LLM 推理，瓶颈是从内存加载权重，而不是用它们计算（内存带宽绑定状态）。仅权重量化将权重量化为 INT4 或 INT3，同时保持激活在 FP16。计算在 FP16 中发生（即时去量化权重），但内存消耗和带宽减少 4-8 倍。

### GPTQ

- GPTQ（Frantar 等人，2022）一次一列量化权重，通过调整后续列来补偿每列的误差。它使用Hessian（来自校准集的二阶信息）来确定最优量化顺序和误差补偿：

$$\hat{W}_{:,j} = \text{quant}(W_{:,j}), \quad W_{:,j+1:} \mathrel{-}= \frac{(\hat{W}_{:,j} - W_{:,j}) \cdot H_{j,j+1:}}{H_{j,j}}$$

- 关键见解：量化第 $j$ 列引入误差。GPTQ 立即通过调整所有剩余列来补偿，使层的整体输出（$XW$）变化尽可能小。这是应用于 transformer 的最优大脑量化（OBQ）。

- GPTQ 与 4 位组量化（组大小 128）在大多数 LLM 上实现 <1% 困惑度退化。量化在单个 GPU 上每个 70B 模型花费约 1 小时。

### AWQ

- AWQ（激活感知权重量化，Lin 等人，2023）观察到一小部分权重通道（1-3%）比其他通道重要得多 —— 它们对应于具有大振幅的激活通道。保护这些显著通道大幅减少量化误差。

- AWQ 在量化之前将这些重要通道缩放因子 $s$（使它们更大，从而受舍入影响更小），并将相应激活缩放 $1/s$（以保持输出不变）。每组缩放 $s$ 被优化以最小化整体量化误差。

- AWQ 比 GPTQ 更简单（无 Hessian 计算）、运行更快，并实现可比质量。它已成为许多开源 LLM 量化管道的默认。

### GGUF / llama.cpp 量化

- GGUF（GGML 通用格式）是 llama.cpp 用于 CPU 推理的格式。它支持许多量化方案：
    - Q4_0：4 位，32 元素块，对称。
    - Q4_K_M：4 位，混合精度重要通道（k-quants）。
    - Q5_K_M：5 位，k-quants（更高质量）。
    - Q8_0：8 位，简单快速。

- "K" 变体（k-quants）为重要权重块分配更多位，类似于 AWQ 的见解但在格式级别实现。Q4_K_M 是大多数模型的最佳点：4 位平均，质量损失最小。

### QuIP 和 QuIP#

- QuIP（Chee 等人，2023）引入不相干处理：在量化之前使用随机正交变换旋转权重矩阵。这将信息分散到所有权重中，防止少数异常权重主导量化误差。

- 直觉：如果一个权重是 100，其余约 1，用相同缩放量化所有权重将大部分 INT4 范围浪费在异常值上。正交旋转后（保持矩阵的数学属性），所有权重具有相似幅度，均匀量化效果好得多。

- QuIP# 使用格码本扩展：不是映射到均匀整数网格，而是映射到最优格中的点（8D 中的 E8 格）。格码在相同比特数内将更多量化点打包到相同比特数，实现比均匀量化更好的率失真。QuIP# 在2 位精度下实现可用质量 —— 典型 INT4 方法比特数的一半。

### SpQR

- SpQR（Dettmers 等人，2023）观察到一小部分权重（0.1-1%）是异常值，对输出质量贡献不成比例。不是将所有内容量化到相同精度，SpQR：

    1. 使用敏感性分析识别异常权重（量化这个权重对层输出有多大变化）。
    2. 以全精度（FP16）稀疏格式存储异常值。
    3. 将所有剩余权重量化为 INT3 或 INT4。

- 结果：约 99% 的权重被激进化量化（小），而关键的 1% 保留全精度（准确）。稀疏异常存储添加最小开销（总大小的 <5%）。

### HQQ

- HQQ（半二次量化，Badri & Shaji，2023）是零样本权重量化方法，根本不需要校准数据。它将量化表述为半二次优化问题，迭代求解最优量化权重和缩放因子。

- 优势：无需校准集意味着无数据依赖、即时量化、无校准数据不匹配风险。HQQ 对于代表性校准数据不可用或敏感时特别有用。

### AQLM

- AQLM（Egiazarian 等人，2024）将加性量化（多码本向量量化）应用于 LLM。AQLM 不是独立量化每个权重，而是将权重分组为向量，并将每个向量表示为来自多个学习码本的条目之和：

$$\mathbf{w} \approx \mathbf{c}_1^{(1)} + \mathbf{c}_2^{(2)} + \cdots + \mathbf{c}_M^{(M)}$$

- 其中 $\mathbf{c}_i^{(m)}$ 是来自码本 $m$ 的条目。使用 $M = 2$ 个各有 256 个条目的码本，8 元素向量编码为两个 8 位索引 = 2 字节 8 个权重 = 每权重 2 位有效。AQLM 在 2 位精度下实现最先进质量，在此极端压缩级别上优于 GPTQ 和 AWQ。

### BitNet 和 1 位 LLM

- BitNet（Wang 等人，2023）将量化推向极端：权重是三进制的（$\{-1, 0, +1\}$），只需要约 1.58 位每权重。矩阵乘法变为仅加法和减法 —— 无需浮点乘法。

- BitNet b1.58（Ma 等人，2024）约束每个权重为 $\{-1, 0, +1\}$。"1.58 位"来自 $\log_2(3) \approx 1.58$。在此精度下，70B 模型适合约 15 GB，推理无需乘法操作 —— 只有加法、减法和符号翻转。

- 矩阵乘法变为：

$$y_j = \sum_i W_{ij} \cdot x_i = \sum_{i: W_{ij}=+1} x_i - \sum_{i: W_{ij}=-1} x_i$$

- 这在任何硬件上都比 FP16 矩阵乘法便宜得多，并可能使 LLM 推理在没有浮点单元的设备上运行。质量权衡对当前模型显著，但随规模和训练时感知意识而提高。

### 微缩放（MX）格式

- 微缩放（MX）格式是新的行业标准（AMD、Arm、Intel、Meta、Microsoft、NVIDIA、Qualcomm 支持），使用块浮点：一组元素共享单个指数，每个元素有自己的尾数。

| 格式 | 共享指数 | 元素位 | 总计（每元素） | 等价 |
|--------|----------------|-------------|--------------------|----|
| MXFP8 | 每块 8 位 | 8 (E4M3/E5M2) | ~8 | 像 FP8 但有更好范围 |
| MXFP6 | 每块 8 位 | 6 | ~6.5 | FP8 和 INT4 之间 |
| MXFP4 | 每块 8 位 | 4 | ~4.5 | 像 INT4 但有类浮点行为 |
| MXINT8 | 每块 8 位 | 8 (整数) | ~8.5 | 带共享缩放的 INT8 |

- 共享指数将指数成本摊销到块（通常 16-32 元素）。每个元素保留比单独指数更多的尾数位，每比特提供更好的精度。MX 格式预计将在未来硬件中取代单独的 FP8 和 INT8 格式。

### FP8 训练

- 在 NVIDIA Hopper 和 Blackwell GPU 上，FP8 训练（不仅是推理）现在是实用的。方案：

    - 前向传播：权重和激活在 E4M3（更高精度、更窄范围）。Transformer Engine 使用延迟缩放动态计算每张量缩放因子（跟踪前一次迭代的统计，将它们应用到当前）。

    - 反向传播：梯度在 E5M2（更宽范围、更低精度）。梯度比权重/激活有更宽的数值范围，所以额外的指数位防止溢出。

    - 主权重：在 FP32 中维护用于优化器状态（像标准混合精度训练与 FP16，第 6 章）。FP8 计算仅用于 matmul，不用于权重更新。

    - 损失缩放：仍然需要 FP8，就像 FP16 一样。动态损失缩放器调整缩放因子，使梯度值保持在 FP8 的可表示范围内。

- FP8 训练对大多数模型大小实现与 BF16 训练相当的质量，吞吐量提高约 2 倍。它是 H100 集群上新的大规模训练运行的默认。

## 激活量化

- 激活（层之间流动的中间张量）也可以量化，实现完全 INT8 计算（权重和激活都在 INT8，INT32 累加）。

- 动态量化：从实际激活值在运行时计算缩放因子。更准确（适应每个输入）但增加开销（在每层计算最小/最大或百分位数）。

- 静态量化：在校准期间计算缩放因子并固定。推理时更快（无运行时统计）但如果校准数据不具有代表性则较不准确。

- 每 token 量化：为序列中每个 token 计算单独的缩放。对 LLM 至关重要，因为不同 token 可以有非常不同的激活幅度（某些 token 产生的激活比其他大 100 倍）。

- 激活量化比权重量化更难，因为激活是数据依赖的（它们随每个输入变化），而权重是固定的。"异常值"问题尤其严重：少数激活通道有极端值（平均值的 100 倍），用与正常通道相同的缩放量化它们浪费精度。

- SmoothQuant（Xiao 等人，2022）通过数学上将量化难度从激活（由于异常值难以量化）迁移到权重（易于量化）来解决异常值：激活乘以 $1/s$，权重乘以 $s$，其中 $s$ 平衡难度。输出 $XW = (X \cdot \text{diag}(s^{-1})) \cdot (\text{diag}(s) \cdot W)$ 不变。

## 混合精度量化

- 并非所有层对量化同样敏感。注意力层通常容忍 INT4，而嵌入层和最终分类器需要更高精度。

- 敏感性分析：单独量化每层并测量精度影响。高敏感度层获得更多位；不敏感层获得更少位。

- Transformer Engine（第 16 章，NVIDIA Hopper）在操作级别实现动态混合精度：每个 matmul 根据张量统计在 FP8 和 FP16 之间选择，在保持质量的同时最大化吞吐量。

## KV-Cache 量化

- 在 LLM 生成期间，KV-cache 存储所有先前 token 的键和值张量。对于长序列，这主导内存：

$$\text{KV-cache 大小} = 2 \times n_{\text{layers}} \times n_{\text{heads}} \times d_{\text{head}} \times \text{seq\_len} \times \text{bytes\_per\_element}$$

- 70B 模型，80 层、64 头、128 维头、序列长度 128K，FP16：$2 \times 80 \times 64 \times 128 \times 131072 \times 2 = 330$ GB。这超过 GPU 内存。

- KV-cache 量化通过在 INT8 或 INT4 中存储缓存的键和值而不是 FP16 来减少这个。量化误差在序列上累积（每个新 token 关注所有缓存的 K/V），但使用每通道或每头量化，退化是可接受的。

- KV-cache 量化是乘法有益：它支持更长的序列（更多上下文）、更大的批量大小（更多并发用户）和更快的推理（加载缓存的内存带宽更少）。这是 LLM 服务影响最大的优化之一。

## 编程任务（使用 CoLab 或 notebook）

1. 从头实现对称 INT8 量化。量化权重矩阵、去量化，并测量重构误差作为值分布的函数。
```python
import jax.numpy as jnp
import jax

def quantise_int8(tensor):
    scale = jnp.max(jnp.abs(tensor)) / 127.0
    quantised = jnp.clip(jnp.round(tensor / scale), -127, 127).astype(jnp.int8)
    return quantised, scale

def dequantise(quantised, scale):
    return quantised.astype(jnp.float32) * scale

# 正常权重（训练模型的典型）
key = jax.random.PRNGKey(0)
weights = jax.random.normal(key, (1024, 1024)) * 0.02

q, s = quantise_int8(weights)
recon = dequantise(q, s)

print(f"原始：     {weights.nbytes / 1024:.0f} KB")
print(f"量化：    {q.nbytes / 1024:.0f} KB ({weights.nbytes / q.nbytes:.0f}x 更小)")
print(f"平均绝对误差：{jnp.abs(weights - recon).mean():.6f}")
print(f"最大绝对误差：  {jnp.abs(weights - recon).max():.6f}")
print(f"相对误差：{jnp.abs(weights - recon).mean() / jnp.abs(weights).mean():.4%}")
```

2. 演示异常值问题。创建具有少数极端通道的激活，并展示每张量量化如何失败而每通道成功。
```python
import jax.numpy as jnp
import jax

key = jax.random.PRNGKey(42)

# 激活：大多数通道正常，2 个通道有 100 倍异常值
activations = jax.random.normal(key, (32, 512)) * 0.1
activations = activations.at[:, 0].set(activations[:, 0] * 100)   # 异常值通道
activations = activations.at[:, 1].set(activations[:, 1] * 50)    # 异常值通道

# 每张量量化（整个张量一个缩放）
scale_tensor = jnp.max(jnp.abs(activations)) / 127.0
q_tensor = jnp.clip(jnp.round(activations / scale_tensor), -127, 127)
recon_tensor = q_tensor * scale_tensor

# 每通道量化（每通道一个缩放）
scales_channel = jnp.max(jnp.abs(activations), axis=0) / 127.0
q_channel = jnp.clip(jnp.round(activations / scales_channel), -127, 127)
recon_channel = q_channel * scales_channel

err_tensor = jnp.abs(activations - recon_tensor).mean()
err_channel = jnp.abs(activations - recon_channel).mean()

print(f"每张量误差：{err_tensor:.6f}")
print(f"每通道误差：{err_channel:.6f}")
print(f"每通道好 {err_tensor / err_channel:.1f}x 倍")
print(f"\n异常值通道浪费 {(activations.shape[1] - 2) / activations.shape[1]:.0%} "
      f"的量化范围用于 {2 / activations.shape[1]:.1%} 的通道")
```
