# 自动语音识别

*自动语音识别将口语音频转换为书面文本，架起了人类语音和机器可读语言之间的桥梁。本文件涵盖 GMM-HMM、CTC 损失、RNN-Transducer、基于注意力的编码器 - 解码器模型 (LAS)、Whisper 和端到端 ASR，从经典流水线到现代神经架构。*

- **自动语音识别** (ASR) 是将口语音频转换为书面文本的任务。它是最古老的 AI 问题之一（第一个系统在 1950 年代识别单个数字），也是部署最广泛的 AI 技术之一（语音助手、转录服务、字幕）。

- 困难来自于语音的巨大变异性：不同的说话者、口音、语速、背景噪声、麦克风特性，以及将连续声学信号映射到离散单词的根本歧义性。

- 将 ASR 想象成法庭速记员。速记员听到连续的声音流，在心理上将其分割成单词，使用上下文解决歧义（"they're" vs "their" vs "there"），然后输入结果。ASR 系统做同样的事情，但分阶段进行，这些阶段可以独立或联合优化。

- **经典 ASR 流水线**以不同阶段链处理音频：原始音频被转换为特征（MFCCs 或 log-mel 频谱图，来自文件 01），**声学模型**评分每个特征帧与每个语音单元的匹配程度，**发音模型**（词典）将语音单元映射到单词，**语言模型**评分单词序列的可能性，**解码器**搜索最大化组合分数的单词序列。每个组件分别训练和调优。

![ASR 流水线从原始音频通过特征提取、声学模型、解码器和语言模型输出文本](../images/asr_pipeline.svg)

- **音素**是区分语言中单词的最小声音单位。英语大约有 39-44 个音素（确切数量取决于使用的方言和音素清单）。例如，"bat" 和 "pat" 在一个音素上不同（/b/ vs /p/）。大多数 ASR 系统建模称为**三音素**的**上下文相关音素**：由左右邻居定义的音素（例如，"b_t" 上下文中的"a"是不同于"c_t"上下文中"a"的单元），因为音素的声学实现受到其邻居的强烈影响（这称为**协同发音**）。

- 三音素的数量巨大（40 个音素的三次方 = 64,000），因此**决策树聚类**将声学相似的三音素分组为**senones**（通常 2000-10,000 个类别）。每个 senone 获得自己的声学模型。这种聚类是来自第 06 章的决策树算法的一种形式。

- **GMM-HMM**（高斯混合模型 - 隐马尔可夫模型）是 1980 年代到 2010 年代初主导的声学建模方法。HMM（来自第 05 章）建模语音的时间结构：每个音素是一个有 3-5 个状态的从左到右的 HMM，其中每个状态代表一个亚音素片段（起始、中间、结束）。状态到状态的转换隐式建模持续时间。

- 在每个 HMM 状态，发射概率（给定该状态特定特征向量的可能性）由**高斯混合模型** (GMM) 建模：多元高斯分布的加权和（来自第 05 章）：

```math
p(\mathbf{x} | s) = \sum_{m=1}^{M} w_m \cdot \mathcal{N}(\mathbf{x} ; \boldsymbol{\mu}_m, \boldsymbol{\Sigma}_m)
```

- 其中 $\mathbf{x}$ 是特征向量（例如，39 维 MFCCs），$s$ 是 HMM 状态，$M$ 是混合组件的数量（通常 8-64），$w_m$ 是混合权重，$\boldsymbol{\mu}_m$、$\boldsymbol{\Sigma}_m$ 是每个高斯组件的均值和协方差。协方差矩阵通常是对角线以计算效率（假设特征维度独立，由于 DCT 去相关，这对 MFCCs 近似正确）。

- 训练使用**Baum-Welch 算法**（EM 的特例，来自第 05 章）从转录语音数据迭代估计 GMM 参数和 HMM 转移概率。解码（找到最可能的状态序列）使用**Viterbi 算法**（动态规划，来自第 05 章）：

```math
\delta_t(j) = \max_{i} \left[ \delta_{t-1}(i) \cdot a_{ij} \right] \cdot b_j(\mathbf{x}_t)
```

- 其中 $\delta_t(j)$ 是在时间 $t$ 结束于状态 $j$ 的最佳路径概率，$a_{ij}$ 是从状态 $i$ 到状态 $j$ 的转移概率，$b_j(\mathbf{x}_t)$ 是状态 $j$ 中特征 $\mathbf{x}_t$ 的发射概率。

- **DNN-HMM** (Hinton et al., 2012) 用深度神经网络 (DNN，来自第 06 章) 替换 GMM 发射模型，从特征帧窗口预测 senone 后验概率 $p(s | \mathbf{x})$。HMM 仍然处理时间结构和排序，但神经网络提供更具判别力的发射分数。这种混合方法相比 GMM 将词错误率降低了 20-30%，是 2012-2016 年的主导范式。

- **WFST 解码**（加权有限状态换能器）是传统 ASR 的标准解码框架。每个组件（HMM 拓扑 H、上下文依赖 C、词典 L、语法/语言模型 G）表示为加权有限状态换能器，它们组合成单个搜索图 $H \circ C \circ L \circ G$。Viterbi 搜索然后找到该组合图中的最低成本路径。WFST 允许知识源的模块化组合和高效的动态规划搜索。数学框架来自有限自动机理论（与第 05 章中的状态机相关）。

- **端到端 ASR** 消除单独的组件（发音模型、音素清单、WFST 解码器）并训练单个神经网络，直接从音频特征映射到字符或词块。关键挑战是**对齐问题**：输入（每秒数百个特征帧）和输出（每秒几个字符）长度非常不同，训练期间它们之间的对齐未知。

- **Connectionist Temporal Classification** (CTC) (Graves et al., 2006) 通过引入特殊的**blank** 令牌解决对齐问题，并允许网络输出任何字符和空白序列，只要折叠连续重复和移除空白产生正确的转录。例如，转录 "cat" 可以由输出序列 "--cc-aa-t--" 产生（其中"-"是空白）。

- 形式上，CTC 定义从所有长度 -$T$ 输出序列（字母加空白）到标签序列的多对一映射 $\mathcal{B}$。标签序列 $\mathbf{y}$ 的概率是对折叠到它的所有对齐的总和：

$$P(\mathbf{y} | \mathbf{x}) = \sum_{\boldsymbol{\pi} \in \mathcal{B}^{-1}(\mathbf{y})} \prod_{t=1}^{T} p(\pi_t | \mathbf{x})$$

![CTC 对齐显示通过空白和字符令牌的许多可能路径都折叠到相同的输出文本](../images/ctc_alignment.svg)

- 朴素计算该总和需要枚举指数级多的对齐，但**CTC 前向 - 后向算法**使用动态规划在 $O(T \cdot |\mathbf{y}|)$ 中高效计算它，类似于第 05 章的 HMM 前向 - 后向算法。

- CTC 做出**条件独立性假设**：给定输入，每个时间步的输出独立于所有其他输出。这意味着 CTC 不能建模输出依赖（例如，它不能学习"q"几乎总是后跟"u"）。必须使用外部语言模型处理这种依赖。

- **CTC 解码**选项：
    - **贪婪解码**：在每个时间步取最可能的令牌，然后折叠。快速但次优。
    - **束搜索**：在每个步维护 top-$k$ 部分假设，合并折叠到相同前缀的假设。可以整合语言模型分数。
    - **前缀束搜索**：修改的束搜索正确处理 CTC 空白合并，确保假设在折叠后比较。

- **RNN-Transducer** (RNN-T) (Graves, 2012) 通过添加显式的**预测网络**（类似语言模型的 RNN）扩展 CTC，该网络将每个输出条件于先前的输出，移除条件独立性假设。RNN-T 有三个组件：
    - **编码器**：处理音频特征产生隐藏表示 $\mathbf{h}_t^\text{enc}$（通常是 LSTM 或 Conformer 层的堆栈）。
    - **预测网络**：自回归 RNN 从先前发出的标签产生隐藏表示 $\mathbf{h}_u^\text{pred}$。
    - **联合网络**：在每个（时间，标签）位置组合编码器和预测网络输出，产生下一个令牌的分布（包括空白）：

$$p(y | t, u) = \text{softmax}(W \cdot \text{tanh}(W_\text{enc} \mathbf{h}_t^\text{enc} + W_\text{pred} \mathbf{h}_u^\text{pred} + b))$$

- RNN-T 每个时间步可以发出零个或多个标签（通过在前进到下一个时间步之前发出非空白令牌，或发出空白不输出前进）。训练使用在 2D（时间，标签）晶格上的前向 - 后向算法，复杂度 $O(T \cdot U)$ 其中 $U$ 是输出长度。RNN-T 是设备上流式 ASR 的主导架构（用于 Google 的 Pixel 手机和类似产品），因为它自然支持流式：编码器从左到右处理音频，预测网络增量生成输出。

- **Listen, Attend and Spell** (LAS) (Chan et al., 2016) 是基于注意力的编码器 - 解码器模型（来自第 06 章的序列到序列架构）。它有三个组件：
    - **Listener**（编码器）：金字塔双向 LSTM 处理完整输入序列并通过 8 倍降采样（通过每层连接连续隐藏状态对），产生更短的编码器隐藏状态序列。
    - **Attention**：在每个解码器步，对所有编码器状态计算注意力权重形成上下文向量（与第 07 章相同的注意力机制）。
    - **Speller**（解码器）：自回归 LSTM 一次生成一个字符的输出转录，条件于上下文向量和先前生成的字符。

- LAS 取得强劲结果但需要完整话语在解码前可用（因为注意力关注所有编码器状态），使其不适合流式应用。它也难以处理非常长话语，因为长序列注意力变得弥散。

- **Conformer** (Gulati et al., 2020) 结合卷积的局部模式捕捉能力和自注意力的全局依赖建模。每个 Conformer 块有四个模块夹心结构：
    1. **前馈模块**（半步）：带残差连接的前馈网络，使用一半残差权重。
    2. **多头自注意力模块**：标准 transformer 自注意力（来自第 07 章）带相对位置编码。
    3. **卷积模块**：逐点卷积、门控线性单元 (GLU)、1D 深度卷积、批归一化、Swish 激活和另一个逐点卷积。深度卷积捕捉局部上下文（像特征序列上的 n-gram）。
    4. **前馈模块**（半步）：与模块 1 相同。

- 输出是：$\mathbf{y} = \text{LayerNorm}(\mathbf{x} + \frac{1}{2}\text{FFN}_1 + \text{MHSA} + \text{Conv} + \frac{1}{2}\text{FFN}_2)$。类似马卡龙的结构 (FFN-Attention-Conv-FFN) 带半步残差经验发现优于其他排序。Conformer 已成为 CTC 和 RNN-T 系统的默认编码器，优于纯 transformer 和纯 LSTM 编码器。

![Conformer 块显示前馈、自注意力、卷积和前馈模块的夹心结构](../images/conformer_block.svg)

- **Whisper** (Radford et al., 2023) 是 OpenAI 的大规模基于注意力的 ASR 模型。它使用标准编码器 - 解码器 transformer 架构（来自第 07 章）在从互联网爬取的 680,000 小时弱监督数据上训练（音频配近似转录）。关键设计选择：
    - 输入：80 通道 log-mel 频谱图（来自文件 01）带 25 ms 窗口和 10 ms 跳，归一化为零均值和单位方差。
    - 编码器：标准 transformer 编码器带正弦位置编码和预激活层归一化。
    - 解码器：transformer 解码器自回归生成令牌使用字节级 BPE 分词器（来自第 07 章）。
    - 多任务：单个模型处理转录、翻译、语言识别和时间戳预测，条件于解码器提示中的特殊任务令牌。
    - 训练数据规模（而非架构新颖性）是 Whisper 跨领域、口音和语言强泛化的主要驱动力。

- **wav2vec 2.0** (Baevski et al., 2020) 是语音表示的**自监督**预训练框架。核心思想是从大量无标签音频学习语音表示，然后用少量标记数据微调。遵循与 BERT（来自第 07 章）相同的自监督范式但适应连续音频信号。

- wav2vec 2.0 架构有三部分：
    - **特征编码器**：多层 1D CNN 处理原始波形样本，以 20 ms 帧率产生潜在表示 $\mathbf{z}_t$（16 kHz 下每 320 个样本一个向量）。
    - **量化模块**：使用**乘积量化**将潜在表示离散化为有限码本（将向量分成组并独立量化每组，从 $G$ 个码本每组 $V$ 个条目中选择）。这产生对比学习目标的靶标 $\mathbf{q}_t$。
    - **上下文网络**：transformer 编码器接受（部分掩码的）潜在表示产生上下文表示 $\mathbf{c}_t$。

![wav2vec 2.0 架构显示 CNN 特征编码器、掩码、transformer 上下文网络和带量化靶标的对比学习](../images/wav2vec2_pretraining.svg)

- 预训练期间，随机跨度的潜在表示被**掩码**（用学习的掩码嵌入替换），模型必须从干扰物（从同一话语其他位置采样的负样本）中识别掩码位置的真实量化表示。对比损失是：

$$\mathcal{L} = -\log \frac{\exp(\text{sim}(\mathbf{c}_t, \mathbf{q}_t) / \kappa)}{\sum_{\tilde{\mathbf{q}} \in Q_t} \exp(\text{sim}(\mathbf{c}_t, \tilde{\mathbf{q}}) / \kappa)}$$

- 其中 $\text{sim}$ 是余弦相似度，$\kappa$ 是温度参数，$Q_t$ 包括真实量化靶标加干扰物。附加的**多样性损失**鼓励均衡使用所有码本条目。该损失本质上是 InfoNCE 对比损失，与视觉自监督学习中使用的对比目标相同家族。

- 预训练后，线性投影和 CTC 头添加在上面，模型在标记数据上微调。wav2vec 2.0 仅用 10 分钟标记数据实现接近最先进的结果（使用 53,000 小时无标签音频预训练），展示自监督学习对低资源语音识别的力量。

- **HuBERT** (Hsu et al., 2021) 是另一种自监督方法，用**掩码预测**目标替换对比目标（预测掩码帧的离散聚类分配）。靶标由离线聚类步骤产生（第一次迭代在 MFCCs 上 k-means，然后在后续迭代在 HuBERT 特征上 k-means）。HuBERT 简化训练流水线相比 wav2vec 2.0（不需要量化模块或对比采样），实现可比或更好结果。

- **Fast Conformer** (Rekesh et al., 2023, NVIDIA NeMo) 用**降采样注意力**机制替换标准 Conformer 中的二次自注意力：在计算注意力前压缩输入序列（通常通过 strided convolution 8 倍），然后扩展回来。这将注意力成本从 $O(T^2)$ 减少到 $O(T^2/64)$ 同时保留全局上下文，能够在非常长话语（长达几分钟）上训练而无内存问题。Fast Conformer 是 NVIDIA NeMo 工具包中的默认编码器，形成其生产级模型的主干。

- **Parakeet** (NVIDIA, 2024) 是基于 Fast Conformer 编码器带 CTC 和 RNN-T 解码器的高精度英语 ASR 模型家族，在 64,000 小时英语语音上训练。Parakeet 模型（0.6B 和 1.1B 参数）在发布时在标准基准上实现最低词错误率，在大多数英语测试集上超越 Whisper large-v3。关键成分是高效 Fast Conformer 架构、激进数据增强（SpecAugment、速度扰动、噪声混合）和大规模监督训练数据——展示已知组件的精心工程仍能推动最先进水平。

- **Canary** (NVIDIA, 2024) 将 NeMo 框架扩展到多语言和多任务 ASR。它使用 Fast Conformer 编码器带基于注意力的解码器（而非 CTC 或 RNN-T），在单个模型中处理跨多种语言的转录加翻译（类似 Whisper 的多任务设计但带更高效的 Fast Conformer 主干）。Canary 模型支持英语、德语、西班牙语和法语带竞争力精度。

- **Moonshine** (Useful Sensors, 2024) 是专门为**设备上和边缘部署**优化的 ASR 模型家族。编码器使用混合架构，用小型 CNN 后跟少数 transformer 层替换初始 transformer/conformer 层，显著减少模型大小（基础模型小于 30M 参数）。Moonshine 针对 CPU 和低功耗设备上的实时流式，Whisper 在这些设备上太大太慢，以 5-10 倍更低延迟和内存占用换取一些精度。

- **Distil-Whisper** (Gandhi et al., 2023) 应用**知识蒸馏**（第 06 章）将 Whisper 压缩成更小更快的模型。学生模型仅用 2 个解码器层（相比 Whisper 的 32 个）同时保持完整编码器，训练匹配 Whisper 的输出分布。Distil-Whisper 实现与教师模型 1% WER 内同时快 6 倍，使全 Whisper 模型太慢的实时应用实用。

- **Universal Speech Model (USM)** (Zhang et al., 2023, Google) 将自监督预训练扩展到 300+ 种语言的 1200 万小时无标签音频，然后监督微调。USM 展示 wav2vec 2.0 / 自监督范式扩展到真正大规模数据机制，在非常有限标记数据下在低资源语言上实现强性能。

- **Massively Multilingual Speech (MMS)** (Pratap et al., 2023, Meta) 将 wav2vec 2.0 预训练扩展到 1,100+ 种语言使用宗教录音和其他多语言音频源。MMS 覆盖比任何先前 ASR 系统更多的语言，首次为许多资源不足语言启用语音识别。

- 现代 ASR 格局正收敛于几个主导模式：(1) Conformer 家族编码器带 CTC 或 RNN-T 用于流式，(2) 编码器 - 解码器 transformers 用于离线/多任务，(3) 自监督预训练用于低资源设置，(4) 规模——更多数据和更大模型一致提高精度。这些之间的选择取决于部署约束：延迟预算、可用计算、语言数量和应用程序是流式还是批处理。

- **语言模型整合**通过整合超出声学模型捕获的语言知识改进 ASR。基本思想是在解码期间组合声学模型分数 $p(\mathbf{x} | \mathbf{y})$（音频匹配转录的程度）和语言模型分数 $p(\mathbf{y})$（转录作为句子的可能性）。

- **浅层融合**在束搜索时组合分数：

$$\hat{\mathbf{y}} = \arg\max_\mathbf{y} \left[ \log p_\text{AM}(\mathbf{y} | \mathbf{x}) + \lambda \log p_\text{LM}(\mathbf{y}) \right]$$

- 其中 $\lambda$ 是可调权重，$p_\text{LM}$ 是外部语言模型（通常是来自第 07 章的 n-gram 或神经 LM）。这简单有效但要求 LM 与 ASR 模型操作相同令牌词汇。

- **深层融合** (Gulcehre et al., 2015) 将语言模型集成到解码器网络内：LM 隐藏状态与解码器隐藏状态连接并通过门控机制在输出投影前传递。整个系统（包括预训练 LM）联合微调。这允许更深整合但训练更复杂。

- **冷融合** (Sriram et al., 2018) 类似深层融合但从头训练 ASR 解码器带集成的语言模型，而非微调预训练解码器。这强制声学模型学习互补信息而非复制 LM 已知的内容。

- **重评分**（N-best 重评分）是两遍方法：首先使用束搜索生成 $N$ 个候选转录，然后用更强大的语言模型（例如，大型 transformer LM）重新排序它们。这实现简单并允许使用对第一遍解码太慢的非常大型 LM。

- **内部语言模型估计** (ILME) 解决微妙问题：端到端模型从训练转录隐式学习内部 LM，在浅层融合期间可能与外部 LM 冲突（本质上双重计数语言先验）。ILME 估计内部 LM 并在融合期间减去其分数：

$$\hat{\mathbf{y}} = \arg\max_\mathbf{y} \left[ \log p_\text{E2E}(\mathbf{y} | \mathbf{x}) - \beta \log p_\text{ILM}(\mathbf{y}) + \lambda \log p_\text{LM}(\mathbf{y}) \right]$$

- **流式与离线 ASR** 是基本架构选择。离线（或批处理）ASR 在处理任何输出前处理完整话语。流式 ASR 在音频到达时增量产生输出，带有限延迟。

- 流式对实时应用必要：实时字幕、语音助手（用户期望在他们说完前响应）、电话呼叫转录。挑战是一些未来上下文有助于识别（知道下一个词是"York"消歧"New"），但流式系统不能等待任意长的未来上下文。

- **单向编码器**（从左到右 LSTMs、因果卷积、因果 transformers）自然支持流式因为每个输出仅依赖过去和当前输入。双向编码器（查看未来上下文）不直接支持流式。

- **分块注意力**（也称为分块或分段注意力）将输入分成固定长度块并对每个块内（和可选地少数前续块）应用自注意力。这将延迟限制为块大小加处理时间，同时仍然允许每个块内一些局部双向上下文。权衡是精度随块大小减少而下降。

- **前瞻**允许流式编码器在产生当前帧输出前查看少量未来帧（例如，300-900 ms）。这通过添加小右上下文到单向计算实现。前瞻窗口增加延迟但显著提高精度。

- 流式 ASR 的**延迟**有几个组件：
    - **算法延迟**：从音频到达时到模型能处理它的延迟（由块大小、前瞻和特征提取决定）。
    - **计算延迟**：运行模型前向传播的时间。
    - **端点延迟**：检测用户说完话的延迟。
    - **首令牌延迟**：第一个词出现多快。**完成延迟**：最终输出确认多快（流式系统通常产生临时输出，随着更多音频到达被纠正）。

- ASR 的**评估指标**：

- **词错误率** (WER) 是主要指标。它通过对齐假设（系统输出）到参考（真实转录）使用编辑距离（将一个转换为另一个的最小编辑数：替换、删除和插入）计算，然后：

$$\text{WER} = \frac{S + D + I}{N}$$

- 其中 $S$ 是替换，$D$ 是删除，$I$ 是插入，$N$ 是参考中总词数。WER 可能超过 100% 如果有许多插入。对于清晰阅读语音，5% WER 被认为大约人类水平；会话或噪声语音难得多（10-20%+）。

- **字符错误率** (CER) 是在字符级别而非词级别应用的相同公式。CER 对无清晰词边界的语言（中文、日语）和评估接近失误的程度（"cat" vs "bat" 是 100% WER 但 33% CER）更具信息性。

- **词信息丢失** (WIL) 和**词信息保留** (WIP) 是信息论替代，比 WER 更精确地考虑参考和假设之间的相关性，但它们较少被报告。

- **实时因子** (RTF) 测量计算效率：处理时间与音频持续时间的比率。RTF < 1 意味着系统运行快于实时；RTF > 1 意味着无法跟上实时音频。流式系统必须维持 RTF < 1。

- **数据增强**对鲁棒 ASR 关键。常用技术：
    - **速度扰动**：以 0.9x 和 1.1x 速度重采样音频（改变音高和持续时间）。
    - **SpecAugment** (Park et al., 2019)：在频谱图中掩码随机频带和时间步。这是音频类的 dropout，是 ASR 最有效的正则化技术之一。它不需要额外数据。
    - **噪声增强**：以各种信噪比将干净语音与录制噪声混合。
    - **房间脉冲响应模拟**：用模拟房间声学卷积干净语音模拟混响环境。

- ASR 的**分词**决定模型的输出词汇。选项包括：
    - **字符**：简单、小词汇（英语约 30），但长输出序列和无隐式语言建模。
    - **词块 / BPE**（来自第 07 章）：子词单元平衡词汇大小和序列长度。现代系统的标准（Whisper 使用字节级 BPE 约 50,000 令牌）。
    - **词**：大词汇（50,000+）、短输出序列，但不能处理词汇外词。
    - **音素**：语言学习动机、紧凑，但需要发音词典。

- ASR 的演变可总结为从重度工程模块化系统（GMM-HMM + WFST 解码，1990 年代 -2010 年代）到混合系统（DNN-HMM，2012-2016）到端到端系统（吸收越来越多流水线到单个神经网络：CTC、RNN-T、LAS，2016-2020）到利用大量无标签或弱标签数据的大规模预训练模型（wav2vec 2.0、Whisper，2020 至今）的进展。每次转变简化工程同时提高精度，遵循机器学习中从数据学习表示而非手工设计它们的更广泛趋势（第 06 章中图像特征被 CNNs 替换，第 07 章中 NLP 特征被 transformers 替换讲述的相同故事）。

1. 用 JAX 从头实现 CTC 损失。创建短序列 logits 和目标标签的玩具示例，计算 CTC 前向算法获得总概率，计算负对数似然损失。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def ctc_forward(log_probs, targets):
    """
    CTC 前向算法（对数域数值稳定）。
    log_probs: (T, V) 词汇表上的对数概率（索引 0 = 空白）
    targets: (U,) 目标标签索引（无空白）
    返回：CTC 下目标序列的对数概率。
    """
    T, V = log_probs.shape
    U = len(targets)

    # 用空白构建扩展标签序列：[空白，y1, 空白，y2, ..., yU, 空白]
    S = 2 * U + 1
    labels = jnp.zeros(S, dtype=jnp.int32)  # 全空白
    for i in range(U):
        labels = labels.at[2 * i + 1].set(targets[i])

    # 初始化 alpha（对数域）
    NEG_INF = -1e30
    alpha = jnp.full((T, S), NEG_INF)
    alpha = alpha.at[0, 0].set(log_probs[0, labels[0]])        # 从空白开始
    alpha = alpha.at[0, 1].set(log_probs[0, labels[1]])        # 或第一个标签

    # 填充前向
    for t in range(1, T):
        for s in range(S):
            # 相同状态
            a = alpha[t - 1, s]
            # 从前一状态
            if s > 0:
                a = jnp.logaddexp(a, alpha[t - 1, s - 1])
            # 跳过空白（如果当前和两个后的标签不同）
            if s > 1 and labels[s] != 0 and labels[s] != labels[s - 2]:
                a = jnp.logaddexp(a, alpha[t - 1, s - 2])
            alpha = alpha.at[t, s].set(a + log_probs[t, labels[s]])

    # 总对数概率：最后时间步最后两个状态的总和
    log_prob = jnp.logaddexp(alpha[T - 1, S - 1], alpha[T - 1, S - 2])
    return log_prob, alpha

# --- 玩具示例 ---
T = 12   # 输入长度（时间步）
V = 5    # 词汇大小（0=空白，1='c', 2='a', 3='t', 4='x'）
targets = jnp.array([1, 2, 3])  # "c", "a", "t"

# 创建随机 logits 并转换为对数概率
key = jax.random.PRNGKey(42)
logits = jax.random.normal(key, (T, V))
log_probs = jax.nn.log_softmax(logits, axis=-1)

log_prob, alpha = ctc_forward(log_probs, targets)
ctc_loss = -log_prob

print(f"目标序列：{targets.tolist()} ('c', 'a', 't')")
print(f"输入长度 T={T}, 词汇大小 V={V}")
print(f"CTC 对数概率：{log_prob:.4f}")
print(f"CTC 损失（负对数概率）：{ctc_loss:.4f}")

# 可视化前向变量（alpha）晶格
fig, ax = plt.subplots(figsize=(12, 5))
# 从对数转换为线性用于可视化
alpha_linear = jnp.exp(alpha - jnp.max(alpha))  # 归一化用于可见性
im = ax.imshow(alpha_linear.T, aspect='auto', origin='lower', cmap='viridis')
ax.set_xlabel('时间步 (t)')
ax.set_ylabel('扩展标签索引 (s)')

label_names = ['_', 'c', '_', 'a', '_', 't', '_']  # _ = 空白
ax.set_yticks(range(len(label_names)))
ax.set_yticklabels(label_names)
ax.set_title(f'CTC 前向变量（alpha 晶格）| 损失 = {ctc_loss:.2f}')
plt.colorbar(im, ax=ax, label='归一化概率')
plt.tight_layout(); plt.show()
```

2. 在 JAX 中构建简单编码器 - 解码器基于注意力的 ASR 模型（最小 LAS 类架构）。使用 1D 卷积编码器和带点积注意力的单层解码器。在合成数据上运行并可视化注意力权重。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# --- 最小基于注意力的编码器 - 解码器用于 ASR ---

def init_params(key, input_dim, hidden_dim, vocab_size):
    """初始化最小 LAS 类模型的参数。"""
    keys = jax.random.split(key, 8)
    scale = 0.1
    params = {
        # 编码器：简单线性投影（模拟 conv 输出）
        'enc_w': jax.random.normal(keys[0], (input_dim, hidden_dim)) * scale,
        'enc_b': jnp.zeros(hidden_dim),
        # 注意力：查询、键、值投影
        'attn_q': jax.random.normal(keys[1], (hidden_dim, hidden_dim)) * scale,
        'attn_k': jax.random.normal(keys[2], (hidden_dim, hidden_dim)) * scale,
        'attn_v': jax.random.normal(keys[3], (hidden_dim, hidden_dim)) * scale,
        # 解码器 RNN（简单 Elman RNN 用于说明）
        'dec_wh': jax.random.normal(keys[4], (hidden_dim, hidden_dim)) * scale,
        'dec_wx': jax.random.normal(keys[5], (vocab_size, hidden_dim)) * scale,
        'dec_wc': jax.random.normal(keys[6], (hidden_dim, hidden_dim)) * scale,
        'dec_b': jnp.zeros(hidden_dim),
        # 输出投影
        'out_w': jax.random.normal(keys[7], (hidden_dim, vocab_size)) * scale,
        'out_b': jnp.zeros(vocab_size),
    }
    return params

def encode(params, x):
    """编码器：线性投影（conv/LSTM 堆栈的占位符）。"""
    return jnp.tanh(x @ params['enc_w'] + params['enc_b'])

def attend(params, query, enc_out):
    """对编码器输出的点积注意力。"""
    q = query @ params['attn_q']                   # (hidden,)
    k = enc_out @ params['attn_k']                 # (T_enc, hidden)
    v = enc_out @ params['attn_v']                 # (T_enc, hidden)
    d_k = q.shape[-1]
    scores = (k @ q) / jnp.sqrt(d_k)              # (T_enc,)
    weights = jax.nn.softmax(scores)               # (T_enc,)
    context = weights @ v                          # (hidden,)
    return context, weights

def decode_step(params, h_prev, y_prev_onehot, enc_out):
    """单解码器步：RNN + 注意力。"""
    # 嵌入前一个令牌
    y_emb = y_prev_onehot @ params['dec_wx']       # (hidden,)
    # 关注编码器
    context, attn_w = attend(params, h_prev, enc_out)
    # RNN 更新
    h = jnp.tanh(h_prev @ params['dec_wh'] + y_emb + context @ params['dec_wc']
                  + params['dec_b'])
    # 输出 logits
    logits = h @ params['out_w'] + params['out_b']
    return h, logits, attn_w

# --- 设置 ---
key = jax.random.PRNGKey(0)
input_dim = 40       # 例如，40 mel 波段
hidden_dim = 64
vocab_size = 10      # 小词汇演示
T_enc = 30           # 编码器时间步
T_dec = 8            # 解码器步

params = init_params(key, input_dim, hidden_dim, vocab_size)

# 合成输入：随机 mel 类特征
key, subkey = jax.random.split(key)
x = jax.random.normal(subkey, (T_enc, input_dim))

# 编码
enc_out = encode(params, x)

# 解码（教师强制带随机目标）
key, subkey = jax.random.split(key)
targets = jax.random.randint(subkey, (T_dec,), 0, vocab_size)

h = jnp.zeros(hidden_dim)
all_logits = []
all_attn = []

for t in range(T_dec):
    y_prev = jax.nn.one_hot(targets[t] if t > 0 else 0, vocab_size)
    h, logits, attn_w = decode_step(params, h, y_prev, enc_out)
    all_logits.append(logits)
    all_attn.append(attn_w)

all_attn = jnp.stack(all_attn)  # (T_dec, T_enc)
all_logits = jnp.stack(all_logits)  # (T_dec, vocab_size)

# --- 可视化注意力权重 ---
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

im = axes[0].imshow(all_attn, aspect='auto', cmap='Blues', origin='lower')
axes[0].set_xlabel('编码器时间步')
axes[0].set_ylabel('解码器步')
axes[0].set_title('注意力权重（解码器 -> 编码器）')
plt.colorbar(im, ax=axes[0])

# 显示每个解码器步的预测令牌分布
im2 = axes[1].imshow(jax.nn.softmax(all_logits, axis=-1), aspect='auto',
                      cmap='Oranges', origin='lower')
axes[1].set_xlabel('词汇索引')
axes[1].set_ylabel('解码器步')
axes[1].set_title('输出令牌概率')
plt.colorbar(im2, ax=axes[1])

plt.suptitle('最小基于注意力的 ASR 模型（未训练）')
plt.tight_layout(); plt.show()
```

3. 使用动态规划（编辑距离）从头计算词错误率 (WER)，评估多个假设与参考。可视化编辑距离矩阵。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt
import numpy as np

def compute_wer(reference, hypothesis):
    """
    使用动态规划计算 WER（词上的 Levenshtein 距离）。
    返回 WER、替换数、删除、插入和 DP 矩阵。
    """
    ref_words = reference.split()
    hyp_words = hypothesis.split()
    N = len(ref_words)
    M = len(hyp_words)

    # DP 矩阵：d[i][j] = ref[:i] 和 hyp[:j] 之间的编辑距离
    d = np.zeros((N + 1, M + 1), dtype=np.int32)
    # 回溯矩阵计数 S, D, I
    ops = np.zeros((N + 1, M + 1, 3), dtype=np.int32)  # [sub, del, ins]

    for i in range(N + 1):
        d[i][0] = i  # 全删除
    for j in range(M + 1):
        d[0][j] = j  # 全插入

    for i in range(1, N + 1):
        for j in range(1, M + 1):
            if ref_words[i - 1] == hyp_words[j - 1]:
                sub_cost = d[i - 1][j - 1]  # 匹配，无编辑
            else:
                sub_cost = d[i - 1][j - 1] + 1  # 替换
            del_cost = d[i - 1][j] + 1      # 删除
            ins_cost = d[i][j - 1] + 1      # 插入

            d[i][j] = min(sub_cost, del_cost, ins_cost)

    # 回溯计数操作
    i, j = N, M
    S, D, I = 0, 0, 0
    while i > 0 or j > 0:
        if i > 0 and j > 0 and d[i][j] == d[i-1][j-1] and ref_words[i-1] == hyp_words[j-1]:
            i -= 1; j -= 1  # 正确
        elif i > 0 and j > 0 and d[i][j] == d[i-1][j-1] + 1:
            S += 1; i -= 1; j -= 1  # 替换
        elif i > 0 and d[i][j] == d[i-1][j] + 1:
            D += 1; i -= 1  # 删除
        elif j > 0 and d[i][j] == d[i][j-1] + 1:
            I += 1; j -= 1  # 插入
        else:
            break

    wer = (S + D + I) / N if N > 0 else 0.0
    return wer, S, D, I, d

# --- 测试用例 ---
reference = "the cat sat on the mat"
hypotheses = [
    "the cat sat on the mat",          # 完美
    "the cat sit on the mat",          # 1 替换
    "the cat on the mat",              # 1 删除
    "the big cat sat on the mat",      # 1 插入
    "a dog sat in a rug",              # 多个错误
]

print(f"参考：'{reference}'\n")
print(f"{'假设':<40s} {'WER':>6s} {'S':>3s} {'D':>3s} {'I':>3s}")
print("-" * 60)
results = []
for hyp in hypotheses:
    wer, S, D, I, dp = compute_wer(reference, hyp)
    results.append((hyp, wer, S, D, I, dp))
    print(f"'{hyp}':<40s} {wer:>6.1%} {S:>3d} {D:>3d} {I:>3d}")

# 可视化最坏情况的 DP 矩阵
worst = results[-1]
hyp_words = worst[0].split()
ref_words = reference.split()
dp_matrix = worst[5]

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# DP 矩阵
im = axes[0].imshow(dp_matrix, cmap='YlOrRd', origin='upper')
axes[0].set_xticks(range(len(hyp_words) + 1))
axes[0].set_xticklabels([''] + hyp_words, rotation=45, ha='right', fontsize=9)
axes[0].set_yticks(range(len(ref_words) + 1))
axes[0].set_yticklabels([''] + ref_words, fontsize=9)
axes[0].set_xlabel('假设词')
axes[0].set_ylabel('参考词')
axes[0].set_title(f'编辑距离矩阵\nWER = {worst[1]:.1%}')
for i in range(dp_matrix.shape[0]):
    for j in range(dp_matrix.shape[1]):
        axes[0].text(j, i, str(dp_matrix[i, j]), ha='center', va='center', fontsize=8)
plt.colorbar(im, ax=axes[0])

# WER 比较条形图
names = [f'Hyp {i+1}' for i in range(len(results))]
wers = [r[1] * 100 for r in results]
colors = ['#27ae60' if w == 0 else '#f39c12' if w < 30 else '#e74c3c' for w in wers]
axes[1].barh(names, wers, color=colors)
axes[1].set_xlabel('WER (%)')
axes[1].set_title('词错误率比较')
for i, (w, r) in enumerate(zip(wers, results)):
    axes[1].text(w + 1, i, f'{w:.0f}% (S={r[2]}, D={r[3]}, I={r[4]})',
                 va='center', fontsize=9)
axes[1].set_xlim(0, max(wers) * 1.4)
```

4. 在 log-mel 频谱图上实现 SpecAugment（频率掩码和时间掩码）并可视化原始与增强版本。从合成信号生成频谱图。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# --- 生成合成 log-mel 频谱图 ---
key = jax.random.PRNGKey(42)
fs = 16000
duration = 2.0
t = jnp.arange(0, duration, 1.0 / fs)

# 模拟语音：带谐波的啁啾信号
f0 = 120.0
x = sum(jnp.sin(2 * jnp.pi * f0 * k * t * (1 + 0.1 * t)) / k for k in range(1, 10))
key, subkey = jax.random.split(key)
x = x + 0.05 * jax.random.normal(subkey, t.shape)

# 计算 log-mel 频谱图（简化）
frame_len = 400  # 25 ms
hop_len = 160    # 10 ms
n_fft = 512
n_mels = 80

n_frames = (len(x) - frame_len) // hop_len + 1
hamming = 0.54 - 0.46 * jnp.cos(2 * jnp.pi * jnp.arange(frame_len) / (frame_len - 1))

frames = jnp.stack([x[i * hop_len : i * hop_len + frame_len] for i in range(n_frames)])
windowed = frames * hamming
spectra = jnp.abs(jnp.fft.rfft(windowed, n=n_fft)) ** 2

# 简单 mel 滤波器组
def hz_to_mel(f): return 2595 * jnp.log10(1 + f / 700)
def mel_to_hz(m): return 700 * (10 ** (m / 2595) - 1)

mel_points = jnp.linspace(hz_to_mel(0), hz_to_mel(fs / 2), n_mels + 2)
hz_pts = mel_to_hz(mel_points)
bins = jnp.floor((n_fft + 1) * hz_pts / fs).astype(jnp.int32)

n_freqs = n_fft // 2 + 1
fb = jnp.zeros((n_mels, n_freqs))
for m in range(n_mels):
    lo, mid, hi = int(bins[m]), int(bins[m+1]), int(bins[m+2])
    for k in range(lo, mid):
        if mid != lo:
            fb = fb.at[m, k].set((k - lo) / (mid - lo))
    for k in range(mid, hi):
        if hi != mid:
            fb = fb.at[m, k].set((hi - k) / (hi - mid))

log_mel = jnp.log(spectra @ fb.T + 1e-10)

# --- SpecAugment ---
def spec_augment(spec, key, n_freq_masks=2, freq_mask_width=15,
                 n_time_masks=2, time_mask_width=25):
    """应用 SpecAugment：频率和时间掩码。"""
    augmented = spec.copy()
    T, F = spec.shape

    # 频率掩码
    for _ in range(n_freq_masks):
        key, k1, k2 = jax.random.split(key, 3)
        f_width = jax.random.randint(k1, (), 1, freq_mask_width + 1)
        f_start = jax.random.randint(k2, (), 0, max(1, F - freq_mask_width))
        mask = (jnp.arange(F) >= f_start) & (jnp.arange(F) < f_start + f_width)
        augmented = jnp.where(mask[None, :], 0.0, augmented)

    # 时间掩码
    for _ in range(n_time_masks):
        key, k1, k2 = jax.random.split(key, 3)
        t_width = jax.random.randint(k1, (), 1, time_mask_width + 1)
        t_start = jax.random.randint(k2, (), 0, max(1, T - time_mask_width))
        mask = (jnp.arange(T) >= t_start) & (jnp.arange(T) < t_start + t_width)
        augmented = jnp.where(mask[:, None], 0.0, augmented)

    return augmented

key, subkey = jax.random.split(key)
log_mel_aug = spec_augment(log_mel, subkey)
```
