## 🔑 第二部分：常见排序算法

### 1. 冒泡排序（Bubble Sort）

**Tom 的"气泡上升"法**：

```
原理：相邻的两个奶酪比较，大的往后放

初始：[5🧀] [2🧀] [8🧀] [1🧀] [9🧀]

第 1 轮：
  [5🧀] [2🧀] → [2🧀] [5🧀]  (5>2，交换)
  [5🧀] [8🧀] → [5🧀] [8🧀]  (5<8，不交换)
  [8🧀] [1🧀] → [1🧀] [8🧀]  (8>1，交换)
  [8🧀] [9🧀] → [8🧀] [9🧀]  (8<9，不交换)
  结果：[2🧀] [5🧀] [1🧀] [8🧀] [9🧀]  ← 最大的 9 已经"冒泡"到右边

第 2 轮：
  继续... 第二大的 8 会到倒数第二的位置

重复直到全部有序！
```

**代码实现**：

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        # 最后 i 个元素已经有序
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                # 交换
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
```

**特点**：
- ⏱️ 时间复杂度：O(n²)
- 📦 空间复杂度：O(1)
- ✅ 稳定排序（相同元素顺序不变）
- 😐 效率较低，适合小数据

### 2. 选择排序（Selection Sort）

**Tom 的"选最小的放前面"法**：

```
原理：每轮选择最小的，放到已排序区的末尾

初始：[5🧀] [2🧀] [8🧀] [1🧀] [9🧀]
      ↑未排序区

第 1 轮：找到最小的 1，和第一个位置交换
       [1🧀] [2🧀] [8🧀] [5🧀] [9🧀]
       ↑已排序  ↑未排序

第 2 轮：在未排序区找最小的 2，已在正确位置
       [1🧀] [2🧀] [8🧀] [5🧀] [9🧀]
       ↑↑已排序

第 3 轮：找到最小的 5，和第三个位置交换
       [1🧀] [2🧀] [5🧀] [8🧀] [9🧀]
       ↑↑↑已排序

继续直到全部有序！
```

**代码实现**：

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        # 找到未排序部分的最小值
        min_idx = i
        for j in range(i+1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        # 交换到已排序区的末尾
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

**特点**：
- ⏱️ 时间复杂度：O(n²)
- 📦 空间复杂度：O(1)
- ❌ 不稳定排序
- 😐 交换次数少，适合写入昂贵的场景

### 3. 插入排序（Insertion Sort）

**Tom 的"整理扑克牌"法**：

```
原理：每次拿起一张牌，插入到已排序部分的正确位置

初始：[5🧀] [2🧀] [8🧀] [1🧀] [9🧀]
      ↑已排序 ↑未排序

第 1 步：拿起 2，插入到 5 前面
       [2🧀] [5🧀] [8🧀] [1🧀] [9🧀]
       ↑↑↑已排序

第 2 步：拿起 8，已在正确位置
       [2🧀] [5🧀] [8🧀] [1🧀] [9🧀]
       ↑↑↑↑↑已排序

第 3 步：拿起 1，插入到最前面
       [1🧀] [2🧀] [5🧀] [8🧀] [9🧀]
       ↑↑↑↑↑↑↑已排序

完成！
```

**代码实现**：

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]  # 当前要插入的元素
        j = i - 1
        # 把比 key 大的元素往后移
        while j >= 0 and arr[j] > key:
            arr[j+1] = arr[j]
            j -= 1
        arr[j+1] = key  # 插入到正确位置
    return arr
```

**特点**：
- ⏱️ 时间复杂度：O(n²)
- 📦 空间复杂度：O(1)
- ✅ 稳定排序
- 😊 对"几乎有序"的数据效率高（O(n)）

### 4. 归并排序（Merge Sort）

**Tom 的"分而治之"法**：

```
原理：先拆分，再合并

初始：[5🧀] [2🧀] [8🧀] [1🧀] [9🧀] [3🧀]

【分解阶段】
拆分：[5🧀] [2🧀] [8🧀] | [1🧀] [9🧀] [3🧀]
      ↓
拆分：[5🧀] [2🧀] | [8🧀] | [1🧀] [9🧀] | [3🧀]
      ↓
拆分：[5🧀] | [2🧀] | [8🧀] | [1🧀] | [9🧀] | [3🧀]
      (不能再分了)

【合并阶段】
合并：[2🧀 5🧀] | [8🧀] | [1🧀 9🧀] | [3🧀]
      ↓
合并：[2🧀 5🧀 8🧀] | [1🧀 3🧀 9🧀]
      ↓
合并：[1🧀 2🧀 3🧀 5🧀 8🧀 9🧀]
      (完成！)
```

**代码实现**：

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    
    # 分解
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    # 合并
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    # 添加剩余元素
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**特点**：
- ⏱️ 时间复杂度：O(n log n)
- 📦 空间复杂度：O(n)
- ✅ 稳定排序
- 😊 适合大数据

### 5. 快速排序（Quick Sort）

**Tom 的"选基准分左右"法**：

```
原理：选一个基准，小的放左边，大的放右边，递归处理

初始：[5🧀] [2🧀] [8🧀] [1🧀] [9🧀] [3🧀]
       选基准：5🧀

第 1 轮分区：
  小于 5: [2🧀] [1🧀] [3🧀]
  等于 5: [5🧀]
  大于 5: [8🧀] [9🧀]
  
  结果：[2🧀 1🧀 3🧀] [5🧀] [8🧀 9🧀]

第 2 轮：递归处理左右两边
  左边 [2🧀 1🧀 3🧀] 选基准 2:
    小于 2: [1🧀]
    等于 2: [2🧀]
    大于 2: [3🧀]
    结果：[1🧀] [2🧀] [3🧀]
  
  右边 [8🧀 9🧀] 选基准 8:
    小于 8: []
    等于 8: [8🧀]
    大于 8: [9🧀]
    结果：[8🧀] [9🧀]

合并：[1🧀 2🧀 3🧀] [5🧀] [8🧀 9🧀]
完成！
```

**代码实现**：

```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    
    # 选基准（中间元素）
    pivot = arr[len(arr) // 2]
    
    # 分区
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    # 递归
    return quick_sort(left) + middle + quick_sort(right)
```

**特点**：
- ⏱️ 平均时间复杂度：O(n log n)
- ⏱️ 最坏情况：O(n²)
- 📦 空间复杂度：O(log n)（递归栈）
- ❌ 不稳定排序
- 🚀 实际使用最快（缓存友好）

## 🔍 第三部分：搜索算法

### 1. 线性搜索（Linear Search）

**Tom 的"一个一个找"法**：

```
场景：仓库还没整理，要找大小为 5 的奶酪

[5🧀] [2🧀] [8🧀] [1🧀] [9🧀] [3🧀]
 ↑
 找到了！在第 1 个位置

最坏情况：要找的在最后或不存在
要检查所有 n 个元素
```

**代码实现**：

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i  # 返回索引
    return -1  # 没找到
```

**特点**：
- ⏱️ 时间复杂度：O(n)
- 📦 空间复杂度：O(1)
- 😊 无需排序，简单直接

### 2. 二分搜索（Binary Search）

**Tom 的"猜数字"法**：

```
场景：仓库已整理好，要找大小为 5 的奶酪

[1🧀] [2🧀] [3🧀] [5🧀] [8🧀] [9🧀]
      ↑              ↑
    left           right

第 1 步：看中间 (3+5)/2=4 位置，值为 5
       5 == 5，找到了！

如果没找到中间值：
如果目标 < 中间：搜索左半部分
如果目标 > 中间：搜索右半部分

每次排除一半的数据！
```

**代码实现**：

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1  # 没找到
```

**特点**：
- ⏱️ 时间复杂度：O(log n)
- 📦 空间复杂度：O(1)
- ⚠️ 要求数据已排序
- 🚀 非常快！100 万元素只需约 20 次比较
