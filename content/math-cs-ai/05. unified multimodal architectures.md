# 统一多模态架构

*统一多模态架构用单个系统取代独立的专家模型，该系统可以读取、推理和生成文本、图像、音频和视频。本章涵盖任意到任意模型（CoDi、NExT-GPT）、原生多模态 LLM（Gemini、GPT-4o）、多模态标记化策略以及统一的架构权衡。*

## 统一的理由

- 想象一个会说五种语言的翻译员，可以在句子中间切换语言而无需停顿。早期多模态系统更像是五个独立的翻译员坐在不同的房间里，每人处理一种语言并通过墙上的插槽传递笔记。统一多模态架构是单个通晓多种语言的人：一个具有共享权重的模型，在单次前向传递中读取、写入和推理文本、图像、音频、视频甚至动作。

- 动机既有实际的也有理论的。在实际方面，为每个模态对（文本到图像、图像到文本、音频到文本等）维护独立的专家模型会导致组合爆炸：$k$ 个模态需要多达 $k(k-1)$ 个有向管道。统一模型将所有这些折叠成单个系统。在理论方面，人类认知不在孤立的模块中处理视觉和语言；跨模态绑定发生得早且深，统一试图反映这一点。

- 共享权重鼓励跨模态迁移。一个已经学习了文本中时间模式（主语在动词之前，原因在效果之前）的 transformer 可以重新利用这些相同的注意力电路来处理视频中的时间模式（物体在移动之前出现）或音频（开始之前持续）。这是你在第 7 章看到的语言模型微调和第 8 章看到的 ImageNet 预训练的多模态类似物。

- 形式化地，让 $\mathcal{M} = \{m_1, m_2, \ldots, m_k\}$ 是一组模态。统一模型定义单个参数化函数 $f_\theta$，将任何输入模态子集映射到任何输出模态子集：

$$f_\theta : \mathcal{P}(\mathcal{M}) \rightarrow \mathcal{P}(\mathcal{M})$$

- 其中 $\mathcal{P}(\mathcal{M})$ 是模态的幂集（所有子集）。关键约束是 $\theta$ 大部分是共享的；只有薄的、特定于模态的适配器层不同。

![高级图表显示多个模态（文本、图像、音频、视频）输入到单个共享 transformer 骨干并产生任何模态的输出](../images/unified_multimodal_overview.svg)

- 统一的承诺伴随着一个基本张力：模态在结构上是不同的。文本是离散标记的一维序列。图像是连续像素值的二维网格。音频是一维连续波形，具有与文本非常不同的时间尺度。视频为图像添加了时间轴。将这些不同的结构调和成 transformer 可以消化的单个序列是该领域的核心工程挑战。

## 任意到任意模型

- 想想一个可以操作电视、空调和音响系统的万能遥控器，都通过相同的界面。任意到任意模型是 AI 等价物：它们接受任何模态组合作为输入并产生任何组合作为输出。

- CoDi（可组合扩散）通过训练特定于模态的扩散模型然后通过共享调节机制对齐它们的潜在空间来实现任意到任意生成。每个模态都有自己的扩散过程（回顾本章文件 04 的扩散模型），但噪声预测网络通过联合交叉注意力层调节，该层同时查看所有输入模态的嵌入。这让 CoDi 能够在单次传递中从文本提示生成例如图像和匹配的音频。

- NExT-GPT 采用不同的架构方法。它通过轻量级投影层将 LLM 骨干（"大脑"）连接到输入侧的特定于模态的编码器和输出侧的特定于模态的解码器。输入编码器（例如来自 CLIP 的图像编码器，来自 CLAP 的音频编码器）将每个模态翻译成 LLM 的嵌入空间。LLM 在组合标记序列上推理并发射特殊的"模态信号标记"，将信息路由到适当的解码器（例如用于图像的 Stable Diffusion，用于音频的 AudioLDM）。只有投影层被训练；LLM 和专家编码器/解码器保持冻结。

- Gemini（Google DeepMind）从预训练开始就是原生多模态。与 NExT-GPT 的即插即用方法不同，Gemini 的 transformer 从头开始在交错的文本、图像、音频和视频标记序列上进行训练。这意味着跨模态注意力模式在预训练期间有机地发展，而不是事后强行添加。该模型使用 SentencePiece 标记器处理文本，并学习类似于本章文件 03 中讨论的 VQ 方法的视觉标记器。

- GPT-4o（"o" 代表"omni"）代表了另一种模式：端到端模型，其中所有模态共享相同的 transformer 和相同的下一标记预测目标。音频输入作为频谱标记处理，图像作为补丁标记处理，文本作为子词标记处理，全部输入到单个序列中。该模型生成由特定于模态的头解码的输出标记。关键创新是通过移除早期系统如 GPT-4V 依赖的单独 ASR、LLM 和 TTS 模型的级联实现的低延迟。

![CoDi（对齐扩散）、NExT-GPT（具有冻结专家的 LLM 枢纽）和 Gemini 风格（原生交错预训练）的架构模式比较](../images/any_to_any_architectures.svg)

- 这些模型位于集成深度的谱系上：

    - 浅层集成（NExT-GPT）：通过训练的适配器连接的冻结专家。构建快速，跨模态推理有限。
    - 中等集成（CoDi）：跨特定于模态的生成器共享调节。更好的对齐，仍然模块化。
    - 深层集成（Gemini、GPT-4o）：在所有模态上端到端训练的单个模型。最丰富的跨模态推理，训练成本最高。

## 具有共享骨干的特定于模态编码器和解码器

- 想象一个工厂，具有单个装配线（共享骨干），但原材料有不同的装载码头（编码器），成品有不同的运输部门（解码器）。每个码头专门用于其货物，但一旦进入工厂，一切都沿着相同的传送带移动。

- 统一模型的主要架构模式使用这个三部分结构：

    - 模态编码器 $E_m$ 将来自模态 $m$ 的原始输入转换为嵌入向量序列 $\mathbf{h}_1^m, \mathbf{h}_2^m, \ldots, \mathbf{h}_{n_m}^m$，每个维度为 $d$。
    - 共享 transformer 骨干 $T_\theta$ 使用自注意力处理来自所有输入模态的连接或交错嵌入。
    - 模态解码器 $D_m$ 将骨干的输出嵌入转换回模态 $m$ 的本机格式（文本标记、图像像素、音频波形）。

- 对于文本，编码器通常是嵌入查找表 $E_\text{text}(w) = \mathbf{W}_e[w]$，其中 $w$ 是标记索引，与你在第 7 章看到的 transformer 相同。对于图像，编码器通常是 Vision Transformer（ViT），将图像分割成补丁并将每个补丁线性投影，如第 8 章所述。对于音频，编码器计算 mel 语谱图并使用卷积前端或音频语谱图 transformer（AST）处理它，如第 9 章所讨论的。

- 共享骨干是具有跨所有模态标记自注意力的标准 transformer。给定连接的输入序列 $\mathbf{H} = [\mathbf{h}_1^{m_1}, \ldots, \mathbf{h}_{n_1}^{m_1}, \mathbf{h}_1^{m_2}, \ldots, \mathbf{h}_{n_2}^{m_2}]$，自注意力允许每个标记关注任何其他标记，无论模态如何：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V}$$

- 这是第 7 章的相同注意力公式，但现在 $\mathbf{Q}$、$\mathbf{K}$ 和 $\mathbf{V}$ 包含来自多个模态的标记。图像补丁标记可以关注文本标记，实现跨模态推理，无需任何单独的交叉注意力模块。

- 模态嵌入添加到每个标记，以便骨干知道标记来自哪个模态。这类似于位置嵌入，但编码模态标识而不是序列位置。可学习向量 $\mathbf{e}_m \in \mathbb{R}^d$ 添加到来自模态 $m$ 的每个标记：

$$\tilde{\mathbf{h}}_i^m = \mathbf{h}_i^m + \mathbf{e}_m + \mathbf{p}_i$$

- 其中 $\mathbf{p}_i$ 是位置 $i$ 的位置嵌入。

## 多模态标记化

- 想象你正在写一封包含英文文本和手绘草图的信。你可能会写一个句子， sketch 一个图表，写另一个引用图表的句子，然后粘贴一个乐谱。这封信是单个线性流，交错不同的"模态"。多模态标记化正是这样做的：它将文本、图像、音频和视频转换为 transformer 从左到右处理的单个扁平标记序列。

- 对于文本，标记化已确立：字节对编码（BPE）或 SentencePiece 产生子词标记词汇表，如第 7 章所述。挑战是将这个想法扩展到连续模态。

- 对于图像，有两种广泛的方法。离散方法使用 VQ-VAE 或 VQ-GAN（本章文件 03 详细介绍）将每个图像映射到码本索引序列。如果码本有 $|\mathcal{C}|$ 个条目，图像编码为 $n$ 个码，则图像变成 $n$ 个离散标记，从大小为 $|\mathcal{C}|$ 的词汇表中提取，与文本词汇表直接兼容。连续方法使用 ViT 或 CNN 编码器产生 $n$ 个连续嵌入向量，线性投影到 transformer 的嵌入维度。Gemini 和 GPT-4o 使用连续方法的变体；自回归图像生成器如 Parti 和 LlamaGen 更喜欢离散路线。

- 对于音频，信号通常转换为 mel 语谱图，然后用神经过音频编解码器（例如 EnCodec、SoundStream，产生分层离散标记）离散化，或通过学习的编码器连续投影。例如，AudioLM 将音频表示为来自多个码本级别的离散标记序列，然后自回归地建模它们。

- 对于视频，标记化建立在图像标记化之上，但也必须压缩时间维度。常见策略使用 3D VQ-VAE（如文件 03 中的 VideoGPT 或 Cosmos Tokenizer），将时空补丁量化为离散标记。时间压缩因子至关重要：原始视频在 24 fps 下每秒产生太多标记，如果没有积极的时间下采样。

- 一旦所有模态都被标记化，它们就交错成单个序列，特殊分隔符标记标记模态边界。典型格式如下：

```
[TEXT] 猫坐在垫子上 [/TEXT] [IMAGE] <img_tok_1> <img_tok_2> ... <img_tok_n> [/IMAGE] [AUDIO] <aud_tok_1> ... <aud_tok_m> [/AUDIO]
```

- 然后 transformer 使用其标准因果（或双向）注意力机制处理整个混合序列。模态分隔符标记具有双重职责：它们告知模型模态边界并充当"汇合点"，其表示总结每个模态段。

![交错标记序列显示文本标记、离散图像标记和音频编解码器标记流经单个 transformer，具有模态边界标记](../images/multimodal_tokenisation_sequence.svg)

- 关键设计选择是标记预算。以 256 个标记标记化的单个图像和 50 个标记的文本标题意味着图像消耗 5 倍多的上下文窗口。模型必须平衡分辨率（更多标记 = 更多细节）与上下文长度（更多标记 = 更高内存和计算成本）。标记合并（逐渐组合相似标记）和自适应标记化（对简单区域使用更少的标记，对复杂区域使用更多标记）等技术有助于管理这种权衡。

## 训练配方：阶段预训练和联合微调

- 你不会在算术之前教孩子微积分。同样，你不能从随机初始化开始在所有模态上同时训练统一多模态模型并期望它很好地收敛。主要方法是阶段训练，模型在精心排序的阶段中学习逐渐更复杂的跨模态能力。

- 阶段 1：单模态预训练。每个模态编码器在大型单模态数据集上独立训练。文本骨干在数万亿文本标记上使用标准语言建模目标（下一标记预测）进行预训练，与第 7 章完全相同。视觉编码器在图像分类或自监督目标（MAE、DINO）上进行预训练，如第 8 章所述。音频编码器在语音识别或音频分类数据上进行预训练，如第 9 章所述。此阶段产生强大的单模态特征提取器。

- 阶段 2：跨模态对齐。预训练的编码器连接到共享骨干，模型在配对多模态数据（图像 - 标题对、音频 - 转录对）上使用对比或生成目标进行训练。在此阶段，编码器权重可能冻结（以保留单模态知识），而只有投影层和骨干被更新。这是本章文件 01 中的 CLIP 风格对齐被折叠到统一模型中的阶段。

- 阶段 3：联合多模态预训练。所有参数（或大多数）被解冻，模型在单模态和多模态数据的混合上使用单个下一标记预测目标进行训练，跨所有模态标记。损失函数为：

$$\mathcal{L} = -\sum_{t=1}^{T} \log p_\theta(x_t \mid x_{<t})$$

- 其中 $x_t$ 可以是文本标记、图像标记或音频标记。模型必须学习预测下一个标记，无论模态如何，这迫使它发展真正的跨模态理解。

- 阶段 4：指令微调和调整。预训练模型在包含多模态指令的精选指令跟随数据集上进行微调（例如，"详细描述这张图像"、"这个视频发出什么声音？"、"生成 X 的图像"）。此阶段通常使用来自人类反馈的强化学习（RLHF）或直接偏好优化（DPO）来使模型的输出与人类偏好保持一致。

- 特定于模态的预热是在阶段内用于防止模态崩溃的技术。如果一个模态（通常是文本，具有最多的训练数据）主导梯度信号，模型可能会"忘记"较弱的模态。预热策略包括：

    - 梯度平衡：缩放每个模态的梯度，以便它们对参数更新的贡献相等。
    - 数据比例调度：逐渐增加多模态数据相对于单模态数据的比例。
    - 损失加权：分配特定于模态的权重 $\lambda_m$，以便总损失为 $\mathcal{L} = \sum_m \lambda_m \mathcal{L}_m$，其中 $\lambda_m$ 调整为平衡跨模态的学习率。

![四阶段训练管道图显示单模态预训练、跨模态对齐、联合多模态预训练和指令微调，箭头指示在每个阶段哪些参数冻结或可训练](../images/staged_multimodal_training.svg)

- 为什么不跳过阶段？从头开始联合训练所有内容在理论上很诱人，但在实践中因几个原因而失败。首先，模型必须同时学习低级特征（边缘检测、音素识别）和高级跨模态推理，它们具有非常不同的学习动态。其次，跨模态的数据分布极不平衡（数万亿文本标记与数十亿图像标记与数亿音频剪辑）。第三，优化 landscape 高度非凸，阶段训练提供课程，引导模型走向更好的盆地，类似于第 6 章的课程学习想法。

## 多模态思维链推理

- 当你解决几何问题时，你可能会 sketch 一个图表，标记角度，写出一个方程，然后逐步解决。你不会直接从问题陈述跳到答案。多模态思维链（CoT）推理使模型能够做同样的事情：生成中间推理步骤，可能涉及文本、视觉注释，甚至生成的图表，然后得出最终答案。

- 在纯文本 CoT（如第 7 章讨论提示策略中探索的）中，模型用自然语言生成推理步骤序列。多模态 CoT 通过允许中间步骤引用或生成视觉内容来扩展这一点。例如，给定图表图像和问题"哪一年销售额最高？"，多模态 CoT 模型可能首先描述图表（"图表显示 2018 年到 2023 年的销售额..."），然后识别相关视觉特征（"最高的条形出现在 2021 年..."），最后输出答案（"2021"）。

- 形式化地，让 $\mathbf{x}$ 是多模态输入，$y$ 是目标答案。标准预测直接模型 $p(y \mid \mathbf{x})$。思维链引入中间推理 $\mathbf{r} = (r_1, r_2, \ldots, r_L)$ 并将预测分解为：

$$p(y \mid \mathbf{x}) = \sum_{\mathbf{r}} p(y \mid \mathbf{r}, \mathbf{x}) \cdot p(\mathbf{r} \mid \mathbf{x})$$

- 在实践中，总和通过贪婪或 beam-search 解码推理链来近似。推理步骤 $r_i$ 可以是文本标记、对图像区域的引用，甚至生成的视觉标记（例如，叠加在输入图像上的边界框注释）。

- 训练多模态 CoT 通常涉及精选数据集，其中人类注释者提供逐步多模态推理轨迹，然后在这些轨迹上微调模型。一些方法从更大的教师模型蒸馏 CoT 能力：教师为大型数据集生成推理轨迹，较小的学生模型在输入和教师的轨迹上进行训练。

- 多模态 CoT 对于需要空间推理的任务特别强大（例如，"红球在蓝立方体的左边吗？"）、对图表的数学推理（例如，几何问题）和多步骤视觉问答，其中答案取决于组合图像多个区域的信息。

## 多模态智能体

- 想想厨房里的机器人厨师。它看柜台上的食材（视觉），阅读平板电脑上的食谱（文本），听计时器哔哔声（音频），然后实际拿起刀切洋葱（动作）。多模态智能体是这个的数字版本：一个通过多种模态感知世界、推理要做什么并采取基于其感知的行动的模型。

- 智能体循环遵循经典的观察 - 推理 - 行动循环：

    1. 观察：智能体从环境中接收多模态输入（截图、用户的口头指令、视频源）。
    2. 推理：统一模型处理多模态输入，可能使用思维链来规划一系列步骤。
    3. 行动：模型输出行动（文本回答、工具调用、坐标 $(x, y)$ 处的鼠标点击、机器人电机命令）。

- 工具使用是多模态智能体的关键能力。模型被训练识别何时不能直接回答问题，而必须调用外部工具：计算器、代码解释器、Web 浏览器或搜索引擎。模型作为其输出标记序列的一部分生成结构化工具调用（例如 `search("伦敦当前天气")`），系统执行调用，结果作为额外输入标记反馈给模型处理。

- 视觉接地将语言连接到图像或视频中的特定区域。当智能体说"点击右上角的蓝色按钮"时，它必须将短语"右上角的蓝色按钮"接地到像素坐标。在架构上，这是通过训练模型输出边界框坐标作为特殊标记或通过让模型在图像上生成热图来指示所指区域来实现的。这将本章文件 02（视觉语言模型）中讨论的接地和指代工作扩展到行动领域。

- 如 WebVoyager 和 SeeAct 等 Web 智能体演示了多模态智能体导航网站。智能体接收网页的截图，识别交互元素（按钮、文本字段、链接），并输出行动（点击、输入、滚动）以实现用户指定的目标。关键挑战是巨大的行动空间：典型网页有数百个可能的点击目标。

![多模态智能体的观察 - 推理 - 行动循环，显示来自屏幕的视觉输入、统一模型内的推理过程和输出行动如点击、输入或调用工具](../images/multimodal_agent_loop.svg)

- 体现智能体将此扩展到物理环境。具有相机和麦克风的机器人接收视觉和音频输入，通过统一模型处理它，并输出电机命令。如 PaLM-E（Google）等项目将机器人传感器数据直接嵌入到语言模型的标记序列中，使机器人能够通过将指令接地在其视觉观察中并生成一系列电机动作来遵循指令如"拿起碗附近的绿色块"。

- 智能体的训练配方在标准阶段预训练之上添加强化学习（RL）阶段。智能体与环境（模拟桌面、Web 浏览器、机器人模拟器）交互，接收任务完成的奖励，并使用 PPO 或 REINFORCE 等算法更新其策略。奖励信号通常是稀疏的（任务成功为 1，否则为 0），这使得优化具有挑战性，并严重依赖于多模态预训练的强先验。

## 基准测试和评估

- 评估可以看到、听到、阅读和行动的模型需要多样化的基准测试套件。没有单个指标捕捉多模态能力，因此该领域依赖于一系列专门的评估。

- MMLU（大规模多任务语言理解）测试 57 个学术科目的知识。虽然最初是纯文本的，但它作为基线：统一多模态模型在获得视觉能力时不应失去纯文本性能。多模态训练后 MMLU 下降表明灾难性遗忘。

- MMBench 评估跨 20 个细粒度能力维度的视觉语言理解，包括属性识别、空间关系理解和 OCR。每个问题呈现图像和多项选择题。基准系统地测试模型是否真正理解图像或依赖于纯文本捷径。

- SEED-Bench 提供 19,000 个多项选择题，涵盖 12 个评估维度，用于图像和视频理解。它专门测试时间理解（在给定帧之前/之后发生了什么）和组合推理（组合多个视觉属性）。

- MM-Vet 通过要求模型同时使用多种技能来评估集成多模态能力：识别、OCR、空间意识、语言生成和知识检索，都在单个问题中。

- MathVista 测试对视觉输入的数学推理：几何图表、统计图表、函数图、科学图形。此基准专门针对多模态思维链能力。

- 音频 - 视觉基准如 AVQA（音频 - 视觉问答）测试模型是否可以推理它们看到和听到的内容之间的关系。例如："说话的人是左边的还是右边的？"

- 智能体基准如 WebArena、OSWorld 和 SWE-bench 评估交互式环境中的任务完成。指标通常是成功率：智能体正确完成的任务比例是多少？这些基准特别具有挑战性，因为它们需要长期规划和错误恢复。

- 如 LMSYS Chatbot Arena 等整体评估框架使用面对面对比中的人类偏好判断。两个模型显示相同的多模态输入，人类评判者选择哪个响应更好。Elo 评级从数千次此类比较中计算，提供与整体模型质量良好相关的单个标量。

- 多模态评估中的一个持续挑战是数据污染：因为这些模型在互联网规模的数据上训练，基准图像和问题可能出现在训练集中。仔细的重复数据删除和创建保留测试集是必要但不完善的保障措施。

## 世界模型

- 想象闭上眼睛并可视化如果你将玻璃杯从桌子边缘推下会发生什么。你"看到"它落下，"听到"破碎声，并"感觉"这将是一个坏主意。你的大脑正在运行世界模型：环境和物理和因果结构的内部模拟，可以预测跨多个模态的未来状态。

- 在 AI 上下文中，世界模型是学习函数，给定当前状态和行动预测世界的下一个状态：

$$\hat{s}_{t+1} = g_\phi(s_t, a_t)$$

- 其中 $s_t$ 是当前状态表示（可能包括视觉、听觉和本体感受信息），$a_t$ 是行动，$\hat{s}_{t+1}$ 是预测的下一个状态。状态 $s_t$ 存在于学习的潜在空间中，而不是原始像素空间，使预测问题可处理。

- 如 Sora（OpenAI）和 Genie（Google DeepMind）等视频预测模型代表了向世界模型迈出的重要一步。它们学习在文本提示和/或行动序列的条件下生成时间连贯的视频帧。虽然它们通常作为视频生成器讨论，但底层能力更接近世界模拟：模型已经内化了足够的物理（重力、碰撞、遮挡、流体动力学）来渲染合理的未来。

- 与多模态架构的联系很深。仅预测像素的世界模型是有限的；真正有用的世界模型跨模态预测。如果你推玻璃杯，世界模型应该预测视觉轨迹（玻璃杯落下）、听觉事件（玻璃杯破碎）和语义后果（地板上现在有碎玻璃）。统一多模态架构是世界模型的自然候选者，因为它们已经在共享空间中表示所有模态。

- 形式化地，多模态世界模型优化：

$$\mathcal{L}_\text{world} = \mathbb{E}\left[\sum_{m \in \mathcal{M}} \lambda_m \| s_{t+1}^m - g_\phi^m(s_t, a_t) \|^2 \right]$$

- 其中 $s_{t+1}^m$ 是模态 $m$ 中的真实下一个状态表示，$g_\phi^m$ 是世界模型的特定于模态的预测头。共享潜在动态 $g_\phi$ 在联合多模态空间中操作，而特定于模态的头将预测解码到每个模态的本机格式。

![世界模型图显示潜在状态被行动更新，具有解码器头预测未来视觉帧、音频波形和语义描述](../images/multimodal_world_model.svg)

- JEPA（联合嵌入预测架构），由 Yann LeCun 提出，为世界模型提供框架，避免像素级预测的陷阱。JEPA 不是在原始像素中预测（这浪费容量在无关细节如确切纹理上），而是在嵌入空间中预测。模型学习将观测映射到嵌入的编码器和预测未来嵌入的预测器：

$$\hat{\mathbf{z}}_{t+1} = h_\psi(\mathbf{z}_t, a_t), \quad \mathbf{z}_t = \text{Enc}(s_t)$$

- 损失比较嵌入而不是原始观测，这对感知混叠（许多不同的像素配置可能表示相同的语义状态）更稳健。这种方法对于多模态世界模型特别有希望，因为它自然地在统一架构已经提供的共享嵌入空间中操作。

- 世界模型具有超越学术兴趣的实际应用。在基于模型的强化学习中，智能体使用其世界模型在采取行动之前"想象"行动的后果，显著减少所需的现实世界交互数量（回顾第 11 章关于基于模型的 RL 的讨论）。在自动驾驶中，世界模型预测场景在未来几秒内如何演变，给定不同的转向决策。在机器人技术中，世界模型允许机器人在执行之前 mentally 排练操作序列。

- 世界模型研究的前沿正朝着实时运行并响应用户任意操作的交互式世界模型发展，基本上成为完全从数据学习的通用模拟器。Genie 2（Google DeepMind）为 3D 环境演示了这一点：给定单个图像，它生成用户可以探索的交互式、可控制的 3D 世界。世界模型和统一多模态架构的融合表明了一个未来，其中单个模型可以在所有模态中感知、预测、模拟和行动。

任务 1：构建最小多模态标记交错器

- 编写一个函数，接受文本字符串和虚拟"图像"（小型 2D 数组）并将它们的标记化表示交错成具有模态嵌入的单个扁平序列。

```python
import jax
import jax.numpy as jnp

# 模拟多模态标记化：文本标记 + "图像补丁"标记
def interleave_modalities(text_tokens, image_patches, embed_dim=32, key=jax.random.PRNGKey(0)):
    """交错文本和图像标记与学习的模态嵌入。"""
    k1, k2, k3 = jax.random.split(key, 3)
    n_text = text_tokens.shape[0]
    n_img = image_patches.shape[0]
    # 随机投影矩阵（真实编码器的替代）
    W_text = jax.random.normal(k1, (text_tokens.shape[-1], embed_dim)) * 0.02
    W_img = jax.random.normal(k2, (image_patches.shape[-1], embed_dim)) * 0.02
    # 模态嵌入：一个用于文本，一个用于图像
    mod_emb = jax.random.normal(k3, (2, embed_dim)) * 0.02
    text_embs = text_tokens @ W_text + mod_emb[0]  # (n_text, embed_dim)
    img_embs = image_patches @ W_img + mod_emb[1]   # (n_img, embed_dim)
    # 交错：[IMG] 标记优先，然后 [TEXT] 标记（像 LLaVA）
    combined = jnp.concatenate([img_embs, text_embs], axis=0)
    print(f"组合序列：{n_img} 图像 + {n_text} 文本 = {combined.shape[0]} 标记")
    return combined

# 尝试：5 个文本标记（dim 16）和 4 个图像补丁（dim 64）
text = jax.random.normal(jax.random.PRNGKey(1), (5, 16))
image = jax.random.normal(jax.random.PRNGKey(2), (4, 64))
seq = interleave_modalities(text, image)
# 实验：改变 embed_dim，交换交错顺序，添加第三个模态
```

任务 2：可视化跨模态注意力模式

- 创建合成多模态序列并计算自注意力分数，以查看图像标记如何关注文本标记，反之亦然。

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def cross_modal_attention(n_text=6, n_img=4, d=32, key=jax.random.PRNGKey(42)):
    """计算和可视化文本和图像标记之间的注意力。"""
    k1, k2, k3 = jax.random.split(key, 3)
    # 模拟两个模态的标记嵌入
    text_embs = jax.random.normal(k1, (n_text, d))
    img_embs = jax.random.normal(k2, (n_img, d))
    seq = jnp.concatenate([img_embs, text_embs], axis=0)  # (n_img+n_text, d)
    # 学习的 Q, K 投影
    Wq = jax.random.normal(k3, (d, d)) * 0.1
    Wk = jax.random.normal(jax.random.PRNGKey(99), (d, d)) * 0.1
    Q, K = seq @ Wq, seq @ Wk
    scores = Q @ K.T / jnp.sqrt(d)
    attn = jax.nn.softmax(scores, axis=-1)
    # 绘图
    labels = [f"img_{i}" for i in range(n_img)] + [f"txt_{i}" for i in range(n_text)]
    fig, ax = plt.subplots(figsize=(7, 6))
    ax.imshow(attn, cmap="viridis")
    ax.set_xticks(range(len(labels))); ax.set_xticklabels(labels, rotation=45, fontsize=8)
    ax.set_yticks(range(len(labels))); ax.set_yticklabels(labels, fontsize=8)
    ax.set_xlabel("Key (被关注)"); ax.set_ylabel("Query (关注从)")
    ax.set_title("跨模态自注意力图")
    plt.colorbar(ax.images[0], ax=ax, shrink=0.8)
    plt.tight_layout(); plt.show()

cross_modal_attention()
# 实验：增加 d，添加因果掩码，观察注意力模式如何变化
```

任务 3：模拟具有特定于模态损失加权的阶段训练

- 演示特定于模态的损失权重如何影响玩具多模态训练循环。观察损失平衡如何防止一个模态主导。

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def staged_training_sim(steps=200, key=jax.random.PRNGKey(7)):
    """模拟具有可调节模态损失权重的多模态训练。"""
    # 两个"模态"具有不同的损失规模（文本损失约 10 倍大于图像损失）
    losses_text, losses_img = [], []
    param = jnp.array([0.0, 0.0])  # 被两个模态损失更新的共享参数
    lr = 0.05
    # 尝试改变这些权重以观察对收敛平衡的影响
    lambda_text, lambda_img = 1.0, 5.0  # 加重较弱的模态

    for step in range(steps):
        k1, k2, key = jax.random.split(key, 3)
        noise_t = jax.random.normal(k1, ()) * 0.3
        noise_i = jax.random.normal(k2, ()) * 0.1
        loss_t = (param[0] - 3.0) ** 2 + noise_t  # 文本目标 = 3.0
        loss_i = 0.1 * (param[1] - 1.0) ** 2 + noise_i  # 图像目标 = 1.0（较小规模）
        # 加权组合梯度
        grad_t = lambda_text * 2 * (param[0] - 3.0)
        grad_i = lambda_img * 0.2 * (param[1] - 1.0)
        param = param - lr * jnp.array([grad_t, grad_i])
        losses_text.append(float(loss_t)); losses_img.append(float(loss_i))

    fig, ax = plt.subplots(figsize=(8, 4))
    ax.plot(losses_text, label=f"文本损失 (weight={lambda_text})", alpha=0.7)
    ax.plot(losses_img, label=f"图像损失 (weight={lambda_img})", alpha=0.7)
    ax.set_xlabel("训练步骤"); ax.set_ylabel("损失"); ax.legend()
    ax.set_title("阶段训练期间的模态损失平衡")
    plt.tight_layout(); plt.show()

staged_training_sim()
# 实验：设置 lambda_img=1.0 并观察图像损失收敛慢得多
```
