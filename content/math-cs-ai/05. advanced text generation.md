# 高级文本生成

*高级文本生成超越了朴素的自回归解码，以提高质量、可控性和速度。本文件涵盖文本扩散模型（D3PM、MDLM）、OCR、用于对齐的 RLHF 和 DPO、长上下文方法（RoPE 缩放、环形注意力）、检索增强生成和用于更快推理的推测解码。*

- 标准自回归生成（文件 04）一次生成一个 token，从左到右。这简单有效，但本质上是顺序的，不允许全局规划，并对输出控制有限。本文件涵盖超越朴素自回归解码的方法：文本扩散模型、光学字符识别、通过人类反馈的可控生成、处理长上下文、检索增强生成和用于更快推理的推测解码。

- **文本扩散模型**将扩散框架（第 08 章为图像引入）应用于离散文本。核心挑战是文本是离散的：你不能像对像素添加噪声那样对 token 添加连续高斯噪声。几种方法解决了这个问题。

- **D3PM**（离散去噪扩散概率模型，Austin 等人，2021）使用转移矩阵直接在离散 token 上定义前向破坏过程。在每個前向步骤中，token 有一定概率被另一个 token 替换（均匀噪声）、被掩码（吸收态）或保持不变。反向学习去噪，从破坏的 token 预测 clean token。步骤 $t$ 的转移矩阵 $Q_t$ 控制破坏：

$$q(x_t \mid x_{t-1}) = \text{Cat}(x_t ; \, x_{t-1} Q_t)$$

- 其中 $\text{Cat}$ 表示分类分布，$x$ 是 one-hot 向量。多步前向过程 $q(x_t \mid x_0)$ 有闭式解：$q(x_t \mid x_0) = \text{Cat}(x_t ; \, x_0 \bar{Q}_t)$，其中 $\bar{Q}_t = Q_1 Q_2 \cdots Q_t$ 是直到步骤 $t$ 的所有转移矩阵的乘积。训练最小化跨时间步分解的变分下界（ELBO），类似于连续情况（第 08 章）：

$$\mathcal{L}_{\text{D3PM}} = D_{\text{KL}}(q(x_T \mid x_0) \| p(x_T)) + \sum_{t=2}^{T} D_{\text{KL}}(q(x_{t-1} \mid x_t, x_0) \| p_\theta(x_{t-1} \mid x_t)) - \log p_\theta(x_0 \mid x_1)$$

- 第一项确保完全破坏的分布匹配先验（均匀或全掩码）。KL 项之和训练模型逆转每个破坏步骤：真实反向后验 $q(x_{t-1} \mid x_t, x_0)$ 可以使用贝叶斯规则和已知转移矩阵用闭式计算，模型 $p_\theta(x_{t-1} \mid x_t)$ 训练匹配它。

- 由于两个分布都是分类的，KL 散度是词汇表条目的简单求和。最后一项衡量从最少破坏状态的重建质量。

- **MDLM**（掩码扩散语言模型，Sahoo 等人，2024）通过使用掩码作为唯一破坏操作简化了 D3PM：前向过程逐渐用 [MASK] token 替换 token，反向过程预测原始 token。这将文本扩散与掩码语言建模（BERT，文件 04）联系起来，扩散时间步控制掩码 token 的比例。在 $t = 0$ 时文本完全干净；在 $t = T$ 时完全掩码。

- **连续文本扩散**通过在连续嵌入空间工作绕过离散问题。Token 首先映射到它们的嵌入向量（第 06 章），噪声在这个连续空间添加，去噪模型（通常是 Transformer）学习逆转这个过程。生成时，模型产生连续向量，通过找到最近嵌入映射回离散 token。挑战是连续空间的小错误可能映射到完全错误的 token，因此需要仔细舍入和截断。

![文本扩散过程](../images/text_diffusion.svg)

- 文本扩散的吸引力在于它通过迭代细化同时生成所有 token，而非从左到右。这允许全局连贯性和轻松填充（在段落中间生成缺失文本），但当前文本扩散模型在长形式文本的生成质量上仍落后于自回归模型。

- **文本 OCR**（光学字符识别）是从图像提取文本的任务。虽然传统上不与语言生成分组，但现代 OCR 系统与 NLP 深度集成并越来越多地使用语言模型组件。

- **场景文本检测**定位自然图像中的文本区域（街道标志、产品标签、车牌）。这很有挑战性，因为野外文本以任意角度、尺度、字体出现，并在杂乱背景前。检测方法通常使用 CNN 或 Transformer 骨干在文本区域周围产生边界框或分割掩码。

- **CRNN**（卷积循环神经网络，Shi 等人，2017）是经典文本识别架构。CNN 从文本图像提取视觉特征，特征图被切片成列序列（每个水平位置一列），双向 LSTM 读取这个序列以建模上下文。输出使用**CTC**（Connectionist 临时分类）解码，处理输入列和输出字符之间的对齐，无需显式分割。

- CTC 解决的基本问题：模型产生 $T$ 个输出分布（每输入列一个），但目标文本有 $L \leq T$ 个字符。

- 我们不知道哪些列对应哪些字符。CTC 引入**空白 token**$\epsilon$并定义多对一映射 $\mathcal{B}$ 折叠重复字符并移除空白：$\mathcal{B}(\text{"HH-ee-ll-ll-oo"}) = \text{"Hello"}$（其中"-"是空白）。

- 目标序列 $y$ 的概率是折叠到 $y$ 的所有输入对齐的求和：

$$P(y \mid x) = \sum_{\pi \in \mathcal{B}^{-1}(y)} \prod_{t=1}^{T} P(\pi_t \mid x)$$

- 其中 $\pi$ 是长度 $T$ 的对齐路径（每列一个标签，包括空白）。朴素求和所有路径是指数级的，但**前向算法**（第 05 章 HMM）使用动态规划在 $O(T \cdot L)$ 时间内高效计算这个求和。

- 空白 token 至关重要：没有它，"Hello"中的重复字符"ll"与单个"l"无法区分。训练最大化 $\log P(y \mid x)$，推理时，最佳路径通过束搜索或 CTC 输出上的贪婪解码找到。

- **文档 OCR**处理结构化文档（发票、表单、科学论文），必须理解布局以及识别字符。像 LayoutLM 这样的现代系统将文本识别与空间位置特征结合：每个 token 获得文本嵌入和编码其在页面上 $(x, y)$ 坐标的位置嵌入。这使模型理解出现在"Total:"下方的数字是总金额。

![CRNN OCR 管道](../images/crnn_ocr_pipeline.svg)

- **视觉语言 OCR**模型如 TrOCR 将文本识别视为图像到文本生成：Vision Transformer 编码器处理图像，语言模型解码器逐字符生成文本。这利用预训练视觉和语言模型的力量，无需手工特征工程即可处理多种脚本、字体和布局。

- **可控生成**是引导语言模型产生具有所需属性输出的挑战：特定风格、主题、情感、安全级别或事实准确性。模型应遵循指令，同时保持流畅和连贯。

- **无分类器引导（CFG）**用于文本，采用图像生成的技术。训练期间，条件信号（例如提示）随机丢弃一定比例的时间，在一个模型中同时训练条件和无条件模型。推理时，输出 logits 插值：

$$\text{logits}_{\text{guided}} = (1 + w) \cdot \text{logits}_{\text{conditional}} - w \cdot \text{logits}_{\text{unconditional}}$$

- 其中 $w > 0$ 放大的条件影响。更高 $w$ 使输出更强烈遵循提示但降低多样性。

- **RLHF**（人类反馈强化学习，Ouyang 等人，2022）是将语言模型与人类偏好对齐的主要方法。过程有三个阶段：

- 首先，**监督微调（SFT）**：在人类撰写的高质量响应数据集上微调基础语言模型。

- 其次，**奖励模型训练**：收集人类比较（给定提示 $x$ 和两个响应 $y_1, y_2$，哪个更好？）并训练奖励模型 $r_\phi(x, y)$ 预测人类偏好。奖励模型用成对排序损失训练：

$$\mathcal{L}_{\text{RM}} = -\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

- 其中 $y_w$ 是优选响应，$y_l$ 是较不优选的响应。

- 第三，**RL 微调**：优化语言模型以最大化奖励，同时保持接近 SFT 模型（防止模式崩溃）。这使用带有 KL 惩罚的 PPO（近端策略优化，第 06 章）：

$$\mathcal{L}_{\text{RL}} = -\mathbb{E}\left[r_\phi(x, y) - \beta \, D_{\text{KL}}(\pi_\theta \| \pi_{\text{SFT}})\right]$$

- KL 项防止模型漂移太远并利用奖励模型的怪癖（"奖励黑客"）。

![RLHF 管道](../images/rlhf_pipeline.svg)

- **DPO**（直接偏好优化，Rafailov 等人，2023）通过完全消除奖励模型简化了 RLHF。关键数学洞察是上述 KL 约束 RL 目标有闭式最优策略：

$$\pi^\ast(y \mid x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y \mid x) \exp\!\left(\frac{r(x, y)}{\beta}\right)$$

- 其中 $Z(x)$ 是归一化配分函数。重新排列得到奖励 $r(x, y) = \beta \log \frac{\pi^\ast(y \mid x)}{\pi_{\text{ref}}(y \mid x)} + \beta \log Z(x)$。将这个隐式奖励代入 Bradley-Terry 偏好模型 $P(y_w \succ y_l) = \sigma(r(x, y_w) - r(x, y_l))$ 导致难以处理的 $Z(x)$ 项抵消，直接产生 DPO 损失：

$$\mathcal{L}_{\text{DPO}} = -\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)}\right)$$

- 这在数学上等同于 RLHF，但将奖励模型和 RL 训练折叠到单个监督步骤中。

- sigmoid 内部的表达式可以读作："增加优选响应的相对概率并减少较不优选响应的相对概率，相对于参考模型测量。"

- $\beta$ 参数控制策略可以偏离参考多少。实践中，DPO 更简单实现（仅为两个补全计算当前和参考模型下的对数概率）并避免 PPO 训练的不稳定性。

- **宪法 AI**（Bai 等人，2022）自动化对齐过程的部分。不是收集人类比较，它使用语言模型本身根据一组原则（"宪法"）批评和修改自己的输出，如"选择较不有害的响应"。AI 生成的比较随后用于偏好训练（RLAIF：AI 反馈的 RL）。

- **长上下文方法**解决标准自注意力的 $O(n^2)$ 内存和计算成本，这限制了序列长度。随着 $n$ 增长到数万 token，标准注意力变得不可行。

- **稀疏注意力**用稀疏模式替换稠密 $n \times n$ 注意力矩阵，每个 token 只关注其他 token 的子集。常见模式包括**局部注意力**（每个 token 关注固定大小邻居窗口）、**跨步注意力**（关注每 $k$ 个 token）和**随机注意力**（关注随机子集）。这些模式的组合（用于 BigBird、Longformer）实现 $O(n)$ 或 $O(n \sqrt{n})$ 复杂度，同时保持捕捉局部和全局依赖的能力。

![稀疏注意力模式](../images/sparse_attention_patterns.svg)

- **滑动窗口注意力**限制每个 token 只关注前 $w$ 个 token（其局部窗口）。这是 $O(nw)$ 而非 $O(n^2)$，但长程信息必须跨层通过重叠窗口传播。$L$ 层和窗口大小 $w$，有效感受野是 $L \times w$ token。

- **环形注意力**通过环形拓扑在多个设备上分布长序列。每个设备持有序列块并为其块计算注意力，同时将键值块发送到环中的下一个设备。这重叠计算与通信，允许任意长度序列，仅受所有设备总内存限制，而非任何单个设备的内存。

- **记忆增强模型**通过为 Transformer 配备外部记忆库扩展上下文。每层，模型可以使用注意力从这个记忆读写。记忆化 Transformer 缓存来自先前块的键值对，并在后续块关注它们，有效扩展上下文超出训练窗口。检索是近似的（使用缓存键上的 $k$ 近邻）以保持高效。

- 上述方法是长上下文的**架构**解决方案。同样重要的是模型如何**训练**以有效使用长上下文。

- **渐进上下文扩展**是标准方法。从一开始就在非常长序列上训练成本过高（$O(n^2)$ 注意力成本），因此模型在短上下文长度（通常 4K-8K token）预训练，然后**继续预训练**分阶段扩展到目标长度。

- Llama 3.1 用逐渐增加的序列长度从 8K 扩展到 128K，共 800B token。DeepSeek-V3 在 4K 训练，然后扩展到 32K，然后 128K。

- 每个阶段使用适度数量的 token（相对于完整预训练预算），因为模型只需要学习如何使用更长位置，而非重新学习语言本身。

- 位置编码必须在扩展期间调整。**RoPE 插值**缩小位置索引，使模型看到相同的旋转角度，只是分布在更长序列上。如果模型在长度 $L$ 训练并想扩展到 $L' = 4L$，你将所有位置索引除以 4。

- 这意味着模型从不会遇到未遇到的旋转角度，但相邻位置间的有效分辨率下降。

- **RoPE 外推**保持原始位置索引不变，简单将 RoPE 应用于超出 $L$ 的位置，依赖模型泛化到未见角度。

- 插值更稳定；没有基础频率调整（ABF）的外推迅速退化。

- **YaRN**（又一个 RoPE 扩展）通过认识并非所有 RoPE 维度应同等对待来改进朴素插值。

- 高频维度（$\theta_i = \theta_{\text{base}}^{-2i/d}$ 中小 $i$）在训练长度内旋转多次并能很好地外推。

- 低频维度（大 $i$）旋转缓慢并对长度扩展更敏感。

- YaRN 仅插值低频维度，外推高频维度，并应用温度缩放 $t$ 到注意力 logits 以补偿分布偏移：

$$\text{score}'_{ij} = \frac{q_i^T k_j}{t \sqrt{d_k}}$$

- 其中 $t > 1$ 展平注意力分布，防止模型在位置信号压缩时过锐地关注附近 token。

- **长上下文数据筛选**是关键且常被低估的挑战。大多数预训练语料库由短文档组成（新闻文章、网页、社交媒体帖子）。

- 长上下文训练需要实际利用完整上下文窗口的数据混合：书籍、代码仓库、长形式科学文章、多轮对话日志和主题相关文档的连接。

- 如果模型仅在短文档上训练（填充或打包以填充上下文窗口），它学会忽略远处 token，因为它们从无关。

- **序列打包**是训练效率技术：多个文档连接成单个训练序列以避免填充浪费，注意力掩码防止跨文档注意力。

- 对于长上下文训练，打包策略很重要：打包许多无关短文档教导模型远处 token 是噪声，而打包更少、真正长的文档教导它使用完整上下文。

- 已知失败模式是**"中间丢失"**现象（Liu 等人，2023）：语言模型倾向于有效使用上下文窗口开头和结尾的信息，但难以处理中间的信息。

- 这类似于人类记忆中的系列位置效应（首因和近因效应）。

- 这部分源于训练数据分布（重要信息通常在文档开头或结尾），部分源于注意模式集中在附近和初始 token。

- 用关键信息的多样化位置进行长上下文训练缓解但不完全解决这个问题。

- **大海捞针**评估测试模型是否可以检索放在长干扰上下文（"大海"）内不同位置的特定事实（"针"）。

- 具有真正长上下文能力的模型应实现近乎完美的检索，无论针放在哪里。

- 此测试清楚揭示中间丢失效应，用于基准上下文扩展方法。

- **长上下文微调**在预训练后使用有针对性的 SFT 数据：长多轮对话、证据分散在数千 token 中的文档问答、长形式摘要和仓库级代码理解。

- Qwen3 在此阶段使用**双块注意力（DCA）**，将长序列作为块对处理，块内注意力完整，块间注意力高效，在微调期间实现 4 倍有效序列容量。

- **状态空间模型（SSM）**提供长序列建模的根本不同方法。它们不是修改注意力，而是用受连续时间控制理论启发的线性动力系统完全替换它。

- SSM 通过由以下控制的潜在状态 $x(t) \in \mathbb{R}^N$ 将输入序列 $u(t)$ 映射到输出 $y(t)$：

$$x'(t) = Ax(t) + Bu(t), \quad y(t) = Cx(t) + Du(t)$$

- 其中 $A \in \mathbb{R}^{N \times N}$ 是状态转移矩阵，$B \in \mathbb{R}^{N \times 1}$ 是输入投影，$C \in \mathbb{R}^{1 \times N}$ 是输出投影，$D$ 是跳过连接。

- 将此应用于离散序列（token），连续系统用步长 $\Delta$**离散化**。零阶保持离散化给出：

$$\bar{A} = \exp(\Delta A), \quad \bar{B} = (\Delta A)^{-1}(\exp(\Delta A) - I) \cdot \Delta B$$

- 离散递归变为 $x_k = \bar{A} x_{k-1} + \bar{B} u_k$，$y_k = C x_k + D u_k$，看起来像 RNN：一次处理一个 token 与隐藏状态。

- 与 RNN 不同，这个递归也可以展开为**全局卷积**：因为系统是线性的，输出是 $y = \bar{K} \ast u$，其中核 $\bar{K} = (C\bar{B}, \, C\bar{A}\bar{B}, \, C\bar{A}^2\bar{B}, \ldots)$ 仅依赖固定参数。

- 这个**双重观点** — 用于高效自回归推理的递归（每步 $O(1)$）和用于高效并行训练的卷积（通过 FFT 的 $O(n \log n)$） — 是 SSM 的核心洞察。

![SSM 双重观点：推理用递归，训练用卷积，以及 Mamba 的选择性扩展](../images/ssm_dual_view.svg)

- **S4**（序列建模的结构化状态空间，Gu 等人，2022）通过解决关键数值挑战使 SSM 实用：状态矩阵 $A$ 必须捕捉长程依赖，但朴素参数化导致消失或爆炸动态（与朴素 RNN 相同的问题）。

- S4 使用**HiPPO**（高阶多项式投影算子）矩阵初始化 $A$，它源自连续信号最优多项式近似理论。HiPPO 矩阵具有特定结构，证明能使状态以优雅衰减维护整个输入历史的压缩表示：

```math
A_{nk} = -\begin{cases} (2n+1)^{1/2}(2k+1)^{1/2} & \text{if } n > k \\ n+1 & \text{if } n = k \\ 0 & \text{if } n < k \end{cases}
```

- 这个下三角结构确保状态使用 Legendre 多项式作为输入信号的在线近似。计算长核的 $\bar{A}^k$ 很昂贵，因此 S4 使用 HiPPO 矩阵可以分解为低秩和对角项之和的事实，实现 $O(n \log n)$ 核计算。

- **Mamba**（Gu 和 Dao，2023）引入**选择性状态空间**的关键创新：使 SSM 参数依赖输入。在 S4 中，矩阵 $A$、$B$、$C$ 和步长 $\Delta$ 是固定的 — 相同动态应用于每个 token 而不论内容。Mamba 使 $B$、$C$ 和 $\Delta$ 成为输入的函数：

$$B_k = \text{Linear}(u_k), \quad C_k = \text{Linear}(u_k), \quad \Delta_k = \text{softplus}(\text{Linear}(u_k))$$

- 这种选择性允许模型在每个位置决定什么信息存储在状态中以及忽略什么 — 类似于注意力如何选择相关 token，但没有二次成本。步长 $\Delta_k$ 控制"门"：大 $\Delta$ 导致状态强烈整合当前输入（连续动态前进大步，有效重置状态），而小 $\Delta$ 保持现有状态并忽略当前输入。

- 权衡是依赖输入的参数打破卷积观点（核不再固定），因此 Mamba 不能使用基于 FFT 的训练。相反，它使用**硬件感知并行扫描**算法，利用递归的结合性：状态更新 $(x_k, u_k) \mapsto x_{k+1}$ 可以表示为一系列结合操作，并使用前缀和（扫描）并行化，类似于硬件设计中的并行前缀加法。这在 GPU 上以 $O(n)$ 时间和 $O(\log n)$ 深度运行，几乎匹配卷积的效率。

- Mamba 实现每 token 真正 $O(1)$ 推理（仅更新固定大小状态，无视上下文增长的 KV 缓存），使其在长序列长度上从根本上比 Transformer 更内存高效。状态大小 $N$（通常 16）远小于 Transformer 的 KV 缓存，后者存储 $O(n \cdot d)$ 值。实践中，Mamba 在相同参数 count 的语言建模基准上匹配或超过 Transformer 质量，长序列推理明显更快。

- **混合架构**结合 SSM 层与注意力层，对大多数层使用 SSM（高效长程传播）并零星添加一些注意力层（精确基于内容的检索）。像 Jamba 和 Zamba 这样的模型交错 Mamba 和 Transformer 块，实现比纯 SSM 更好的质量，同时保持大部分推理效率优势。这表明注意力和 SSM 捕捉互补能力：SSM 擅长平滑、长程状态传播，而注意力擅长精确、内容依赖的查找。

- **检索增强生成（RAG）**通过给语言模型在推理时访问外部知识库来解决其知识限制。RAG 不是仅依赖训练期间模型参数编码的知识，而是检索相关文档并在其上调节生成。

- 经典**检索器 - 阅读器架构**有两个组件。**检索器**获取查询并从语料库获取最相关的 top-$k$ 段落。**阅读器**（语言模型）在查询和检索段落上条件生成答案。检索器可以使用稀疏方法（BM25，扩展文件 02 的 TF-IDF）或稠密方法。

- **稠密段落检索（DPR）**使用双编码器架构：一个编码器将问题映射到向量，另一个将段落映射到向量。两者通常基于 BERT。索引时，所有段落被编码并存储。查询时，问题被编码，使用近似最近邻搜索（如 FAISS）找到最近段落。相似性度量是问题和段落向量之间的点积。

- **分块策略**显著影响检索质量。文档必须分割成检索器可以处理的足够小段落，但要足够大以包含完整思想。固定大小分块（例如 256 token 带 50 token 重叠）简单但可能尴尬地分割句子。语义分块在段落或章节边界分割。分层分块在不同粒度创建摘要树。

![RAG 架构](../images/rag_architecture.svg)

- RAG 提供几个优势：知识库可以在不重新训练模型的情况下更新，模型可以引用来源，幻觉减少，因为模型可以在检索文本中基于其答案。主要挑战是检索质量（如果检索错误段落，模型可能自信产生错误答案）和延迟（检索为推理添加步骤）。

- **推测解码**通过使用小型、快速**草稿模型**并行提出多个 token 加速自回归生成，然后在单个前向传递中由大型**目标模型**验证。

- 算法工作如下：草稿模型自回归生成 $k$ 个候选 token（这很快，因为草稿模型小）。

- 目标模型然后在单个前向传递中同时评分所有 $k$ 个 token（这高效，因为工作是批处理的）。

- 对于从草稿分布 $p_d(t)$ 采样的每个候选 token $t$，它以概率 $\min(1, \, p_{\text{target}}(t) / p_d(t))$ 接受。如果拒绝，从**调整分布**$p_{\text{adj}}(t) = \max(0, \, p_{\text{target}}(t) - p_d(t))$ 重新采样校正 token，归一化。

- 这个接受 - 拒绝方案保证输出分布与仅目标模型相同。

- 理解为什么，考虑发出 token $t$ 的有效概率。它可以直接接受（概率 $p_d(t) \cdot \min(1, p_{\text{target}}(t)/p_d(t))$）或通过重新采样产生。

- 对于 $p_{\text{target}}(t) \leq p_d(t)$ 的 token，直接接受贡献 $p_{\text{target}}(t)$。对于 $p_{\text{target}}(t) > p_d(t)$ 的 token，直接接受贡献 $p_d(t)$，重新采样贡献剩余 $p_{\text{target}}(t) - p_d(t)$（考虑拒绝概率后）。

- 在两种情况下，发出 $t$ 的总概率等于 $p_{\text{target}}(t)$。草稿模型仅影响速度，不影响质量。

![推测解码](../images/speculative_decoding.svg)

- 加速取决于接受率：如果草稿模型与目标模型良好对齐，大多数 token 被接受，挂钟时间大致是草稿模型的时间。典型加速是 2-3 倍，无质量下降。

- **Medusa**（Cai 等人，2024）采用不同方法：不是单独草稿模型，它在目标模型本身添加多个轻量级预测头。每个头同时预测不同未来 token 位置（$k = 1, 2, 3, \ldots$ 步 ahead）。每步，Medusa 使用树结构提出几个候选延续，目标模型注意力层的单个前向传递验证哪些候选一致。这完全避免了对单独草稿模型的需求。

- **并行生成**方法更广泛旨在打破自回归解码的顺序瓶颈。Jacobi 解码用猜测初始化所有位置并并行迭代细化它们直到收敛，将生成为不动点迭代。非自回归模型（NAT）在单个前向传递中同时生成所有 token，但通常遭受质量下降，需要迭代细化、CTC 损失或从自回归教师知识蒸馏等技术来缩小差距。

- 上述技术 — 对齐、长上下文、检索、高效解码、状态空间模型 — 在现代生产 LLM 中汇集在一起。

- 本文件其余部分调查前沿模型的架构创新，展示文件 01-04 的理论思想和上述方法如何在实践中结合。

- **分组查询注意力（GQA）**是最广泛采用的注意力效率技术。标准多头注意力（MHA）为每个头维护独立的键和值投影，需要每 token 缓存 $n_{\text{heads}} \times d_{\text{head}}$ 值。GQA 分组多个查询头共享单个键值头。

- 有 64 个查询头和 8 个 KV 头（Llama 3、Qwen、Gemma 的常见配置），每个 KV 头由 8 个查询头共享，与 MHA 相比将 KV 缓存减少 8 倍。

- 输出质量几乎与 MHA 相同，因为查询仍然可以关注不同模式，它们只是共享相同键值子空间。多查询注意力（MQA）是所有查询单个 KV 头的极端情况，但 GQA 提供更好的质量 - 效率权衡。

- **多头潜在注意力（MLA）**在 DeepSeek-V2 中引入，实现更激进的 KV 缓存压缩。MLA 不是缓存完整键值投影（即使有 GQA），而是将隐藏状态下投影到低秩**潜在向量**$c_t \in \mathbb{R}^{d_c}$，其中 $d_c \ll n_{\text{heads}} \times d_{\text{head}}$：

$$c_t = W_{\text{down}} \, h_t$$

- 仅这个压缩向量被缓存。注意力时，完整键和值表示通过上投影重建：$k_t = W_{\text{up}}^K c_t$，$v_t = W_{\text{up}}^V c_t$。在 DeepSeek-V3（671B 总参数，37B 激活）中，压缩维度是 $d_c = 512$，而完整 MHA 是 $128 \times 128 = 16{,}384$，KV 缓存减少 93%。

- 微妙之处：标准 RoPE 依赖位置并与共享压缩不兼容，因此 MLA 使用**解耦 RoPE**：查询和键的小型单独流（每头 64 维）通过 RoPE 携带位置信息，而大部分表示通过压缩潜在路径流动。

![注意力 KV 缓存策略：MHA、GQA 和 MLA 比较](../images/mla_vs_gqa.svg)

- **大规模位置编码**已与原始正弦方案显著分化。所有前沿模型使用**RoPE**（文件 04），但有长上下文的关键修改。原始 RoPE 公式 $\theta_i = \theta_{\text{base}}^{-2i/d}$ 中的基础频率 $\theta_{\text{base}}$ 通常是 10,000，限制超出训练长度的外推。

- **调整基础频率（ABF）**简单增加 $\theta_{\text{base}}$ 到 500,000（Llama 3）或 1,000,000（Qwen3、Gemma 3），拉伸旋转周期，使模型在训练期间遇到更少完整旋转并能更远外推。

- **YaRN**（又一个 RoPE 扩展）应用频率依赖插值：低频维度插值（缩小），高频维度外推，温度因子调整注意力分布。DeepSeek-V3、Qwen 和 Kimi K2 都使用基于 YaRN 的扩展，从 4K-8K 预训练的模型达到 128K 上下文。

- **iRoPE**（交错 RoPE）在 Llama 4 中引入，采用更激进方法：每 4 层注意力层**根本不使用位置编码**（NoPE），而其他层使用带块注意力的标准 RoPE。

- NoPE 层可以关注所有位置，无任何位置偏差，而 RoPE 层提供局部排序。结合推理时的温度缩放，这实现 Llama 4 Scout 的 10M token 上下文窗口 — 比任何纯 RoPE 方法高几个数量级。

- **大规模混合专家**已成为前沿模型的主要架构（文件 04 介绍 MoE 基础）。关键设计选择是专家数量、路由稀疏性和负载平衡。

- **路由稀疏性**变化显著：DeepSeek-V3 使用 256 个专家带 top-8 路由（32 倍稀疏性），Qwen3 使用 128 个专家带 top-8（16 倍稀疏性），Mixtral 使用 8 个专家带 top-2（4 倍稀疏性），Llama 4 Maverick 使用 128 个专家带 top-1 加共享专家（128 倍稀疏性）。

- 更高稀疏性意味着相同激活计算更多总参数，但需要更仔细的负载平衡和通信基础设施。

- **无辅助损失负载平衡**（DeepSeek-V3）替换传统负载平衡损失（文件 04），发现会降低模型质量。相反，每个专家维护每训练步调整动态偏置项：过载专家偏置减少（接收更少 token），欠载专家偏置增加。这实现平衡路由，无任何辅助损失污染主训练信号。

- **共享专家**出现在大多数 MoE 设计中：一个或多个专家 FFN 处理每个 token 而不论路由。这些处理所有 token 需要的常见模式（基本语法、功能词），释放路由专家专业化。Llama 4 使用 1 个共享专家加每 token 1 个路由专家（非常稀疏）；DeepSeek-V3 使用 1 个共享加 8 个路由。

- **交替稠密和 MoE 层**提供另一个设计轴。Gemma 2 和 3 交替局部/全局注意力层（Gemma 3 中 5:1 比例，其中局部层使用 1,024 token 滑动窗口，仅全局层缓存完整 128K 上下文）。

- Llama 4 Maverick 交错稠密 FFN 层与 MoE 层。Kimi K2 使用混合稀疏层（专家层中穿插一个稠密层）。这个异构设计允许不同层服务不同功能。

- **多 token 预测（MTP）**在 DeepSeek-V3 中使用，训练模型不仅预测下一个 token 还预测之后的 token。每个位置，辅助预测模块（共享主模型的嵌入）预测一个额外未来 token。MTP 损失相对于主下一个 token 损失加权为 0.1-0.3。除了训练期间提高表示质量，MTP 头可以在推理时作为推测解码的草稿头，提供免费加速。

- **知识蒸馏**是训练策略，大型"教师"模型的输出指导较小"学生"模型的训练。Gemma 2 和 3 广泛使用蒸馏：较小模型（2B、4B）在教师概率分布作为软目标的 50 倍计算最优数据量上训练。这就是为什么 Gemma 3-4B 匹配 Gemma 2-27B 质量。

- 蒸馏损失替换或补充标准交叉熵：学生最小化其输出分布与教师之间的 KL 散度：

$$\mathcal{L}_{\text{distill}} = D_{\text{KL}}(p_{\text{teacher}}(\cdot \mid x) \| p_{\text{student}}(\cdot \mid x))$$

- DeepSeek-R1 用 80 万精选思维链样本将其 671B 推理模型蒸馏到小至 1.5B 的稠密模型，产生不成比例强大推理的小模型。

- **通过强化学习的推理**代表 LLM 能力中最近的重大进展。DeepSeek-R1 证明在基础模型上纯强化学习（无需监督微调）可以引出思维链推理、自我验证和错误纠正，当模型因正确答案获得奖励时，这些行为自发涌现。

- DeepSeek-R1 使用**GRPO**（组相对策略优化），消除 PPO 需要的价值网络。每个提示，GRPO 采样 $G$ 个输出组，计算它们的奖励，并在组内归一化优势：

$$A_i = \frac{r_i - \text{mean}(r_1, \ldots, r_G)}{\text{std}(r_1, \ldots, r_G)}$$

- 策略梯度然后使用这些组相对优势，带截断目标（类似于 PPO 的截断）。

- 消除评论家网络将 RL 训练的内存和计算需求减半，使其实际可以用 RL 训练 671B 参数模型。

- 关键设计选择：DeepSeek-R1 使用**基于规则的奖励**（针对真实值检查数学答案、运行代码测试用例），而非神经奖励模型，因为发现神经奖励模型在这个规模易受奖励黑客攻击。

- **Qwen3 的混合思考模式**将推理（带 `<think>` 标签的逐步思维链）和快速直接响应集成到单个模型中，允许用户控制"思考预算"，在延迟和推理深度之间权衡。

- 这是通过在思考和非思考数据上训练实现的，而非通过单独模型检查点。

- **大规模训练稳定**需要超越标准实践的新技术。**Logit 软上限**（Gemma 2）通过 $s \cdot \tanh(\text{logits} / s)$ 传递注意力分数，软上限 $s$（通常 30-50）防止无界增长。

- **QK-Norm**（Qwen3）在计算注意力分数前对查询和键向量应用 RMSNorm，替换对 QKV 偏置的需求。**QK-Clip**（Kimi K2 的 MuonClip 优化器）监控训练期间最大注意力 logit，并在超过阈值时重新缩放查询 - 键权重矩阵，实现 1T 参数模型稳定预训练，零不稳定事件。

- **FP8 混合精度训练**（DeepSeek-V3）对前向和反向传递中计算密集型矩阵乘法使用 8 位浮点，同时保持主权重在更高精度。

- 与 BF16/FP16 训练相比，这大约加倍吞吐量，质量损失可忽略。DeepSeek-V3 训练其 671B 参数模型仅用 280 万 H800 GPU 小时 — 可比模型的一小部分 — 主要归功于此和其他工程优化。

1. 从头实现简单检索增强生成管道。使用 TF-IDF（文件 02）索引一组文档，为查询检索最相关段落，并将其前置到提示。
```python
import jax.numpy as jnp
import math
from collections import Counter

# Knowledge base: a set of short passages
knowledge_base = [
    "The Eiffel Tower is a wrought-iron lattice tower in Paris, France. It was constructed from 1887 to 1889 as the centerpiece of the 1889 World's Fair.",
    "The Great Wall of China is a series of fortifications built along the northern borders of China. Construction began in the 7th century BC.",
    "Photosynthesis is the process by which plants convert sunlight, water, and carbon dioxide into glucose and oxygen using chlorophyll.",
    "The theory of general relativity, published by Albert Einstein in 1915, describes gravity as the curvature of spacetime caused by mass and energy.",
    "Python is a high-level programming language known for its simple syntax and readability. It was created by Guido van Rossum and released in 1991.",
    "The mitochondria are organelles found in eukaryotic cells. They generate most of the cell's supply of ATP, used as a source of chemical energy.",
]

# Build TF-IDF index (reusing concepts from file 02)
def tokenise(text):
    return text.lower().split()

vocab = sorted(set(w for doc in knowledge_base for w in tokenise(doc)))
word2idx = {w: i for i, w in enumerate(vocab)}
V = len(vocab)
N = len(knowledge_base)

# Document frequencies
doc_freq = Counter()
for doc in knowledge_base:
    for w in set(tokenise(doc)):
        doc_freq[w] += 1

def tfidf_vector(text):
    words = tokenise(text)
    counts = Counter(words)
    vec = jnp.zeros(V)
    for w, c in counts.items():
        if w in word2idx:
            tf = 1 + math.log(c)
            idf = math.log(N / (doc_freq.get(w, 0) + 1))
            vec = vec.at[word2idx[w]].set(tf * idf)
    return vec

# Index all documents
doc_vectors = jnp.stack([tfidf_vector(doc) for doc in knowledge_base])

def cosine_sim(a, b):
    return jnp.dot(a, b) / (jnp.linalg.norm(a) * jnp.linalg.norm(b) + 1e-8)

def retrieve(query, top_k=2):
    """Retrieve top-k most relevant passages for a query."""
    q_vec = tfidf_vector(query)
    sims = jnp.array([cosine_sim(q_vec, doc_vectors[i]) for i in range(N)])
    top_indices = jnp.argsort(-sims)[:top_k]
    return [(int(i), float(sims[i]), knowledge_base[int(i)]) for i in top_indices]

# Test retrieval
queries = [
    "Who built the Eiffel Tower?",
    "How do plants make food?",
    "What did Einstein discover?",
]

for query in queries:
    results = retrieve(query, top_k=1)
    print(f"\nQuery: '{query}'")
    for idx, sim, passage in results:
        print(f"  Retrieved (sim={sim:.3f}): '{passage[:80]}...'")

    # RAG-style prompt construction
    context = results[0][2]
    rag_prompt = f"Context: {context}\n\nQuestion: {query}\nAnswer:"
    print(f"  RAG prompt:\n    {rag_prompt[:120]}...")
```

2. 用玩具草稿和目标模型实现推测解码。显示接受的输出匹配目标模型的分布。
```python
import jax
import jax.numpy as jnp

# Simulate a draft model (fast, less accurate) and target model (slow, accurate)
vocab_size = 8
seq_len = 5

key = jax.random.PRNGKey(42)

# Target model: returns logits given a sequence
def target_model(seq, key):
    """Simulated target model: produces token logits (expensive)."""
    # In practice this would be a large Transformer forward pass
    k1, k2 = jax.random.split(key)
    logits = jax.random.normal(k1, (len(seq), vocab_size)) * 2
    # Make it somewhat predictable: bias toward token (seq[-1] + 1) % vocab_size
    for i in range(len(seq)):
        logits = logits.at[i, (seq[i] + 1) % vocab_size].add(3.0)
    return logits

def draft_model(seq, key):
    """Simulated draft model: similar but noisier (cheap)."""
    k1, k2 = jax.random.split(key)
    logits = jax.random.normal(k1, (len(seq), vocab_size))
    for i in range(len(seq)):
        logits = logits.at[i, (seq[i] + 1) % vocab_size].add(2.0)
    return logits

def sample_token(logits, key):
    return jax.random.categorical(key, logits)

def speculative_decode(prefix, draft_steps=3, key=jax.random.PRNGKey(0)):
    """Speculative decoding: draft proposes, target verifies."""
    seq = list(prefix)
    total_accepted = 0
    total_proposed = 0

    for _ in range(4):  # generate 4 rounds
        key, *subkeys = jax.random.split(key, draft_steps + 3)

        # Draft model proposes draft_steps tokens
        draft_tokens = []
        draft_probs = []
        draft_seq = list(seq)
        for i in range(draft_steps):
            d_logits = draft_model(jnp.array(draft_seq), subkeys[i])
            d_probs = jax.nn.softmax(d_logits[-1])
            tok = sample_token(d_logits[-1], subkeys[i])
            draft_tokens.append(int(tok))
            draft_probs.append(d_probs)
            draft_seq.append(int(tok))

        # Target model scores all draft tokens in one pass
        target_logits = target_model(jnp.array(draft_seq), subkeys[draft_steps])
        target_start = len(seq) - 1  # position of last prefix token

        # Accept/reject each draft token
        accepted = 0
        for i in range(draft_steps):
            t_probs = jax.nn.softmax(target_logits[target_start + i])
            d_prob = draft_probs[i][draft_tokens[i]]
            t_prob = t_probs[draft_tokens[i]]

            # Accept with probability min(1, target_prob / draft_prob)
            accept_prob = jnp.minimum(1.0, t_prob / (d_prob + 1e-10))
            key, accept_key = jax.random.split(key)
            if jax.random.uniform(accept_key) < accept_prob:
                seq.append(draft_tokens[i])
                accepted += 1
            else:
                # Reject: sample from adjusted distribution
                key, resample_key = jax.random.split(key)
                adjusted = jnp.maximum(0, t_probs - draft_probs[i])
                adjusted = adjusted / (adjusted.sum() + 1e-10)
                new_tok = jax.random.categorical(resample_key, jnp.log(adjusted + 1e-10))
                seq.append(int(new_tok))
                break

        total_accepted += accepted
        total_proposed += draft_steps

    return seq, total_accepted, total_proposed

# Run speculative decoding
prefix = [0, 1]
result_seq, accepted, proposed = speculative_decode(prefix)
acceptance_rate = accepted / proposed if proposed > 0 else 0

print(f"Prefix: {prefix}")
print(f"Generated sequence: {result_seq}")
print(f"Draft proposals: {proposed}")
print(f"Accepted: {accepted}")
print(f"Acceptance rate: {acceptance_rate:.1%}")
print(f"Speedup potential: {(accepted + proposed) / proposed:.2f}x")
```

3. 构建简单 DPO 训练循环。给定优选和较不优选补全对，使用 DPO 损失更新小型模型。
```python
import jax
import jax.numpy as jnp

# Tiny language model: linear projection from one-hot to logits
vocab_size = 10
seq_len = 4

key = jax.random.PRNGKey(42)
k1, k2 = jax.random.split(key)

# Current policy parameters (trainable)
theta = jax.random.normal(k1, (vocab_size, vocab_size)) * 0.1
# Reference policy parameters (frozen copy of initial theta)
theta_ref = theta.copy()

def log_prob_sequence(params, sequence):
    """Compute log P(sequence) under a simple autoregressive model."""
    total = 0.0
    for t in range(1, len(sequence)):
        # Simple: logits at position t depend on token at t-1
        logits = params[sequence[t-1]]
        log_probs = jax.nn.log_softmax(logits)
        total += log_probs[sequence[t]]
    return total

def dpo_loss(theta, theta_ref, preferred, dispreferred, beta=0.1):
    """Direct Preference Optimisation loss for one pair."""
    log_pi_w = log_prob_sequence(theta, preferred)
    log_pi_l = log_prob_sequence(theta, dispreferred)
    log_ref_w = log_prob_sequence(theta_ref, preferred)
    log_ref_l = log_prob_sequence(theta_ref, dispreferred)

    # DPO objective
    return -jax.nn.log_sigmoid(
        beta * ((log_pi_w - log_ref_w) - (log_pi_l - log_ref_l))
    )

# Preference dataset: (prompt_prefix, preferred_completion, dispreferred_completion)
preferences = [
    (jnp.array([1, 3, 5, 7]), jnp.array([1, 3, 5, 2])),  # prefer 7 over 2 at end
    (jnp.array([0, 2, 4, 6]), jnp.array([0, 2, 4, 9])),  # prefer 6 over 9
    (jnp.array([3, 3, 3, 3]), jnp.array([3, 3, 3, 0])),  # prefer repeating over 0
    (jnp.array([5, 6, 7, 8]), jnp.array([5, 6, 7, 1])),  # prefer 8 over 1
]

grad_fn = jax.jit(jax.grad(dpo_loss))
lr = 0.05

print("Training DPO...")
for epoch in range(100):
    total_loss = 0.0
    for preferred, dispreferred in preferences:
        loss = dpo_loss(theta, theta_ref, preferred, dispreferred)
        grads = grad_fn(theta, theta_ref, preferred, dispreferred)
        theta = theta - lr * grads
        total_loss += loss
    if (epoch + 1) % 20 == 0:
        avg_loss = total_loss / len(preferences)
        print(f"  Epoch {epoch+1}: avg DPO loss = {avg_loss:.4f}")

# Check: the model should now prefer the preferred completions
print("\nPreference check after DPO training:")
for preferred, dispreferred in preferences:
    lp_w = log_prob_sequence(theta, preferred)
    lp_l = log_prob_sequence(theta, dispreferred)
    print(f"  Preferred {list(preferred.astype(int))}: logP={lp_w:.3f}  "
          f"Dispreferred {list(dispreferred.astype(int))}: logP={lp_l:.3f}  "
          f"{'correct' if lp_w > lp_l else 'WRONG'}")
```
