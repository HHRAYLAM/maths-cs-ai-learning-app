## 🌀 文本扩散模型 - 同时生成所有 token

扩散模型（第 8 章图像）也可以用于文本！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 核心思想

```
自回归：从左到右逐词生成
扩散：从噪声开始迭代细化

t=T: [MASK] [MASK] [MASK] [MASK]
         ↓ 去噪
t=T-1: The  [MASK] [MASK] cat
         ↓ 去噪
t=T-2: The  sat   [MASK] cat
         ↓ 去噪
t=0:   The  sat   on   cat  → The cat sat on mat
```

### D3PM（离散去噪扩散）

问题：文本是离散的，不能像图像那样添加高斯噪声！

解决：用转移矩阵直接定义离散 token 的破坏过程：

$$q(x_t \mid x_{t-1}) = \text{Cat}(x_t ; \, x_{t-1} Q_t)$$

- $Q_t$：转移矩阵，控制 token 被替换/掩码的概率
- $\text{Cat}$：分类分布

</div>
  <div class="lesson-visual-column">

```
D3PM 前向过程：

原始："The cat sat on the mat"
         ↓ 随机替换/掩码
t=1:   "The [M] sat [M] the [M]"
         ↓ 继续破坏
t=2:   "[M] [M] [M] on [M] [M]"
...
t=T:   全部掩码

反向过程（学习去噪）：
从完全掩码逐步恢复原文！
```

### 连续文本扩散

另一种方法：在连续嵌入空间工作！

```
步骤：
1. Token → 嵌入向量（连续空间）
2. 在嵌入空间添加噪声
3. 去噪模型（Transformer）学习逆转
4. 生成时映射回离散 token（最近邻）

挑战：小误差可能映射到错误 token！
```

优势：
- 同时生成所有 token
- 允许全局规划
- 轻松填充（在中间生成缺失文本）

劣势：
- 长文本质量仍落后自回归

## 📷 OCR - 从图像读取文本

OCR（光学字符识别）从图像提取文本，现代 OCR 深度集成 NLP！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### CRNN（卷积循环神经网络）

```
文本图像
    ↓
CNN 提取视觉特征
    ↓
特征图切片成列序列
    ↓
双向 LSTM 建模上下文
    ↓
CTC 解码 → 输出文本
```

### CTC 解码

问题：输入列数 $T$ ≠ 输出字符数 $L$！

解决：引入空白 token $\epsilon$，定义映射 $\mathcal{B}$：

$$\mathcal{B}(\text{"H-e-ll-l-ll-o"}) = \text{"Hello"}$$

目标概率：

$$P(y \mid x) = \sum_{\pi \in \mathcal{B}^{-1}(y)} \prod_{t=1}^{T} P(\pi_t \mid x)$$

用前向算法（HMM）高效计算！

</div>
  <div class="lesson-visual-column">

```
CTC 对齐例子：

输入列：1   2   3   4   5   6   7
输出：  H   H   e   l   l   o   ϵ
         └─┘   └───┘   └─┘
        折叠重复  移除空白

结果："Hello"

空白 token 关键：区分 "Hello" vs "Helo"
```

## 🤖 RLHF - 人类反馈强化学习

RLHF 将语言模型与人类偏好对齐！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 三阶段流程

1. 监督微调（SFT）
- 在人类撰写的高质量响应上微调

2. 奖励模型训练
- 收集人类比较：$(x, y_w, y_l)$ → 哪个更好？
- 训练奖励模型 $r_\phi(x, y)$：

$$\mathcal{L}_{\text{RM}} = -\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

3. RL 微调
- 用 PPO 最大化奖励，带 KL 惩罚：

$$\mathcal{L}_{\text{RL}} = -\mathbb{E}\left[r_\phi(x, y) - \beta \, D_{\text{KL}}(\pi_\theta \| \pi_{\text{SFT}})\right]$$

</div>
  <div class="lesson-visual-column">

```
RLHF 流程：

阶段 1: SFT
基础模型 → 人类数据 → SFT 模型

阶段 2: 奖励模型
(x, y_w ≻ y_l) → 训练 → r_φ(x,y)

阶段 3: RL 微调
SFT 模型 + PPO + KL 惩罚 → 对齐模型

最终：模型输出符合人类偏好！
```

### DPO - 直接偏好优化

DPO 简化 RLHF：无需单独奖励模型！

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的 DPO 理解</div>
  <div class="story-card-content">
    <strong>关键洞察</strong>：KL 约束 RL 目标有闭式解
    <br><br>
    <strong>最优策略</strong>：$\pi^\ast(y \mid x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y \mid x) \exp\!\left(\frac{r(x, y)}{\beta}\right)$
    <br><br>
    <strong>DPO 损失</strong>：
    $$\mathcal{L}_{\text{DPO}} = -\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)}\right)$$
    <br><br>
    直观：增加优选响应概率，减少劣选响应概率！
  </div>
</div>

---

### DPO vs RLHF 对比

| 特性 | RLHF | DPO |
|------|------|-----|
| 奖励模型 | 需要单独训练 | 隐式，无需训练 |
| RL 算法 | 需要 PPO | 无需 RL，仅监督 |
| 实现复杂度 | 高（不稳定） | 低（稳定） |
| 效果 | 好 | 同等或更好 |

```
DPO 直观理解：

给定优选 y_w 和劣选 y_l：

DPO 鼓励：
log π(y_w) - log π_ref(y_w) ↑ 增加
log π(y_l) - log π_ref(y_l) ↓ 减少

相对参考模型，
拉开优选和劣选的差距！
```

## 📏 长上下文方法 - 处理超长文本

问题：标准注意力 $O(n^2)$ 复杂度，序列太长放不下！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 稀疏注意力

| 模式 | 说明 | 复杂度 |
|------|------|--------|
| 局部注意力 | 关注固定窗口邻居 | $O(nw)$ |
| 跨步注意力 | 关注每 $k$ 个 token | $O(nk)$ |
| 随机注意力 | 关注随机子集 | $O(nr)$ |
| 组合模式 | BigBird、Longformer | $O(n \sqrt{n})$ |

### 滑动窗口注意力

```
每个 token 只关注前 w 个 token：

token i 关注：[i-w, i-w+1, ..., i-1]

复杂度：O(nw) 而非 O(n²)

L 层后，有效感受野 = L × w
```

## 🧠 状态空间模型（SSM）- 注意力替代方案

SSM 用线性动力系统替代注意力！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 连续时间 SSM

$$x'(t) = Ax(t) + Bu(t), \quad y(t) = Cx(t) + Du(t)$$

- $x(t)$：潜在状态
- $u(t)$：输入
- $y(t)$：输出
- $A, B, C$：系统矩阵

### 离散化

零阶保持离散化（步长 $\Delta$）：

$$\bar{A} = \exp(\Delta A), \quad \bar{B} = (\Delta A)^{-1}(\exp(\Delta A) - I) \cdot \Delta B$$

离散递归：

$$x_k = \bar{A} x_{k-1} + \bar{B} u_k, \quad y_k = C x_k + D u_k$$

看起来像 RNN！

</div>
  <div class="lesson-visual-column">

```
SSM 双重观点：

观点 1：递归（推理）
x₀ → x₁ → x₂ → ... → xₙ
每步 O(1)，无 KV 缓存增长

观点 2：卷积（训练）
y = K̄ * u
核 K̄ = (CB, CAB, CA²B, ...)
用 FFT：O(n log n) 并行！

相同数学，两种实现！
```

<div class="visual-caption">图 5：SSM 双重观点</div>

</div>
</div>

### S4（结构化状态空间）

S4 使 SSM 实用：

问题：朴素 $A$ 矩阵导致消失/爆炸动态！

解决：用HiPPO矩阵初始化：

$$A_{nk} = -\begin{cases} (2n+1)^{1/2}(2k+1)^{1/2} & n > k \\ n+1 & n = k \\ 0 & n < k \end{cases}$$

- 源自连续信号最优多项式近似
- 状态以优雅衰减维护历史压缩表示

### Mamba 选择性

$$B_k = \text{Linear}(u_k), \quad C_k = \text{Linear}(u_k), \quad \Delta_k = \text{softplus}(\text{Linear}(u_k))$$

步长 $\Delta_k$ 的作用：
- 大 $\Delta$：状态强烈整合当前输入（重置）
- 小 $\Delta$：保持现有状态（忽略当前输入）

硬件感知并行扫描：
- 递归可表示为结合操作
- 用前缀和并行化
- GPU 上 $O(n)$ 时间，$O(\log n)$ 深度

---

### Mamba vs Transformer

| 特性 | Transformer | Mamba |
|------|-------------|-------|
| 推理复杂度 | $O(n \cdot d)$ KV 缓存 | $O(1)$ 固定状态 |
| 训练并行 | 完全并行 | 并行扫描 |
| 长序列 | 内存瓶颈 | 天然高效 |
| 质量 | 优秀 | 匹配或超越 |

```
混合架构（Jamba、Zamba）：
交错 Mamba 层 + 注意力层

Mamba：高效长程传播
注意力：精确基于内容检索

最佳组合！
```

## ⚡ 推测解码 - 加速推理

推测解码用小模型加速大模型！

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的理解</div>
  <div class="story-card-content">
    <strong>核心思想</strong>：小模型草稿，大模型验证
    <br><br>
    <strong>流程</strong>：
    1. 草稿模型自回归生成 k 个 token（快）
    2. 目标模型单前向传递评分所有 k 个（高效批处理）
    3. 接受 - 拒绝采样：以 $\min(1, p_{target}/p_{draft})$ 概率接受
    <br><br>
    <strong>保证</strong>：输出分布与仅目标模型相同！
  </div>
</div>

---

### 推测解码算法

```
草稿模型生成 3 个 token：
"The" → "cat" → "sat"

目标模型单前向传递评分：
P("cat" | "The")
P("sat" | "The cat")
P("on" | "The cat sat")

接受 - 拒绝：
对每个 token t，以 min(1, P_target(t)/P_draft(t)) 接受

如果拒绝，从调整分布重采样！
```

加速：2-3 倍，无质量下降

---

### Medusa（无草稿模型）

Medusa 不用单独草稿模型：

```
目标模型本身添加多个预测头：
头 1：预测 +1 步
头 2：预测 +2 步
头 3：预测 +3 步
```

## 🏗️ 前沿架构创新

### 分组查询注意力（GQA）

GQA 减少 KV 缓存：

<div class="lesson-two-column">
  <div class="lesson-text-column">

```
MHA：每头独立 KV 投影
64 头 × 128 维 = 8192 维缓存

GQA：多查询头共享单 KV 头
64 查询头 ÷ 8 KV 头 = 每组 8 个
缓存减少 8 倍！

Llama 3、Qwen、Gemma 都用 GQA
```

### MLA（多头潜在注意力）

MLA 更激进压缩：

$$c_t = W_{\text{down}} \, h_t$$

- $c_t$：低秩潜在向量（$d_c \ll$ 完整 KV）
- 仅缓存 $c_t$
- 注意力时重建：$k_t = W_{\text{up}}^K c_t$

DeepSeek-V3：
- 完整 MHA：128×128 = 16,384 维
- MLA 压缩：$d_c = 512$ 维
- 缓存减少93%！

</div>
  <div class="lesson-visual-column">

```
KV 缓存策略对比：

MHA:
Q1→K1,V1  Q2→K2,V2  Q3→K3,V3  ... (独立)

GQA:
Q1,Q2→K1,V1  Q3,Q4→K2,V2  ... (共享)

MLA:
Q1,Q2,Q3,Q4→c→重建 K,V (压缩)

缓存：MHA > GQA >> MLA
质量：MLA ≈ GQA ≈ MHA
```
