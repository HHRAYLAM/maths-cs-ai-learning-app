# 图像和视频分词

*图像和视频分词将连续视觉数据转换为离散标记序列，使变换器能够像处理文本一样处理它们。本文件涵盖 VQ-VAE、VQ-GAN、码本学习、DALL-E 的 dVAE、视频分词和无查找表量化*

## 为什么要对图像进行分词

- 将语言想象成有限的字母表：英语大约有 26 个字母，现代语言模型将文本分割成 30,000-100,000 个子词标记。每个句子都变成一个离散符号序列，变换器可以逐个预测这些符号。另一方面，图像存在于连续的高维空间中：单个 256x256 RGB 图像是 $\mathbb{R}^{256 \times 256 \times 3} \approx \mathbb{R}^{196{,}608}$ 中的一个点。如果你想让语言模型使用它与说英语相同的机制来"说"图像，你需要将这些连续像素数组转换为从有限词汇表中抽取的可管理离散标记序列。这种转换就是图像分词。

- 想象你是一位马赛克艺术家。你没有无限的瓷砖色调；你只有固定的调色板，比如说 8192 种不同的瓷砖颜色。要将照片再现为马赛克，你必须 (1) 决定每个瓷砖代表照片的哪个区域，(2) 为每个区域选择最接近的瓷砖颜色，(3) 接受一些细节丢失但整体图像可识别。图像分词正是这样做的：编码器将空间补丁压缩为潜在向量，码本将每个向量映射到其最接近的条目，结果是一个整数索引网格，每个补丁一个，离散模型可以处理。

- 分词的好处有三方面。首先，它显著压缩图像：256x256 图像可能变成 16x16 标记网格，将序列长度从 65,536 个像素减少到 256 个标记，这对于注意力机制模型（其成本随序列长度二次方增长）是可行的。其次，它统一了表示：文本标记和图像标记存在于同一个离散词汇表中，使单个自回归变换器能够生成交错的文本和图像。第三，它施加了有用的瓶颈，迫使模型学习语义上有意义的代码，而不是记忆像素噪声。

![图像分词管道概述：连续图像进入编码器，潜在向量针对码本量化，产生离散标记索引网格](../images/image_tokenisation_overview.svg)

- 回想第 8 章卷积神经网络如何从图像中提取层次化特征图，以及第 7 章文本分词器如何将字符串转换为整数序列。图像分词位于交叉点：它使用 CNN 或视觉变换器编码器（第 8 章）产生空间特征，然后借用离散词汇表的概念（第 7 章）将这些特征转换为标记索引。

## VQ-VAE：向量量化

- 正如我们在第 6 章中看到的，标准变分自编码器（VAE）将输入编码为连续潜在分布，并从该分布解码样本回重构。潜在空间是连续的，这使得将其送入离散序列模型很尴尬。向量量化变分自编码器（VQ-VAE），由 van den Oord 等人（2017）引入，通过引入可学习的嵌入向量码本并将每个编码器输出 snap 到其最近的码本条目，用离散潜在替换连续潜在。

- 想象一个图书馆正好有 $K$ 个标标签的书架。当新书（编码器输出）到达时，图书管理员将其放在与其现有书籍（码本向量）最相似的书架上，并记录书架编号。稍后，要检索书籍，你只需要书架编号：该书架上的码本条目是足够好的替代品。这就是向量量化。

- 形式上，VQ-VAE 有三个组件：

- 一个编码器 $E$，将输入图像 $\mathbf{x} \in \mathbb{R}^{H \times W \times 3}$ 映射到空间连续潜在向量网格 $\mathbf{z}_e = E(\mathbf{x}) \in \mathbb{R}^{h \times w \times d}$，其中 $h \times w$ 是下采样空间分辨率，$d$ 是嵌入维度。

- 一个码本 $\mathcal{C} = \{\mathbf{e}_1, \mathbf{e}_2, \ldots, \mathbf{e}_K\} \subset \mathbb{R}^d$，包含 $K$ 个可学习的嵌入向量。典型码本大小范围为 512 到 16,384 个条目。

- 一个解码器 $D$，从量化潜在重构图像。

- 量化步骤将每个空间位置 $(i, j)$ 的编码器输出 $\mathbf{z}_e(\mathbf{x})$ 替换为其最近的码本条目：

$$\mathbf{z}_q(i,j) = \mathbf{e}_{k^\ast} \quad \text{where} \quad k^\ast = \arg\min_k \|\mathbf{z}_e(i,j) - \mathbf{e}_k\|_2$$

- 这是嵌入空间中的最近邻查找，与 k-means 分配操作完全相同（第 6 章）。索引 $k^\ast$ 是空间位置 $(i,j)$ 的离散标记，完整图像表示为来自 $\{1, \ldots, K\}$ 的整数 $h \times w$ 网格。

![VQ-VAE 架构：编码器产生连续潜在，每个潜在向量匹配最近的码本条目，解码器从量化代码重构](../images/vqvae_architecture.svg)

- 挑战在于 $\arg\min$ 不可微分：你无法通过离散选择反向传播。VQ-VAE 通过直通估计器解决这个问题：在前向传播期间，解码器接收 $\mathbf{z}_q$（量化向量）；在后向传播期间，重构损失相对于 $\mathbf{z}_q$ 的梯度直接复制到 $\mathbf{z}_e$，就像量化步骤是恒等函数一样。这可以简洁地写为：

$$\mathbf{z}_q = \mathbf{z}_e + \text{sg}(\mathbf{z}_q - \mathbf{z}_e)$$

- 其中 $\text{sg}(\cdot)$ 是停止梯度操作符。在前向传播中，这评估为 $\mathbf{z}_q$；在后向传播中，梯度仅通过 $\mathbf{z}_e$ 项流动。

- 完整的 VQ-VAE 损失有三项：

$$\mathcal{L} = \underbrace{\|\mathbf{x} - D(\mathbf{z}_q)\|_2^2}_{\text{重构}} + \underbrace{\|\text{sg}(\mathbf{z}_e) - \mathbf{e}\|_2^2}_{\text{码本 (VQ)}} + \underbrace{\beta \|\mathbf{z}_e - \text{sg}(\mathbf{e})\|_2^2}_{\text{承诺}}$$

- 重构损失训练编码器和解码器忠实地再现输入。码本损失（也称为 VQ 损失）将码本向量拉向编码器输出；注意 $\text{sg}(\mathbf{z}_e)$ 意味着编码器不从该项接收梯度，因此它只更新码本。承诺损失则相反：它鼓励编码器输出保持在码本向量附近，防止编码器"逃离"码本。超参数 $\beta$（通常为 0.25）控制码本和承诺项之间的平衡。

- 在实践中，码本通常使用指数移动平均（EMA）更新而不是梯度下降，这更稳定。设 $\mathbf{n}_k$ 是分配给码本条目 $k$ 的编码器输出计数，$\mathbf{s}_k$ 是它们的和。EMA 更新为：

$$\mathbf{n}_k \leftarrow \gamma \mathbf{n}_k + (1 - \gamma) |\{(i,j) : k^\ast_{ij} = k\}|$$

$$\mathbf{s}_k \leftarrow \gamma \mathbf{s}_k + (1 - \gamma) \sum_{(i,j) : k^\ast_{ij} = k} \mathbf{z}_e(i,j)$$

$$\mathbf{e}_k \leftarrow \frac{\mathbf{s}_k}{\mathbf{n}_k}$$

- 其中 $\gamma$ 是衰减率（通常为 0.99）。这等效于对编码器输出运行在线 k-means 算法。

### 码本坍塌

- VQ-VAE 的一个臭名昭著的失败模式是码本坍塌（也称为索引坍塌）：模型学习只使用 $K$ 个码本条目的一小部分，留下大多数条目"死亡"。想象一个图书馆，90% 的书架是空的，因为图书管理员总是将书籍路由到相同的几个流行书架。这浪费了表示容量。

- 码本坍塌发生是因为编码器、码本和解码器在训练期间协同适应。如果一个条目几轮未被选择，它会漂离编码器流形，使其更不可能被选择，产生正反馈循环。

- 几种技术减轻码本坍塌：
    - 码本重置：定期重新初始化死亡条目，通过复制随机采样的编码器输出。这给死亡条目一个在潜在空间活跃区域附近的新鲜开始。
    - 带 Laplace 平滑的 EMA 更新：向 $\mathbf{n}_k$ 添加小常数，防止任何条目有零计数，确保所有条目接收梯度信号。
    - 承诺损失调优：增加 $\beta$ 迫使编码器输出更紧密地聚集在码本条目周围，更均匀地分布分配。
    - 因子化代码：将码本查找分解为较小组合的乘积（例如，两个大小为 $\sqrt{K}$ 的码本），这通过减少每次查找的有效码本大小来提高利用率。
    - 熵正则化：添加鼓励码本使用均匀分布的惩罚，最大化熵 $H = -\sum_k p_k \log p_k$，其中 $p_k$ 是经验分配概率。

## VQ-GAN：对抗训练提高保真度

- VQ-VAE 产生不错的重构，但像素级 $\ell_2$ 损失倾向于生成模糊输出，因为它同等地惩罚每个像素偏差，在 plausible 细节上平均而不是选择清晰的细节。想象要求某人绘制一张脸，使与所有可能脸的平均差异最小化——他们会绘制一张模糊的平均脸，而不是一张清晰的个体脸。

- VQ-GAN（Esser 等人，2021）通过结合 VQ-VAE 框架与来自生成对抗网络的判别器（第 6 章）来解决这个问题。判别器是基于补丁的卷积网络，判断局部图像补丁是真实的（来自训练数据）还是伪造的（来自解码器）。这个对抗损失鼓励解码器产生感知上清晰、真实的纹理，而不是像素级平均。

- VQ-GAN 目标将两项添加到 VQ-VAE 损失：

$$\mathcal{L}_\text{VQ-GAN} = \mathcal{L}_\text{VQ-VAE} + \lambda_\text{adv} \mathcal{L}_\text{adv} + \lambda_\text{perc} \mathcal{L}_\text{perc}$$

- 对抗损失 $\mathcal{L}_\text{adv}$ 是应用于解码器输出的标准 GAN 目标。判别器 $\mathcal{D}$ 试图区分真实补丁和解码补丁，解码器（生成器）试图欺骗它。非饱和公式为：

$$\mathcal{L}_\text{adv} = -\mathbb{E}[\log \mathcal{D}(D(\mathbf{z}_q))]$$

- 感知损失 $\mathcal{L}_\text{perc}$ 比较原始和重构图像之间预训练网络（通常是 VGG 或 LPIPS）的特征激活：

$$\mathcal{L}_\text{perc} = \sum_l \|\phi_l(\mathbf{x}) - \phi_l(D(\mathbf{z}_q))\|_2^2$$

- 其中 $\phi_l$ 表示预训练网络第 $l$ 层的特征图。这个损失捕捉高级结构相似性，而不是像素级精度。

- 权重 $\lambda_\text{adv}$ 自适应设置，使对抗梯度和重构梯度平衡，防止对抗损失在训练早期（当重构较差时）主导。

![VQ-GAN 训练：编码器和解码器通过量化步骤连接，补丁判别器对解码输出提供对抗反馈](../images/vqgan_training.svg)

- 结果是一个分词器，在相同码本大小下产生比 VQ-VAE 显著更清晰的重构。VQ-GAN 是许多主要图像生成系统背后的骨干分词器，包括原始 DALL-E、Parti 和众多文本到图像模型。它将 256x256 图像转换为 16x16 或 32x32 离散标记网格，来自大小为 1024-16384 的码本，实现每个空间维度 16 倍到 64 倍的压缩比。

## 残差量化和多尺度码本

- 单个码本对重构质量施加强硬上限：每个空间位置正好由一个码本向量表示，任何比码本能表达的更精细的细节都丢失了。想象用固定调色板中的单个词描述颜色："青色"接近但不精确。如果你可以添加细化——"青色，但稍微更蓝一点且亮一点"——你会更接近。

- 残差量化（RQ）迭代地应用这个想法。在第一个量化步骤产生 $\mathbf{z}_q^{(1)}$ 后，计算残差 $\mathbf{r}^{(1)} = \mathbf{z}_e - \mathbf{z}_q^{(1)}$，然后针对第二个码本量化残差得到 $\mathbf{z}_q^{(2)}$，依此类推 $T$ 层：

$$\mathbf{r}^{(0)} = \mathbf{z}_e$$

$$\mathbf{z}_q^{(t)} = \text{Quantise}(\mathbf{r}^{(t-1)}, \mathcal{C}^{(t)})$$

$$\mathbf{r}^{(t)} = \mathbf{r}^{(t-1)} - \mathbf{z}_q^{(t)}$$

- 最终的量化表示是 $\hat{\mathbf{z}} = \sum_{t=1}^{T} \mathbf{z}_q^{(t)}$。使用 $T$ 层，每层使用大小为 $K$ 的码本，有效词汇表大小是 $K^T$，但你只需要存储 $T \times K$ 向量而不是 $K^T$。例如，8 层，$K = 1024$ 给出有效 $1024^8 \approx 10^{24}$ 条目，而只存储 8192 个向量。

- 每个连续层捕捉更精细的细节：第一个码本捕捉粗略结构，第二个捕捉中频校正，依此类推。这类似于 JPEG 中的连续近似或 Web 图像中的渐进式渲染，其中粗略版本首先出现，细节逐渐填充。

![残差量化：原始向量在连续阶段近似，每个阶段量化前一阶段的残差](../images/residual_quantisation.svg)

- 多尺度码本通过在不同空间分辨率操作扩展这个想法。你不是重复量化相同的空间网格，而是在多个尺度上量化：粗略网格捕捉全局结构，更精细的网格捕捉局部细节。这类似于第 8 章目标检测部分中的特征金字塔思想，其中不同尺度的特征捕捉不同级别的细节。

- 乘积量化是相关技术，其中 $d$ 维潜在向量分裂为 $M$ 个 $d/M$ 维子向量，每个子向量用自己的码本独立量化。这给出 $K^M$ 的有效词汇表，而只存储 $M \times K$ 向量。乘积量化广泛用于近似最近邻搜索（第 13 章），并已适应图像分词。

- 有限标量量化（FSQ），由 Mentzer 等人（2023）引入，采用完全不同的方法：不是学习码本，它只是将潜在向量的每个维度舍入到固定整数级别集（例如，$\{-2, -1, 0, 1, 2\}$）。使用每维度 $L$ 级别和 $d$ 维度，隐式码本大小是 $L^d$。FSQ 完全避免码本坍塌，因为没有学习的码本向量，只有确定性舍入的编码器输出。直通估计器处理舍入的不可微分性。

## 实践中的图像分词器

- 从 VQ-VAE 到 VQ-GAN 到残差量化的进展催生了一系列用于最先进生成模型的实用图像分词器。

### DALL-E 分词器 (dVAE)

- 原始DALL-E（Ramesh 等人，2021）使用离散 VAE（dVAE）将 256x256 图像分词为 32x32 标记网格，来自大小为 8192 的码本。dVAE 用 Gumbel-Softmax 松弛替换硬 $\arg\min$ 量化，使前向传播在训练期间可微分。在推理时，使用 $\arg\max$ 产生硬标记分配。dVAE 用重构损失、针对均匀先验的 KL 散度和 Gumbel-Softmax 的学习温度计划组合训练。DALL-E 然后训练 120 亿参数自回归变换器来建模 256 个文本标记和 1024 个图像标记（32x32）的联合分布。

### LlamaGen

- LlamaGen（Sun 等人，2024）表明你可以重新利用标准 Llama 风格语言模型架构（第 7 章）用于自回归图像生成，前提是你有一个好的图像分词器。LlamaGen 使用改进的 VQ-GAN 分词器，大码本（16,384 个条目），训练纯自回归变换器（除了分词器外没有特殊的图像特定修改）以光栅扫描顺序从左到右预测图像标记。关键见解是，一旦图像被分词为离散序列，与语言相同的下一个标记预测范式也适用于图像，验证分词真正弥合了模态差距。

### Cosmos 分词器

- Cosmos 分词器（NVIDIA，2024）设计用于统一框架中的图像和视频。它使用因果 3D 架构，将图像视为单帧视频，使同一个分词器能够处理两种模态。Cosmos 支持连续和离散分词模式：连续模式输出实值潜在向量（用于扩散模型后端），而离散模式应用有限标量量化产生整数标记（用于自回归模型后端）。编码器使用因果 3D 卷积，使每帧的标记仅依赖于当前和先前的帧，实现流式视频分词。

## 视频分词

- 视频为图像的空间维度添加了第三轴——时间。视频是帧序列，通常每秒 24-30 帧，相邻帧高度冗余，因为视觉世界在 33 毫秒内不会发生巨大变化。视频分词利用这种时间冗余，实现比独立分词每帧高得多的压缩。

- 将视频压缩想象成翻页书。如果你从零开始绘制每一页，你需要数千张详细的图纸。但大多数页面几乎与它们的邻居相同，所以你可以每 10 页绘制一个完整的"关键帧"，只在中间的页面上注意小的变化。视频分词器自动学习这个技巧。

### 3D VQ-VAE

- VQ-VAE 到视频最直接的扩展是3D VQ-VAE，它将编码器和解码器中的 2D 卷积替换为同时在空间和时间维度上操作的 3D 卷积。如果编码器在空间上下采样 $f_s$ 倍，在时间上下采样 $f_t$ 倍，$T \times H \times W$ 的视频剪辑变成 $(T/f_t) \times (H/f_s) \times (W/f_s)$ 的标记网格。

- 例如，使用 $f_s = 16$ 和 $f_t = 4$，16 帧 256x256 视频剪辑变成 $4 \times 16 \times 16 = 1024$ 标记序列。这足够紧凑，变换器可以自回归地建模，而原始像素数将是 $16 \times 256 \times 256 \times 3 \approx 310$ 万个值。

- 3D 卷积联合学习空间和时间特征。早期层捕捉局部运动（边缘在帧之间移动），而更深层捕捉更高级动力学（对象出现、消失或改变形状）。这与第 8 章卷积网络中的层次化特征提取原则相同，沿时间轴扩展。

### 因果视频分词器

- 标准 3D 卷积查看过去、当前和未来帧，这意味着你需要整个视频剪辑才能分词任何部分。因果视频分词器约束时间卷积，使每个输出仅依赖于当前和先前的帧，从不依赖未来帧。这类似于自回归变换器中的因果掩码（第 7 章）：信息向前时间流动，但从不向后。

- 因果分词对两个用例至关重要。首先，流式传输：你可以实时分词视频，因为帧到达，无需缓冲未来帧。其次，自回归生成：当变换器逐帧生成视频时，帧 $t$ 的标记必须是可计算的，无需知道帧 $t+1$，因为帧 $t+1$ 尚未生成。

- 因果约束通过非对称填充时间卷积实现：时间大小 $k$ 的内核在过去侧填充 $k-1$ 个零，在未来侧填充零个零，确保时间 $t$ 的输出仅依赖于时间 $t-k+1, \ldots, t$ 的输入。

- 因果视频分词器的一个优雅属性是它们可以分词单个图像（"一帧视频"），无需特殊处理。第一帧没有过去上下文，所以它的标记仅从帧本身计算。这个图像 - 视频统一意味着单个分词器服务于两种模态，简化架构，使模型能够用同一个解码器生成图像和视频。

### 时间压缩策略

- 不同的应用需要不同的时间压缩比。对于动作识别（微妙的运动很重要），温和压缩（$f_t = 2$）保留时间细节。对于长格式视频生成（存储数千帧是禁止的），需要激进压缩（$f_t = 8$ 或更高）。

- 一些分词器使用因子化压缩：空间和时间压缩分阶段执行。首先，2D 编码器独立压缩每帧，产生每帧潜在网格。然后，1D 时间编码器跨时间维度压缩。这个因子化计算成本比完整 3D 卷积便宜，允许空间和时间不同的压缩比。权衡是它不能像联合 3D 编码那样有效地捕捉时空模式（如球对角移动）。

- 时间插值标记是最近的创新，分词器只完整编码关键帧，将中间帧表示为轻量级插值代码，描述如何在关键帧之间变形。这反映了经典视频压缩（H.264/HEVC 中的 I 帧和 P 帧），但在学习的潜在空间中。

## 连续与离散标记

- 不是每个下游模型都需要离散标记。扩散模型（第 10 章，文件 04）原生使用连续值工作——它们迭代地去噪高斯样本，它们的损失函数（去噪得分匹配）定义在连续空间上。对于扩散后端，分词器编码器产生从不清化的连续潜在向量。潜在扩散模型（Stable Diffusion、DALL-E 3、Flux）使用 VQ-GAN 类似的编码器 - 解码器，但完全跳过码本，在连续潜在空间中操作。

- 自回归模型（GPT 风格）相反，使用有限词汇表的 softmax 预测下一个标记。它们根本上需要离散标记。每个使用自回归变换器的图像生成系统（DALL-E、Parti、LlamaGen、Chameleon）依赖于离散分词器。

- 因此，连续与离散标记的选择由生成后端驱动：

- 使用离散标记当：模型是自回归的（下一个标记预测带交叉熵损失），你想与文本标记共享词汇表用于统一多模态模型，或者你需要精确的标记级控制（例如，通过标记替换进行检索或编辑）。

- 使用连续标记当：模型是扩散模型或流匹配模型，任务需要非常高保真重构（连续潜在完全避免量化误差），或者你想使用在实值向量上操作的回归损失。

- 一些最近的架构支持两种模式。例如，Cosmos 分词器可以从同一个编码器输出连续潜在（用于其扩散模式）或 FSQ 离散标记（用于其自回归模式），带有一个轻量级量化头，可以打开或关闭。

- 软量化是中间立场：不是硬 $\arg\min$ 分配，计算前 $k$ 个最近码本条目的加权平均，权重由负距离的 softmax 给出。这比硬量化保留更多信息，同时仍然是近似离散的。一些系统在训练期间使用软量化，在推理时使用硬量化。

## 应用

### 自回归图像生成

- 一旦图像是离散标记序列，你可以训练标准自回归变换器来建模它们。图像标记被扁平化为 1D 序列（通常按光栅扫描顺序：从左到右，从上到下），变换器学习 $p(\text{token}_i | \text{token}_1, \ldots, \text{token}_{i-1})$，带标准交叉熵损失。在生成时，标记逐个采样，完成的网格通过分词器的解码器产生像素。

- 条件文本很简单：在图像标记序列前添加文本标记，所以模型学习 $p(\text{image tokens} | \text{text tokens})$。这正是 DALL-E、Parti 和 LlamaGen 执行文本到图像生成的方式。文本和图像标记共享同一个变换器、同一个注意力机制，通常同一个嵌入表（文本和图像标记占据不同的索引范围）。

- 光栅扫描顺序引入人为不对称性：图像的左上角最先生成，没有关于右下角的上下文。几部作品解决了这个问题。掩码图像建模（MaskGIT）训练双向变换器同时生成所有标记，但具有不同的置信度，迭代地解掩码最自信的标记。多尺度生成最先生成粗略标记（捕捉全局构图），然后用残差标记细化。这些方法以纯左到右生成的简单性为代价，换取更好的全局连贯性。

### 统一视觉 - 语言标记

- 图像分词最深层的动机是统一：将视觉和语言放入相同的表示格式，使单个模型架构处理两者。正如我们在第 7 章中讨论的，语言模型是非凡的序列到序列机器。通过将图像表示为标记序列，我们免费继承语言建模的所有基础设施——预训练配方、缩放定律、RLHF、上下文长度扩展。

- Chameleon（Meta，2024）是一个突出的例子：它使用 VQ-GAN 分词器，8192 码本条目，将图像转换为标记，与文本标记交错在单个约 65,000 个条目的词汇表中（文本 + 图像）。标准变换器在混合文本 - 图像序列上训练，使它能够生成给定图像的文本、给定文本的图像，或交错的文本和图像内容，所有都用同一个前向传播。

- Gemini（Google，2024）采用类似方法，大规模，原生理解和生成图像、音频和文本在单个变换器内，带特定模态的分词器馈送到共享序列。

- 统一模型中的关键工程挑战是词汇表平衡：如果 65,000 个词汇表条目中有 8192 个是图像标记，模型可能不足分配容量给视觉。解决方案包括每个模态的独立嵌入层（仅在注意力级别共享）、特定模态的损失加权，以及预训练期间仔细的数据混合比例。

![统一视觉 - 语言模型：来自独立分词器的文本和图像标记交错为单个序列，由一个变换器处理](../images/unified_vision_language_tokens.svg)

1. 在 JAX 中实现最小 VQ 层：给定一批编码器输出向量，执行最近邻码本查找，计算 VQ-VAE 损失（重构 + 码本 + 承诺）。将码本利用率可视化为直方图。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# --- 最小 VQ 层 ---
key = jax.random.PRNGKey(42)
d = 8          # 嵌入维度
K = 64         # 码本大小
n_vectors = 256  # 编码器输出批次

# 随机编码器输出和码本
k1, k2 = jax.random.split(key)
z_e = jax.random.normal(k1, (n_vectors, d))       # 编码器输出
codebook = jax.random.normal(k2, (K, d)) * 0.1     # 码本（小初始化）

# 最近邻查找：为每个 z_e 找到最近的码本条目
# distances[i, k] = ||z_e[i] - codebook[k]||^2
distances = (
    jnp.sum(z_e  2, axis=1, keepdims=True)
    - 2 * z_e @ codebook.T
    + jnp.sum(codebook  2, axis=1, keepdims=True).T
)
indices = jnp.argmin(distances, axis=1)       # 标记索引
z_q = codebook[indices]                        # 量化向量

# VQ-VAE 损失项
beta = 0.25
loss_codebook = jnp.mean((jax.lax.stop_gradient(z_e) - z_q)  2)
loss_commit   = jnp.mean((z_e - jax.lax.stop_gradient(z_q))  2)
loss_total    = loss_codebook + beta * loss_commit
print(f"码本损失：{loss_codebook:.4f}, 承诺损失：{loss_commit:.4f}")

# 码本利用率
unique, counts = jnp.unique(indices, return_counts=True, size=K, fill_value=-1)
plt.figure(figsize=(10, 4))
plt.bar(range(K), counts, color='#3498db', alpha=0.8)
plt.xlabel('码本索引'); plt.ylabel('分配计数')
plt.title(f'码本利用率 ({jnp.sum(counts > 0)}/{K} 个条目已使用)')
plt.grid(True, alpha=0.3); plt.tight_layout(); plt.show()
# 尝试：增加 K 到 512 并观察坍塌。然后添加码本重置逻辑。
```

2. 构建玩具 2D 向量量化器，学习平铺 2D 分布。生成随机 2D 点，通过 EMA 更新学习码本，可视化 Voronoi 区域。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# 从高斯混合生成 2D 数据
key = jax.random.PRNGKey(0)
n_points = 2000
K = 16  # 码本条目
gamma = 0.99  # EMA 衰减

# 四个簇
keys = jax.random.split(key, 5)
centres = jnp.array([[2, 2], [-2, 2], [-2, -2], [2, -2]], dtype=jnp.float32)
data = jnp.concatenate([
    jax.random.normal(keys[i], (n_points // 4, 2)) * 0.5 + centres[i]
    for i in range(4)
])

# 从随机数据点初始化码本
idx = jax.random.choice(keys[4], n_points, (K,), replace=False)
codebook = data[idx]
ema_count = jnp.ones(K)
ema_sum = codebook.copy()

# 运行基于 EMA 的码本学习几个 epoch
for epoch in range(30):
    # 将每个点分配到最近的码本条目
    dists = jnp.sum((data[:, None, :] - codebook[None, :, :])  2, axis=2)
    assignments = jnp.argmin(dists, axis=1)
    # EMA 更新
    for k in range(K):
        mask = (assignments == k)
        count_k = jnp.sum(mask)
        ema_count = ema_count.at[k].set(gamma * ema_count[k] + (1 - gamma) * count_k)
        if count_k > 0:
            sum_k = jnp.sum(data[mask], axis=0)
            ema_sum = ema_sum.at[k].set(gamma * ema_sum[k] + (1 - gamma) * sum_k)
    codebook = ema_sum / ema_count[:, None]

# 可视化分配和码本
fig, ax = plt.subplots(1, 1, figsize=(8, 8))
colors = plt.cm.tab20(jnp.linspace(0, 1, K))
for k in range(K):
    mask = assignments == k
    ax.scatter(data[mask, 0], data[mask, 1], c=[colors[k]], s=5, alpha=0.3)
ax.scatter(codebook[:, 0], codebook[:, 1], c='black', s=120, marker='X',
           edgecolors='white', linewidths=1.5, zorder=10, label='码本')
ax.set_title(f'在 2D 数据上学习的 VQ 码本 ({K} 个条目)')
ax.legend(); ax.set_aspect('equal'); ax.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()
# 尝试：增加 K 到 64 并观察更精细的平铺。减小 gamma 并观察不稳定性。
```

3. 演示残差量化：用 $T$ 个连续量化阶段编码一批向量，测量重构误差如何随每个级别降低。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

key = jax.random.PRNGKey(7)
d = 16         # 嵌入维度
K = 32         # 每层码本大小
T = 8          # 残差级别数
n_vectors = 512

# 要量化的随机数据
k1, *cb_keys = jax.random.split(key, T + 1)
z = jax.random.normal(k1, (n_vectors, d))

# 每层独立的随机码本
codebooks = [jax.random.normal(cb_keys[t], (K, d)) * (0.5  t)
             for t in range(T)]

# 残差量化循环
residual = z.copy()
z_hat = jnp.zeros_like(z)
errors = []

for t in range(T):
    cb = codebooks[t]
    dists = (jnp.sum(residual  2, axis=1, keepdims=True)
             - 2 * residual @ cb.T
             + jnp.sum(cb  2, axis=1, keepdims=True).T)
    indices = jnp.argmin(dists, axis=1)
    z_q_t = cb[indices]
    z_hat = z_hat + z_q_t
    residual = residual - z_q_t
    mse = jnp.mean(jnp.sum((z - z_hat)  2, axis=1))
    errors.append(float(mse))
    print(f"级别 {t+1}: MSE = {mse:.4f}")

plt.figure(figsize=(8, 5))
plt.plot(range(1, T + 1), errors, 'o-', color='#e74c3c', linewidth=2, markersize=8)
plt.xlabel('残差量化级别')
plt.ylabel('重构 MSE')
plt.title('残差量化的误差减少')
plt.xticks(range(1, T + 1)); plt.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()
# 尝试：使用单个大小为 K*T 的码本并与 RQ 比较。哪个赢？
```

4. 模拟简单 1D"视频分词器"：生成 1D 信号序列（模仿视频帧），应用因果时间压缩，并在重构质量方面与非因果压缩比较。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

key = jax.random.PRNGKey(99)
n_frames = 16
frame_len = 64

# 生成"视频"：高斯 bumps 在帧之间缓慢移动
x_axis = jnp.linspace(-3, 3, frame_len)
frames = jnp.stack([
    jnp.exp(-0.5 * (x_axis - (-2 + 4 * t / n_frames))  2)
    for t in range(n_frames)
])  # 形状：(n_frames, frame_len)

# 因果时间压缩：每帧的代码仅依赖于过去的帧
# 简单方法：用过去的指数衰减平均当前帧
alpha_causal = 0.6
causal_codes = jnp.zeros_like(frames)
causal_codes = causal_codes.at[0].set(frames[0])
for t in range(1, n_frames):
    causal_codes = causal_codes.at[t].set(
        alpha_causal * frames[t] + (1 - alpha_causal) * causal_codes[t - 1]
    )

# 非因果：用过去和未来平均（双边平滑）
kernel = jnp.array([0.2, 0.6, 0.2])  # 过去，当前，未来
padded = jnp.concatenate([frames[:1], frames, frames[-1:]], axis=0)
noncausal_codes = jnp.stack([
    kernel[0] * padded[t] + kernel[1] * padded[t+1] + kernel[2] * padded[t+2]
    for t in range(n_frames)
])

# 重构误差
mse_causal = jnp.mean((frames - causal_codes)  2)
mse_noncausal = jnp.mean((frames - noncausal_codes)  2)
print(f"因果 MSE: {mse_causal:.6f}, 非因果 MSE: {mse_noncausal:.6f}")

fig, axes = plt.subplots(1, 3, figsize=(15, 5))
for ax, data, title in zip(axes,
    [frames, causal_codes, noncausal_codes],
    ['原始帧', f'因果 (MSE={mse_causal:.5f})',
     f'非因果 (MSE={mse_noncausal:.5f})']):
    ax.imshow(data, aspect='auto', cmap='viridis', origin='lower')
    ax.set_xlabel('空间位置'); ax.set_ylabel('帧索引')
    ax.set_title(title)
plt.tight_layout(); plt.show()
# 尝试：改变 alpha_causal 和内核权重。alpha=1.0 会发生什么？
```
