# 高效架构

*让模型更快不仅关于更低精度，还关于更聪明的架构每令牌做更少工作。本文件涵盖 StreamingLLM、稀疏和线性注意力、多查询和组查询注意力、推理时的专家混合、知识蒸馏、剪枝和神经架构搜索*

- 量化（文件 1）让每个操作更便宜。本文件让更少操作首先发生。两者互补：架构上高效且量化的模型可以比原始模型快 10-100 倍。

## StreamingLLM：无限长度生成

- 标准 transformers 存储 KV-cache 中所有先前令牌，它随序列长度线性增长。在某点，缓存超过 GPU 内存且生成失败。StreamingLLM (Xiao et al., 2023) 用固定大小滚动 KV-cache 解决这个问题。

- 关键观察：序列中前几个令牌获得不成比例高注意力分数无论其内容。这些称为注意力汇。如果从缓存驱逐它们，注意力分布崩溃且生成质量灾难性下降。

- StreamingLLM 的解决方案：永久在缓存中保留少量汇令牌（前 1-4 个令牌），加滚动窗口的最近 $w$ 个令牌。总缓存大小是 $\text{sink} + w$，固定无论已生成多少令牌。

$$\text{Cache} = [\text{token}_0, \text{token}_1, \text{token}_{t-w+1}, \ldots, \text{token}_t]$$

- 注意力汇锚定 softmax 分布，滚动窗口提供最近上下文。这启用无限长度生成带恒定内存，代价是丢失序列中间上下文访问。

- StreamingLLM 无需任何重新训练对自然发展注意力汇的模型工作（大多数预训练 LLM 如此）。对不这样的模型，训练期间添加单个可学习汇令牌修复它。

## 稀疏注意力

- 完整自注意力在序列长度 $n$ 上是 $O(n^2)$ 因为每个令牌关注每个其他令牌。对于 $n = 128K$，注意力矩阵有 $128K^2 = 160$ 亿条目。稀疏注意力模式通过限制哪些令牌关注哪些来减少这个。

![注意力稀疏模式：完整注意力是 O(n²)，滑动窗口是 O(n·w)，局部 + 全局添加长程令牌](../images/attention_sparsity_patterns.svg)

- 滑动窗口注意力 (Mistral, Gemma)：每个令牌仅关注前 $w$ 个令牌（例如，$w = 4096$）。注意力是 $O(n \cdot w)$ 而非 $O(n^2)$。信息通过多层传播：$L$ 层后，有效上下文是 $L \times w$。

- 局部 + 全局注意力 (Longformer, BigBird)：大多数令牌使用滑动窗口注意力（局部），但少数指定令牌（例如，[CLS]、每 512 个令牌）关注所有令牌（全局）。这捕捉局部模式和长程依赖。

- 膨胀注意力：在窗口内关注每 $k$ 个令牌，创建用相同数量注意力分数覆盖更大范围的稀疏模式。跨层增加膨胀创建类似膨胀卷积的层次模式（第 8 章）。

- 现代 LLM 的实际赢家是滑动窗口 + 完整注意力交错：一些层使用滑动窗口（便宜，处理局部上下文），一些层使用完整注意力（昂贵，捕捉长程）。Mistral/Mixtral 使用这个模式。

## 线性注意力和状态空间模型

- 我们能完全替换 $O(n^2)$ 注意力吗？线性注意力和状态空间模型 (SSM) 通过避免显式注意力矩阵以 $O(n)$ 时间处理序列。

- 线性注意力用核近似替换 softmax 注意力：

$$\text{标准：} O = \text{softmax}(QK^T / \sqrt{d}) V$$
$$\text{线性：} O = \phi(Q) (\phi(K)^T V)$$

- 通过先结合 $K^T V$ 乘积（这是 $d \times d$，独立于序列长度），计算变成 $O(n \cdot d^2)$ 而非 $O(n^2 \cdot d)$。对于长序列其中 $n \gg d$，这是巨大节省。

- RWKV 结合 RNN 和 transformers 的思想。它使用循环公式顺序处理令牌（像 RNN）但在训练期间可并行化（像 transformer）。推理每令牌 $O(1)$（恒定内存，无 KV-cache 增长）。

- Mamba (Gu & Dao, 2023) 是选择性状态空间模型。它通过学习状态转移处理序列：

$$h_t = \bar{A} h_{t-1} + \bar{B} x_t, \quad y_t = C h_t$$

- 其中 $\bar{A}$ 和 $\bar{B}$ 是输入依赖（选择性），允许 Mamba 动态聚焦或忽略输入部分。与固定 SSM 不同，选择性使 Mamba 在语言任务上与 transformers 竞争同时维持 $O(n)$ 缩放。

- 权衡：线性注意力和 SSM 对长序列更快但通常对需要精确长程检索的任务能力不如完整注意力。混合架构（一些 transformer 层 + 一些 Mamba 层）通常给出两方面最好的。

## 多查询和组查询注意力

- 标准多头注意力 (MHA，第 7 章) 对每个头使用单独 $K$、$V$ 投影。对于 $h$ 个头，这意味着 KV-cache 中 $h$ 个单独键和值张量。多查询注意力 (MQA) 和组查询注意力 (GQA) 减少这个。

- MQA (Shazeer, 2019)：所有头共享单个 $K, V$ 投影集。每个头仍有自己 $Q$ 投影。KV-cache 缩小 $h$ 倍（例如，32 头 32 倍）。

- GQA (Ainslie et al., 2023)：中间立场。头分组，每组共享一组 $K, V$ 投影。带 $h = 32$ 个头和 $g = 8$ 组，每 4 个头组共享 K/V。KV-cache 缩小 $h/g = 4$ 倍。

$$\text{MHA: } h \text{ 个头，} h \text{ 个 K/V 集} \quad \to \quad \text{GQA: } h \text{ 个头，} g \text{ 个 K/V 集} \quad \to \quad \text{MQA: } h \text{ 个头，} 1 \text{ 个 K/V 集}$$

![MHA vs GQA vs MQA：MHA 给每个头自己 KV，GQA 跨组共享 KV，MQA 对所有头用单个 KV——大幅减少 KV-cache 大小](../images/mha_gqa_mqa.svg)

- 大多数现代 LLM 使用 GQA（Llama 2/3、Gemma、Mistral）。它减少 KV-cache 内存和推理延迟与 MHA 相比可忽略质量损失。

### 多头潜在注意力 (MLA)

- MLA (DeepSeek-V2, 2024) 比 GQA 更进一步通过压缩 KV-cache 到低秩潜在空间。MLA 不缓存完整键和值向量，而是缓存每令牌压缩潜在向量 $\mathbf{c}_t$ 并在注意力期间即时重建 K/V：

$$\mathbf{c}_t = W_{\text{compress}} \cdot [\mathbf{k}_t; \mathbf{v}_t], \quad \mathbf{k}_t = W_K^{\text{up}} \cdot \mathbf{c}_t, \quad \mathbf{v}_t = W_V^{\text{up}} \cdot \mathbf{c}_t$$

- 压缩向量 $\mathbf{c}_t$ 比原始 K 和 V 组合小得多。DeepSeek-V2 实现相比 MHA 93.3% 减少 KV-cache 大小，甚至优于 MQA，同时维持 MHA 级别质量。

- 权衡：从潜在重建 K/V 对每次注意力操作添加小计算成本。但因为 LLM 解码是内存带宽受限（非计算受限），这是净赢：加载更少内存 > 每令牌稍多计算。

### Flash Attention

- Flash Attention (Dao et al., 2022，在第 16 章文件 05 详细涵盖) 不是架构改变而是实现优化，属于任何高效注意力讨论。它计算精确标准注意力带：

    - O(n) 内存 而非 O(n²)（注意力矩阵从不在 HBM 中实体化）。
    - 2-4 倍更快 比标准注意力（通过平铺和在线 softmax 保持数据在 SRAM 中）。
    - 无质量损失——输出与标准注意力数学相同。

- Flash Attention 现在是 PyTorch 中默认注意力实现 (`torch.nn.functional.scaled_dot_product_attention`)、JAX 和所有主要推理框架。如果你在 2024+ 运行注意力，你几乎肯定使用 Flash Attention。

### Ring Attention

- Ring Attention (Liu et al., 2023) 跨多个设备分布注意力计算用于即使有 Flash Attention 也太长无法放入单个 GPU 内存的序列。

- 思想：跨 $N$ 个设备划分序列。每个设备保存 $n/N$ 令牌的 Q、K、V。设备排列成环。每步中：
    1. 每个设备计算局部注意力（其 Q 对其局部 K/V）。
    2. 每个设备发送其 K/V 块到环中下一个设备。
    3. 每个设备从前一设备接收 K/V 并计算对那些的注意力。
    4. $N$ 步后，每个设备已关注每个 K/V 块。

- 通信重叠与计算：当计算当前 K/V 块的注意力时，下一个块正在传输。这几乎完全隐藏通信延迟。

- Ring Attention 启用百万令牌上下文窗口通过跨 GPU 环分布 KV-cache。每设备内存是 O(n/N)，使任意长序列可行（仅受设备数量限制）。

## 推理时的专家混合

- MoE 模型（第 7 章）每令牌仅激活其参数的一部分（通常 8 个专家中 2 个）。在推理时，独特挑战是专家缓存：所有专家必须在内存中（因为任何令牌可能路由到任何专家），但每令牌仅 2 个活跃。

- 对于 Mixtral 8x7B 模型：总参数 = 47B（8 × 7B 专家，但有共享组件）。每令牌活跃参数 ≈ 13B（2 个专家 + 共享层）。该模型有 LLM-70B 级别质量带 LLM-13B 级别推理成本，但需要 47B 参数在内存中。

- 专家卸载：对 GPU 内存受限部署，保持非活跃专家在 CPU 或 SSD 上并按需加载。这有效因为令牌路由足够可预测可以预取可能专家。

- 专家缓存：在 GPU 内存中维护最近使用专家的 LRU 缓存。如果相同专家重复激活（域内数据常见），缓存命中率高。

## 知识蒸馏

- 蒸馏（第 06 章）训练小"学生"模型模仿大"教师"。学生从教师的软预测（类上的概率分布）学习，比硬标签单独包含更多信息。

$$\mathcal{L} = \alpha \cdot \text{KL}(p_{\text{teacher}}^{T} \| p_{\text{student}}^{T}) + (1 - \alpha) \cdot \mathcal{L}_{\text{CE}}(y, p_{\text{student}})$$

- 其中 $T$ 是温度（更高 $T$ 软化分布，揭示教师的不确定性），$\alpha$ 平衡蒸馏损失与标准交叉熵损失。

- 对于 LLM：蒸馏用于从大、能力模型创建小、快模型。GPT-4 → 7B 学生捕获 GPT-4 对特定任务的大部分行为。学生服务便宜 10-100 倍。

- 任务特定蒸馏：仅在与你部署任务相关的数据上蒸馏。7B 模型在 70B 教师医学问答上蒸馏可以对该特定任务优于 70B 模型（因为学生有限容量完全聚焦目标域）。

## 剪枝

- 剪枝移除不必要权重（设它们为零），减少模型大小和计算。

- 非结构化剪枝（基于幅度）：移除最小绝对值的单个权重。这创建稀疏权重矩阵。对压缩简单有效，但当前硬件（GPU）无法高效加速稀疏操作除非稀疏性遵循特定模式。

- 结构化剪枝：移除整个单元——注意力头、MLP 神经元或层。这产生更小稠密模型在标准硬件上直接加速。权衡是更粗粒度（移除完整头可能一起移除有用和无用权重）。

- 2:4 稀疏性（NVIDIA Ampere+）：硬件支持的稀疏模式其中每 4 个权重中 2 个为零。GPU 的稀疏 Tensor Core 跳过零乘法，实现约 2 倍加速。这是当今唯一有实际硬件加速的稀疏模式。

- 彩票假设 (Frankle & Carlin, 2019)：在随机初始化网络内，存在子网络（"中奖彩票"）可以隔离训练匹配完整网络的性能。找到这些子网络（通过训练、剪枝和回退）昂贵，但洞察激发剪枝研究。

## 神经架构搜索 (NAS)

- NAS 通过在可能架构空间搜索找到在硬件约束（延迟、内存、功率）下最大化准确率的架构来自动化架构设计。

- EfficientNet（第 8 章）由 NAS 发现：复合缩放规则（平衡深度、宽度、分辨率）从搜索出现，而非来自人类直觉。

- 对于推理效率，NAS 可以找到针对特定硬件目标优化的架构："在 iPhone Neural Engine 上找到 <5ms 延迟和 >80% ImageNet 准确率的模型。"搜索空间包括层类型、宽度、激活函数和注意力模式。

- 一次型网络训练单个过参数化网络并为不同部署目标提取子网络。一次训练产生云 GPU、移动 GPU 和 CPU 的模型，每个针对其目标优化。

1. 实现滑动窗口注意力并与完整注意力比较内存使用。
```python
import jax
import jax.numpy as jnp

def full_attention(Q, K, V):
    """标准 O(n^2) 注意力。"""
    scores = Q @ K.T / jnp.sqrt(Q.shape[-1])
    weights = jax.nn.softmax(scores, axis=-1)
    return weights @ V

def sliding_window_attention(Q, K, V, window_size=128):
    """滑动窗口注意力：每个令牌关注前 window_size 个令牌。"""
    n = Q.shape[0]
    d = Q.shape[-1]
    output = jnp.zeros_like(Q)

    for i in range(n):
        start = max(0, i - window_size + 1)
        k_window = K[start:i+1]
        v_window = V[start:i+1]
        scores = Q[i] @ k_window.T / jnp.sqrt(d)
        weights = jax.nn.softmax(scores)
        output = output.at[i].set(weights @ v_window)

    return output

n, d = 512, 64
key = jax.random.PRNGKey(0)
Q = jax.random.normal(key, (n, d))
K = jax.random.normal(jax.random.PRNGKey(1), (n, d))
V = jax.random.normal(jax.random.PRNGKey(2), (n, d))

print(f"完整注意力内存：    O(n^2) = {n*n} 条目")
print(f"窗口 (w=128) 内存：   O(n*w) = {n*128} 条目")
print(f"减少：{n*n / (n*128):.1f}x")
```

2. 比较 MHA、GQA 和 MQA 的 KV-cache 大小。显示为什么 GQA 是实际最佳点。
```python
def kv_cache_size(n_heads, n_kv_heads, d_head, seq_len, bytes=2):
    """KV-cache 大小以 MB 计。"""
    return 2 * n_kv_heads * d_head * seq_len * bytes / 1e6

n_heads = 32
d_head = 128
seq_len = 32768

mha = kv_cache_size(n_heads, n_heads, d_head, seq_len)       # 32 KV 头
gqa = kv_cache_size(n_heads, 8, d_head, seq_len)              # 8 KV 头
mqa = kv_cache_size(n_heads, 1, d_head, seq_len)              # 1 KV 头

print(f"MHA (32 KV 头): {mha:.0f} MB 每层")
print(f"GQA (8 KV 头):  {gqa:.0f} MB 每层 ({mha/gqa:.0f}x 更小)")
print(f"MQA (1 KV 头):   {mqa:.0f} MB 每层 ({mha/mqa:.0f}x 更小)")
```

3. 通过从随机注意力层移除最不重要注意力头并测量输出变化来模拟结构化剪枝。
```python
import jax
import jax.numpy as jnp

key = jax.random.PRNGKey(0)
n_heads, seq_len, d_head = 8, 64, 32

# 随机多头注意力输出（每头一个）
head_outputs = jax.random.normal(key, (n_heads, seq_len, d_head))

# 完整输出：连接所有头
full_output = head_outputs.reshape(seq_len, n_heads * d_head)

# 重要性：通过范数测量每头的贡献
head_norms = jnp.linalg.norm(head_outputs, axis=(1, 2))
print("头重要性（按范数）：", jnp.round(head_norms, 2))

# 剪枝最不重要头
for n_keep in [8, 6, 4, 2]:
    top_heads = jnp.argsort(head_norms)[-n_keep:]
    pruned = head_outputs[top_heads].reshape(seq_len, n_keep * d_head)

    # 填充到原始大小用于比较（置零剪枝头）
    full_pruned = jnp.zeros_like(head_outputs)
    full_pruned = full_pruned.at[top_heads].set(head_outputs[top_heads])
    full_pruned = full_pruned.reshape(seq_len, n_heads * d_head)

    error = jnp.linalg.norm(full_output - full_pruned) / jnp.linalg.norm(full_output)
    print(f"保留 {n_keep}/{n_heads} 头：相对误差 = {error:.4f}, "
          f"内存 = {n_keep/n_heads:.0%}")
```
