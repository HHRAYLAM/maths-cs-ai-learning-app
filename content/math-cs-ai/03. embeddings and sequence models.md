## 🎯 Word2Vec - 用预测学习嵌入

Word2Vec 通过简单预测任务训练浅层神经网络学习词嵌入：

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 两种架构

| 架构 | 输入 | 预测目标 | 特点 |
|------|------|---------|------|
| CBOW | 上下文词 | 中心词 | 快、对高频词好 |
| Skip-gram | 中心词 | 上下文词 | 对稀有词好 |

### CBOW（连续词袋）

```
上下文："the cat ___ on the"
         ↓ 平均嵌入
    向量表示
         ↓
    预测 "sat"

从周围词预测中心词！
```

### Skip-gram

```
中心词："sat"
         ↓
预测每个上下文词：
"the"、"cat"、"on"、"the"

从中心词预测周围词！
```

</div>

```
架构对比：

CBOW：
  [the] [cat] [on] [the]
      \   |   /   /
        平均
          ↓
       [sat] ← 预测

Skip-gram：
       [sat] ← 输入
      /  |  \  \
   [the][cat][on][the] ← 预测

CBOW 多对一，Skip-gram 一对多！
```

### 负采样 - 加速训练

问题：完整 softmax 需要对所有词汇求和（10 万 + 词），太慢！

解决：负采样 — 转化为二分类问题：

$$\mathcal{L} = \log \sigma(v_{w_O}^T v_{w_I}) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n} [\log \sigma(-v_{w_i}^T v_{w_I})]$$

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的理解</div>
  <div class="story-card-content">
    <strong>完整 softmax</strong>：区分所有 10 万词
    <br><br>
    <strong>负采样</strong>：
    - 正样本：真实的上下文词
    <br>
    - 负样本：随机采样的噪声词
    <br><br>
    只需更新少数词（1 正 + k 负），快得多！
  </div>
</div>

### Word2Vec 为什么有效？

理论解释（Levy & Goldberg, 2014）：

带负采样的 Skip-gram 隐式分解移位 PMI 矩阵：

$$v_w^T v_c \approx \text{PMI}(w, c) - \log k$$

其中 $\text{PMI}(w, c) = \log \frac{P(w, c)}{P(w) P(c)}$

| PMI 值 | 含义 |
|--------|------|
| PMI > 0 | 共现比偶然多 → 语义相关 |
| PMI = 0 | 独立出现 |
| PMI < 0 | 共现比偶然少 |

结论：Word2Vec 在做与经典分布语义（如 LSA）相同的事，但更可扩展！

### 向量算术 - 类比推理

```
Word2Vec 最神奇的性质：

king - man + woman ≈ queen

可视化：
     man ──────→ king
      │           │
      │ "royalty" │ "royalty"
      ↓           ↓
    woman ────→ queen

语义关系 = 向量方向！
```

更多类比：

| 类比 | 向量运算 |
|------|---------|
| 巴黎 - 法国 + 意大利 | ≈ 罗马 |
| 跑 - 人 + 狗 | ≈ 追 |
| 大 - 小 + 高 | ≈ 低（反义关系） |

## 🌍 GloVe - 全局共现矩阵分解

GloVe（Global Vectors）采用不同方法：

<div class="lesson-two-column">
  <div class="lesson-text-column">

### GloVe vs Word2Vec

| 特性 | Word2Vec | GloVe |
|------|---------|-------|
| 学习对象 | 局部上下文窗口 | 全局共现矩阵 |
| 方法 | 预测任务 | 矩阵分解 |
| 训练 | 在线 SGD | 批优化 |
| 效果 | 相似质量 | 相似质量 |

### GloVe 目标

构建共现矩阵 $X$，$X_{ij}$ = 词 $j$ 出现在词 $i$ 上下文的次数：

$$w_i^T \tilde{w}_j + b_i + \tilde{b}_j = \log X_{ij}$$

损失函数：

$$\mathcal{L} = \sum_{i,j} f(X_{ij}) (w_i^T \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij})^2$$

- $f(X_{ij})$：截断函数，防止高频共现主导

</div>
  <div class="lesson-visual-column">

```
GloVe 流程：

1. 构建共现矩阵 X：
         the  cat  sat  on
   the    -    5    2    3
   cat    5    -    4    1
   sat    2    4    -    6
   on     3    1    6    -

2. 学习嵌入使点积 ≈ log 共现

3. 结果：共现多的词向量靠近！
```

## ⚡ FastText - 子词嵌入

FastText 将词表示为字符 n-gram 袋：

```
词 "where"，n=3：

<wh, whe, her, ere, re> + <where>（整词）

词的嵌入 = 所有 n-gram 嵌入的和
```

## 🔄 RNN 用于语言建模

RNN 语言模型逐词读取并预测下一个词：

```
序列："The cat sat on the ___"

t=1: 输入"The" → h₁ → 预测"cat"
t=2: 输入"cat" → h₂ → 预测"sat"
t=3: 输入"sat" → h₃ → 预测"on"
t=4: 输入"on" → h₄ → 预测"the"
t=5: 输入"the" → h₅ → 预测"mat"

隐藏状态 h_t 压缩历史 w₁...w_t
```

问题：固定大小隐藏状态 → 信息逐渐丢失！

### 双向 RNN

双向 RNN 从两个方向处理序列：

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 结构

```
前向 RNN：从左到右
   → h₁ → h₂ → h₃ → ...

后向 RNN：从右到左
   ← h₁ ← h₂ ← h₃ ← ...

输出：连接两个方向
   h_t = [h_t→ ; h_t←]
```

### 应用

| 任务 | 为什么用双向 |
|------|------------|
| POS 标注 | 词性依赖前后文 |
| NER | 实体识别需要完整上下文 |
| 情感分析 | 整句理解 |

不能用于语言建模：预测未来时不能偷看！

</div>
  <div class="lesson-visual-column">

```
双向 RNN 可视化：

时间：  1     2     3     4
       ↓     ↓     ↓     ↓  前向
  → h₁ → h₂ → h₃ → h₄ →
       
  ← h₁ ← h₂ ← h₃ ← h₄ ←
       ↑     ↑     ↑     ↑
      后向

输出：[h₁→;h₁←] [h₂→;h₂←] [h₃→;h₃←] [h₄→;h₄←]

每个位置都有完整上下文！
```

### 深度堆叠 RNN

```
单层 RNN：
输入 → RNN 层 1 → 输出

双层 RNN：
输入 → RNN 层 1 → RNN 层 2 → 输出

三层 RNN：
输入 → RNN 层 1 → RNN 层 2 → RNN 层 3 → 输出
```

典型深度：2-4 层

超过 4 层的问题：
- 梯度消失
- 过拟合

解决：残差连接

## 🔄 Seq2Seq - 序列到序列

Seq2Seq 将可变长度输入映射到可变长度输出：

<div class="story-card">
  <div class="story-card-title">🐱 Tom 的 Seq2Seq</div>
  <div class="story-card-content">
    <strong>编码器</strong>：读取输入序列，压缩为上下文向量
    <br><br>
    <strong>解码器</strong>：从上下文向量生成输出序列
    <br><br>
    <strong>应用</strong>：机器翻译、文本摘要、对话系统
    <br><br>
    <strong>例子</strong>：法语→英语翻译
  </div>
</div>

---

### 编码器 - 解码器架构

```
翻译："Le chat est noir" → "The cat is black"

编码器（读取法语）：
   输入："Le" "chat" "est" "noir"
           ↓     ↓     ↓     ↓
         h₁ → h₂ → h₃ → h₄ (最终状态=上下文向量)
                           ↓
解码器（生成英语）：
         上下文向量
             ↓
   输出："<EOS>" "The" "cat" "is" "black" "<EOS>"
```

技巧：反转输入序列（"noir est chat Le"）缩短梯度路径！

---

### 瓶颈问题

```
短句子：
"Le chat est noir" → 单向量 ✓ 可以捕捉

长句子：
"Le chat noir est assis sur le tapis rouge devant la cheminée..."
→ 单向量 ✗ 信息丢失！

需要更好的方法...
```

## 🎯 注意力机制 - 动态查看输入

注意力让解码器在每个时间步"回看"编码器的相关部分：

<div class="lesson-two-column">
  <div class="lesson-text-column">

### Bahdanau 注意力（加性）

$$e_{ti} = v^T \tanh(W_s s_{t-1} + W_h h_i)$$

$$\alpha_{ti} = \frac{\exp(e_{ti})}{\sum_j \exp(e_{tj})}$$

$$c_t = \sum_i \alpha_{ti} h_i$$

- $e_{ti}$：解码器状态 $s_{t-1}$ 与编码器状态 $h_i$ 的对齐分数
- $\alpha_{ti}$：注意力权重（softmax 归一化）
- $c_t$：上下文向量（加权和）

### Luong 注意力（乘性）

| 变体 | 分数计算 |
|------|---------|
| dot | $e_{ti} = s_t^T h_i$ |
| general | $e_{ti} = s_t^T W h_i$ |
| concat | 类似 Bahdanau |

更快：矩阵乘法 vs 前馈网络！

</div>
  <div class="lesson-visual-column">

```
注意力可视化（热图）：

源句：Le   chat  est  noir
译句：
The   ██   ▓▓   ░░   ░░
cat   ▓▓   ██   ░░   ░░
is    ░░   ▓▓   ██   ░░
black ░░   ▓▓   ▓▓   ██

黑色 = 高注意力
每行显示生成该词时关注源句哪些部分

"black" 关注 "noir"（语义对应）！
```

### 注意力 vs 固定上下文

```
无注意力（固定上下文）：
编码器 → 单向量 → 解码器所有步骤
         ↑
      信息瓶颈

有注意力（动态上下文）：
编码器 → [h₁, h₂, h₃, h₄]
            ↓ 不同权重
解码器 t=1: c₁ = 0.7h₁ + 0.2h₂ + 0.1h₃ + 0h₄
解码器 t=2: c₂ = 0.1h₁ + 0.6h₂ + 0.2h₃ + 0.1h₄
...

每步不同的上下文向量！
```

## 🔍 解码策略

### 贪心解码

```
每步选择最高概率词：

t=1: P("The")=0.9, P("A")=0.1 → 选"The"
t=2: P("cat")=0.8, P("dog")=0.2 → 选"cat"
...

问题：局部最优≠全局最优！
```

### 束搜索（Beam Search）

束搜索维护前 $k$ 个部分序列：

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的理解</div>
  <div class="story-card-content">
    <strong>束宽 k=1</strong>：贪心解码

### 束搜索过程

```
束宽 k=2：

步骤 1:
"The" (0.9), "A" (0.1)
保留：["The"], ["A"]

步骤 2:
"The cat" (0.9×0.8=0.72)
"The dog" (0.9×0.2=0.18)
"A cat" (0.1×0.7=0.07)
"A dog" (0.1×0.3=0.03)
保留：["The cat"], ["The dog"]

步骤 3:
继续扩展并保留前 2 个...

最终选择总分最高的序列！
```

## 📡 TextCNN - 用 CNN 处理文本

TextCNN 用 1D 卷积检测局部 n-gram 模式：

<div class="lesson-two-column">
  <div class="lesson-text-column">

### TextCNN 架构

```
输入嵌入（序列长度 n，嵌入维度 d）
    ↓
多个并行卷积滤波器：
- 滤波器 1：宽度 3，检测 3-gram
- 滤波器 2：宽度 4，检测 4-gram
- 滤波器 3：宽度 5，检测 5-gram
    ↓
每个滤波器输出特征图
    ↓
时间上最大池化（取每个特征图的最大值）
    ↓
连接所有池化特征
    ↓
全连接分类器
```

### 优点

- 快：并行处理
- 有效：情感分析等任务

### 缺点

- 无法捕捉长程依赖
- 滤波器宽度有限

</div>
  <div class="lesson-visual-column">

```
TextCNN 可视化：

嵌入矩阵（n 行×d 列）
    ↓
┌───────────┐
│ 滤波器 1   │ 宽度 3
└───────────┘
    ↓ 最大池化
   [f₁]

┌─────────────┐
│ 滤波器 2     │ 宽度 4
└─────────────┘
    ↓ 最大池化
   [f₂]

连接：[f₁; f₂; ...] → 分类

每个滤波器检测一种 n-gram 模式！
```

## 🎭 ELMo - 上下文嵌入

ELMo（Embeddings from Language Models）生成上下文相关的词表示：

```
传统嵌入（Word2Vec、GloVe）：
"bank" → 单个向量 v（无论上下文）

ELMo 嵌入：
"I deposited money at the bank" → v₁（金融机构）
"We sat by the river bank" → v₂（河岸）

相同词，不同向量！
```

---

### ELMo 结构

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 双向 LSTM 语言模型

```
前向 LSTM：从左到右，预测下一个词
后向 LSTM：从右到左，预测前一个词

在每位置 k，组合所有 L 层隐藏状态：

ELMoₖ = γ Σⱼ sⱼ hₖ,ⱼ

- hₖ,ⱼ：位置 k、层 j 的隐藏状态
- sⱼ：学习权重（softmax 归一化）
- γ：任务特定缩放因子
```

### 层次表示

| 层 | 捕获信息 |
|----|---------|
| 低层 | 语法（POS、形态） |
| 高层 | 语义（词义、角色） |

优势：通过混合所有层，适应不同任务！

</div>
  <div class="lesson-visual-column">

```
ELMo 流程：

输入："I deposited money at the bank"
    ↓
前向 LSTM：→ h₁ → h₂ → h₃ → h₄ → h₅
后向 LSTM：← h₁ ← h₂ ← h₃ ← h₄ ← h₅
    ↓
组合所有层（加权）
    ↓
输出：ELMo₁, ELMo₂, ELMO₃, ELMo₄, ELMo₅

每个词都有上下文相关表示！
```

### 预训练-微调范式

ELMo 开启的范式：

```
步骤 1：预训练
在大規模无标签文本上训练语言模型
（如 Wikipedia + BooksCorpus）

步骤 2：提取表示
用预训练模型为下游任务生成嵌入

步骤 3：微调（可选）
在任务数据上轻度调整模型参数

这种范式后来被 BERT、GPT 进一步发展！
```

## 📝 本课总结

<div class="key-point">
  <div class="key-point-title">🎓 知识回顾</div>
  <ul class="key-point-list">
    <li><strong>Word2Vec</strong>：CBOW（多对一）、Skip-gram（一对多）、负采样</li>
    <li><strong>GloVe</strong>：全局共现矩阵分解</li>
    <li><strong>FastText</strong>：字符 n-gram 袋，处理未知词</li>
    <li><strong>向量算术</strong>：语义关系 = 向量平移（king - man + woman = queen）</li>
    <li><strong>RNN</strong>：序列建模，隐藏状态压缩历史</li>
    <li><strong>双向 RNN</strong>：完整上下文，用于标注任务</li>
    <li><strong>Seq2Seq</strong>：编码器 - 解码器，处理变长序列</li>
    <li><strong>注意力</strong>：Bahdanau（加性）、Luong（乘性），动态上下文</li>
    <li><strong>束搜索</strong>：维护前 k 个序列，平衡质量与速度</li>
    <li><strong>TextCNN</strong>：1D 卷积检测 n-gram 模式</li>
    <li><strong>ELMo</strong>：双向 LSTM，上下文嵌入，预训练范式</li>
  </ul>
</div>
