# 分布式深度学习

*分布式训练将计算分散到多个 GPU 和机器上，训练对单个设备来说太大或太慢的模型。本文件涵盖混合精度、数据并行、模型并行、流水线并行、ZeRO、FSDP、张量并行和通信原语如 all-reduce，这些对于大规模训练 LLM 至关重要。*

- 在单个 GPU 上训练大型神经网络最终会碰到墙。模型可能不适合内存，或者训练可能需要数月。分布式训练将工作分散到多个设备（GPU、TPU 或整台机器）以更快地训练和训练更大的模型。本文件涵盖使这成为可能的技术。

- 理解为什么分布很重要，从训练的**计算成本**开始。通过具有 $d_{\text{in}}$ 输入和 $d_{\text{out}}$ 输出的稠密层的单个前向传递在 $B$ 个示例的批次上大约需要 $2 \cdot B \cdot d_{\text{in}} \cdot d_{\text{out}}$ FLOPs（浮点运算）：输出矩阵的每个元素一次乘法和一次加法。反向传递成本大约是前向传递的两倍（计算关于输入和权重的梯度），所以稠密层上的一个训练步骤大约是 $6 \cdot B \cdot d_{\text{in}} \cdot d_{\text{out}}$ FLOPs。

- 对于具有隐藏维度 $d$ 的 transformer 层，自注意力块涉及四个投影（Q、K、V 和输出），每个成本 $O(B \cdot n \cdot d^2)$ FLOPs（其中 $n$ 是序列长度），加上注意力矩阵计算 $O(B \cdot n^2 \cdot d)$。前馈块有两个稠密层，通常扩展到 $4d$ 并返回：$O(B \cdot n \cdot 8d^2)$。每层总计：大约 $O(B \cdot n \cdot 12d^2 + B \cdot n^2 \cdot d)$。乘以层数你就明白为什么训练 GPT 规模模型需要数千 GPU 小时。

- **内存墙**通常是更紧的约束。训练期间，GPU 内存必须同时容纳四件事：

![堆叠条形图显示训练内存细分：参数、梯度、优化器状态、激活](../images/training_memory_breakdown.svg)

- **参数**：模型权重。FP32（每个参数 4 字节）的 70 亿参数模型仅权重就需要 28 GB。
- **梯度**：与参数大小相同。另一个 28 GB。
- **优化器状态**：Adam 维护两个额外缓冲区（一阶和二阶矩估计），每个与参数大小相同。即使模型使用较低精度，这些也保持在 FP32 以用于数值稳定性。对于我们的 7B 模型，这是 $2 \times 28 = 56$ GB。
- **激活**：前向传递期间保存的中间值，用于反向传递。大小取决于批次大小、序列长度和模型宽度。这通常是最大的组件，随批次大小线性增长。

- 对于我们的 7B 模型和 FP32 Adam：28（参数）+ 28（梯度）+ 56（优化器）= 112 GB，甚至还没计算激活。单个 80 GB A100 GPU 无法容纳这个。这就是为什么分布式策略至关重要。

- **混合精度训练**是第一道防线。不是将所有内容存储在 FP32（32 位浮点）中，而是使用 FP16 或 BF16（16 位）进行前向和反向传递，同时在 FP32 中保持权重的主副本用于优化器更新。

- **FP16** 具有高精度（10 位尾数）但范围有限，可能导致溢出/下溢。损失缩放（在反向传递前将损失乘以大因子，然后将梯度除以相同因子）缓解这个问题。

- **BF16**（brain float）具有与 FP32 相同的指数范围（8 位指数）但精度较低（7 位尾数）。它几乎从不过出，很少需要损失缩放，使其更简单使用。BF16 是现代 transformer 训练的默认值。

- 混合精度大约将激活和梯度的内存减半（前向/反向传递期间的 dominant 成本），同时在 FP32 中保持优化器状态以用于数值稳定性。

- **数据并行**是最简单的分布式策略。你在 $N$ 个 GPU 上复制整个模型，将每个小批次分成 $N$ 个相等的块，并将一个块发送到每个 GPU。每个 GPU 独立在其块上运行前向和反向传递。然后梯度在所有 GPU 上平均（使用 all-reduce 操作），每个 GPU 更新其模型的本地副本。

- 从模型的角度来看，这等同于用 $N$ 倍大的小批次训练。如果每个 GPU 处理大小为 $B$ 的批次，有效批次大小是 $N \cdot B$。

![并排比较：数据并行复制模型并分割数据，模型并行分割模型并共享数据](../images/data_model_parallelism.svg)

- 梯度平均可以同步或异步完成。**同步 SGD** 等待所有 GPU 完成后再平均，确保数学上等同于具有大批次的单 GPU 训练。缺点是最慢的 GPU（"straggler"）拖累所有人。

- **异步 SGD** 让每个 GPU 独立更新共享参数服务器，无需等待。这消除了 straggler 问题但引入"陈旧梯度"：GPU 可能基于稍过时的参数计算梯度。陈旧梯度增加噪声并可能减慢收敛。实践中，具有高效通信的同步 SGD 更受青睐。

- **梯度累积**是在有限硬件上模拟更大批次大小的软件技巧。不是每个小批次进行一次更新，而是运行几次前向/反向传递并累积梯度，然后进行一次更新。这给出与更大批次相同的结果，无需更多 GPU 内存用于激活（一次只有一个批次的激活在内存中）。

- 当模型本身太大无法适应单个 GPU 时，你需要**模型并行**。有两种主要风格。

- **张量并行**跨 GPU 分割单个层。大矩阵乘法 $Y = XW$ 可以按列分割：将 $W$ 分割为 $[W_1, W_2]$ 跨两个 GPU，并行计算 $Y_1 = XW_1$ 和 $Y_2 = XW_2$，然后连接。这适用于注意力投影和前馈层。它需要 GPU 之间的快速通信（通常节点内的 NVLink），因为部分结果必须在每层组合。

- **流水线并行**将不同层分配给不同 GPU。GPU 0 运行层 1-4，GPU 1 运行层 5-8，依此类推。数据像流水线一样流经管道。朴素方法有"流水线气泡"：当 GPU 0 为微批次 1 处理前向传递时，GPU 1-3 空闲。**微批次**通过将小批次分成更小的微批次按顺序流经管道来缓解这个问题，使所有 GPU 大部分时间忙碌。

- **混合并行**结合数据、张量和流水线并行。典型的大模型设置可能在节点内使用张量并行（8 个 GPU 通过快速 NVLink 连接），跨节点使用流水线并行，跨节点组使用数据并行。这就是 GPT-4 和 Llama 等模型的训练方式。

- 分布式训练的效率严重依赖**通信**。关键操作是**all-reduce**：给定 $N$ 个 GPU 上每个的值，计算总和（或平均）并将结果分发给所有 GPU。

- 朴素的 all-reduce 将所有数据发送到一个 GPU，求和，然后广播回来。这在通信中是 $O(N)$ 并在根处创建瓶颈。

- **环形 all-reduce** 更高效。将 $N$ 个 GPU 排列成环。每个 GPU 将数据分成 $N$ 块。在 $N - 1$ 步中，每个 GPU 发送一块到其邻居并从另一个邻居接收一块，累积部分和。再经过 $N - 1$ 步后，完整和传播到所有 GPU。每个 GPU 传输的总数据：$2(N-1)/N$ 倍数据大小，随着 $N$ 增长接近 $2\times$。关键是这不随 $N$ 增加，使其带宽最优。

![四个 GPU 排列成环，每个将梯度块传递给邻居直到所有都有完整和](../images/ring_allreduce.svg)

- **参数服务器**是替代架构，专用服务器节点保存模型参数。工作器计算梯度并将它们发送到服务器，服务器更新参数并发送回来。这更简单但可能在服务器处创建通信瓶颈。

- **NCCL**（NVIDIA 集体通信库）是 GPU 到 GPU 通信的标准库。它提供 all-reduce、all-gather、广播和其他集体操作的优化实现，自动为网络拓扑选择最佳算法。

- **缩放定律**描述模型性能如何随计算、数据和模型大小改进。原始 Kaplan 等人（2020）缩放定律发现损失随每个呈幂律减少：

$$L(N) \propto N^{-\alpha_N}, \quad L(D) \propto D^{-\alpha_D}, \quad L(C) \propto C^{-\alpha_C}$$

- 其中 $N$ 是参数数量，$D$ 是数据集大小，$C$ 是计算预算。

- **Chinchilla 缩放定律**（Hoffmann 等人，2022）表明大多数模型训练不足：对于给定计算预算，你应该在更多数据上训练更小模型，而非之前认为的。最优比例大约是每参数 20 token。7B 模型应该看到约 140B token，而非 Llama 1 用 65B 模型使用的 300B token。这个发现将领域转向"计算最优"训练。

- **混合专家（MoE）** 是缩放模型容量而不按比例缩放计算的架构。不是每个 transformer 层一个前馈网络，而是有 $N$ 个"专家"网络（每个是标准 FFN）。**门控网络**（路由器）检查每个 token 并将其发送到 top-$K$ 专家（通常 $K = 1$ 或 $K = 2$）。

![token 通过门控网络路由到选择的专家，带 top-K 稀疏路由和输出的加权组合](../images/moe_routing.svg)

- 总参数 count 大得多（因为有 $N$ 个专家），但每 token 的 FLOPs 保持大致恒定（因为每 token 只激活 $K$ 个专家）。例如，Mixtral 8x7B 有 47B 总参数但每个前向传递只使用约 13B，给出更大模型的性能但成本更小。

- MoE 引入挑战。**负载平衡**：如果路由器将大多数 token 发送到同一专家，其他专家被浪费。辅助损失鼓励均匀路由。**通信**：不同专家可能位于不同 GPU 上，所以路由 token 需要 all-to-all 通信，这很昂贵。

- **容错**在训练运行持续数周或数月数千 GPU 时至关重要。如果单个 GPU 故障，你不想丢失所有进度。**检查点**定期保存模型权重、优化器状态和训练状态（学习率、步数、数据位置）到磁盘。如果发生故障，你从最后一个检查点重新启动。

- **梯度检查点**（也称为激活重计算）是内存优化，而非容错机制。前向传递期间，不是为反向传递保存所有激活，你只在某些检查点保存激活。反向传递期间，你从检查点重新计算缺失的激活。这是用计算换内存：它将前向传递成本增加约 33% 但可以将激活内存减少 $\sqrt{L}$ 因子（其中 $L$ 是层数）。

- 综合在一起，训练前沿模型结合所有这些技术：BF16 混合精度、带环形 all-reduce 的数千 GPU 数据并行、节点内张量并行、跨节点流水线并行、减少内存的梯度检查点、用于参数效率的 MoE，以及用于容错的定期检查点。系统工程与算法设计一样具有挑战性。

- 总结分布式训练工具包：

| 技术 | 作用 | 权衡 |
|---|---|---|
| 混合精度 (BF16) | 激活/梯度内存减半 | 轻微数值差异 |
| 数据并行 | 跨 GPU 缩放批次大小 | 梯度同步的通信开销 |
| 张量并行 | 跨 GPU 分割层 | 需要快速互连 |
| 流水线并行 | 跨 GPU 分割模型阶段 | 流水线气泡（浪费计算） |
| 梯度累积 | 模拟大批次 | 更慢（多次前向/反向传递） |
| 梯度检查点 | 减少激活内存 | 约 33% 更多计算 |
| 环形 all-reduce | 高效梯度平均 | 大模型的带宽限制 |
| MoE | 更多容量，相同 FLOPs | 负载平衡、路由复杂性 |
| 缩放定律 | 指导计算分配 | 经验的，可能不适用于所有规模 |

## 编码任务（使用 CoLab 或 notebook）

1. 计算 transformer 层的 FLOPs 和内存需求。给定隐藏维度 $d$、序列长度 $n$、批次大小 $B$ 和层数，估计总训练成本。
```python
import jax.numpy as jnp

def transformer_layer_flops(d, n, B):
    """Approximate FLOPs for one transformer layer forward pass."""
    # QKV projections: 3 * (B * n * d * d) * 2 (multiply-add)
    qkv_flops = 3 * 2 * B * n * d * d
    # Attention: (B * n * n * d) * 2 for QK^T, (B * n * n * d) * 2 for attn*V
    attn_flops = 2 * 2 * B * n * n * d
    # Output projection: (B * n * d * d) * 2
    out_flops = 2 * B * n * d * d
    # FFN: two layers, d->4d and 4d->d: 2 * (B * n * d * 4d) * 2
    ffn_flops = 2 * 2 * B * n * d * 4 * d
    return qkv_flops + attn_flops + out_flops + ffn_flops

def transformer_layer_memory(d, n, B, dtype_bytes=2):
    """Approximate activation memory (bytes) for one layer."""
    # QKV: 3 * B * n * d
    qkv_mem = 3 * B * n * d * dtype_bytes
    # Attention weights: B * heads * n * n (approx B * n * n * sizeof)
    attn_mem = B * n * n * dtype_bytes
    # FFN intermediate: B * n * 4d
    ffn_mem = B * n * 4 * d * dtype_bytes
    return qkv_mem + attn_mem + ffn_mem

# Example: GPT-2 scale
d, n, B, L = 1024, 1024, 8, 24
fwd_flops = transformer_layer_flops(d, n, B)
total_flops = 3 * L * fwd_flops  # 3x for forward + backward
act_mem = L * transformer_layer_memory(d, n, B)
param_count = L * (12 * d * d + 13 * d)  # approximate

print(f"Model: d={d}, n={n}, B={B}, L={L}")
print(f"Parameters: {param_count / 1e6:.0f}M")
print(f"FLOPs per step: {total_flops / 1e12:.2f} TFLOPs")
print(f"Activation memory: {act_mem / 1e9:.2f} GB (BF16)")
print(f"Parameter memory (FP32): {param_count * 4 / 1e9:.2f} GB")
print(f"Adam optimizer memory: {param_count * 8 / 1e9:.2f} GB")
print(f"Total training memory: {(param_count * 16 + act_mem) / 1e9:.2f} GB")
```

2. 模拟数据并行训练。将数据集分割到多个"虚拟 GPU"，独立计算梯度，平均它们，并验证结果匹配单 GPU 训练。
```python
import jax
import jax.numpy as jnp

# Simple linear model: y = wx + b
key = jax.random.PRNGKey(0)
X = jax.random.normal(key, (64, 4))
w_true = jnp.array([1.0, -2.0, 3.0, 0.5])
y = X @ w_true + 0.1 * jax.random.normal(key, (64,))

def loss_fn(w, X, y):
    return jnp.mean((X @ w - y) ** 2)

grad_fn = jax.grad(loss_fn)

# Single GPU: full batch gradient
w = jnp.zeros(4)
grad_single = grad_fn(w, X, y)

# Data parallel: split across 4 "GPUs"
n_gpus = 4
chunk_size = len(X) // n_gpus
grads = []
for i in range(n_gpus):
    X_chunk = X[i*chunk_size:(i+1)*chunk_size]
    y_chunk = y[i*chunk_size:(i+1)*chunk_size]
    grads.append(grad_fn(w, X_chunk, y_chunk))

# All-reduce: average gradients
grad_parallel = jnp.mean(jnp.stack(grads), axis=0)

print("Single-GPU gradient:", grad_single)
print("Data-parallel gradient (avg):", grad_parallel)
print(f"Match: {jnp.allclose(grad_single, grad_parallel, atol=1e-5)}")

# Train both and compare
w_single, w_parallel = jnp.zeros(4), jnp.zeros(4)
lr = 0.1
for step in range(100):
    w_single = w_single - lr * grad_fn(w_single, X, y)

    grads = [grad_fn(w_parallel, X[i*chunk_size:(i+1)*chunk_size],
                     y[i*chunk_size:(i+1)*chunk_size]) for i in range(n_gpus)]
    avg_grad = jnp.mean(jnp.stack(grads), axis=0)
    w_parallel = w_parallel - lr * avg_grad

print(f"\nAfter 100 steps:")
print(f"Single-GPU weights: {w_single}")
print(f"Data-parallel weights: {w_parallel}")
print(f"Max difference: {jnp.max(jnp.abs(w_single - w_parallel)):.2e}")
```

3. 实现简单混合专家层。创建门控网络将 token 路由到 top-K 专家并组合它们的输出。
```python
import jax
import jax.numpy as jnp

def expert_fn(x, W1, b1, W2, b2):
    """Simple 2-layer FFN expert."""
    h = jnp.maximum(0, x @ W1 + b1)  # ReLU
    return h @ W2 + b2

def moe_layer(x, gate_W, experts_params, top_k=2):
    """
    MoE forward pass.
    x: (batch, d_model)
    gate_W: (d_model, n_experts)
    experts_params: list of (W1, b1, W2, b2) per expert
    """
    n_experts = len(experts_params)

    # Gating: compute routing scores
    gate_logits = x @ gate_W  # (batch, n_experts)
    gate_probs = jax.nn.softmax(gate_logits, axis=-1)

    # Top-K selection
    top_k_indices = jnp.argsort(-gate_probs, axis=-1)[:, :top_k]
    top_k_probs = jnp.take_along_axis(gate_probs, top_k_indices, axis=-1)
    # Renormalise
    top_k_probs = top_k_probs / jnp.sum(top_k_probs, axis=-1, keepdims=True)

    # Compute expert outputs (simplified: run all experts, mask later)
    expert_outputs = jnp.stack([
        expert_fn(x, *experts_params[i]) for i in range(n_experts)
    ], axis=1)  # (batch, n_experts, d_model)

    # Gather top-K expert outputs and weight them
    batch_idx = jnp.arange(x.shape[0])[:, None]
    selected_outputs = expert_outputs[batch_idx, top_k_indices]  # (batch, top_k, d_model)
    output = jnp.sum(selected_outputs * top_k_probs[:, :, None], axis=1)

    return output, gate_probs

# Setup
key = jax.random.PRNGKey(42)
batch, d_model, d_ff, n_experts = 8, 16, 32, 4

# Initialise experts
experts_params = []
for i in range(n_experts):
    k1, k2, key = jax.random.split(key, 3)[0], jax.random.split(key, 3)[1], jax.random.split(key, 3)[2]
    experts_params.append((
        jax.random.normal(k1, (d_model, d_ff)) * 0.1,
        jnp.zeros(d_ff),
        jax.random.normal(k2, (d_ff, d_model)) * 0.1,
        jnp.zeros(d_model),
    ))

key, subkey = jax.random.split(key)
gate_W = jax.random.normal(subkey, (d_model, n_experts)) * 0.1
x = jax.random.normal(key, (batch, d_model))

output, gate_probs = moe_layer(x, gate_W, experts_params, top_k=2)

print(f"Input shape: {x.shape}")
print(f"Output shape: {output.shape}")
print(f"Gate probabilities (first sample): {gate_probs[0]}")
print(f"Expert usage (avg across batch):")
for i in range(n_experts):
    usage = jnp.mean(gate_probs[:, i])
    print(f"  Expert {i}: {usage:.3f}")
```
