## 为什么是模式而非记忆

- LeetCode、NeetCode 和 HackerRank 上有数千道编程题。没有人能记住所有题目，试图记住也是必输的策略。面试官不会从固定列表中选择问题，他们会修改、组合和伪装问题。死记硬背"两数之和"的解法在面试官提出你从未见过的变体时将毫无帮助。

- 好消息是：只有大约**15-20 个核心模式**（双指针、滑动窗口、BFS/DFS、动态规划、回溯等）。每道问题，无论表面看起来多么新颖，都可以归结为这些模式之一或其组合。面试不是在测试你是否见过这道具体题目，而是在测试你能否**剥离上下文**、故事、具体数据类型、边界情况，并识别底层模式。

- 考虑这三道问题：
    - "在数组中找到两个数，使它们的和等于目标值。"
    - "找到两个分子，使它们的结合能之和等于阈值。"
    - "给定账户余额列表，找到两个账户，使它们的总和等于一笔债务。"

- 它们看起来不同，但其实是同一个问题：**两数之和**。上下文（数字、分子、账户）无关紧要。结构是：在集合中搜索补数 → 哈希表查找。

- 这就是为什么本章通过直觉教授**模式**，而非通过重复教授解法。对于每个模式，我们都会解释：
    - **问题的什么结构性质**提示使用该模式（有序输入 → 双指针；子数组约束 → 滑动窗口；最优子结构 + 重叠子问题 → 动态规划）。
    - **为什么该模式有效** —— 数学或逻辑推理，而不仅仅是"它能给出正确答案"。
    - **如何适配它** —— 通过展示相同核心思想应用于不同上下文的简单、中等和困难变体。

- 当你深刻理解*为什么*滑动窗口有效（约束的单调性意味着扩展/收缩就足够了），你就可以将其应用于具有该结构的任何问题，即使你从未见过。当你只死记硬背"无重复字符的最长子串"的代码时，问题一旦变化你就会束手无策。

- 实际策略：
    1. **学习模式**（本章）。
    2. **练习识别**伪装问题（每个文件末尾的 NeetCode 课后作业）。
    3. **练习实现**（在时间压力下）。
    4. 面试中：阅读问题 → 剥离上下文 → 识别模式 → 实现。

## 大 O 表示法

- 当说一个算法"快"或"慢"时，我们需要一种精确的衡量方式。**大 O 表示法**描述算法的运行时（或空间使用）如何随输入大小 $n$ 增长而增长，忽略常数因子和低阶项。

- 形式化定义：$f(n) = O(g(n))$ 意味着存在常数 $c > 0$ 和 $n_0$，使得对于所有 $n \geq n_0$，都有 $f(n) \leq c \cdot g(n)$。通俗地说：对于大输入，$f$ 的增长速度不超过 $g$。

- 为什么忽略常数？因为 $2n$ 算法和 $5n$ 算法都是 $O(n)$：它们以相同方式扩展。在更快的计算机上，常数会变化，但扩展方式不变。大 O 捕捉了问题的**内在**难度，独立于硬件。

### 增长率层次结构

- 从最快到最慢：

| 大 O | 名称 | 示例 | $n = 10^6$ 时的操作数 |
|-------|------|---------|----------------------|
| $O(1)$ | 常数 | 数组访问、哈希查找 | 1 |
| $O(\log n)$ | 对数 | 二分查找 | 20 |
| $O(n)$ | 线性 | 线性扫描、单层循环 | $10^6$ |
| $O(n \log n)$ | 线性对数 | 归并排序、高效排序 | $2 \times 10^7$ |
| $O(n^2)$ | 二次 | 嵌套循环、暴力枚举对 | $10^{12}$（太慢） |
| $O(n^3)$ | 三次 | 三层嵌套循环、矩阵乘法 | $10^{18}$（极慢） |
| $O(2^n)$ | 指数 | 所有子集、暴力回溯 | $10^{301030}$（不可能） |
| $O(n!)$ | 阶乘 | 所有排列 | 荒谬 |

- **经验法则**：现代计算机每秒大约执行 $10^8$–$10^9$ 次简单操作。对于 1 秒时间限制：
    - $O(n)$ 适用于 $n \leq 10^8$
    - $O(n \log n)$ 适用于 $n \leq 10^7$
    - $O(n^2)$ 适用于 $n \leq 10^4$
    - $O(2^n)$ 适用于 $n \leq 25$

- 这个表格让你立即判断你的方法是否足够快。如果 $n = 10^5$ 而你的解法是 $O(n^2)$，那将是 $10^{10}$ 次操作 —— 太慢。你需要更好的算法。

### 如何分析大 O

- **单层循环**遍历 $n$ 个元素：$O(n)$。

```python
total = 0
for x in arr:   # n 次迭代
    total += x   # 每次迭代 O(1)
# 总计：O(n)
```

- **嵌套循环**：迭代次数相乘。

```python
for i in range(n):       # n 次迭代
    for j in range(n):   # 每次 n 次迭代
        process(i, j)    # O(1)
# 总计：O(n^2)
```

- **循环中对半缩小**：$O(\log n)$。每次迭代将问题规模减半，因此需要 $\log_2 n$ 次迭代。

```python
i = n
while i > 0:
    process(i)
    i //= 2
# 总计：O(log n)
```

- **内部循环依赖于外部循环**：

```python
for i in range(n):
    for j in range(i):   # j 从 0 到 i-1
        process(i, j)
# 总计：0 + 1 + 2 + ... + (n-1) = n(n-1)/2 = O(n^2)
```

- **递归**：写出递推关系并求解（第 13 章介绍了主定理）。例如归并排序：$T(n) = 2T(n/2) + O(n) = O(n \log n)$。

### 常见陷阱

- **隐藏的循环**：Python 中 `x in list` 是 $O(n)$（线性扫描），但 `x in set` 是 $O(1)$。在循环内对列表使用 `in` 会得到 $O(n^2)$，而非 $O(n)$。

```python
# 糟糕：O(n^2) — 在列表上用 "in" 是 O(n)
for x in arr:
    if x in another_list:
        process(x)

# 良好：O(n) — 先转换为集合
another_set = set(another_list)
for x in arr:
    if x in another_set:
        process(x)
```

- **字符串拼接**：Python 中 `s += c` 每次复制整个字符串。在 $n$ 次循环内：$O(1 + 2 + \cdots + n) = O(n^2)$。

- **排序主导**：如果算法排序（$O(n \log n)$）然后线性扫描（$O(n)$），总计是 $O(n \log n)$ —— 排序主导。

- **摊还复杂度**：某些操作偶尔昂贵但平均便宜。动态数组追加是 $O(1)$ 摊还，因为罕见的 $O(n)$ 扩容被分摊到 $n$ 次廉价追加中。不要将摊还 $O(1)$ 与最坏情况 $O(1)$ 混淆。

### 空间复杂度

- 空间复杂度遵循相同的大 O 规则，应用于内存使用而非时间。

- **原地**算法使用 $O(1)$ 额外空间（不计输入）。快速排序是 $O(\log n)$ 空间（递归栈深度）。归并排序是 $O(n)$ 空间（用于合并的临时数组）。

- **递归栈**：每次递归调用使用栈空间。$n$ 层深的递归使用 $O(n)$ 空间，即使每次调用不分配额外内存。这就是为什么在 $n$ 个节点的图上进行递归 DFS 使用 $O(n)$ 空间。

- 面试时始终同时说明时间和空间复杂度。$O(n)$ 时间、$O(n)$ 空间的解法通常可接受，但 $O(n)$ 时间、$O(1)$ 空间的解法更好。面试官可能会要求你优化其中之一。

## 递归

- **递归**是函数调用自身来求解同一问题的更小实例。它是具有递归结构问题的最自然方法：树、嵌套结构、分治和数学序列。

- 每个递归函数都有两部分：
    1. **基本情况**：可以直接求解（无需递归）的最小实例。这是停止递归的条件。
    2. **递归情况**：将问题分解为更小的子问题，递归求解它们，并组合结果。

### 示例：阶乘

```python
def factorial(n):
    if n <= 1:        # 基本情况
        return 1
    return n * factorial(n - 1)  # 递归情况
```

- `factorial(4)` 的执行过程：
    - `factorial(4)` 调用 `factorial(3)`
    - `factorial(3)` 调用 `factorial(2)`
    - `factorial(2)` 调用 `factorial(1)`
    - `factorial(1)` 返回 `1`（基本情况）
    - `factorial(2)` 返回 `2 * 1 = 2`
    - `factorial(3)` 返回 `3 * 2 = 6`
    - `factorial(4)` 返回 `4 * 6 = 24`

- 每次调用都被推入**调用栈**。栈增长直到达到基本情况，然后随着每次调用返回而展开。如果递归太深（例如 Python 中的 `factorial(1000000)`），栈会溢出（`RecursionError`）。Python 的默认递归限制是 1000。

### 如何递归思考

- 关键思维转变：**信任递归**。编写递归函数时，假设递归调用为更小的子问题返回正确答案。你的工作只是：
    1. 处理基本情况。
    2. 将问题分解为更小的部分。
    3. 组合结果。

- 你不需要在脑海中追踪每次递归调用。这就像试图通过心算执行每次迭代来理解循环。相反，验证："如果递归调用为更小的输入给出正确答案，我的组合步骤是否为完整输入给出正确答案？"

### 示例：链表上的递归

- 递归反转链表：

```python
def reverse(head):
    if not head or not head.next:   # 基本情况：0 或 1 个节点
        return head

    new_head = reverse(head.next)   # 反转其余部分
    head.next.next = head           # 将下一个节点指回我
    head.next = None                # 我现在是尾部
    return new_head
```

- **信任递归**：`reverse(head.next)` 正确反转其余部分并返回新头节点。我们只需要将当前节点附加到末尾。

### 示例：树上的递归

- 计算二叉树的高度：

```python
def height(root):
    if not root:           # 基本情况：空树高度为 0
        return 0
    left_h = height(root.left)    # 左子树高度
    right_h = height(root.right)  # 右子树高度
    return 1 + max(left_h, right_h)  # 当前节点增加 1 层
```

- 这种模式 —— "递归左子树，递归右子树，组合" —— 解决了绝大多数树问题（参见文件 03）。

### 递归 vs 迭代

- 每个递归算法都可以转换为迭代算法（使用显式栈或循环）。迭代避免了调用栈开销和栈溢出风险。

- **何时优先递归**：问题具有自然递归结构（树、嵌套数据、分治）。递归解法更清晰、更易于推理。

- **何时优先迭代**：递归深度可能非常大（例如处理 $10^6$ 个节点的链表）。迭代解法避免栈溢出。

- **尾递归**：如果递归调用是函数中的最后一个操作（递归调用返回后不执行任何工作），则称该递归调用为"尾递归"。某些语言（Scheme、Scala）优化尾调用以使用恒定栈空间。Python**不**优化尾调用，因此 Python 中的尾递归仍然使用 $O(n)$ 栈空间。

### 常见陷阱

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 缺少基本情况 | 无限递归 → 栈溢出 | 始终定义何时停止 |
| 基本情况错误 | 递归分解中的差一错误 | 用最小输入测试（0、1、2） |
| 未缩小问题 | `f(n)` 调用 `f(n)` 而非 `f(n-1)` | 确保子问题严格更小 |
| 冗余计算 | 斐波那契：`f(n) = f(n-1) + f(n-2)` 指数级重复计算 | 使用记忆化（→ 动态规划） |
| Python 递归限制 | `factorial(10000)` 崩溃 | 使用 `sys.setrecursionlimit` 或转换为迭代 |

## 回溯

- **回溯**是一种系统化的方法，通过增量构建所有可能解，并放弃不可能产生有效答案的部分解。

- 想象穿越迷宫。在每个路口，你选择一条路径。如果走进死胡同，你返回最后一个路口尝试不同的路径。你不是从头开始 —— 你**回溯**到最近的决策点。

### 三个步骤

每个回溯算法都遵循相同的模式：

1. **选择**：选择一个候选项来扩展当前部分解。
2. **探索**：递归尝试从该候选项构建完整解。
3. **取消选择**：撤销选择（回溯）并尝试下一个候选项。

```python
def backtrack(state, choices, result):
    if is_complete(state):
        result.append(state.copy())
        return

    for choice in choices:
        if is_valid(choice, state):
            state.add(choice)           # 1. 选择
            backtrack(state, choices, result)  # 2. 探索
            state.remove(choice)        # 3. 取消选择（回溯）
```

- **取消选择**步骤是回溯与普通递归的区别所在。没有它，状态会累积所有选择，你无法探索替代路径。

### 何时使用回溯

- 问题要求**枚举所有有效配置**：所有排列、所有子集、所有有效排列（如 N 皇后问题）。
- 问题要求**找到任意有效配置**：数独求解、迷宫路径查找。
- 搜索空间很大但可以**剪枝**：大多数部分解可以在不完全探索的情况下提前拒绝。

### 剪枝如何加速

- 没有剪枝时，回溯探索每种可能的组合 —— 指数级时间。**剪枝**提前剪除分支：

```python
for choice in choices:
    if not is_valid(choice, state):
        continue  # 剪枝：跳过整个子树

    state.add(choice)
    backtrack(state, choices, result)
    state.remove(choice)
```

- 在 N 皇后问题（文件 05）中，在放置皇后之前检查列和对角线冲突将搜索树从 $n^n$ 剪枝到大约 $n!$ 个候选项。对于 $n = 8$，这是 1600 万 → 4 万。良好的剪枝使指数级算法对于中等规模的 $n$ 变得实用。

### 生成所有子集（最简单的回溯）

```python
def subsets(nums):
    result = []

    def backtrack(start, path):
        result.append(path[:])  # 每个部分解都是有效子集

        for i in range(start, len(nums)):
            path.append(nums[i])        # 选择
            backtrack(i + 1, path)       # 探索（i+1：不重用）
            path.pop()                   # 取消选择

    backtrack(0, [])
    return result
```

- 对于 `[1, 2, 3]`，递归树：
    - `[]` → `[1]` → `[1,2]` → `[1,2,3]`（回溯）→ `[1,3]`（回溯）→ `[2]` → `[2,3]`（回溯）→ `[3]`

- 树中的每个节点都是对 `backtrack` 的一次调用。每个叶子节点（和中间节点）生成一个子集。子集总数：$2^n$。

### 生成所有排列

```python
def permutations(nums):
    result = []

    def backtrack(path, remaining):
        if not remaining:
            result.append(path[:])
            return

        for i in range(len(remaining)):
            path.append(remaining[i])                    # 选择
            backtrack(path, remaining[:i] + remaining[i+1:])  # 探索
            path.pop()                                   # 取消选择

    backtrack([], nums)
    return result
```

- 排列总数：$n!$。每个需要 $O(n)$ 工作来构造 `remaining`，因此总计是 $O(n \cdot n!)$。

### 常见陷阱

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 忘记复制路径 | `result.append(path)` — 所有条目共享同一列表 | `result.append(path[:])` 或 `path.copy()` |
| 未回溯（取消选择） | 状态持续增长，后续候选项看到过时状态 | 始终在递归调用后 `path.pop()` 或 `state.remove()` |
| 循环起始错误 | 子集有重复，或排列有不必要的重用 | 使用 `start` 参数避免重新访问早期索引 |
| 跳过剪枝 | 探索明显无效的分支 | 在递归调用前添加 `if not is_valid: continue` |

## 动态规划

- **动态规划（DP）**是一种优化技术，用于相同子问题被重复求解的问题。DP 不是重复计算，而是求解每个子问题一次并存储结果。

- DP 适用于具有两个性质的问题：
    1. **最优子结构**：最优解可以从子问题的最优解构建。
    2. **重叠子问题**：相同的子问题在递归中多次出现。

### 斐波那契动机

- 朴素递归斐波那契：

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)
```

- 对于 `fib(5)`，递归树：
    - `fib(5)` 调用 `fib(4)` 和 `fib(3)`
    - `fib(4)` 调用 `fib(3)` 和 `fib(2)`
    - `fib(3)` 被计算**两次**，`fib(2)` 被计算**三次**

- 这是 $O(2^n)$，因为树在每一层都分支，大多数分支重新计算相同的值。对于 `fib(50)`，它需要超过 $10^{15}$ 次操作 —— 不可行。

- 使用**记忆化**（自顶向下 DP）：

```python
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]
```

- 现在 `fib(3)` 只计算一次，存储起来，并在后续调用中查找。总计：$O(n)$ 时间，$O(n)$ 空间。

- 使用**制表法**（自底向上 DP）：

```python
def fib_tab(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
```

- 同样是 $O(n)$ 时间，但自底向上构建解而无需递归。可以进一步优化到 $O(1)$ 空间，因为每个值只依赖于前两个值。

### DP 方法

对于任何 DP 问题，遵循这些步骤：

1. **定义状态**：`dp[i]`（或 `dp[i][j]`）代表什么？这是最难的一步。状态必须捕获足够的信息以做出最优决策。

2. **写出递推关系**：`dp[i]` 如何与更小的子问题关联？这是转移公式。

3. **确定基本情况**：什么是最小的可以直接求解的子问题？

4. **确定迭代顺序**：哪些子问题必须在哪些之前解决？自底向上：以确保依赖项已解决的顺序迭代。自顶向下：递归自动处理这个问题。

5. **优化空间**（可选）：如果 `dp[i]` 只依赖于前一行或前几个条目，你不需要完整的表。

### 示例：思考过程

**问题**：给定正整数数组，找到不相邻元素的最大和（打家劫舍）。

**第 1 步 — 定义状态**：`dp[i]` = 考虑元素 `nums[0..i]` 时的最大和。

**第 2 步 — 写出递推关系**：对于元素 $i$，我们要么：
- 跳过它：`dp[i] = dp[i-1]`（不取元素 $i$ 的最佳和）。
- 取它：`dp[i] = dp[i-2] + nums[i]`（必须跳过元素 $i-1$，然后加上元素 $i$）。

所以：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`。

**第 3 步 — 基本情况**：`dp[0] = nums[0]`，`dp[1] = max(nums[0], nums[1])`。

**第 4 步 — 迭代顺序**：从左到右（每个状态依赖于前两个状态）。

**第 5 步 — 空间优化**：只需要最后两个值。

```python
def rob(nums):
    if len(nums) == 1:
        return nums[0]

    prev2, prev1 = nums[0], max(nums[0], nums[1])

    for i in range(2, len(nums)):
        curr = max(prev1, prev2 + nums[i])
        prev2, prev1 = prev1, curr

    return prev1
```

### 如何识别 DP 问题

- 问题要求**最优值**（最小成本、最大利润、最长序列）或**计数**（方法数）。
- 问题在**每步都有选择**（取/跳过、向左/向右、使用或不使用这枚硬币），并且整体最佳答案依赖于子问题的最佳答案。
- 绘制递归树会揭示**重复的子问题**。
- 暴力解法是指数级的，但**不同状态**的数量远少于递归调用次数。

### DP 的类别

- **1D DP**：状态依赖于单个索引。示例：爬楼梯、打家劫舍、最大子数组。

- **2D DP**：状态依赖于两个索引。示例：最长公共子序列（`dp[i][j]` 表示字符串 1 的前 $i$ 个字符和字符串 2 的前 $j$ 个字符）、编辑距离、网格路径问题。

- **区间 DP**：状态是范围 `dp[i][j]`，表示 `arr[i..j]` 上的子问题。示例：矩阵链乘法、戳气球。

- **背包 DP**：状态是物品索引和容量。示例：0/1 背包、零钱兑换、子集和问题。

- **状态压缩 DP**：状态包括表示哪些元素已被使用的位掩码。示例：旅行商问题、分配问题。状态空间是 $O(2^n \cdot n)$，对于 $n \leq 20$ 是可行的。

### 自顶向下 vs 自底向上

| | 自顶向下（记忆化） | 自底向上（制表法） |
|--|---|---|
| 实现 | 递归 + 缓存 | 迭代 + 表 |
| 计算 | 仅实际需要的子问题 | 直到目标的所有子问题 |
| 栈溢出风险 | 有（深层递归） | 无 |
| 空间优化 | 较难 | 较易（使用滚动数组） |
| 编码容易度 | 通常更自然（编写递归，添加缓存） | 需要考虑迭代顺序 |

- 面试中，自顶向下通常编码更快。在生产中，自底向上通常更受青睐，因为性能更好（无递归开销，更好的缓存行为）。

### 常见陷阱

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 状态定义错误 | `dp[i]` 未捕获足够的决策信息 | 添加维度（如 `dp[i][j]` 而非 `dp[i]`） |
| 缺少基本情况 | `dp[0]` 错误 → 所有后续值都错误 | 手动验证基本情况 |
| 迭代顺序错误 | 在依赖项之前计算 `dp[i]` | 绘制依赖箭头并相应迭代 |
| 未正确初始化 `dp` | 在应该用无穷大时使用 0（最小化问题） | 最小化用 `float('inf')`，最大化用 `float('-inf')` |
| 忘记考虑"跳过"选项 | 总是取当前元素 | 递推通常有 `max(取，跳过)` |
| 可变默认参数 | `def f(memo={})` 跨调用共享缓存 | `def f(memo=None): if memo is None: memo = {}` |
| 2D DP 中的差一错误 | `dp` 是 1 索引时访问 `text1[i]` | `dp` 大小为 `(m+1) x (n+1)`，访问 `text1[i-1]` |

## 综合运用

- 这四个概念形成一个递进：
    1. **大 O** 告诉你方法是否足够快。
    2. **递归** 将问题分解为子问题。
    3. **回溯** 是递归 + 选择 + 撤销，用于穷举搜索。
    4. **DP** 是递归 + 缓存，用于重叠子问题上的优化。

- 当你看到新问题时：
    - 估计输入大小 $n$。什么样的大 O 是可接受的？
    - 如果暴力解法是指数级的且问题要求枚举/查找配置：**回溯**（带剪枝使其实用）。
    - 如果暴力解法是指数级的且问题要求最优值或计数，并且你看到重叠子问题：**DP**。
    - 如果问题具有将搜索空间减半的结构：**二分查找**或**分治**。
    - 如果问题是序列上带有子数组约束：**滑动窗口**或**双指针**。
    - 如果问题需要快速查找：**哈希表**。
