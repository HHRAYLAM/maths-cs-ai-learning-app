# 跨模态生成

*跨模态生成产生输出在一个模态，以另一个模态的输入为条件；文本到图像、图像到文本、文本到音频，以及 beyond。本文件涵盖 DALL-E、Stable Diffusion、无分类器引导、ControlNet、图像描述生成、文本到视频（Sora）和文本到音频生成。*

- 在本章的文件 01-03 中，你学习了如何表示、对齐和标记化不同模态。现在是创作行为：从一个模态生成另一个模态。跨模态生成是文本到图像工具、视频合成系统、音乐作曲模型和图像描述生成的引擎。把它想象为教机器成为多媒体艺术家——你用词语描述你想要的，它绘画、动画或作曲。

- 核心思想是**条件生成**：给定来自模态$A$ 的输入（例如，文本），在模态$B$（例如，图像）中产生输出。形式上，你学习模型$p_\theta(y \mid x)$，其中$x$ 是条件信号，$y$ 是生成输出。挑战是这个条件分布极其复杂和高维——512x512 图像生活在$\mathbb{R}^{786432}$ 中，单个文本提示有许多有效图像。

## 文本到图像生成

- 想象你向法庭素描艺术家描述场景。艺术家必须解释你的话，回忆对象看起来像什么，空间组合它们，并渲染最终图片。文本到图像模型正是做这个，但它们必须从数据而不是多年艺术学校学习所有这些技能。

### DALL-E：自回归图像生成

- **DALL-E**（Ramesh 等人，2021）将图像生成视为序列预测问题——与语言模型（第 07 章）相同的范式。关键洞察是，如果你可以将图像表示为离散标记（回想文件 03 的 VQ-VAE），那么生成图像只是逐个生成标记序列。

- 管道有两个阶段。首先，**离散 VAE（dVAE）** 将 256x256 图像压缩为 32x32 网格的离散标记，来自 8192 条目的码本，将图像减少到 1024 标记序列。其次，**变压器解码器** 训练为建模 256 个文本标记（BPE 编码）与 1024 个图像标记连接的联合分布，总共 1280 标记：

$$p(x_{\text{text}}, x_{\text{img}}) = \prod_{i=1}^{1280} p(x_i \mid x_1, \ldots, x_{i-1})$$

- 生成时，你输入文本标记，模型自回归地逐个采样图像标记。这很优雅，因为它重用语言建模的确切机制——注意力、因果掩码、top-k 采样——用于图像合成。

- 缺点是自回归生成本质上是顺序的：逐个生成 1024 标记很慢，序列中早期任何错误会复合。DALL-E 通过生成多个候选图像并用 CLIP（来自文件 01）重新排名它们以找到最佳匹配文本提示来缓解。

![DALL-E 管道：文本标记和图像标记连接成单个序列，由变压器解码器处理，自回归预测以文本标记为条件的图像标记](../images/dalle_autoregressive_pipeline.svg)

### Stable Diffusion：带文本条件的潜在扩散

- **Stable Diffusion**（Rombach 等人，2022）采用根本不同的方法。它不是逐个预测标记，而是从纯噪声开始，逐渐去噪成图像，由文本提示引导。回想第 8 章的扩散模型——Stable Diffusion 在压缩潜在空间中操作而不是像素空间，使其显著更高效。

- 架构有三个组件协同工作。**VAE 编码器** 将图像从像素空间（$512 \times 512 \times 3$）压缩到潜在表示（$64 \times 64 \times 4$），将维度减少 48 倍。**文本编码器**（通常 CLIP 或 OpenCLIP）将文本提示转换为嵌入向量序列。**U-Net 去噪器** 采用噪声潜在、时间步和文本嵌入，并预测每个步骤要减去的噪声。文本条件通过**交叉注意力**层进入 U-Net：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

- 其中$Q$ 来自噪声图像特征，$K, V$ 来自文本嵌入。这使模型能够在每个空间位置关注相关单词——在去噪应该出现"红球"的区域时，模型关注标记"红"和"球"。

- 推理时，你在潜在空间中采样$z_T \sim \mathcal{N}(0, I)$，使用 U-Net 迭代去噪$T$ 步（通常 DDIM 调度 20-50 步），并用 VAE 解码器将干净潜在$z_0$ 解码回像素空间。整个前向传递在消费 GPU 上几秒内生成 512x512 图像。

![Stable Diffusion 架构：文本提示由 CLIP 编码，潜在空间中的随机噪声由带交叉注意力到文本嵌入的 U-Net 迭代去噪，然后由 VAE 解码生成最终图像](../images/stable_diffusion_architecture.svg)

### 实践中的无分类器引导

- **无分类器引导（CFG）** 是使文本到图像模型产生实际匹配提示的图像的秘诀。回想第 8 章，CFG 训练模型既有条件又有无条件，然后在采样时放大有条件信号：

$$\hat{\epsilon} = \epsilon_\theta(x_t, \varnothing) + s \cdot (\epsilon_\theta(x_t, c) - \epsilon_\theta(x_t, \varnothing))$$

- 其中$s$ 是引导尺度。想象项$(\epsilon_\theta(x_t, c) - \epsilon_\theta(x_t, \varnothing))$ 为"朝向提示的方向"——它捕获条件预测与无条件预测的不同之处。乘以$s > 1$ 夸大这个方向，推动图像更接近文本描述，以多样性为代价。

- 在实践中，$s = 7.5$ 是 Stable Diffusion 的常见默认值。在$s = 1.0$ 你得到原始模型输出（多样但与提示松散匹配）。在$s = 20+$ 图像变得过饱和和重复但与文本非常一致。最优$s$ 取决于应用：创意探索 favor 较低引导，而精确提示遵循需要较高引导。

### Imagen：级联扩散与语言理解

- **Imagen**（Saharia 等人，2022）证明强大文本编码器比更大图像模型更重要。Imagen 不使用 CLIP，而是使用冻结的**T5-XXL** 语言模型（来自第 07 章）作为文本编码器，它对语言语义、组合性和空间关系（"蓝色立方体在红色球体顶部"）有更丰富理解。

- Imagen 使用**级联扩散**方法：基础扩散模型生成 64x64 图像，第一超分辨率模型升级到 256x256，第二超分辨率模型达到 1024x1024。每个阶段是单独扩散模型，以文本和（对于上采样器）较低分辨率图像为条件。这个级联避免在基础分辨率建模精细细节，允许基础模型专注于组合和语义，而上采样器处理纹理和清晰度。

- Imagen 还引入**动态阈值**：在每个去噪步骤，预测像素值被裁剪到基于百分位数的范围而不是固定范围$[-1, 1]$。这防止高引导尺度下的饱和伪影，扩散模型中的常见问题。

### Parti：大规模自回归

- **Parti**（Pathways Autoregressive Text-to-Image，Yu 等人，2022）以大规模复兴自回归方法。像 DALL-E 一样，它将图像转换为离散标记（使用 ViT-VQGAN）并用变压器顺序生成它们。但 Parti 使用 200 亿参数编码器 - 解码器变压器（基于 Pathways 架构），并表明自回归模型在足够缩放时可以匹配扩散质量。

- Parti 的编码器 - 解码器架构是与 DALL-E 的仅解码器设计的关键区别。文本通过编码器；解码器在生成图像标记时交叉关注编码文本。这镜像机器翻译（第 07 章）——你从"文本语言"翻译到"图像语言"。

### DiT 和基于流的生成

- **Diffusion Transformers (DiT)**（Peebles 和 Xie，2023）用普通变压器替换扩散模型中的 U-Net 骨干。每个噪声潜在图块被视为标记（类似于第 8 章的 ViT），变压器用自注意力和对文本条件的交叉注意力处理这些标记。DiT 表明变压器对扩散比 U-Net 更可预测地缩放——加倍计算可靠地减半 FID 分数。

- **流匹配**（从第 8 章回想）已成为扩散噪声预测范式的替代方案。模型不是预测要减去的噪声$\epsilon$，而是预测速度$v_\theta(x_t, t)$，沿直路径从噪声传输样本到数据。**Stable Diffusion 3** 和**Flux** 采用流匹配与**多模态 DiT（MM-DiT）** 架构，其中文本和图像标记由带双向注意力的变压器块共同处理——两个模态相互关注，而不是文本仅通过交叉注意力调节图像特征。

![DiT 架构：噪声潜在图像块像 ViT 一样标记化，由带自适应层归一化的变压器块处理用于时间步和类别条件，然后解码回空间潜在](../images/dit_architecture.svg)

## 文本到视频生成

- 文本到视频是文本到图像，带无情的额外约束：**时间相干性**。每帧必须内部一致（有效图像），但连续帧也必须平滑连接——对象应该自然移动，照明应该连续变化，"相机"应该遵循物理合理轨迹。想想绘制单个景观与导演电影的区别。

### 时间挑战

- 视频引入三个超越图像生成的挑战。**时间一致性** 要求对象跨帧维护其身份——第 1 帧的狗应该仍然是第 100 帧的同一只狗。**运动建模** 需要学习物理动力学：对象如何移动、重力如何工作、流体如何流动。**计算成本** 严重：10 秒视频在 24 fps 和 512x512 分辨率包含$10 \times 24 \times 512 \times 512 \times 3 \approx 1.88$ 亿值，约 240 倍于单个图像的数据。

### Make-A-Video 和扩展到视频方法

- **Make-A-Video**（Singer 等人，2022）采用实用方法：从预训练文本到图像模型开始并添加时间层。关键洞察是你已经有强大的文本 - 图像模型在数十亿图像 - 文本对上训练，你只需要从（无标签）视频数据学习运动。

- Make-A-Video 插入**时间注意力**和**时间卷积**层到预训练空间 U-Net。空间层（在图像上预训练）处理外观，而新时间层（在视频上训练）处理运动。空间自注意力在每帧内操作；时间注意力在每个空间位置跨帧操作。这个因式分解是高效的，因为时间和空间模式大部分可分离。

- 生成管道镜像 Imagen 的级联：基础模型生成 16 帧 64x64，然后空间和时间超分辨率模型升级到最终分辨率和帧率。帧插值网络增加时间平滑性。

### VideoPoet 和基于标记的视频模型

- **VideoPoet**（Kondratyuk 等人，2024）在语言建模范式下统一视频生成。所有模态——文本、图像、视频、音频——都标记化为离散序列，单个大语言模型（LLM）训练为自回归预测所有模态的标记。这启用零样本能力：文本到视频、图像到视频、视频到音频、视频编辑和修复都从同一模型出现。

- VideoPoet 用 MAGVIT-v2 编码器（3D VQ-VAE，来自文件 03）标记化视频，共同压缩空间和时间维度。音频用 SoundStream 标记化。LLM 骨干在文本上预训练并在多模态标记序列上微调，学习跨模态的联合分布。

### Sora 风格时间扩散

- **Sora**（OpenAI，2024）将时间扩散带入主流关注，带其生成长、相干、物理合理视频的能力。虽然完整架构细节未发布，关键思想涉及将 DiT 缩放到时空：视频帧分解为**时空图块**（跨越高度、宽度和时间的 3D 块），被视为大变压器的标记。

- 时空图块方法意味着模型将视频作为原生 3D 信号处理，而不是 2D 帧序列。这允许它捕获长程时间依赖——模型可以"提前计划"跨越整个视频持续时间，而不是逐帧生成。

- Sora 可以处理可变持续时间、分辨率和纵横比，通过调整时空图块数量。在本机分辨率上训练数据（而不是将所有裁剪为方形）改进组合和构图质量。

### Wan：开源视频生成

- **Wan**（Wan 等人，2025）是开源视频生成模型家族（1.3B 和 14B 参数），构建在 DiT 骨干上带 3D VAE 时间压缩。Wan 使用**流匹配**而不是传统 DDPM 风格扩散，学习从噪声到视频潜在的直传输路径。3D VAE 在空间和时间上压缩视频（4 倍时间压缩），DiT 用完整 3D 注意力处理结果时空潜在标记。

- Wan 支持文本到视频、图像到视频（动画静止图像）和视频编辑。14B 模型生成相干视频长达 5 秒 720p 分辨率，表明当架构和训练配方选择仔细制作时，开源模型可以接近专有系统的质量。

![文本到视频管道：文本由语言模型编码，时空噪声由时间扩散变压器去噪关注文本嵌入，由 3D VAE 解码为视频帧](../images/text_to_video_pipeline.svg)

## 文本到音频生成

- 想象电影作曲家阅读剧本并配乐。文本到音频模型做类似事情：给定文本描述（"带大雨和远处雷声的雷暴"），它们生成对应音频波形。挑战是 bridging 文本的离散、符号性质与声音的连续、时间性质之间的差距。

### AudioLM：音频的语言建模

- **AudioLM**（Borsos 等人，2023）通过自回归预测离散音频标记生成音频，借鉴 DALL-E 用于图像的相同语言建模范式。它使用分层标记结构：**语义标记**（来自自监督模型如 w2v-BERT，回想第 9 章）捕获高级内容（说什么或播放什么），而**声学标记**（来自 SoundStream，神经音频编解码器）捕获细粒度声学细节（如何听起来——音色、录音质量）。

- 生成分两个阶段进行。首先，变压器预测给定可选音频提示的语义标记，建立高级内容计划。其次，另一个变压器预测以语义标记为条件的声学标记，填充声学细节。这个分层镜像文本到语音管道（第 9 章）——语义标记扮演音素角色，声学标记扮演 mel 频谱图帧角色。

- AudioLM 可以生成语音延续（给定 3 秒语音，生成下一个 10 秒）、音乐延续和音效，都来自单个在仅音频数据上训练的模型（预训练不需要文本标签）。

### MusicLM：文本条件音乐

- **MusicLM**（Agostinelli 等人，2023）扩展 AudioLM 到文本条件音乐生成。它添加文本 - 音频联合嵌入（来自**MuLan**，CLIP 风格模型在音乐 - 文本对上训练）来条件生成。MuLan 嵌入捕获文本描述的语义意义（"带萨克斯独奏的欢快爵士"）并引导分层标记生成。

- MusicLM 以 24 kHz 生成音乐任意持续时间，在数分钟长的作品上维护旋律和节奏相干性。它也可以条件在哼唱旋律（使用音高跟踪器提取的旋律标记）加文本描述，生成遵循哼唱曲调的完整编曲，风格由文本描述。

### MusicGen：高效单阶段生成

- **MusicGen**（Copet 等人，2023）简化多阶段方法。MusicGen 不使用单独的语义和声学模型，而是使用单个自回归变压器，直接从音频编解码器生成多个码本级别。关键创新是**交错码本模式**：不是在移动到下一个之前生成一个时间步的所有码本级别，MusicGen 跨码本和时间步交错标记，模式允许某些码本级别并行解码。

- 条件简单：文本由 T5 编码器编码，文本嵌入前置到音频标记序列（如语言模型中的前缀提示）或通过交叉注意力注入。MusicGen 也支持旋律条件：参考旋律的色谱图（来自第 9 章讨论的频谱图特征）编码并与文本条件一起使用。

$$p(a_1, \ldots, a_T) = \prod_{t=1}^{T} \prod_{k=1}^{K} p(a_{t,k} \mid a_{<t}, c_{\text{text}})$$

- 其中$a_{t,k}$ 是时间步$t$ 和码本级别$k$ 的音频标记，$c_{\text{text}}$ 是文本条件。乘积超过$k$ 因式分解取决于码本模式——某些级别并行预测。

## 图像到文本生成

- 现在翻转方向：给定图像，生成自然语言描述。这是**图像描述生成**，是条件文本生成的形式，其中图像是条件。想象博物馆向导描述绘画——他们必须感知视觉内容，理解对象间关系，并用流畅语言表达观察。

### 描述生成为条件生成

- 经典方法使用**编码器 - 解码器** 架构（第 07 章）。预训练 CNN 或 ViT（第 8 章）将图像编码为特征向量集。语言模型解码器逐字生成描述，在每步关注图像特征：

$$p(w_1, \ldots, w_L \mid I) = \prod_{l=1}^{L} p(w_l \mid w_1, \ldots, w_{l-1}, I)$$

- 其中$w_l$ 是描述词，$I$ 是图像表示。交叉注意力连接文本解码器到图像特征，允许模型在生成不同词时"看"图像不同区域——生成"狗"时关注狗区域，生成"公园"时关注公园区域。

- **CoCa**（Contrastive Captioners，Yu 等人，2022）在单个模型中统一对比学习（文件 01 的 CLIP 风格目标）与描述生成。图像编码器产生特征既用于与文本的对比对齐又用于描述解码器中的交叉注意力。这个多任务训练给 CoCa 强零样本识别（来自对比学习）和强生成（来自描述生成）。

### 现代视觉语言描述生成

- 现代方法通常使用**大多模态模型**（文件 02）进行描述生成。模型如 LLaVA、Qwen-VL 和 GPT-4V 将描述生成视为视觉问答的特殊情况——"问题"隐式是"描述这个图像"。视觉编码器（CLIP ViT 或 SigLIP）产生图块标记投影到 LLM 的嵌入空间，LLM 生成自由形式描述。

- 基于 LLM 描述生成的优势是**指令遵循**：你可以请求不同详细级别（"用一句话描述"与"提供详细段落"）、关注特定方面（"描述颜色"）或生成结构化输出（"列出所有对象及其位置"）。这个灵活性来自 LLM 的指令调优（第 07 章）。

## 视频 - 音频共同生成

- 想象看电影无声——体验空洞。视觉内容和音频深度耦合：弹跳球有节奏砰砰声、雨产生滴答声、人群产生欢呼。**视频 - 音频共同生成** 旨在一起产生两个模态，维护你看到和听到之间的时间对齐。

### 联合时间建模

- 核心挑战是**时间同步**：鼓击的音频必须与显示鼓槌击鼓的视觉帧完全重合。这需要两个模态可以引用的共享时间表示。

- 一种方法是从共享潜在时间线生成视频和音频。模型如**CoDi**（Composable Diffusion，Tang 等人，2023）对每个模态使用单独扩散模型但通过共享潜在空间对齐它们。训练期间，跨模态注意力层学习在每个时间步同步视觉和音频特征。生成期间，两个扩散过程同时运行，通过共享对齐相互条件。

- VideoPoet（上面讨论）采用更统一方法：因为所有模态标记化为单个序列，LLM 自然学习视频和音频标记之间的时间对应。吠叫狗的视频片段后跟对应音频标记教模型关联视觉吠叫动作与吠叫声音。

- **时间对齐损失**函数显式强制执行同步。一个公式在帧级别使用对比学习：时间$t$ 的音频段应该比其它时间帧更类似于时间$t$ 的视频帧：

$$\mathcal{L}_{\text{sync}} = -\mathbb{E}_t \left[\log \frac{\exp(\text{sim}(v_t, a_t) / \tau)}{\sum_{t'} \exp(\text{sim}(v_t, a_{t'}) / \tau)}\right]$$

- 其中$v_t$ 和$a_t$ 是时间$t$ 的视频和音频表示，$\tau$ 是温度参数。这在结构上等同于文件 01 的 InfoNCE 损失，但应用于时间帧级别而不是剪辑级别。

## 指令遵循生成

- 想象告诉艺术家"使天空更戏剧性"或"用皇冠替换帽子"。**指令遵循生成** 让你使用自然语言命令编辑图像，而不是精确空间掩码或画笔笔触。

### InstructPix2Pix：通过描述编辑

- **InstructPix2Pix**（Brooks 等人，2023）训练条件扩散模型，采用输入图像和文本指令，然后产生编辑图像。巧妙部分是训练数据如何创建：GPT-3 生成编辑指令（"使它冬季"、"将猫变成狗"）与输入 - 输出文本描述配对，文本到图像模型（Stable Diffusion）生成对应图像对。

- 模型是修改的 Stable Diffusion U-Net，接收文本指令（通过交叉注意力）和输入图像潜在（与噪声潜在通道连接）。它使用**双无分类器引导**带两个引导尺度——一个用于文本指令（$s_T$）和一个用于输入图像（$s_I$）：

$$\hat{\epsilon} = \epsilon_\theta(x_t, \varnothing, \varnothing) + s_I \cdot (\epsilon_\theta(x_t, c_I, \varnothing) - \epsilon_\theta(x_t, \varnothing, \varnothing)) + s_T \cdot (\epsilon_\theta(x_t, c_I, c_T) - \epsilon_\theta(x_t, c_I, \varnothing))$$

- 其中$c_I$ 是输入图像条件，$c_T$ 是文本指令。第一引导项控制多少保留输入图像；第二控制多强遵循指令。这给用户二维旋钮：高$s_I$ 密切保留原始，而高$s_T$ 做更戏剧性编辑。

![InstructPix2Pix：输入图像和文本指令输入修改扩散模型，产生遵循指令同时保留未编辑区域的编辑图像](../images/instructpix2pix_pipeline.svg)

### SDEdit 和基于噪声的编辑

- **SDEdit**（Meng 等人，2022）提供更简单编辑方法，不需要特殊训练。你采用输入图像，添加噪声到它（运行前向扩散过程到中间时间步$t_0$），然后用描述期望输出的文本提示去噪。噪声量控制编辑强度：低噪声保留结构（颜色变化、风格转换），而高噪声允许主要重构（对象替换、布局变化）。

- 权衡精确：在时间步$t_0$，噪声图像保留$\bar{\alpha}_{t_0}$ 分数原始信号。去噪过程根据新文本提示填充损坏细节。这在数学上有根据：扩散模型从后验$p(x_0 \mid x_{t_0}, c)$ 采样，其中$x_{t_0}$ 约束生成为"接近"原始。

### ControlNet：空间条件

- **ControlNet**（Zhang 等人，2023）添加细粒度空间控制到文本到图像扩散。预训练 U-Net 编码器的副本训练为接受额外输入条件——边缘图（Canny 边缘）、深度图、姿势骨架、分割图——而原始 U-Net 权重冻结。ControlNet 编码器的输出通过**零卷积**（1x1 卷积初始化为零）添加到冻结 U-Net 的跳过连接，确保训练从预训练模型行为开始并逐渐学习新条件。

- 这个架构让你提供草图、深度图或人体姿势作为结构指南，文本提示填充外观。预训练权重处理真实感和文本理解；ControlNet 层处理空间保真度到条件。

## 一致性和对齐指标

- 如何测量生成图像是否好？"好"有至少两个维度：**质量**（看起来像真实图像吗？）和**对齐**（匹配文本提示吗？）。已开发几种指标量化这些。

### Frechet Inception Distance（FID）

- **Frechet Inception Distance（FID）**（Heusel 等人，2017）测量预训练 Inception 网络特征空间中生成图像分布和真实图像分布之间的距离。把它想象为比较两个图像集合的"指纹"而不是比较单个图像。

- 真实和生成图像集都通过 Inception-v3，收集倒数第二层的激活。这些激活建模为多元高斯$\mathcal{N}(\mu_r, \Sigma_r)$ 和$\mathcal{N}(\mu_g, \Sigma_g)$。FID 是这些高斯之间的 Frechet 距离（Wasserstein-2 距离）：

$$\text{FID} = \|\mu_r - \mu_g\|^2 + \text{Tr}\left(\Sigma_r + \Sigma_g - 2(\Sigma_r \Sigma_g)^{1/2}\right)$$

- 更低 FID 更好。FID = 0 意味着分布相同。FID 捕获质量（如果生成图像模糊，它们的特征将不同于真实图像）和多样性（如果模型遭受模式崩溃，$\Sigma_g$ 将小于$\Sigma_r$）。ImageNet 256x256 上典型最先进值是 FID < 2.0。

- FID 有已知限制：它假设高斯特征分布（这是近似的），它需要数千样本用于稳定估计，它使用 Inception 特征（可能不捕获所有感知相关差异）。

### Inception Score（IS）

- **Inception Score（IS）**（Salimans 等人，2016）测量两个属性：每个生成图像应该自信可分类（条件类别分布$p(y \mid x)$ 应该峰值），生成图像集应该覆盖许多类别（边缘$p(y) = \mathbb{E}_x[p(y \mid x)]$ 应该均匀）。IS 通过 KL 散度组合这些：

$$\text{IS} = \exp\left(\mathbb{E}_x \left[D_{\text{KL}}(p(y \mid x) \| p(y))\right]\right)$$

- 更高 IS 更好。最大 IS 等于类别数（ImageNet 为 1000）。IS 奖励质量（清晰、可识别图像）和多样性（类别覆盖），但它有显著限制：它完全忽略真实数据分布，它不能检测类别内模式丢弃，它偏向 ImageNet 类图像因为它使用 Inception 的类别预测。

### CLIPScore：测量文本 - 图像对齐

- **CLIPScore**（Hessel 等人，2021）直接使用预训练 CLIP 模型（文件 01）测量生成图像如何匹配其文本提示。分数是 CLIP 图像嵌入和 CLIP 文本嵌入之间的余弦相似度：

$$\text{CLIPScore}(I, T) = \max(0, \cos(E_I(I), E_T(T)))$$

- 其中$E_I$ 和$E_T$ 是 CLIP 图像和文本编码器。CLIPScore 是无参考的——它不需要真实图像，只有文本提示。它与人类对文本 - 图像对齐的判断良好相关，已成为评估文本到图像模型中提示保真度的标准指标。

- 与参考描述比较，**RefCLIPScore** 包含参考图像：

$$\text{RefCLIPScore} = \text{HarmonicMean}(\text{CLIPScore}(I, T), \max(0, \cos(E_I(I), E_I(I_{\text{ref}}))))$$

- 这平衡文本对齐与视觉相似性到参考。

### 人类评估

- 自动指标是代理；人类判断仍然是黄金标准。常见协议包括**成对比较**（两个图像哪个更好匹配提示？）、**Likert 量表**（从 1-5 评价质量和对齐）和**Elo 评级**（跨模型锦标赛风格排名）。DrawBench 和 PartiPrompts 基准提供标准化提示集用于系统人类评估。

## 伦理考虑

- 跨模态生成是 AI 中伦理影响最深远的领域之一。从文本创建逼真图像、视频和音频的能力提出从业者必须认真对待的深刻担忧。

### 深度伪造和错误信息

- **深度伪造**是生成或操纵的媒体，旨在描绘从未发生的事件。文本到图像和文本到视频模型可以创建公众人物的可信假照片、伪造证据和误导性新闻图像。危险不仅在于假的存在，而在于它们的存在破坏对所有媒体的信任——如果任何图像可能是假的，没有图像完全可信。

- 检测方法包括在真实与生成图像上训练分类器、分析统计伪影（GAN 生成图像有微妙光谱签名）和嵌入不可见水印（Stable Diffusion 的不可见水印、Google 的 SynthID）。然而，检测是军备竞赛：随着生成器改进，检测器必须不断更新。

### 生成中的偏倚

- 在互联网规模数据上训练的模型继承和放大社会偏倚。文本到图像模型不成比例地生成较浅肤色面孔，将某些职业与特定性别关联，并默认西方文化规范用于未明确说明的提示。这些偏倚根植于训练数据分布和 CLIP/T5 文本编码器，它们从自己的训练语料库编码偏倚。

- 缓解策略包括策划更具代表性的训练数据、对文本编码器应用去偏技术、使用安全分类器过滤有问题输出，并允许用户控制人口统计属性。这些都不是完整解决方案，持续审计至关重要。

### 内容过滤和安全

- 负责任部署需要多层保护。**输入过滤** 在生成前阻止有害提示。**输出过滤** 分类生成内容并拒绝有害材料。**NSFW 分类器** 检测性明确、暴力或其他有害内容。Stable Diffusion 的安全检查器例如计算生成图像的 CLIP 嵌入与一组预定义有害概念嵌入的余弦相似度，标记超过阈值的图像。

- 许多生成模型（Stable Diffusion、Wan）的开源性质在民主化访问和防止滥用之间创造紧张。一旦模型权重发布，内容过滤可以绕过。这导致关于适当开放级别和模型开发者责任的辩论。

### 知识产权和同意

- 在互联网数据上训练的生成模型可能复制受版权保护的风格、商标或真实人物的 likeness 未经同意。法律和伦理框架仍在演变，但负责任实践包括尊重选择退出机制、承认嵌入训练数据中的创意贡献，并开发技术防护防止记忆和背诵训练示例。

2. 使用完整 Frechet 距离公式计算两组 2D 样本之间的 FID。变化生成分布并观察 FID 如何变化。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def compute_fid(real, generated):
    """计算两个 2D 样本集之间的 Frechet 距离。"""
    mu_r, mu_g = jnp.mean(real, axis=0), jnp.mean(generated, axis=0)
    sigma_r = jnp.cov(real.T)
    sigma_g = jnp.cov(generated.T)
    diff = mu_r - mu_g
    # 通过特征分解的矩阵平方根
    product = sigma_r @ sigma_g
    eigvals, eigvecs = jnp.linalg.eigh(product)
    sqrt_product = eigvecs @ jnp.diag(jnp.sqrt(jnp.maximum(eigvals, 0))) @ eigvecs.T
    fid = jnp.sum(diff ** 2) + jnp.trace(sigma_r + sigma_g - 2 * sqrt_product)
    return fid

key = jax.random.PRNGKey(0)
k1, k2, k3, k4 = jax.random.split(key, 4)

# 真实分布：标准 2D 高斯
real = jax.random.normal(k1, (1000, 2))

# 生成分布带增加发散
shifts = [0.0, 0.5, 1.0, 2.0, 4.0]
fig, axes = plt.subplots(1, len(shifts), figsize=(18, 3.5))
for ax, shift in zip(axes, shifts):
    gen = jax.random.normal(k2, (1000, 2)) * (1 + shift * 0.2) + shift
    fid = compute_fid(real, gen)
    ax.scatter(real[:, 0], real[:, 1], s=3, alpha=0.3, label='Real')
    ax.scatter(gen[:, 0], gen[:, 1], s=3, alpha=0.3, label='Generated')
    ax.set_title(f'Shift={shift}\nFID={fid:.2f}')
    ax.set_xlim(-5, 8); ax.set_ylim(-5, 8)
    ax.set_aspect('equal'); ax.legend(fontsize=7)
plt.suptitle('FID increases as generated distribution diverges from real')
plt.tight_layout(); plt.show()
```

```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

def cosine_similarity(a, b):
    return jnp.dot(a, b) / (jnp.linalg.norm(a) * jnp.linalg.norm(b))

def clip_score(img_emb, txt_emb):
    """CLIPScore：钳位余弦相似度。"""
    return jnp.maximum(0.0, cosine_similarity(img_emb, txt_emb))
```
