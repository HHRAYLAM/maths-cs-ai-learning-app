# 链表、栈和队列

*链表、栈和队列是更复杂数据结构的构建块。本章涵盖它们的机制，然后构建关键模式：快慢指针、单调栈和基于堆的优先队列，通过渐进式难度的问题和每个步骤的常见陷阱。*

- 数组提供快速随机访问但插入成本高。链表提供快速插入但没有随机访问。栈和队列限制对一端或两端的访问，这种限制正是它们强大的原因：通过限制你能做什么，它们简化了你需要思考的内容。

## 链表

- 单链表 是节点的链。每个节点存储一个值和指向下一个节点的指针。最后一个节点指向 `null`。

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
```

- 相对于数组的优势：在已知位置插入或删除是 $O(1)$（只需重新指向指针）。无需移位元素。

- 劣势：访问元素 $i$ 需要 $O(i)$ 遍历（无随机访问）。缓存局部性差（节点分散在内存中）。

- 双链表 添加了 `prev` 指针，支持向后遍历。用于 LRU 缓存（常数时间删除任何节点）和浏览器历史（前进/后退）。

| 操作 | 单链表 | 双链表 |
|-----------|--------|--------|
| 按索引访问 | $O(n)$ | $O(n)$ |
| 在头部插入 | $O(1)$ | $O(1)$ |
| 在尾部插入 | $O(n)$ 或 $O(1)$* | $O(1)$ |
| 删除给定节点 | $O(n)$ | $O(1)$ |
| 搜索 | $O(n)$ | $O(n)$ |

*有尾指针时。需要前驱节点，这需要遍历。

- 哨兵节点（虚拟头/尾）简化边界情况。没有虚拟头时，在头部插入或删除头部需要特殊情况代码。有了虚拟节点，每个真实节点都有前驱。

```python
# 没有虚拟：头部删除的特殊情况
def delete_head(head):
    if not head:
        return None
    return head.next

# 有虚拟：统一逻辑
dummy = ListNode(0)
dummy.next = head
# 现在每次删除都是：prev.next = prev.next.next
```

- 陷阱：忘记处理空列表（`head is None`）或单元素列表。始终测试这些边界情况。

## 模式：快慢指针（Floyd 算法）

- 使用两个以不同速度移动的指针来检测链表的属性。慢指针一次移动一步；快指针一次移动两步。

### 简单：链表环

- 问题：确定链表是否有环。

- 模式：如果有环，快指针最终会追上慢指针（它们会相遇）。如果没有环，快指针到达 `null`。

```python
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```

- 为什么有效：如果环的长度是 $c$，快指针每步缩小 1 个节点的差距。它们必须在慢指针进入环后 $c$ 步内相遇。

- 陷阱：检查 `fast and fast.next`（而不仅仅是 `fast.next`）。如果 `fast` 是 `None`，调用 `fast.next` 会崩溃。

### 中等：找到链表的中间节点

- 问题：返回中间节点。

- 模式：当快指针到达末尾时，慢指针在中间。

```python
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # 慢指针在中间（如果是偶数长度，则在第二个中间）
```

### 中等：链表环 II（找到环起点）

- 问题：返回环开始的节点。

- 模式：快慢指针相遇后，将一个指针重置为头部。两个指针都以速度 1 移动。它们在环起点相遇。

```python
def detect_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            # 重置一个指针到头部
            slow = head
            while slow != fast:
                slow = slow.next
                fast = fast.next
            return slow
    return None
```

- 为什么有效：设从头部到环起点的距离为 $a$，从环起点到相遇点的距离为 $b$。慢指针走了 $a + b$。快指针走了 $2(a + b)$。差值是一个完整环：$a + b = c$（环长度）。所以 $a = c - b$：从头部到环起点的距离等于从相遇点到环起点的距离（沿环向前）。

### 困难：每 K 个节点一组反转链表

- 问题：反转链表中每 $k$ 个连续节点。

```python
def reverse_k_group(head, k):
    # 检查是否还有 k 个节点
    node = head
    for _ in range(k):
        if not node:
            return head
        node = node.next

    # 反转 k 个节点
    prev, curr = None, head
    for _ in range(k):
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # head 现在是反转组的尾部
    # 递归处理剩余部分
    head.next = reverse_k_group(curr, k)
    return prev  # prev 是这组的新头部
```

- 陷阱：原地反转模式（`prev, curr, nxt`）值得记住。画出来：在每一步，你将 `curr.next` 向后指向 `prev`，然后前进所有三个指针。顺序错误会破坏列表。

## 栈

- 栈 是 LIFO（后进先出）：最后添加的元素最先删除。想象一叠盘子。

- 操作：`push(x)` 添加到顶部，`pop()` 从顶部删除，`peek()` 查看顶部而不删除。都是 $O(1)$。

- 栈是递归（调用栈）、表达式求值（中缀转后缀）和撤销操作（每个操作被推入，撤销弹出最后一个）背后的隐式结构。

### 简单：有效括号

- 问题：给定一个括号字符串 `()[]{}`，确定它们是否平衡。

- 模式：将左括号推入栈。当你看到右括号时，检查栈顶是否是匹配的左括号。

```python
def is_valid(s):
    stack = []
    matching = {')': '(', ']': '[', '}': '{'}

    for char in s:
        if char in matching:
            if not stack or stack[-1] != matching[char]:
                return False
            stack.pop()
        else:
            stack.append(char)

    return len(stack) == 0
```

- 陷阱：忘记最后的 `len(stack) == 0`。字符串"((("没有不匹配但无效，因为仍有未闭合的括号。

## 模式：单调栈

- 单调栈 将元素保持排序顺序（递增或递减）。当新元素会违反顺序时，你弹出元素直到顺序恢复。

- 何时使用：问题问"对于每个元素，找到下一个/前一个更大/更小的元素。"栈给出 $O(n)$ 总计，因为每个元素最多被推入和弹出一次。

### 中等：每日温度

- 问题：给定每日温度，对于每一天找到直到更高温度的天数。

- 模式：使用索引栈。当当前温度高于栈顶时，弹出并记录距离。

```python
def daily_temperatures(temperatures):
    n = len(temperatures)
    result = [0] * n
    stack = []  # 索引栈，温度按降序排列

    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev = stack.pop()
            result[prev] = i - prev
        stack.append(i)

    return result
```

- 每个元素被推入一次，最多弹出一次：总计 $O(n)$。

- 陷阱：在栈上存储索引（而不是值）。你需要索引来计算距离。

### 困难：柱状图中最大的矩形

- 问题：给定一个柱高数组，找到最大矩形的面积。

- 模式：对于每个柱，找到它可以向左和向右延伸多远（即每侧最近的更短柱）。单调递增栈高效跟踪这个。

```python
def largest_rectangle(heights):
    stack = []  # 索引栈，高度按升序排列
    max_area = 0
    heights.append(0)  # 哨兵，在末尾刷新栈

    for i, h in enumerate(heights):
        start = i
        while stack and stack[-1][1] > h:
            idx, height = stack.pop()
            max_area = max(max_area, height * (i - idx))
            start = idx  # 这个柱可以延伸回被弹出柱的起点
        stack.append((start, h))

    heights.pop()  # 移除哨兵
    return max_area
```

- 陷阱：`start = idx` 行很微妙。当我们弹出一个比当前柱更高的柱时，当前柱可以向后延伸到被弹出柱的起点（因为中间的所有柱都至少和被弹出柱一样高）。缺少这行会给出错误的面积。

- 陷阱：哨兵 `heights.append(0)` 确保栈中所有剩余的柱都被处理。没有它，那些从未遇到右侧更短柱的柱会被错过。

## 队列

- 队列 是 FIFO（先进先出）：元素在背面添加，从前面删除。想象商店里的队伍。

- 双端队列（deque）支持在两端的 $O(1)$ 插入和删除。Python 的 `collections.deque` 是标准实现。

- 队列是BFS（广度优先搜索，第 13 章第 4 节）、任务调度和消息传递背后的结构。

### 简单：用栈实现队列

- 问题：仅使用两个栈实现队列。

- 模式：使用一个栈用于推入，一个栈用于弹出。当弹出栈为空时，从推入栈转移所有元素（反转顺序）。

```python
class MyQueue:
    def __init__(self):
        self.push_stack = []
        self.pop_stack = []

    def push(self, x):
        self.push_stack.append(x)

    def pop(self):
        if not self.pop_stack:
            while self.push_stack:
                self.pop_stack.append(self.push_stack.pop())
        return self.pop_stack.pop()

    def peek(self):
        if not self.pop_stack:
            while self.push_stack:
                self.pop_stack.append(self.push_stack.pop())
        return self.pop_stack[-1]

    def empty(self):
        return not self.push_stack and not self.pop_stack
```

- 每次操作摊销 $O(1)$：每个元素最多在栈之间移动一次。

## 优先队列和堆

- 优先队列 返回最小（或最大）元素优先，无论插入顺序。标准实现是二叉堆。

- 最小堆 是完全二叉树，其中每个父节点小于其子节点。最小值总是在根部。存储为数组：节点 $i$ 的子节点在位置 $2i + 1$ 和 $2i + 2$。

| 操作 | 时间 |
|-----------|------|
| 插入 | $O(\log n)$ |
| 获取最小值 | $O(1)$ |
| 提取最小值 | $O(\log n)$ |
| 从数组构建堆 | $O(n)$ |

- Python 的 `heapq` 模块提供最小堆。对于最大堆，取负值。

```python
import heapq

# 最小堆
h = []
heapq.heappush(h, 5)
heapq.heappush(h, 2)
heapq.heappush(h, 8)
print(heapq.heappop(h))  # 2（最小）

# 最大堆技巧：取负值
heapq.heappush(h, -10)
print(-heapq.heappop(h))  # 10（最大）
```

### 中等：数组中的第 K 大元素

- 问题：找到第 k 大的元素。

- 模式：维护大小为 $k$ 的最小堆。堆的根是第 k 大的元素。如果堆有 $k$ 个元素且新元素大于根，替换根。

```python
import heapq

def find_kth_largest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)  # O(k)

    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)  # 弹出最小，推入 num：O(log k)

    return heap[0]
```

- $O(n \log k)$ 时间，$O(k)$ 空间。当 $k \ll n$ 时比排序（$O(n \log n)$）好得多。

- 陷阱：使用大小为 $n$ 的最大堆并弹出 $k$ 次也有效但较慢：$O(n + k \log n)$。大小为 $k$ 的最小堆是最优方法。

### 困难：合并 K 个排序链表

- 问题：将 $k$ 个排序链表合并为一个排序列表。

- 模式：使用包含每个列表头部的最小堆。弹出最小的，添加到结果，并将其下一个节点推入堆。

```python
import heapq

def merge_k_lists(lists):
    heap = []
    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))

    dummy = ListNode(0)
    curr = dummy

    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next
```

- $O(n \log k)$，其中 $n$ 是节点总数。堆总是最多有 $k$ 个元素。

- 陷阱：堆元组中的 `i`（索引）是决胜局。没有它，当值相等时，Python 尝试比较 `ListNode` 对象，这会崩溃，因为 `ListNode` 不支持 `<`。索引确保有效比较。

## 常见陷阱总结

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| `fast.next` 上的空指针 | 使用 `while fast.next` 检测环 | 检查 `fast and fast.next` |
| 不处理空列表 | `None` 的反转 | 添加 `if not head` 守卫 |
| 栈下溢 | 从空栈弹出 | 检查 `len(stack) > 0` 或 `if stack` |
| 忘记哨兵 | 柱状图错过最后的柱 | 追加 0 刷新栈 |
| 堆中缺少决胜局 | 比较不可比较的对象 | 在堆元组中添加索引 |
| 遍历时修改列表 | 遍历时删除节点 | 使用 prev/curr 模式或虚拟头 |
