# 语言基础

*语言学提供了 NLP 系统隐式学习和利用的结构化词汇。本文件涵盖形态学、句法学、语义学、语用学、音系学、 constituency 和 dependency 解析，以及分布假设 —— 这些是 AI 中 tokenization、语法和意义基础的人类语言科学。*

- 在构建能够理解或生成语言的系统之前，我们需要理解语言本身是如何工作的。

- 语言学是语言的科学研究，它提供了 NLP 不断借鉴的概念词汇。

- 即使是现代神经模型（从原始数据中学习语言）也隐式地重新发现了语言学家几十年来记录的结构。

- 语言在每个层面都有结构：构成单词的声音、构成单词的部分、将单词组合成句子的规则、这些句子承载的意义，以及上下文如何塑造解释。我们将自底向上地逐一讲解每个层面。

- **形态学**研究单词的内部结构。单词不是原子的；它们由更小的有意义单位**语素**构成。

- 单词"unhappiness"包含三个语素："un-"（表示"不"的前缀）、"happy"（词根）和"-ness"（将形容词转化为名词的后缀）。每个语素都对意义有贡献。

- **词根**（或词干）是承载主要意义的核心语素。"Happy"、"run"、"compute"都是词根。

- **词缀**是附着在词根上以修改它的语素。

- 英语有**前缀**（词根之前：un-、re-、pre-）和**后缀**（之后：-ing、-ed、-tion）。某些语言还有中缀（插入词根内部）和环缀（环绕）。

![语素树："unhappiness"拆分为前缀"un"、词根"happy"、后缀"ness"](../images/morpheme_tree.svg)

- 有两种形态学过程。**屈折**改变单词的语法属性而不改变其核心意义或词性："run"变为"runs"（第三人称）、"running"（进行时）、"ran"（过去式）。单词仍然是动词，意义相同。

- **派生**创造新单词，通常改变词性："happy"（形容词）变为"happiness"（名词）、"compute"（动词）变为"computation"（名词）变为"computational"（形容词）。每个派生都转变意义和语法类别。

- 语言在形态复杂性上差异巨大。英语相对是**分析语**（每个单词语素少，依赖词序）。

- 土耳其语和芬兰语是**黏着语**（单词可以包含许多串在一起的语素）。阿拉伯语和希伯来语使用**模板**形态（词根是辅音骨架，如 k-t-b 表示"写"，插入元音模式创造不同单词：kitab"书"、kataba"他写了"、maktub"已写的"）。

- 形态学对 NLP 很重要，因为它影响 tokenization。单词级 tokeniser 将"run"、"runs"、"running"和"ran"视为四个无关符号。

- 形态学感知系统识别它们共享词根。子词 tokenization（BPE、WordPiece）（我们将在文件 02 中介绍）是形态学分析的统计近似。

- **句法学**研究单词如何组合成短语和句子。每种语言都有支配词序和结构的规则；违反这些规则会产生乱码。

- "The cat sat on the mat"是合乎语法的英语；"Mat the on sat cat the"不是。

- 有两个主要框架来描述句法结构。

- **短语结构语法**（也称 constituency 语法）认为句子是通过将短语嵌套在短语内构建的。句子（S）由名词短语（NP）和动词短语（VP）组成。

- 名词短语可能是限定词（Det）后跟名词（N）。动词短语可能是动词（V）后跟名词短语。这些规则构建树：

![ constituency 树："the cat sat on the mat"的树，S 分支为 NP 和 VP，NP 分支为 Det "the"和 N "cat"，VP 分支为 V "sat"和 PP，PP 分支为 P "on"和 NP](../images/constituency_tree.svg)

- 这棵树称为**constituency 树**（或解析树）。每个内部节点是短语类型，每个叶子是单词。树捕捉层次分组："on the mat"是一个单元（介词短语）、"sat on the mat"是一个单元（动词短语），整体是句子。

- **上下文无关文法（CFG）**形式化这些规则。它由一组产生式规则组成，每个规则形式为 $A \to \alpha$，其中 $A$ 是非终结符（如 NP 或 VP 的短语类型），$\alpha$ 是终结符（单词）和非终结符的序列。例如：

```
S  → NP VP
NP → Det N
NP → Det N PP
VP → V NP
VP → V PP
PP → P NP
Det → "the" | "a"
N  → "cat" | "mat" | "dog"
V  → "sat" | "chased"
P  → "on" | "under"
```

- 从 S 开始并重复应用规则，你可以生成语法允许的所有句子。解析是相反的过程：给定一个句子，找到产生它的树（或多个树）。具有多个有效解析树的句子是**句法歧义**的。"I saw the man with the telescope"有两种解析：我用望远镜看到那个人，或者我看到一个拿着望远镜的人。

- **依存语法**采取不同的视角。它不描述短语嵌套，而是描述单词之间的直接关系。句子中的每个单词都依赖于恰好一个其他单词（其**中心词**），除了句子的根节点。结果是**依存树**，其中边用语法关系标记（主语、宾语、修饰语等）。

![依存树："the cat sat on the mat"的树，箭头从"sat"指向"cat"（nsubj）和"on"（prep），从"on"指向"mat"（pobj），从"cat"指向"the"（det），从"mat"指向"the"（det）](../images/dependency_tree.svg)

- 在依存视角下，"sat"是根节点。"Cat"作为主语（nsubj）依赖于"sat"。"On"作为介词修饰语依赖于"sat"。"Mat"作为介词宾语依赖于"on"。每个单词都挂在一个中心词上，形成树。

- 依存语法已成为现代 NLP 中的主导框架，因为依存树更容易用统计解析器生成，并且关系更直接地映射到语义角色（谁对谁做了什么）。

- **配价**描述动词需要多少个论元。"Sleep"是**不及物**（一个论元：睡眠者）。"Eat"是**及物**（两个：吃者和被吃物）。"Give"是**双及物**（三个：给予者、被给物和接受者）。知道动词的配价约束哪些解析树是有效的。

- **语义学**研究意义。句法学告诉你句子如何结构化；语义学告诉你它意味着什么。

- **词汇语义学**关注单个单词的意义。单词以系统方式相互关联：

    - **同义关系**：具有（几乎）相同意义的单词。"Big"和"large"是同义词。真正的完全同义关系很罕见；几乎总是在内涵或用法上有细微差别。
    - **反义关系**：具有相反意义的单词。"Hot"和"cold"、"buy"和"sell"。
    - **上下义关系**："is-a"关系。"Dog"是"animal"的下位词（狗是动物的一种）。"Animal"是"dog"的上位词。这些形成分类层次。
    - **部分整体关系**："部分 - 整体"关系。"Wheel"是"car"的部分整体词。
    - **多义性**：单个单词具有多个相关意义。"Bank"意思是金融机构或河岸。上下文消除歧义。

- **词义消歧（WSD）**是确定在给定上下文中多义词的哪个意义是预期的任务。在"I deposited money at the bank"中，金融意义是正确的。在"We sat by the river bank"中，地理意义是。WSD 是早期 NLP 的核心问题；现代上下文嵌入（ELMo、BERT）通过为同一单词的不同用法产生不同的向量表示来基本解决这个问题。

- **组合语义学**询问单个单词的意义如何组合形成短语或句子的意义。**组合性原则**（归因于 Frege）指出：复杂表达式的意义由其部分的意义和用于组合它们的规则决定。"The cat chased the dog"与"the dog chased the cat"意义不同，因为句法结构（谁是主语 vs 宾语）与单词意义相互作用。

- 并非所有意义都是组合的。**习语**如"kick the bucket"（意思是"死亡"）具有无法从其部分推导出的意义。这对任何组合方法都是挑战。

- **分布语义学**是支撑现代 NLP 的计算意义方法。**分布假设**（Firth，1957）指出："你应通过它所处的环境来认识一个单词。"出现在相似上下文中的单词往往具有相似意义。这是词嵌入（Word2Vec、GloVe）的理论基础，我们将在文件 03 中探讨。

- **语用学**研究上下文如何影响意义。相同的句子根据谁说的、何时、何地、为什么可以意味着不同的事情。

- "Can you pass the salt?"在句法上是一个关于能力的是/否问题。在语用上，它是一个请求。你不会回答"Yes, I can"然后坐着不动。理解这需要超越字面单词的知识，即**言语行为**的惯例。

- **言语行为理论**（Austin、Searle）区分：
    - **言内行为**：字面内容（"Can you pass the salt?"）
    - **言外行为**：预期功能（请求）
    - **言后行为**：对听者的影响（他们传递盐）

- **含义**（Grice）是隐含但未明确陈述的意义。如果有人问"Is John a good cook?"，你回答"He's British"，你没有直接回答问题，但听者可以推断（通过文化刻板印象，无论公平与否）你意味着"no"。Grice 的**合作原则**说说话者通常试图提供信息、真实、相关和清晰，听者解释话语时假设这些准则成立。

- **指代**是一种语用现象，其中不同表达式指代同一实体。在"Alice went to the store. She bought milk"中，"she"指代 Alice。解析指代对于理解多句文本至关重要，是关键 NLP 任务。

- **话语结构**描述句子如何连接形成连贯文本。叙述有开头、中间和结尾。论证有主张和证据。**修辞结构理论（RST）**分析文本为话语关系（详细阐述、对比、因果等）的树。

- 语用学是 NLP 最困难的地方。现代语言模型通过训练数据隐式处理大部分句法和语义，但语用推理、理解讽刺、含义和上下文相关意义仍然是前沿挑战。

- **音系学**研究语言的语音系统。虽然本章侧重于文本，但简要概述可桥接到音频和语音章节（第 09 章）。

- **音位**是区分意义的最小声音单位。英语有约 44 个音位。单词"bat"和"pat"因一个音位而异（/b/ vs /p/），这完全改变了意义。这称为**最小对立体**。

- **音位变体**是同一音位的不同物理实现，不改变意义。"Pin"中的"p"（送气，带气流）和"spin"中的"p"（不送气）在英语中是/p/的音位变体；母语者将它们视为相同声音。

- **国际音标（IPA）**为所有语言的音位提供标准化记法。单词"cat"转录为/kæt/。IPA 是书面文本和语音系统之间的桥梁。

- **韵律**涵盖语音的节奏、重音和语调。"I didn't say he stole the money"根据哪个单词重音而有七种不同含义。韵律携带文本 alone 丢失的信息，这就是为什么语音合成系统必须仔细建模它。

- 在 NLP 中，音系知识出现在语音合成（字素到音位转换）、语音识别（将声学信号映射到音位），甚至在拼写纠正和音译中。

1. 使用常见前缀和后缀列表构建一个简单的形态分析器，将英语单词拆分为可能的语素。
```python
prefixes = ['un', 're', 'pre', 'dis', 'mis', 'over', 'under', 'out', 'non']
suffixes = ['ing', 'ed', 'ly', 'ness', 'ment', 'tion', 'able', 'ible', 'er', 'est', 'ful', 'less', 'ous']

def analyse_morphemes(word):
    """使用已知词缀进行简单的语素分析。"""
    parts = []
    remaining = word.lower()

    # 检查前缀
    for p in sorted(prefixes, key=len, reverse=True):
        if remaining.startswith(p) and len(remaining) > len(p) + 2:
            parts.append(f"[前缀：{p}]")
            remaining = remaining[len(p):]
            break

    # 检查后缀
    for s in sorted(suffixes, key=len, reverse=True):
        if remaining.endswith(s) and len(remaining) > len(s) + 2:
            root = remaining[:-len(s)]
            parts.append(f"[词根：{root}]")
            parts.append(f"[后缀：{s}]")
            remaining = None
            break

    if remaining is not None:
        parts.append(f"[词根：{remaining}]")

    return parts

for word in ['unhappiness', 'reusable', 'disconnected', 'overreacting', 'kindness']:
    print(f"{word:20s} → {' + '.join(analyse_morphemes(word))}")
```

2. 使用递归下降实现简单的上下文无关文法解析器。定义一个小文法并将句子解析为 constituency 树。
```python
class CFGParser:
    """小型英语语法递归下降解析器。"""
    def __init__(self, tokens):
        self.tokens = tokens
        self.pos = 0

    def peek(self):
        return self.tokens[self.pos] if self.pos < len(self.tokens) else None

    def consume(self, expected=None):
        tok = self.peek()
        if expected and tok != expected:
            return None
        self.pos += 1
        return tok

    def parse_det(self):
        if self.peek() in ('the', 'a'):
            return ('Det', self.consume())
        return None

    def parse_noun(self):
        if self.peek() in ('cat', 'dog', 'mat', 'man'):
            return ('N', self.consume())
        return None

    def parse_verb(self):
        if self.peek() in ('sat', 'chased', 'saw'):
            return ('V', self.consume())
        return None

    def parse_prep(self):
        if self.peek() in ('on', 'under', 'with'):
            return ('P', self.consume())
        return None

    def parse_np(self):
        save = self.pos
        det = self.parse_det()
        noun = self.parse_noun()
        if det and noun:
            # 检查可选 PP
            pp = self.parse_pp()
            if pp:
                return ('NP', det, noun, pp)
            return ('NP', det, noun)
        self.pos = save
        return None

    def parse_pp(self):
        save = self.pos
        prep = self.parse_prep()
        np = self.parse_np()
        if prep and np:
            return ('PP', prep, np)
        self.pos = save
        return None

    def parse_vp(self):
        save = self.pos
        verb = self.parse_verb()
        if verb:
            np = self.parse_np()
            if np:
                return ('VP', verb, np)
            pp = self.parse_pp()
            if pp:
                return ('VP', verb, pp)
        self.pos = save
        return None

    def parse_sentence(self):
        np = self.parse_np()
        vp = self.parse_vp()
        if np and vp and self.pos == len(self.tokens):
            return ('S', np, vp)
        return None

def print_tree(tree, indent=0):
    if isinstance(tree, str):
        print(' ' * indent + tree)
    elif isinstance(tree, tuple):
        print(' ' * indent + tree[0])
        for child in tree[1:]:
            print_tree(child, indent + 2)

sentences = [
    "the cat sat on the mat",
    "a dog chased the cat",
]

for sent in sentences:
    tokens = sent.split()
    parser = CFGParser(tokens)
    tree = parser.parse_sentence()
    print(f"\n'{sent}':")
    if tree:
        print_tree(tree)
    else:
        print("  (未找到解析)")
```

3. 通过构建简单的单词图探索词汇关系。给定具有同义、反义和上下义关系的小词汇，在单词之间找到路径。
```python
relations = {
    ('big', 'large'): 'synonym',
    ('big', 'small'): 'antonym',
    ('small', 'tiny'): 'synonym',
    ('dog', 'animal'): 'hypernym',
    ('cat', 'animal'): 'hypernym',
    ('puppy', 'dog'): 'hypernym',
    ('happy', 'glad'): 'synonym',
    ('happy', 'sad'): 'antonym',
    ('hot', 'cold'): 'antonym',
    ('hot', 'warm'): 'synonym',
}

# 构建邻接表
from collections import defaultdict, deque

graph = defaultdict(list)
for (w1, w2), rel in relations.items():
    graph[w1].append((w2, rel))
    graph[w2].append((w1, rel))

def find_path(start, end):
    """BFS 通过关系图在两个单词之间找到路径。"""
    queue = deque([(start, [(start, None)])])
    visited = {start}
    while queue:
        node, path = queue.popleft()
        if node == end:
            return path
        for neighbor, rel in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [(neighbor, rel)]))
    return None

pairs = [('big', 'tiny'), ('puppy', 'cat'), ('happy', 'sad')]
for w1, w2 in pairs:
    path = find_path(w1, w2)
    if path:
        steps = " → ".join(f"{w}({r})" if r else w for w, r in path)
        print(f"{w1} → {w2}: {steps}")
    else:
        print(f"{w1} → {w2}: 未找到路径")
```
