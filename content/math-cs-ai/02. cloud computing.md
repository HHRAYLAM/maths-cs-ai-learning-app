## 📚 第一部分：云计算基础

### 什么是云计算？

Tom 的比喻：

```
传统 IT vs 云计算：

❌ 传统 IT（自己买菜做饭）：
   - 买服务器 = 买锅碗瓢盆
   - 建机房 = 建厨房
   - 雇运维 = 请厨师
   - 前期投入大，不用也放着

✅ 云计算（下馆子）：
   - 按需点菜
   - 按量付费
   - 不用管厨房
   - 随时可以扩容（多订几桌）
```

### NIST 定义的核心特性

| 特性 | Tom 的比喻 | 技术含义 |
|------|-----------|---------|
| 按需自服务 | 🤳 自己点菜 | 用户可自助获取资源 |
| 广泛网络接入 | 📱 随时随地访问 | 通过网络随时访问 |
| 资源池化 | 🏪 共享资源池 | 多租户共享资源 |
| 快速弹性 | 🎈 能伸能缩 | 快速伸缩资源 |
| 可计量服务 | 💰 按量付费 | 使用量可计量计费 |

## 🔑 第二部分：云服务模型

### 三种服务模型

Tom 的"云计算分层"蛋糕：

```
┌─────────────────────────────────────────────────────────┐
│              云计算服务模型金字塔                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ☁️ SaaS (软件即服务) - 最上层                          │
│     Tom 比喻：直接吃现成的奶酪蛋糕                      │
│     例子：Google Docs, Salesforce, Notion               │
│     你负责：数据和使用                                  │
│     云厂商负责：其他一切                                │
│                                                         │
│  🛠️ PaaS (平台即服务) - 中间层                          │
│     Tom 比喻：租厨房和工具，自己做蛋糕                  │
│     例子：Heroku, Google App Engine, Vercel             │
│     你负责：代码和数据                                  │
│     云厂商负责：运行时、中间件、OS 等                   │
│                                                         │
│  🖥️ IaaS (基础设施即服务) - 最下层                      │
│     Tom 比喻：租厨房和炉灶，自带食材和工具              │
│     例子：AWS EC2, Google Compute Engine, Azure VMs     │

```

| 方面 | IaaS | PaaS | SaaS |
|------|------|------|------|
| 控制程度 | 🔧 高 | 🛠️ 中 | 📱 低 |
| 灵活性 | 🎯 高 | 🎯 中 | 🎯 低 |
| 易用性 | 😓 难 | 😐 中 | 😊 易 |
| 运维负担 | 💪 重 | 🙆 轻 | 🎉 无 |
| 适用场景 | 需要完全控制 | 专注开发 | 直接使用 |

## 🚀 第五部分：Serverless 计算

### 什么是 Serverless？

Tom 的"无服务器"魔法：

```
传统云服务器 vs Serverless：

🖥️ 传统云服务器：
   - 需要选择服务器规格
   - 24 小时运行，不管用不用
   - 按时间付费（秒/小时）
   - Tom 比喻：租房子，不住也要付房租

⚡ Serverless（无服务器）：
   - 不用管服务器
   - 有请求才运行
   - 按使用量付费（毫秒/次数）
   - Tom 比喻：住酒店，住几天付几天
```

### Serverless 适用场景

| 场景 | Tom 比喻 | 为什么适合 |
|------|---------|-----------|
| 事件驱动 | 🔔 有人下单才做奶酪 | 按需触发 |
| 间歇性负载 | 📈 双 11 爆单，平时没事 | 自动伸缩 |
| 异步任务 | 📧 发送邮件、生成报告 | 后台运行 |
| API 后端 | 🌐 REST API 服务 | 按请求计费 |

### 代码示例 - AWS Lambda

```python
# Tom 的 Serverless 奶酪订单处理

import json

def lambda_handler(event, context):
    """
    AWS Lambda 函数
    
    Tom 的说明：
    - event: 触发事件（如 API 请求）
    - context: 运行上下文
    """
    
    # 解析订单
    body = json.loads(event.get('body', '{}'))
    cheese_type = body.get('cheese_type', 'cheddar')
    quantity = body.get('quantity', 1)
    
    # 处理订单
    order_total = calculate_price(cheese_type, quantity)
    
    # 返回响应
    return {
        'statusCode': 200,
        'body': json.dumps({
            'message': '订单处理成功！',
            'cheese': cheese_type,
            'quantity': quantity,
            'total': order_total
        })
    }

def calculate_price(cheese_type, quantity):
    """计算奶酪价格"""
    prices = {
        'cheddar': 10,
        'mozzarella': 8,
        'brie': 12,
        'parmesan': 15
    }
    return prices.get(cheese_type, 10) * quantity

# Tom 的 Lambda 配置：
# - 内存：256MB
# - 超时：30 秒
# - 触发器：API Gateway
# - 计费：按调用次数和执行时间
```
