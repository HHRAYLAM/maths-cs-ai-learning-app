## 🏷️ 分类 vs 回归

**监督学习**的两大任务：

### 分类（Classification）

预测**离散类别**：

| 输入 | 输出 |
|------|------|
| 邮件内容 → | 垃圾邮件/正常邮件 |
| 奶酪特征 → | 喜欢/不喜欢 |
| 图片 → | 猫/狗/鸟 |

### 回归（Regression）

预测**连续值**：

| 输入 | 输出 |
|------|------|
| 房屋特征 → | 价格（$） |
| 历史销量 → | 明天销量 |
| 天气数据 → | 明天气温 |

## 🎲 朴素贝叶斯 - 最简单的分类器

**朴素贝叶斯**直接应用贝叶斯定理：

$$P(C_k \mid x) = \frac{P(x \mid C_k) \, P(C_k)}{P(x)}$$

<div class="lesson-two-column">
  <div class="lesson-text-column">

### "朴素"的假设

假设给定类别后每个特征都是**独立的**：

```
如果知道是"好奶酪"：
  "颜色黄"不影响"气味浓"
```

这在现实中几乎从不正确，但分类器**仍然出奇地有效**！

### 分类规则

选择最大化后验概率的类别：

$$\hat{y} = \arg\max_{k} \; P(C_k) \prod_{i=1}^{n} P(x_i \mid C_k)$$

### 三种变体

| 变体 | 适用数据类型 |
|------|------------|
| **多项式** | 计数数据（词频） |
| **高斯** | 连续数据（身高、体重） |
| **伯努利** | 二元数据（存在/不存在） |

</div>
  <div class="lesson-visual-column">

```
朴素贝叶斯分类：

特征：颜色=黄，重量=100g，气味=浓
        ↓
    P(喜欢 | 黄) × P(喜欢 | 100g) × P(喜欢 | 浓) × P(喜欢)
        ↓
    vs
    P(不喜欢 | 黄) × P(不喜欢 | 100g) × ...
        ↓
    哪个大选哪个！
```

### 拉普拉斯平滑

**问题**：当训练数据中某个特征值从未与某个类别一起出现时，似然变为零，整个后验坍缩为零！

**解决**：拉普拉斯平滑

$$P(x_i \mid C_k) = \frac{\text{count}(x_i, C_k) + \alpha}{\text{count}(C_k) + \alpha \cdot V}$$

- $\alpha$ = 平滑参数（通常为 1）
- $V$ = 特征可能值的数量

**效果**：确保没有概率恰好为零！

## 🌳 决策树 - 二十个问题游戏

**决策树**通过一系列是/否问题来划分特征空间。

想象"二十个问题"游戏：在每一步，你问最能缩小可能性的问题！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 树的结构

```
        根节点
       ╱     ╲
    是╱       ╲否
     ╱         ╲
  内部节点    叶节点
   ╱   ╲
是╱     ╲否
```

### 划分过程

1. 从根节点开始（包含所有数据）
2. 选择一个特征和阈值（如"重量<100g？"）
3. 根据答案流向左或右
4. 递归继续直到叶节点
5. 叶节点持有预测

</div>
  <div class="lesson-visual-column">

```
决策树例子：

重量<100g?
   ╱    ╲
  是     否
  ╱       ╲
气味浓？   颜色黄？
 ╱  ╲      ╱   ╲
是   否    是   否
│    │    │    │
喜欢 不喜欢 喜欢 不喜欢

每个叶节点是一个预测！
```

<div class="visual-caption">图 2：决策树通过问题链分类</div>

</div>
</div>

### 不纯度度量

**关键问题**：应该划分哪个特征？

目标：产生最"纯"的子节点（大多数示例属于同一类别）

#### 基尼不纯度

$$\text{Gini}(S) = 1 - \sum_{k=1}^{K} p_k^2$$

- 纯净节点：Gini = 0
- 二元均衡节点：Gini = 0.5（最大）

#### 熵（信息论）

$$H(S) = -\sum_{k=1}^{K} p_k \log_2 p_k$$

- 纯净节点：熵 = 0
- 二元均衡节点：熵 = 1 比特

---

### 信息增益

**信息增益**是划分所减少的不纯度：

$$\text{IG}(S, \text{split}) = H(S) - \frac{|S_L|}{|S|} H(S_L) - \frac{|S_R|}{|S|} H(S_R)$$

算法在每个节点**贪婪地**选择信息增益最高的划分！

---

### 过拟合与剪枝

**问题**：不加控制的树会持续划分直到每个叶节点纯净 → 过拟合（记忆数据）

**解决**：剪枝

| 方法 | 说明 |
|------|------|
| **预剪枝** | 生长前设限：最大深度、最小样本数 |
| **后剪枝** | 先长满树，然后移除不改善验证集的分支 |

### 袋装法（Bootstrap Aggregating）

在不同的随机数据子集上训练多个模型：

```
原始数据：①②③④⑤⑥⑦⑧
          ↓ 有放回抽样
模型 1: ①③③⑤⑦⑧  → 预测 A
模型 2: ②②④⑤⑥⑧  → 预测 B
模型 3: ①②③⑥⑦⑦  → 预测 A
          ↓ 多数投票
最终预测：A
```

每个模型看到约 63% 的原始数据，平均可以消除方差！

---

### 随机森林

**随机森林** = 袋装法 + 特征随机化

在每个划分处，树只考虑特征的随机子集（通常选择 $\sqrt{d}$ 个特征）！

**优点**：
- 进一步去相关树木
- 最可靠的开箱即用分类器之一
- 超参数少

---

### 提升法（Boosting）

与袋装法相反：顺序训练，每个新模型专注于先前模型出错的示例！

#### AdaBoost（自适应提升）

```
步骤 1: 所有样本权重相等
        ↓
步骤 2: 训练弱学习器，错误样本权重增加
        ↓
步骤 3: 再训练，更关注之前的错误
        ↓
最终：所有学习器的加权投票
```

---

### 梯度提升

**梯度提升**推广了 AdaBoost 的思想：

训练每个新模型来预测当前集成的**残差**（损失函数的负梯度）！

**GBDT**（使用决策树的梯度提升）是许多结构化数据竞赛获胜解决方案背后的技术（XGBoost、LightGBM、CatBoost）！

<div class="key-point">
  <div class="key-point-title">🎯 袋装法 vs 提升法</div>
  <ul class="key-point-list">
    <li><strong>袋装法</strong>：独立训练，减少方差（如随机森林）</li>
    <li><strong>提升法</strong>：顺序训练，减少偏差（如 AdaBoost、GBDT）</li>
    <li>袋装法适合过拟合的模型；提升法适合欠拟合的模型</li>
  </ul>
</div>

## 🎯 K 均值聚类 - 最简单的无监督学习

**K 均值聚类**将数据分成 K 个组，最小化每个点到其聚类中心的距离！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 算法步骤

1. 随机初始化 K 个质心
2. **分配**：每个点分配到最近的质心
3. **更新**：重新计算每个质心为分配点的均值
4. 重复 2-3 直到收敛

### 目标函数

最小化**惯性**（簇内平方和）：

$$J = \sum_{k=1}^{K} \sum_{x \in C_k} \|x - \mu_k\|^2$$

### K 均值++ 初始化

- 随机选第一个质心
- 后续质心以与最近质心的距离平方成正比选择
- 几乎总能给出更好结果

</div>
  <div class="lesson-visual-column">

```
K 均值过程：

初始：      迭代 1:     迭代 2:
  • •         • •  A      • •  A
•     A     •   A  •    •   A  •
  B • •   B   • •    B   • •
      B         B           B
质心随机    分配到最近   质心更新

收敛后：
A 组的点都靠近 A 质心
B 组的点都靠近 B 质心
```

<div class="visual-caption">图 3：K 均值交替分配和更新</div>

</div>
</div>

---

### 选择 K 值

#### 肘部法则

绘制惯性与 K 的关系图，寻找"肘部"（增加 K 不再有明显帮助的位置）！

#### 轮廓分数

度量点与自身聚类相比与最近其他聚类的相似程度：

- +1：良好聚类
- 0：在两个聚类边界
- -1：错误聚类

## 📐 高斯混合模型（GMM）- 软分配聚类

**GMM**将数据建模为 K 个高斯分布的混合：

$$P(x) = \sum_{k=1}^{K} \pi_k \, \mathcal{N}(x \mid \mu_k, \Sigma_k)$$

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的 GMM 理解</div>
  <div class="story-card-content">
    <strong>K 均值</strong>：硬分配（非此即彼）
    <br><br>
    <strong>GMM</strong>：软分配（60% 属于 A，40% 属于 B）
    <br><br>
    GMM 更<strong>灵活</strong>，可以处理椭圆形状的聚类！
  </div>
</div>

---

### EM 算法（期望最大化）

GMM 使用 EM 算法拟合：

| 步骤 | 操作 |
|------|------|
| **E 步** | 计算责任：点属于每个高斯的概率 |
| **M 步** | 更新参数：均值、协方差、混合权重 |

**K 均值实际上是 GMM 的特例**：球形高斯 + 硬分配！

## 🥊 支持向量机（SVM）- 最大间隔分类

**SVM**从几何角度处理分类：找到具有**最大间隔**的超平面！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 最大间隔

```
     间隔边界
      ↓
● ● ═══════ ← 超平面
      ↑
     间隔边界

支持向量：正好在间隔边界上的点
```

最近的点称为**支持向量**，是唯一对定义边界重要的点！

### 优化问题

$$\min_{w, b} \; \frac{1}{2}\|w\|^2 \quad \text{s.t.} \quad y_i(w \cdot x_i + b) \geq 1$$

这是凸二次规划，有**唯一全局解**！

</div>
  <div class="lesson-visual-column">

```
SVM 可视化：

  类 A     间隔    类 B
  ● ●    ════    ○ ○
   ●   ═       ═   ○
  ●  ═  支持向量  ═  ○
   ● ═           ═ ○
  ● ●═══════════○ ○

间隔越宽 → 泛化越好！
```

<div class="visual-caption">图 4：SVM 找到最大间隔超平面</div>

</div>
</div>

### 软间隔 SVM

真实数据很少完美可分，引入松弛变量允许一些点违反间隔：

$$\min_{w, b, \xi} \; \frac{1}{2}\|w\|^2 + C \sum_{i=1}^{n} \xi_i$$

超参数 $C$ 控制权衡：
- **大 C**：严重惩罚错误（紧拟合，过拟合风险）
- **小 C**：允许违规（宽间隔，更多正则化）

---

### 核技巧 - SVM 的超能力

**核技巧**让 SVM 在高维空间中计算点积，而不必显式计算变换！

#### 常见核函数

| 核 | 公式 | 特点 |
|---|------|------|
| **RBF** | $\exp(-\gamma \|x_i - x_j\|^2)$ | 映射到无限维，最常用 |
| **多项式** | $(x_i \cdot x_j + c)^d$ | 捕捉特征交互 |
| **线性** | $x_i \cdot x_j$ | 无变换，快速 |

**RBF 核**的 SVM 在深度学习接管之前是主导的分类器！

## 🤖 经典机器学习算法总结

| 算法 | 类型 | 关键优势 | 关键弱点 |
|------|------|---------|---------|
| **朴素贝叶斯** | 监督（生成式） | 快速，适用于少量数据 | 独立性假设 |
| **决策树** | 监督 | 可解释 | 容易过拟合 |
| **随机森林** | 监督（集成） | 稳健，超参数少 | 可解释性差 |
| **梯度提升** | 监督（集成） | 表格数据 SOTA | 较慢，更多调优 |
| **K 均值** | 无监督（聚类） | 简单，可扩展 | 假设球形聚类 |
| **GMM** | 无监督（聚类） | 软分配，灵活形状 | 对初始化敏感 |
| **SVM** | 监督 | 高维空间有效 | 大数据集慢 |
