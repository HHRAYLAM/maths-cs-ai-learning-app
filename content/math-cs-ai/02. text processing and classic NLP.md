# 文本处理和经典 NLP

*文本处理将原始字符转换为模型可以消费的结构化表示。本文件涵盖分词（词、子词、BPE、WordPiece）、文本归一化、编辑距离、TF-IDF、n-gram 语言模型、POS 标注、NER 和情感分析，仍支撑现代系统的经典 NLP 流水线。*

- 原始文本混乱。在任何 NLP 模型可以使用语言前，文本必须清理、归一化，并转换为结构化表示。本文件涵盖从原始字符到模型可以消费的特征的流水线，连同主导深度学习的经典 NLP 算法。

- **文本归一化**将原始文本转换为规范形式。目标是减少无关变异所以 "Hello"、"hello"、"HELLO" 和 "héllo" 被适当对待。

- **大小写折叠**将文本转换为小写。这折叠 "The" 和 "the" 为一个令牌。它对大多数任务有帮助，但在某些情况下破坏有用信息："US"（国家）vs "us"（代词），或 "Apple"（公司）vs "apple"（水果）。

- **Unicode 归一化**处理相同字符可以用多种方式编码的事实。字符 "é" 可以是单个码点（U+00E9）或基 "e" 加组合重音（U+0065 + U+0301）。NFC 归一化组合它们为一个码点；NFD 分解它们。没有归一化，两个看起来相同的字符串可能不匹配。

- **编辑距离**测量两个字符串多不同。**Levenshtein 距离**计数转换一个字符串为另一个所需的最少单字符插入、删除和替换。"kitten" → "sitting" 有编辑距离 3（k→s、e→i、插入 g）。

- 编辑距离用动态规划计算（我们在算法章节回顾）。定义 $D[i][j]$ 为字符串 $s$ 的前 $i$ 个字符和字符串 $t$ 的前 $j$ 个字符之间的距离：

```math
D[i][j] = \begin{cases} j & \text{if } i = 0 \\ i & \text{if } j = 0 \\ D[i{-}1][j{-}1] & \text{if } s[i] = t[j] \\ 1 + \min(D[i{-}1][j], \; D[i][j{-}1], \; D[i{-}1][j{-}1]) & \text{otherwise} \end{cases}
```

- 编辑距离为拼写纠正、模糊匹配和 DNA 序列比对提供动力。在 NLP 中，它用于处理拼写错误和找相似词。

- **分词**将文本分割为离散单元（令牌），模型可以处理。这是第一个且可以说最重要的预处理步骤。分词策略的选择深刻影响模型行为。

- **空白分词**在空格上分割。简单但天真："New York" 成为两个令牌，"don't" 是一个令牌（或分割为 "don" 和 "'t" 取决于分割器），像中文和日语的语言根本没有词间空格。

- **基于规则的分词**用手工模式（正则表达式）处理缩写、标点符号和特殊情况。"I'm" → "I" + "'m"，"U.S.A." 保持为一个令牌。每种语言需要自己的规则，这是劳动密集的。

- **子词分词**是现代解决方案。不是词边界分割，它从数据学习频繁子词单元的词汇。这优雅处理未知词：如果 "unhappiness" 不在词汇中，它可能分割为 "un" + "happi" + "ness"，保留形态结构。

![词级、字符级和子词分词 "unhappiness" 和 "transformers" 的比较](../images/tokenisation_comparison.svg)

- **字节对编码** (BPE) 从单个字符开始作为词汇。它重复找最频繁相邻对并将它们合并为新令牌。足够合并后，常见词是单个令牌，稀有词分割为频繁子词片段。

- BPE 算法：
    1. 用训练语料中所有单个字符初始化词汇
    2. 计数每个相邻令牌对的频率
    3. 合并最频繁对为新令牌
    4. 重复步骤 2-3 对期望合并数（词汇大小）

- 例如，从 "l o w"（5 次）、"l o w e r"（2 次）、"n e w e s t"（6 次）开始：最频繁对可能是 "e s" → 合并为 "es"。然后 "es t" → "est"。然后 "n e w" → "new"。最终词汇包含完整词和子词片段。

- **WordPiece**（BERT 使用）类似 BPE 但基于似然而非频率选择合并。它合并最大化训练数据语言模型似然的对。非词首子词令牌前缀 "##"（例如，"playing" → "play" + "##ing"）。

- **Unigram**（SentencePiece 使用）采取相反方法：从大词汇开始并迭代移除令牌其移除最不小伤害训练数据似然。最终词汇是最好解释语料库的子词单元集。

- **SentencePiece** 是语言无关分词库，将输入视为原始字节流（无空格上的预分词）。这使它适用于任何语言，包括那些无空格的。它实现 BPE 和 Unigram 算法。

- 词汇大小是关键超参数。典型选择范围从 30,000 到 100,000 令牌。更大词汇意味着每序列更少令牌（更高效）但更大嵌入表。更小词汇意味着更多子词分割和更长序列。

- 两种技术都将词还原为基础形式，但它们方法不同。

- **词干提取**用粗糙规则砍掉后缀。Porter 词干提取器减少 "running" 为 "run"，"happiness" 为 "happi"，"studies" 为 "studi"。它快但不精确："university" 和 "universe" 都词干为 "univers" 尽管不相关。

- **词形还原**用词汇和形态分析找真字典形式（词元）。"Running" → "run"，"better" → "good"，"mice" → "mouse"。它需要知道词性："saw" 作为动词词形还原为 "see"，但作为名词它保持 "saw"。

- 现代子词分词已大体替换神经 NLP 中的词干提取和词形还原，但它们仍在信息检索有用或当与更小模型或有限数据工作时。

- **词性** (POS) **标注**分配语法类别给每个词：名词、动词、形容词、限定词等。这是最古老 NLP 任务之一，对句法分析基础。

- Penn Treebank 标签集对英语最常见，带 36 个标签（NN 单数名词，NNS 复数名词，VB 基动词，VBD 过去时，JJ 形容词等）。

- POS 标注棘手因为许多词歧义。"Book" 可以是名词（"the book"）或动词（"book a flight"）。"Run" 有跨词性数十种意义。上下文 essential。

- 早期标注器用第 05 章的**隐马尔可夫模型** (HMMs)。隐藏状态是 POS 标签，观察是词。转移概率捕捉标签序列（限定词可能后跟名词或形容词），发射概率捕捉哪些词与哪些标签一起出现。Viterbi 算法找最可能标签序列。

- POS 标注的 HMM 模型：

$$\hat{t}_{1:n} = \arg\max_{t_{1:n}} \prod_{i=1}^{n} P(w_i \mid t_i) \cdot P(t_i \mid t_{i-1})$$

- 现代 POS 标注器用神经网络（双向 LSTM 或 transformers）并实现超过 97% 准确率在英语上，接近人类性能。

- **命名实体识别** (NER) 识别和分类文本中专有名称和其他特定实体：人、组织、位置、日期、金额等。

- 在 "Apple CEO Tim Cook announced the event in Cupertino on Monday," NER 系统应识别：Apple (ORG)、Tim Cook (PER)、Cupertino (LOC)、Monday (DATE)。

- NER 通常框架为**序列标注**使用**BIO 标注**（也称为 IOB 标注）。每个令牌获得标签：
    - **B-TYPE**：TYPE 实体开始
    - **I-TYPE**：TYPE 实体内部（继续）
    - **O**：任何实体外部

- "Tim Cook visited New York" 变为：Tim/B-PER Cook/I-PER visited/O New/B-LOC York/I-LOC。B 标签标记新实体何时开始，这重要当两个相同类型实体相邻时。

![带 BIO 标签的句子颜色编码：B-PER（红色）、I-PER（红色）、O（灰色）、B-LOC（蓝色）、I-LOC（蓝色）](../images/bio_tagging.svg)

- 经典 NER 用第 05 章的**条件随机场** (CRFs)，它建模给定输入的整个标签序列的条件概率。不像 HMMs 是生成的 ($P(x, y)$)，CRFs 是判别的并直接建模 $P(y \mid x)$。线性链 CRF 定义：

$$P(y_{1:n} \mid x_{1:n}) = \frac{1}{Z(x)} \exp\!\left(\sum_{i=1}^{n} \left[\sum_k \lambda_k f_k(y_i, x, i) + \sum_j \mu_j g_j(y_i, y_{i-1}, x, i)\right]\right)$$

- 这里 $f_k$ 是**发射特征**（标签 $y_i$ 多可能给定位置 $i$ 的输入）和 $g_j$ 是**转移特征**（标签 $y_i$ 多可能给定先前标签 $y_{i-1}$）。

- 配分函数 $Z(x) = \sum_{y'} \exp(\ldots)$ 对所有可能标签序列求和归一化分布。训练最大化条件对数似然，需要用前向算法（第 05 章）高效计算 $Z(x)$。

- 关键优势超越独立分类每个令牌：CRF 的转移特征强制结构约束（例如，I-PER 应仅跟随 B-PER 或 I-PER，从不在 O 后出现）。

- 现代 NER 堆叠 CRF 在神经编码器上（BiLSTM-CRF 或 BERT-CRF），其中神经网络产生发射分数且 CRF 层学习转移结构。

- **句法分析**将句子转换为其句法结构，要么成分树要么依存树（都来自文件 01）。

- **CYK 算法**（Cocke-Younger-Kasami）用动态规划解析带上下文无关语法的句子。

- 它要求语法是**Chomsky 正规式**（每规则有要么两个非终端要么一个终端在右侧）。它自底向上填充三角表：单元格表示句子的跨度，每个单元格存储可以生成该跨度的非终端。

- CYK 在 $O(n^3 \cdot |G|)$ 时间运行，其中 $n$ 是句子长度，$|G|$ 是语法大小。这是精确但慢对大语法。

- **移进 - 归约分析**从左到右处理句子，维护栈。每步，它要么**移进**（推下一个词到栈）要么**归约**（从栈弹出元素并用短语替换它们）。训练的分类器在每步决定行动。这在 $O(n)$ 时间运行，使它快得多比 CYK。

- **依存分析**现在比成分分析在实践中更常见。基于转移的依存解析器（如移进 - 归约）和基于图的解析器（评分所有可能边并找最大生成树）是两种主要方法。用 BiLSTMs 或 transformers 的神经依存解析器实现最先进结果。

- 在嵌入前，NLP 用简单计数方法表示文档为向量。

- **词袋** (BoW) **模型**表示文档为词计数向量，完全忽略词顺序。如果词汇有 $V$ 个词，每个文档是 $\mathbb{R}^V$ 中的向量（连接回第 01 章的向量空间）。词 $w$ 的条目是 $w$ 在文档中出现次数。

![词袋：文档转换为词计数表，然后到 $\mathbb{R}^V$ 中的稀疏向量带每个词汇词一个条目](../images/bag_of_words.svg)

- BoW 简单但惊人有效对任务如文档分类和垃圾邮件过滤。它主要弱点是它平等对待每个词："the" 和 "revolutionary" 获得相等权重。

- **TF-IDF**（词频 - 逆文档频率）通过基于多信息性加权词修复这个。在一个文档频繁出现但跨语料稀有的词可能对那个文档重要。

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

- **词频** $\text{TF}(t, d)$ 通常是项 $t$ 在文档 $d$ 中的原始计数（或其对数：$1 + \log(\text{count})$）。

- **逆文档频率** $\text{IDF}(t) = \log\frac{N}{|\{d : t \in d\}|}$，其中 $N$ 是文档总数。出现在每个文档的词（如 "the"）获得 IDF 接近 0。稀有词获得高 IDF。

- TF-IDF 向量可以用余弦相似度比较（来自第 01 章）测量文档相似度。这是经典信息检索和搜索引擎的基础。

- **语言模型**分配概率到词序列。它回答：这个句子多可能？语言模型对机器翻译、语音识别、拼写纠正和文本生成中心。

- 句子 $w_1, w_2, \ldots, w_n$ 的概率，由概率链式法则（第 05 章）：

$$P(w_1, w_2, \ldots, w_n) = \prod_{i=1}^{n} P(w_i \mid w_1, \ldots, w_{i-1})$$

- 这是精确但不切实际：你需要存储每个可能历史的概率。**马尔可夫假设**（第 05 章）截断历史到最后 $k-1$ 个词，给出**n-gram 模型**（其中 $n = k$）。

- **bigram 模型** ($n = 2$）仅条件于前一个词：

$$P(w_i \mid w_1, \ldots, w_{i-1}) \approx P(w_i \mid w_{i-1})$$

- **trigram 模型** ($n = 3$）条件于前两个词。N-gram 概率通过语料计数估计：

$$P(w_i \mid w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i)}{\text{count}(w_{i-1})}$$

- **困惑度**测量语言模型多好预测测试集。它是测试集的反概率，按词数归一化：

$$\text{PPL} = P(w_1, \ldots, w_N)^{-1/N} = \exp\!\left(-\frac{1}{N} \sum_{i=1}^{N} \log P(w_i \mid w_{<i})\right)$$

- 更低困惑度意味着模型对测试数据更少"惊讶"因此更好。在 10,000 词词汇上分配均匀概率的模型有困惑度 10,000。好 bigram 模型可能实现困惑度约 200。现代神经语言模型实现困惑度低于 20。

- 注意困惑度是指数化交叉熵（来自第 05 章信息论）。训练期间最小化交叉熵损失直接最小化困惑度。

- **平滑**处理零概率问题：如果 n-gram 从不在训练出现，模型分配它概率 0，这使整个句子概率 0。**拉普拉斯平滑** (add-1) 添加小计数到每个 n-gram：

$$P_{\text{Laplace}}(w_i \mid w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i) + 1}{\text{count}(w_{i-1}) + V}$$

- 这对大词汇太激进（它从观察 n-gram 偷太多概率）。**Kneser-Ney 平滑**是 n-gram 模型的金标准。它组合两个思想：绝对折扣和回退的继续概率。

- 首先，**绝对折扣**从每个观察计数减固定折扣 $d$（通常 $d \approx 0.75$），而非添加伪计数。释放概率质量重新分配给未观察 n-gram。插值形式是：

$$P_{\text{KN}}(w_i \mid w_{i-1}) = \frac{\max(\text{count}(w_{i-1}, w_i) - d, \; 0)}{\text{count}(w_{i-1})} + \lambda(w_{i-1}) \cdot P_{\text{cont}}(w_i)$$

- 其中 $\lambda(w_{i-1})$ 是归一化常数分配折扣质量。关键创新是**继续概率** $P_{\text{cont}}(w_i)$，它测量 $w_i$ 出现在多少不同上下文，而非它多频繁整体出现：

$$P_{\text{cont}}(w_i) = \frac{|\{w' : \text{count}(w', w_i) > 0\}|}{|\{(w', w'') : \text{count}(w', w'') > 0\}|}$$

- 分子计数语料中多少不同词在 $w_i$ 前。像 "Francisco" 的词出现在少上下文（几乎总是在 "San" 后），所以即使 "San Francisco" 非常频繁，"Francisco" 获得低继续概率并将不在其他上下文错误预测。

- 相反，常见词如 "the" 出现在许多不同词后并获得高继续概率。这捕获直觉：词的多样性比其原始频率对回退估计更重要。

- N-gram 模型是几十年的最先进技术。它们快、可解释、无需训练（仅计数）。但它们挣扎长程依赖（"The keys that I left on the table **are** missing" 需要知道主语 "keys" 是复数，它远离动词）。神经语言模型，从 RNNs 开始到 transformers 结束，解决这个限制。

1. 用动态规划实现 Levenshtein 编辑距离。在词对上测试它并用它于简单拼写纠正。
```python
import jax.numpy as jnp

def edit_distance(s, t):
    """用动态规划计算 Levenshtein 编辑距离。"""
    m, n = len(s), len(t)
    D = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(m + 1):
        D[i][0] = i
    for j in range(n + 1):
        D[0][j] = j

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s[i-1] == t[j-1]:
                D[i][j] = D[i-1][j-1]
            else:
                D[i][j] = 1 + min(D[i-1][j], D[i][j-1], D[i-1][j-1])

    return D[m][n]

# 测试
pairs = [("kitten", "sitting"), ("sunday", "saturday"), ("hello", "hallo")]
for s, t in pairs:
    print(f"d('{s}', '{t}') = {edit_distance(s, t)}")

# 简单拼写纠正
dictionary = ["the", "their", "there", "then", "than", "this", "that", "these", "those"]
misspelled = "thier"
corrections = sorted(dictionary, key=lambda w: edit_distance(misspelled, w))
print(f"\n最接近 '{misspelled}': {corrections[:3]}")
```

2. 从头实现 BPE 分词。从字符级令牌开始并迭代合并最频繁对。
```python
from collections import Counter

def get_pairs(corpus):
    """计数所有词中相邻令牌对。"""
    pairs = Counter()
    for word, freq in corpus.items():
        symbols = word.split()
        for i in range(len(symbols) - 1):
            pairs[(symbols[i], symbols[i+1])] += freq
    return pairs

def merge_pair(pair, corpus):
    """合并语料中对的所有出现。"""
    new_corpus = {}
    bigram = ' '.join(pair)
    replacement = ''.join(pair)
    for word, freq in corpus.items():
        new_word = word.replace(bigram, replacement)
        new_corpus[new_word] = freq
    return new_corpus

# 训练语料带词频率
text = "low low low low low lower lower newest newest newest newest newest newest"
word_freqs = Counter(text.split())
# 初始化：分割每个词为字符带词尾标记
corpus = {' '.join(word) + ' _': freq for word, freq in word_freqs.items()}

print("初始语料：")
for word, freq in corpus.items():
    print(f"  {word}: {freq}")

# 运行 BPE 10 次合并
for i in range(10):
    pairs = get_pairs(corpus)
    if not pairs:
        break
    best_pair = max(pairs, key=pairs.get)
    corpus = merge_pair(best_pair, corpus)
    print(f"\n合并 {i+1}: {best_pair} (freq={pairs[best_pair]})")
    for word, freq in corpus.items():
        print(f"  {word}: {freq}")
```

3. 构建 bigram 语言模型并计算测试句子的困惑度。实验拉普拉斯平滑。
```python
from collections import Counter, defaultdict
import math

# 训练语料
train = """the cat sat on the mat . the dog chased the cat .
the cat ran from the dog . a dog sat on a mat .""".split()

# 计数 bigrams 和 unigrams
bigrams = Counter(zip(train[:-1], train[1:]))
unigrams = Counter(train)
vocab_size = len(set(train))

def bigram_prob(w2, w1, alpha=0):
    """P(w2 | w1) 带可选拉普拉斯平滑。"""
    return (bigrams[(w1, w2)] + alpha) / (unigrams[w1] + alpha * vocab_size)

# 计算困惑度
test = "the cat sat on a mat .".split()

for alpha in [0, 1, 0.1]:
    log_prob = 0
    for w1, w2 in zip(test[:-1], test[1:]):
        p = bigram_prob(w2, w1, alpha=alpha)
        if p > 0:
            log_prob += math.log(p)
        else:
            log_prob += float('-inf')

    ppl = math.exp(-log_prob / (len(test) - 1)) if log_prob > float('-inf') else float('inf')
    print(f"平滑 α={alpha}: perplexity = {ppl:.2f}")
```

4. 从头实现 TF-IDF 并用余弦相似度找最相似文档到查询。
```python
import jax.numpy as jnp
import math
from collections import Counter

documents = [
    "the cat sat on the mat",
    "the dog chased the cat around the park",
    "a mat was placed on the floor by the door",
    "the quick brown fox jumped over the lazy dog",
]

# 构建词汇
vocab = sorted(set(word for doc in documents for word in doc.split()))
word_to_idx = {w: i for i, w in enumerate(vocab)}
V = len(vocab)
N = len(documents)

# 计算 TF-IDF 矩阵
doc_freq = Counter()
for doc in documents:
    for word in set(doc.split()):
        doc_freq[word] += 1

tfidf_matrix = jnp.zeros((N, V))
for i, doc in enumerate(documents):
    word_counts = Counter(doc.split())
    for word, count in word_counts.items():
        tf = 1 + math.log(count)
        idf = math.log(N / doc_freq[word])
        j = word_to_idx[word]
        tfidf_matrix = tfidf_matrix.at[i, j].set(tf * idf)

# 查询
query = "cat on the mat"
query_vec = jnp.zeros(V)
query_counts = Counter(query.split())
for word, count in query_counts.items():
    if word in word_to_idx:
        tf = 1 + math.log(count)
        idf = math.log(N / doc_freq.get(word, 1))
        query_vec = query_vec.at[word_to_idx[word]].set(tf * idf)

# 余弦相似度（来自第 01 章）
def cosine_sim(a, b):
    return jnp.dot(a, b) / (jnp.linalg.norm(a) * jnp.linalg.norm(b) + 1e-8)

print(f"查询：'{query}'\n")
for i, doc in enumerate(documents):
    sim = cosine_sim(query_vec, tfidf_matrix[i])
    print(f"  文档 {i} (sim={sim:.3f}): '{doc}'")
```
