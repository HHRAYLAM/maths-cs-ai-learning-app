## 🧹 文本归一化 - 清理混乱文本

文本归一化将原始文本转换为规范形式：

| 操作 | 说明 | 例子 |
|------|------|------|
| 大小写折叠 | 转小写 | "Hello" → "hello" |
| Unicode 归一化 | 统一字符编码 | "é" → 单码点 |
| 去除多余空格 | 标准化空白 | "a   b" → "a b" |
| 处理标点 | 统一或移除 | "!!!" → "!" |

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 大小写折叠的权衡

转小写的好处：
- "The" 和 "the" 视为同一词
- 减少词汇量

问题：
- "US"（美国）vs "us"（我们）
- "Apple"（公司）vs "apple"（水果）

解决：保留大小写作为额外特征！

### Unicode 归一化

```
"é" 可以有两种编码：

1. 单码点：U+00E9（é）
2. 组合：e (U+0065) + ´ (U+0301)

看起来一样，但计算机认为是不同的！

NFC：组合成单码点
NFD：分解成多码点
```

         ↓ 转小写

干净文本 ready for processing！

## 📏 编辑距离 - 测量字符串差异

编辑距离（Levenshtein 距离）：转换一个字符串为另一个所需的最少操作数：

```
"kitten" → "sitting"

步骤 1: k→s  (替换)
步骤 2: e→i  (替换)
步骤 3: 插入 g (在末尾)

编辑距离 = 3
```

### 动态规划公式

$$D[i][j] = \begin{cases} 
j & \text{if } i = 0 \\
i & \text{if } j = 0 \\
D[i-1][j-1] & \text{if } s[i] = t[j] \\
1 + \min(D[i-1][j], D[i][j-1], D[i-1][j-1]) & \text{otherwise}
\end{cases}$$

## ✂️ 分词 - 切分文本

分词将文本分割成离散单元（token），是 NLP 的第一步！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 分词方法对比

| 方法 | 说明 | 问题 |
|------|------|------|
| 空白分词 | 空格分割 | "New York" 变两词 |
| 基于规则 | 正则表达式 | 每语言需要自己的规则 |
| 子词分词 | 学习频繁子词 | 现代方案，处理未知词 |

### 空白分词的问题

```
英文："don't" → "don" + "'t"? 还是保持一个词？
中文："我喜欢你" → 根本没有空格！
日文："こんにちは" → 也没有空格

空白分词太天真！
```

### BPE（字节对编码）

BPE 从字符开始，反复合并最频繁相邻对：

```
BPE 算法步骤：

1. 初始化：词汇 = 所有单个字符
2. 统计相邻对频率
3. 合并最频繁对为新 token
4. 重复 2-3 直到达到词汇大小

例子：
训练语料："low", "low", "low", "low", "lower", "new", "new", "newest"

迭代 1: 最频繁对是 "e"-"s" → 合并为 "es"
迭代 2: 最频繁对是 "es"-"t" → 合并为 "est"
迭代 3: 最频繁对是 "n"-"ew" → 合并为 "new"

最终词汇包含完整词和子词片段！
```

---

### WordPiece（BERT 使用）

WordPiece 类似 BPE，但基于似然而非频率选择合并：

```
WordPiece vs BPE：

BPE：合并频率最高的对
WordPiece：合并最大化语言模型似然的对

WordPiece 标记：
"playing" → "play" + "##ing"
           (词根)   (后缀，带##前缀)

## 前缀表示"这不是词首"
```

---

### SentencePiece

SentencePiece 是语言无关的分词库：

| 特性 | 说明 |
|------|------|
| 无预分词 | 直接处理原始字节流 |
| 支持任何语言 | 包括中文、日文等无空格语言 |
| 实现 BPE 和 Unigram | 两种算法可选 |

Unigram 算法：从大词汇开始，迭代移除对似然影响最小的 token。

---

### 词汇大小选择

```
典型词汇大小：30,000 - 100,000 tokens

大词汇（100k）：
✓ 每序列 token 少，更高效
✗ 嵌入表大，内存占用多

小词汇（30k）：
✓ 嵌入表小
✗ 更多子词分割，序列更长

平衡点：50k 左右常用
```

## 🏷️ 词性标注（POS Tagging）

词性标注给每个词分配语法类别：

| 标签 | 含义 | 例子 |
|------|------|------|
| NN | 名词单数 | cat, dog |
| NNS | 名词复数 | cats, dogs |
| VB | 动词基式 | run, eat |
| VBD | 动词过去式 | ran, ate |
| JJ | 形容词 | happy, big |
| DT | 限定词 | the, a |
| IN | 介词 | in, on, at |

---

### 歧义问题

```
"Book" 可以是：
- NN 名词："the book"（书）
- VB 动词："book a flight"（预订）

"Run" 有数十种用法：
- 动词："I run"
- 名词："a run in my stocking"
- 形容词："runny" 的词根

上下文 essential！
```

### HMM 模型（隐马尔可夫模型）

$$\hat{t}_{1:n} = \arg\max_{t_{1:n}} \prod_{i=1}^{n} P(w_i \mid t_i) \cdot P(t_i \mid t_{i-1})$$

- 转移概率 $P(t_i \mid t_{i-1})$：标签序列的可能性
- 发射概率 $P(w_i \mid t_i)$：哪个词与哪个标签一起出现

Viterbi 算法找到最可能的标签序列！

## 🎯 命名实体识别（NER）

NER 识别文本中专有名称和其他特定实体：

```
输入："Apple CEO Tim Cook announced the event in Cupertino on Monday"

输出：
- Apple → 组织 (ORG)
- Tim Cook → 人名 (PER)
- Cupertino → 地点 (LOC)
- Monday → 日期 (DATE)
```

<div class="lesson-two-column">
  <div class="lesson-text-column">

### BIO 标注体系

| 标签 | 含义 |
|------|------|
| B-TYPE | 实体开始 |
| I-TYPE | 实体内部（继续） |
| O | 非实体 |

例子："Tim Cook visited New York"

```
Tim   → B-PER
Cook  → I-PER
visited → O
New   → B-LOC
York  → I-LOC
```

B 标签很重要：区分相邻同类型实体

"I saw John B-PER Smith I-PER and John B-PER I-PER"
（两个人，都叫 John）

</div>

```
BIO 标注可视化：

句子：Tim Cook visited New York
       ┌─────┐         ┌──────┐
       │PER  │         │ LOC  │
       │B I  │         │ B  I │
       └─────┘         └──────┘
       
颜色编码：
红色 = PER（人名）
蓝色 = LOC（地点）
灰色 = O（非实体）
```

### 条件随机场（CRF）

CRF 是判别式序列标注模型：

$$P(y_{1:n} \mid x_{1:n}) = \frac{1}{Z(x)} \exp\left(\sum_{i=1}^{n} \left[\sum_k \lambda_k f_k(y_i, x, i) + \sum_j \mu_j g_j(y_i, y_{i-1}, x, i)\right]\right)$$

- 发射特征 $f_k$：标签 $y_i$ 给定输入 $x$ 的可能性
- 转移特征 $g_j$：标签 $y_i$ 给定前一个标签 $y_{i-1}$ 的可能性

关键优势：强制结构约束

```
CRF 学习的约束：
✓ I-PER 可以跟随 B-PER 或 I-PER
✗ I-PER 不能跟随 O

这防止无效标注序列！
```

## 📊 句法分析（Parsing）

句法分析将句子转换为句法结构：

### CYK 算法

| 特性 | 说明 |
|------|------|
| 方法 | 动态规划，自底向上 |
| 要求 | 语法需为 Chomsky 正规式 |
| 复杂度 | $O(n^3 \cdot \|G\|)$ |
| 优点 | 精确 |
| 缺点 | 慢 |

---

### 移进 - 归约分析

```
移进 - 归约过程：

栈：[]        输入：["The", "cat", "sat"]
         ↓ 移进
栈：["The"]   输入：["cat", "sat"]
         ↓ 移进
栈：["The", "cat"]  输入：["sat"]
         ↓ 归约 (Det + N → NP)
栈：["NP"]    输入：["sat"]
         ↓ 移进
栈：["NP", "sat"]  输入：[]
         ↓ 归约 (NP + V → VP)
栈：["VP"]    输入：[]
         ↓ 归约 (NP + VP → S)
栈：["S"]     输入：[]

完成！
```

复杂度：$O(n)$，比 CYK 快得多！

## 📐 词袋模型（BoW）

词袋将文档表示为词计数向量：

  <div class="story-card-content">
    <strong>词袋模型</strong>：只看词频，不管顺序
    <br><br>
    "The cat sat" = [the:1, cat:1, sat:1]
    <br><br>
    "Sat cat the" = [the:1, cat:1, sat:1]
    <br><br>
    两者向量相同！丢失了语法结构
  </div>
</div>

---

### BoW 例子

```
词汇：["apple", "banana", "cat", "dog"]

文档 1: "I have an apple and a banana"
向量：[1, 1, 0, 0]

文档 2: "My cat and my dog"
向量：[0, 0, 1, 1]

文档 3: "apple banana cat dog"
向量：[1, 1, 1, 1]

向量空间是 R^4！
```

优点：简单，有效于文档分类、垃圾邮件过滤

缺点：平等对待每个词，"the" 和 "revolutionary" 权重相同

## 📈 TF-IDF - 加权词重要性

TF-IDF 根据词的信息性加权：

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 词频（TF）

$$\text{TF}(t, d) = \text{词 t 在文档 d 中出现的次数}$$

或用对数缩放：$1 + \log(\text{count})$

### 逆文档频率（IDF）

$$\text{IDF}(t) = \log \frac{N}{|\{d : t \in d\}|}$$

- $N$ = 文档总数
- 分母 = 包含词 $t$ 的文档数

直观：
- 常见词（"the"）→ IDF ≈ 0
- 稀有词 → IDF 高

</div>
  <div class="lesson-visual-column">

```
TF-IDF 例子：

语料库有 1000 篇文档：

词 "the"：
- 在 990 篇文档出现
- IDF = log(1000/990) ≈ 0.01
- TF-IDF 很低！

词 "revolutionary"：
- 只在 5 篇文档出现
- IDF = log(1000/5) ≈ 5.3
- TF-IDF 很高！

稀有词更重要！
```

## 📝 N-gram 语言模型

语言模型给词序列分配概率：

$$P(w_1, w_2, \ldots, w_n) = \prod_{i=1}^{n} P(w_i \mid w_1, \ldots, w_{i-1})$$

问题：需要存储每个可能历史的概率！

解决：马尔可夫假设 — 只考虑最后 $k-1$ 个词

---

### N-gram 类型

| N-gram | 条件于 | 例子 |
|--------|--------|------|
| Unigram (n=1) | 无历史 | $P(w_i)$ |
| Bigram (n=2) | 前 1 个词 | $P(w_i \mid w_{i-1})$ |
| Trigram (n=3) | 前 2 个词 | $P(w_i \mid w_{i-1}, w_{i-2})$ |

概率估计：

$$P(w_i \mid w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i)}{\text{count}(w_{i-1})}$$

---

### 困惑度（Perplexity）

困惑度衡量语言模型预测测试集的好坏：

$$\text{PPL} = \exp\left(-\frac{1}{N} \sum_{i=1}^{N} \log P(w_i \mid w_{<i})\right)$$

| 困惑度 | 含义 |
|--------|------|
| 10,000 | 均匀随机猜测（10k 词汇） |
| ~200 | 好的 bigram 模型 |
| <20 | 现代神经语言模型 |

越低越好！困惑度 = 指数化交叉熵
