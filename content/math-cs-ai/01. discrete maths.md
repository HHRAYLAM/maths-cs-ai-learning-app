# 离散数学

*离散数学是计数、分离结构的数学，是计算构建的基础。本文件涵盖命题和谓词逻辑、证明技术、集合、关系、函数、图论基础和递推关系*

- 在前面的章节中，我们研究了连续数学：微积分（第 3 章）、概率分布（第 5 章）和实值参数上的优化（第 6 章）。但计算机本质上是离散机器。它们存储位（0 或 1）、处理整数、遵循分支逻辑，并操作有限数据结构。离散数学为推理这些结构提供了形式语言。

- 本章中的所有内容都依赖于离散数学：处理器逻辑门是布尔代数，调度算法需要正确性证明，内存管理使用集合运算，算法分析需要递推关系。

## 命题逻辑

- 命题逻辑是真/假陈述的代数。命题是一个要么为真（T）要么为假（F）的陈述，从不同时为两者。"正在下雨"是一个命题。"现在几点了？"不是（它是一个问题，不是具有真值的陈述）。

- 命题可以使用逻辑连接词组合：

    - AND（合取，$p \wedge q$）：仅当 $p$ 和 $q$ 都为真时为真。
    - OR（析取，$p \vee q$）：当 $p$ 或 $q$ 至少一个为真时为真。
    - NOT（否定，$\neg p$）：翻转真值。
    - IMPLIES（蕴含，$p \to q$）：仅当 $p$ 为真且 $q$ 为假时为假。"如果下雨，地面是湿的"仅在下雨但地面干燥时被违反。
    - IFF（双条件，$p \leftrightarrow q$）：当两者具有相同真值时为真。

- 真值表详尽列出所有可能的输入组合和结果输出。对于 $n$ 个命题，表有 $2^n$ 行。这是我们验证逻辑等价的方式：

| $p$ | $q$ | $p \wedge q$ | $p \vee q$ | $p \to q$ |
|-----|-----|--------------|------------|-----------|
| T | T | T | T | T |
| T | F | F | T | F |
| F | T | F | T | T |
| F | F | F | F | T |

- 蕴含行中 $p$ 为假的情况值得注意：$F \to q$ 总是为真，无论 $q$ 是什么。这是空真。"如果猪会飞，那么我是英国国王"在逻辑上为真，因为前提是假的。这似乎违反直觉，但对于数学推理至关重要。

- 逻辑等价是对所有真值都成立的恒等式：

    - 德摩根定律：$\neg(p \wedge q) \equiv \neg p \vee \neg q$ 和 $\neg(p \vee q) \equiv \neg p \wedge \neg q$。否定 AND，否定每个部分并切换为 OR（反之亦然）。这些直接出现在编程中：`!(a && b)` 等价于 `(!a || !b)`。

    - 逆否命题：$p \to q \equiv \neg q \to \neg p$。"如果下雨，地面是湿的"等价于"如果地面不湿，则没有下雨"。这是一个强大的证明技术。

    - 双重否定：$\neg(\neg p) \equiv p$。

    - 分配律：$p \wedge (q \vee r) \equiv (p \wedge q) \vee (p \wedge r)$。

- 总是为真（对于所有真值赋值）的公式是重言式。总是为假的是矛盾。有时为真有时为假的是偶然。例如，$p \vee \neg p$ 是重言式，$p \wedge \neg p$ 是矛盾。

## 谓词逻辑和量词

- 命题逻辑不能表达关于集合中*所有*或*某些*元素的陈述。"每个大于 2 的素数都是奇数"需要谓词逻辑，它用变量、谓词和量词扩展命题逻辑。

- 谓词是依赖于变量的陈述：$P(x)$ = "$x$ 是偶数。"当 $x$ 给定特定值时它成为命题：$P(4)$ 为真，$P(7)$ 为假。

- 量词表达范围：

    - 全称量词（$\forall$）："对所有。"$\forall x \, P(x)$ 表示"$P(x)$ 对域中的每个 $x$ 为真。"
    - 存在量词（$\exists$）："存在。"$\exists x \, P(x)$ 表示"存在至少一个 $x$ 使得 $P(x)$ 为真。"

- 否定量词会翻转它们：$\neg(\forall x \, P(x)) \equiv \exists x \, \neg P(x)$。"不是每个人都通过了"意味着"有人失败了。"并且 $\neg(\exists x \, P(x)) \equiv \forall x \, \neg P(x)$。"没有完美的算法"意味着"每个算法都有缺陷。"

- 嵌套量词表达复杂关系。$\forall x \, \exists y \, (y > x)$ 表示"对于每个数，存在一个更大的数"（对整数为真）。顺序很重要：$\exists y \, \forall x \, (y > x)$ 表示"存在一个数大于所有其他数"（对整数为假）。

- 谓词逻辑是形式规范的语言。当我们说算法"正确"时，我们意味着 $\forall \text{inputs} \, x, \, \text{output}(x) = \text{desired}(x)$。当我们说它"终止"时，我们意味着 $\forall x \, \exists t \, \text{halts}(x, t)$。

## 证明技术

- 证明是一个逻辑论证，毫无置疑地建立陈述的真理性。与经验证据（仅显示在测试情况下有效）不同，证明保证它在所有情况下都有效。这是计算机科学中的正确性标准。

- 直接证明：假设前提，通过逻辑步骤推导结论。要证明"如果 $n$ 是偶数，则 $n^2$ 是偶数"：假设 $n = 2k$ 对于某个整数 $k$，则 $n^2 = 4k^2 = 2(2k^2)$，这是偶数。

- 反证法：假设陈述为假并推导矛盾。要证明 $\sqrt{2}$ 是无理数：假设 $\sqrt{2} = a/b$（完全约分）。则 $2 = a^2/b^2$，所以 $a^2 = 2b^2$，意味着 $a^2$ 是偶数，所以 $a$ 是偶数，设 $a = 2c$。则 $4c^2 = 2b^2$，所以 $b^2 = 2c^2$，意味着 $b$ 也是偶数。但我们说过 $a/b$ 是完全约分的——矛盾。

- 数学归纳法证明：通过对所有自然数证明陈述：(1) 基础情况成立（通常 $n = 0$ 或 $n = 1$），和 (2) 归纳步骤：如果陈述对 $n = k$ 成立（归纳假设），则它对 $n = k + 1$ 成立。

- 例如，证明 $\sum_{i=1}^{n} i = \frac{n(n+1)}{2}$：
    - 基础情况：$n = 1$：$1 = \frac{1 \cdot 2}{2} = 1$。为真。
    - 归纳步骤：假设 $\sum_{i=1}^{k} i = \frac{k(k+1)}{2}$。则 $\sum_{i=1}^{k+1} i = \frac{k(k+1)}{2} + (k+1) = \frac{k(k+1) + 2(k+1)}{2} = \frac{(k+1)(k+2)}{2}$。这是 $n = k+1$ 时的公式。完成。

- 归纳法是证明递归算法和数据结构性质的主力。每个递归算法都有一个隐式的归纳正确性证明：基础情况是终止条件，归纳步骤是递归调用。

- 强归纳法假设陈述对直到 $k$ 的所有值成立（而不仅仅是 $k$），然后证明它对 $k + 1$ 成立。当递归依赖于不止前一个值时这很有用。

- 鸽巢原理：如果将 $n+1$ 个物体放入 $n$ 个盒子中，至少一个盒子包含两个物体。简单但令人惊讶地强大。它证明在任何 13 人群体中，至少两人共享生日月份。在网络中，它证明当项目多于桶时哈希碰撞是不可避免的。

## 集合

- 集合是无序的不同元素的集合。集合是数学中最原始的数据结构，它们支撑着从类型系统到数据库查询的一切。

- 集合运算（连接到第 5 章，我们在概率中使用这些）：

    - 并集 $A \cup B$：在 $A$ 或 $B$ 或两者中的元素。
    - 交集 $A \cap B$：同时在 $A$ 和 $B$ 中的元素。
    - 补集 $\bar{A}$：不在 $A$ 中的元素（相对于全集）。
    - 差集 $A \setminus B$：在 $A$ 但不在 $B$ 中的元素。
    - 笛卡尔积 $A \times B$：所有有序对 $(a, b)$，其中 $a \in A, b \in B$。

- 幂集 $\mathcal{P}(A)$ 是 $A$ 的所有子集的集合。如果 $|A| = n$，则 $|\mathcal{P}(A)| = 2^n$。对于 $A = \{1, 2\}$：$\mathcal{P}(A) = \{\emptyset, \{1\}, \{2\}, \{1, 2\}\}$。

- 基数测量集合大小。有限集合具有整数基数。无限集合有不同的大小：自然数 $\mathbb{N}$ 和有理数 $\mathbb{Q}$ 是可数无限（可以列出），而实数 $\mathbb{R}$ 是不可数无限（不能列出，由康托尔对角线论证证明）。这种区别在可计算性理论中很重要：有不可数多个函数但只有可数多个程序，所以大多数函数是不可计算的。

## 关系

- 关系 $R$ 在集合 $A$ 上是 $A \times A$ 的子集：一组指定哪些元素相关的有序对。例如，整数上的 $\leq$ 是集合 $\{(a, b) : a \leq b\}$。

- 关系的重要性质：

    - 自反：每个元素与自身相关。对所有 $a$ 有 $a R a$。示例：$\leq$（每个数字都 $\leq$ 自身）。
    - 对称：如果 $a R b$ 则 $b R a$。示例："是兄弟姐妹。"
    - 反对称：如果 $a R b$ 和 $b R a$ 则 $a = b$。示例：$\leq$。
    - 传递：如果 $a R b$ 和 $b R c$ 则 $a R c$。示例：$<$、$\leq$、"是祖先。"

- 等价关系是自反、对称和传递的。它将集合划分为等价类，其中类内的所有元素彼此相关但不与其他类的元素相关。模算术是等价关系：$a \equiv b \pmod{n}$ 将整数划分为 $n$ 个类。编程语言中的类型等价是等价关系。

- 偏序是自反、反对称和传递的。它定义"小于或等于"结构，可能留下一些元素不可比较。文件系统目录形成偏序（父子关系），但兄弟目录不可比较。全序是每对都可比较的偏序（如整数上的 $\leq$）。

- 偏序在并发中至关重要：事件上的"发生在之前"关系是偏序。不被发生在之前排序的事件是并发的，可以以任何相对顺序执行。

## 函数

- 函数 $f: A \to B$ 将 $A$（定义域）的每个元素映射到 $B$（陪域）的恰好一个元素。函数是确定性计算的数学模型：给定输入，恰好有一个输出。

- 单射（一对一）：不同输入总是给出不同输出。$f(a) = f(b) \implies a = b$。无损压缩是单射：不同输入必须压缩为不同输出（否则无法唯一解压缩）。

- 满射（onto）：$B$ 的每个元素都被 $A$ 的某个元素击中。范围等于陪域。将字符串映射到 256 位哈希的哈希函数如果字符串少于可能的哈希则不是满射。

- 双射：既是单射又是满射。$A$ 和 $B$ 之间完美的一一对应。双射有逆。加密必须是双射：每个明文映射到唯一密文，解密函数是逆。

- 复合 $(g \circ f)(x) = g(f(x))$：先应用 $f$，然后 $g$。函数复合是结合的（第 2 章：与矩阵乘法结合相同）。软件中的流水线是函数复合：数据流过变换链。

## 图论基础

- 我们在第 12 章（图神经网络）中广泛讨论了图，包括邻接矩阵、图类型、拉普拉斯和谱理论。这里我们关注与计算机科学相关的算法和结构性质。

- 树是连通无环图。等价地，它有 $n$ 个节点和 $n-1$ 条边。树是文件系统、XML/HTML 文档、决策过程和递归分解的结构。有根树有指定的根节点；每个其他节点恰好有一个父节点。

- 图 $G$ 的生成树是包含 $G$ 所有节点的树，使用其边的子集。最小生成树（MST）最小化总边权重。Kruskal 算法（对边排序，贪心地添加不产生环的最轻边）和 Prim 算法（从起始节点生长树，总是添加最轻的边到新节点）都在 $O(|E| \log |V|)$ 内找到 MST。

- 平面性：如果图可以在平面中绘制且无边交叉，则是平面的。根据欧拉公式，对于连通平面图：$|V| - |E| + |F| = 2$，其中 $|F|$ 是面数（区域，包括外面）。这意味着平面图的 $|E| \leq 3|V| - 6$，所以平面图是稀疏的。电路板布线和地图着色利用平面性。

- 图着色为节点分配颜色，使得没有两个相邻节点共享颜色。所需的最小颜色数是色数 $\chi(G)$。四色定理指出任何平面图的 $\chi(G) \leq 4$。在计算机科学中，图着色模拟寄存器分配（为变量分配 CPU 寄存器，使得同时存活的变量获得不同寄存器）和调度（为任务分配时间槽，使得冲突任务不重叠）。

- 欧拉路径恰好访问每条边一次。当且仅当图恰好有 0 或 2 个奇度节点时存在。哈密顿路径恰好访问每个节点一次。确定是否存在哈密顿路径是 NP 完全的——计算机科学中的经典难题之一。这种对比（欧拉：多项式，哈密顿：NP 完全）说明了听起来相似的问题如何具有 vastly 不同的计算复杂度。

## 递推关系

- 递推关系定义一个序列，其中每一项依赖于前面的项。它们自然地从递归算法中产生。

- 最简单的例子：$T(n) = T(n-1) + 1$，$T(0) = 0$。展开：$T(n) = T(n-1) + 1 = T(n-2) + 2 = \cdots = n$。这是 $O(n)$，简单循环的时间复杂度。

- 归并排序给出 $T(n) = 2T(n/2) + O(n)$：将数组分成两半（两个大小为 $n/2$ 的子问题），递归排序每一半，然后合并（$O(n)$ 工作）。解是 $T(n) = O(n \log n)$。

- 主定理求解形式为 $T(n) = aT(n/b) + O(n^d)$ 的递推：

    - 如果 $d > \log_b a$：$T(n) = O(n^d)$（每层的工作主导）
    - 如果 $d = \log_b a$：$T(n) = O(n^d \log n)$（工作在各层平衡）
    - 如果 $d < \log_b a$：$T(n) = O(n^{\log_b a})$（子问题数主导）

- 对于归并排序：$a = 2, b = 2, d = 1$。因为 $d = \log_2 2 = 1$，我们处于平衡情况：$T(n) = O(n \log n)$。

- 斐波那契递推 $F(n) = F(n-1) + F(n-2)$，$F(0) = 0, F(1) = 1$ 有闭式解 $F(n) = \frac{\phi^n - \psi^n}{\sqrt{5}}$，其中 $\phi = \frac{1+\sqrt{5}}{2}$（黄金比例）和 $\psi = \frac{1-\sqrt{5}}{2}$。这表明斐波那契序列以 $O(\phi^n)$ 指数增长，这就是为什么朴素递归斐波那契是指数级慢的。

- 组合学（排列、组合、二项式定理和容斥）在第 5 章（概率）中涵盖。这些计数技术对于算法分析至关重要（有多少可能的输入？多少次比较？），但我们这里不再重复。

## 可计算性

- 不是所有东西都可以计算。这是所有数学中最深刻的结果之一，它设定了计算机能做什么的根本限制。

- 图灵机是计算的抽象模型：无限单元带（每个单元保存一个符号）、读写头和具有转换规则的有限状态集。尽管简单，图灵机可以计算任何真实计算机可以计算的东西。这是邱奇 - 图灵论题：任何有效可计算的函数都可以由图灵机计算。

- 每种编程语言（Python、C、Haskell）都是图灵完备的：它可以模拟图灵机，因此可以计算任何可计算的东西。语言之间的差异在于便利性、速度和安全性，而不是它们根本上可以计算什么。

- 停机问题询问：给定程序和输入，程序最终会停止，还是会永远运行？图灵证明（1936）没有算法可以一般地解决这个问题。证明是反证法：假设存在停机检测器 $H(P, x)$。构造一个程序 $D$ 运行 $H(D, D)$ 并做与 $H$ 所说相反的事。如果 $H$ 说 $D$ 停机，$D$ 永远循环。如果 $H$ 说 $D$ 循环，$D$ 停机。矛盾。

- 这不是当前技术的限制；这是数学上的不可能。无论多少计算、聪明或 AI 都永远无法一般地解决停机问题。这是计算机科学的哥德尔不完备定理类比。

- 实际后果：你无法编写完美的死锁检测器、完美的病毒扫描器或完美的优化编译器。这些中的每一个都需要一般地解决停机问题（或等价的不可判定问题）。实际工具使用启发式和近似，在常见情况下有效，但不能保证对所有输入的正确性。

- 如果存在算法总是以正确的是/否答案终止，则问题是可判定的。如果不存在这样的算法，则是不可判定的。停机问题是不可判定的。素数测试是可判定的。大多数编程语言中的类型检查是可判定的（通过设计）。

## 复杂性理论

- 即使在可计算问题中，有些问题也比其他问题难得多。复杂性理论根据解决它们所需的资源（时间、空间）随输入增长来分类问题。

![P、NP 和 NP 完全：P 包含在 NP 中，NP 完全位于边界，P=NP 是否成立是核心开放问题](../images/p_np_complexity.svg)

- P（多项式时间）：可以在 $O(n^k)$ 时间内解决的问题，对于某个常数 $k$。排序（$O(n \log n)$）、最短路径（$O(|V|^2)$）、矩阵乘法（$O(n^3)$）。这些被认为是"高效"或"易处理"的。

- NP（非确定性多项式时间）：问题的提议解可以在多项式时间内验证的问题，即使找到解可能需要指数时间。例如，给定声称的哈密顿路径，你可以通过检查每条边在 $O(n)$ 内验证它。但找到一个可能需要尝试指数级多种可能性。

- P 中的每个问题也在 NP 中（如果你能快速解决它，你当然可以快速验证解）。核心问题是 $P = NP$ 是否成立：每个解可以快速验证的问题也可以快速解决吗？这是计算机科学中最重要的开放问题，有 100 万美元的克莱数学千禧年奖。

- 大多数专家相信 $P \neq NP$，意味着有些问题根本上比验证更难解决。如果 $P = NP$，密码学会崩溃（破解加密在 NP 中），优化、调度和药物设计将变得容易。

- NP 完全问题是 NP 中最难的问题。问题是 NP 完全的如果：(1) 它在 NP 中，和 (2) 每个其他 NP 问题可以在多项式时间内归约到它。如果你能有效地解决任何 NP 完全问题，你可以解决所有问题（并且 $P = NP$）。

- 归约将一个问题转换为另一个问题。如果问题 A 归约到问题 B，则 B 至少和 A 一样难。Cook（1971）表明SAT（布尔可满足性：给定逻辑公式，是否存在变量赋值使其为真？）是 NP 完全的。Karp（1972）通过将 SAT 归约到每个问题，表明 21 个其他经典问题是 NP 完全的。

- 著名的 NP 完全问题：
    - 旅行商问题（TSP）：找到访问每个城市恰好一次的最短路线。
    - 图着色：用 $k$ 种颜色为节点着色，使得没有相邻节点共享颜色（$k \geq 3$）。
    - 子集和：给定一组整数，是否存在子集和为目标值？
    - 布尔可满足性（SAT）：是否存在真值赋值满足逻辑公式？
    - 哈密顿路径（上图论中提到）。

- 当你在实践中遇到 NP 完全问题时，你不会为大型输入精确解决它。相反，你使用：近似算法（找到保证在最优解因子内的解）、启发式（贪心、局部搜索、模拟退火）或特殊情况求解器（许多 NP 完全问题对于受限输入很容易）。例如，现代 SAT 求解器例行解决具有数百万变量的实例，尽管最坏情况指数复杂度，通过利用实际实例中的结构。

- NP 难问题至少和 NP 完全一样难，但可能不在 NP 中（它们的解甚至可能无法在多项式时间内验证）。NP 完全问题的优化版本通常是 NP 难的："找到最短 TSP 路线"是 NP 难的，而"是否存在短于 $k$ 的 TSP 路线？"是 NP 完全的。

```python
import itertools

def truth_table(n_vars, expr_fn):
    """为 n_vars 变量的布尔函数生成真值表。"""
    headers = [f"p{i}" for i in range(n_vars)]
    print(" | ".join(headers + ["result"]))
    print("-" * (len(headers) * 4 + 10))
    for vals in itertools.product([False, True], repeat=n_vars):
        result = expr_fn(*vals)
        row = [str(v)[0] for v in vals] + [str(result)[0]]
        print(" | ".join(f"{r:>2}" for r in row))

# 德摩根定律：NOT(p AND q) == (NOT p) OR (NOT q)
print("德摩根定律验证:")
truth_table(2, lambda p, q: (not (p and q)) == ((not p) or (not q)))
```

```python
import jax.numpy as jnp

def merge_sort_ops(n):
    """计算归并排序中的比较数（递推：T(n) = 2T(n/2) + n）。"""
    if n <= 1:
        return 0
    half = n // 2
    return merge_sort_ops(half) + merge_sort_ops(n - half) + n

for n in [8, 64, 512, 4096, 32768]:
    ops = merge_sort_ops(n)
    predicted = n * jnp.log2(n)
    ratio = ops / predicted
    print(f"n={n:5d}  ops={ops:>10d}  n log n={int(predicted):>10d}  ratio={ratio:.3f}")
```
