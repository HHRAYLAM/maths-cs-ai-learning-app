## 😲 惊喜度（自信息）

**惊喜度**度量一个事件有多令人惊讶：

$$I(x) = \log_2 \frac{1}{p(x)} = -\log_2 p(x)$$

单位：**比特**（bits）

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 直观理解

| 事件 | 概率 | 惊喜度 |
|------|------|--------|
| 沙漠里晴天 | 90% | 0.15 比特 |
| 沙漠里下雪 | 1% | 6.64 比特 |
| 公平硬币正面 | 50% | 1 比特 |
| 骰子掷出 6 | 1/6 | 2.58 比特 |

### 为什么用对数？

1. **确定事件**（$p=1$）应该给出零信息：$\log(1) = 0$ ✓
2. **独立事件**应该具有可加信息：$\log(1/p_1 p_2) = \log(1/p_1) + \log(1/p_2)$ ✓
3. 我们想要平滑、表现良好的函数

</div>
  <div class="lesson-visual-column">

```
惊喜度可视化：

概率 p     惊喜度 I(x)
  1.0  →    0 比特  （完全不惊讶）
  0.5  →    1 比特  （有点惊讶）
  0.1  →    3.3 比特（很惊讶）
  0.01 →    6.6 比特（超级惊讶！）
  
越罕见 → 越惊讶 → 信息越多！
```

## 📊 熵 - 平均惊喜度

**熵**是期望惊喜度，从分布中采样的每个事件获得的平均信息量：

$$H(X) = E[I(X)] = -\sum_{x} p(x) \log_2 p(x)$$

度量分布的**不确定性**或**不可预测性**。

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 例子

**公平硬币**（p=0.5）：
$$H = -0.5\log_2(0.5) - 0.5\log_2(0.5) = 1 \text{ 比特}$$

**偏置硬币**（p=0.9）：
$$H = -0.9\log_2(0.9) - 0.1\log_2(0.1) \approx 0.469 \text{ 比特}$$

**确定事件**（p=1）：
$$H = 0 \text{ 比特}$$（完全没有不确定性）

### 关键性质

- 当所有结果等可能时，熵最大
- 对于 n 个等可能结果：$H = \log_2 n$
- 公平骰子：$H = \log_2 6 \approx 2.585$ 比特

</div>
  <div class="lesson-visual-column">

```
熵的可视化：

公平硬币：          偏置硬币：
  H = 1 比特          H = 0.47 比特
  ╱│╲                ╱ │
 ╱ │ ╲              ╱  │
╱──●──╲            ╱───●──
 50/50              90/10
 最大不确定性        较小不确定性

确定性：
  H = 0 比特
  ● （没有不确定性）
```

<div class="visual-caption">图 2：熵描述分布的不确定性</div>

</div>
</div>

### 熵的实际意义：压缩

**香农信源编码定理**：

> 你不能在不丢失信息的情况下将数据压缩到低于其熵率。

<div class="story-card">
  <div class="story-card-title">🐱 Tom 的压缩理解</div>
  <div class="story-card-content">
    <strong>高熵图像</strong>：每个像素等可能，无法压缩
    <br><br>
    <strong>低熵图像</strong>：大部分是白色，压缩效果好
    <br><br>
    JPEG 压缩有效是因为相邻像素<strong>相关</strong>（熵低）！
  </div>
</div>

### 尺度了解

- 灰度像素（256 值）的最大熵 = 8 比特
- 1080p 灰度图像最多 ≈ 1660 万比特
- 真实图像的熵低得多（像素相关）

## 🔗 互信息 - 变量间的信息共享

**互信息**度量知道一个变量告诉你关于另一个变量的多少信息：

$$I(X; Y) = H(X) - H(X|Y)$$

= 观察 Y 时关于 X 的**不确定性减少量**

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 等价公式

$$I(X; Y) = \sum_{x,y} p(x,y) \log_2 \frac{p(x,y)}{p(x) p(y)}$$

### 关键性质

| 关系 | 互信息 |
|------|--------|
| X 和 Y 独立 | $I(X;Y) = 0$ |
| X 和 Y 完全相关 | $I(X;Y) = H(X)$ |
| X 和 Y 部分相关 | $0 < I(X;Y) < H(X)$ |

### ML 应用

- **特征选择**：选择与目标具有高 MI 的特征
- **信息瓶颈**：压缩输入同时保留目标信息
- **聚类评估**：聚类与真实标签的 MI

</div>
  <div class="lesson-visual-column">

```
互信息可视化：

X 的熵：  ┌─────────┐
          │  H(X)   │
          ├────┬────┤
          │I(X;Y)│  ← 互信息（共享）
          ├────┴────┤
          │ H(X|Y)  │  ← 剩余不确定性
          └─────────┘
          
知道 Y 后，X 的不确定性减少了 I(X;Y)！
```

## 📐 交叉熵 - 用错误编码的成本

**交叉熵**度量使用为分布 $q$ 优化的编码来编码来自分布 $p$ 的事件所需的平均比特数：

$$H(p, q) = -\sum_{x} p(x) \log_2 q(x)$$

<div class="story-card jerry">
  <div class="story-card-title">🐭 Jerry 的编码理解</div>
  <div class="story-card-content">
    <strong>真实分布 p</strong>：实际概率
    <br><br>
    <strong>近似分布 q</strong>：我们用的编码
    <br><br>
    如果 $q$ 与 $p$ 完美匹配 → 交叉熵 = 熵
    <br>
    如果 $q$ 是糟糕近似 → 交叉熵更高！
  </div>
</div>

### 与熵的关系

$$H(p, q) = H(p) + D_{\text{KL}}(p \| q)$$

**"额外"比特**来自不匹配！

---

### 交叉熵损失 - 分类的标准损失函数

这正是为什么交叉熵是 ML 中分类的标准损失函数：

$$\mathcal{L} = -\sum_{c} y_c \log \hat{y}_c$$

- $y_c$ = 真实标签（one-hot 分布 p）
- $\hat{y}_c$ = 模型预测概率（分布 q）

对于单个样本：$\mathcal{L} = -\log \hat{y}_c$

**含义**：损失是模型预测下真实类别的**惊喜度**！

<div class="lesson-two-column">
  <div class="lesson-text-column">

### 例子

真实类别是"猫"：

| 模型预测 | 交叉熵损失 |
|---------|-----------|
| 猫 99% | $-\log(0.99) \approx 0.01$ |
| 猫 50% | $-\log(0.50) \approx 0.69$ |
| 猫 1% | $-\log(0.01) \approx 4.60$ |

模型为正确类别分配高概率 → 损失低！

</div>
  <div class="lesson-visual-column">

```
交叉熵损失可视化：

置信度    损失
 1%  →   4.60  ████████
10%  →   2.30  ████
50%  →   0.69  ██
90%  →   0.15  █
99%  →   0.01  

置信度越高 → 损失越低！
```

<div class="visual-caption">图 4：交叉熵损失随预测改善而减少</div>

</div>

## 📏 KL 散度 - 分布间的差异

**KL 散度**（相对熵）度量一个分布与另一个分布的差异：

$$D_{\text{KL}}(p \| q) = \sum_{x} p(x) \log \frac{p(x)}{q(x)} = H(p, q) - H(p)$$

<div class="key-point">
  <div class="key-point-title">🎯 KL 散度的性质</div>
  <ul class="key-point-list">
    <li>总是非负：$D_{\text{KL}} \ge 0$</li>
    <li>仅当 $p = q$ 时等于零</li>
    <li><strong>不对称</strong>：$D_{\text{KL}}(p \| q) \neq D_{\text{KL}}(q \| p)$</li>
    <li>不是真正的"距离"（不满足三角不等式）</li>
  </ul>
</div>

---

### KL 散度的不对称性

这种不对称性很重要！

| 方向 | 行为 | 应用 |
|------|------|------|
| $D_{\text{KL}}(p \| q)$ | **矩匹配**：q 覆盖 p 的所有模式 | 期望传播 |
| $D_{\text{KL}}(q \| p)$ | **模式寻求**：q 集中在 p 的一个模式 | 变分推断 |

```
p 有两个峰：
    ╱│╲      ╱│╲
   ╱ │ ╲    ╱ │ ╲

最小化 D(p||q)：  最小化 D(q||p)：
   ╱───────╲         ╱│╲
  ╱    q    ╲       ╱ │ ╲
 覆盖两个峰         只覆盖一个峰
```

---

### 为什么最小化交叉熵等价于最小化 KL 散度？

$$H(p, q) = H(p) + D_{\text{KL}}(p \| q)$$

由于 $H(p)$ 相对于模型是**常数**：

$$\min H(p, q) \iff \min D_{\text{KL}}(p \| q)$$

这就是为什么我们可以使用交叉熵损失，并知道我们也在最小化真实分布和预测分布之间的 KL 散度！

---

## 🔄 KL 散度在贝叶斯更新中的应用

**贝叶斯更新**：

> 后验 $P(\theta | D)$ 是与先验 $P(\theta)$ 最接近（在 KL 散度意义上）且与观测数据一致的分布。

每个新观察都会更新后验，减少关于 $\theta$ 的不确定性！

---

### VAE 中的应用

**变分自动编码器（VAE）** 的损失函数有两项：

1. **重建损失**（交叉熵）
2. **KL 散度项**：正则化潜在空间保持接近标准正态分布

$$\mathcal{L} = \underbrace{\text{重建损失}}_{\text{交叉熵}} + \underbrace{D_{\text{KL}}(q(z|x) \| p(z))}_{\text{正则化}}$$
