# 图注意力网络

*图注意力网络用学习到的、数据依赖的加权替换均匀的邻居聚合。本章涵盖 GAT、多头图注意力、GATv2、图 Transformer、位置和结构编码以及可扩展性。*

- 在 GCN（文件 3）中，每个节点使用由图结构确定的固定权重（归一化邻接矩阵）来聚合其特征。一个有三个邻居的节点会给每个邻居大致相等的权重（$\approx 1/3$）。但并非所有邻居都同样重要：来自密切合作者的消息应该比来自遥远熟人的消息更重要。

- **图注意力网络**通过学习**关注哪些邻居**来解决这个问题，使用与驱动 Transformer 相同的注意力机制（第 7 章）。每个节点不是使用固定的、基于结构的权重，而是计算动态的、基于内容的邻居注意力分数。

## GAT：图注意力网络

- **GAT**（Veličković 等人，2018）计算每个节点与其邻居之间的注意力系数。对于节点 $i$ 和邻居 $j$：

$$e_{ij} = \text{LeakyReLU}\left(\mathbf{a}^T \left[W\mathbf{h}_i \| W\mathbf{h}_j\right]\right)$$

- 其中 $W \in \mathbb{R}^{d' \times d}$ 是共享线性变换，$\|$ 表示拼接，$\mathbf{a} \in \mathbb{R}^{2d'}$ 是可学习的注意力向量。分数 $e_{ij}$ 衡量节点 $j$ 的特征对节点 $i$ 的重要性。

- 使用 softmax 对所有邻居的原始分数进行归一化：

$$\alpha_{ij} = \text{softmax}_j(e_{ij}) = \frac{\exp(e_{ij})}{\sum_{k \in \mathcal{N}(i)} \exp(e_{ik})}$$

- 这确保注意力权重在每个节点的邻域内总和为 1，就像 Transformer 注意力一样（第 7 章）。节点的更新特征是：

$$\mathbf{h}_i' = \sigma\left(\sum_{j \in \mathcal{N}(i)} \alpha_{ij} W\mathbf{h}_j\right)$$

![GCN 为所有邻居分配固定相等的权重；GAT 学习数据依赖的注意力权重](../images/gat_attention_weights.svg)

- 与 GCN 的关键区别：权重 $\alpha_{ij}$ 是从**数据中学到的**，而不是由图结构固定的。节点可以学习关注信息量最大的邻居，同时忽略噪声或不相关的邻居。

- 注意，注意力仅在边上计算（节点 $i$ 只关注其邻居 $\mathcal{N}(i)$），而不是在所有节点对上计算。这使计算量与边数成正比，而不是节点数的平方。

## 多头图注意力

- 与 Transformer 一样（第 7 章），**多头注意力**并行运行 $K$ 个独立的注意力机制，每个都有自己独立的参数 $W^k$ 和 $\mathbf{a}^k$。结果被拼接（在中间层）或平均（在最后一层）：

$$\mathbf{h}_i' = \Big\|_{k=1}^{K} \sigma\left(\sum_{j \in \mathcal{N}(i)} \alpha_{ij}^k W^k \mathbf{h}_j\right)$$

- 每个头可以关注邻域的不同方面：一个头可能关注结构特征，另一个关注语义相似性。这与 Transformer 中多头注意力的动机相同：不同的头捕捉不同类型的关系。

- 有 $K$ 个头且每个头的输出维度为 $d'$ 时，拼接输出的维度为 $K \times d'$。最后一层通常平均而不是拼接以产生固定大小的输出。

## GATv2：修复静态注意力

- 原始 GAT 有一个微妙的局限性：其注意力函数是**静态的**（也称为基于排名的）。注意力分数取决于拼接 $[W\mathbf{h}_i \| W\mathbf{h}_j]$，但因为注意力向量 $\mathbf{a}$ 在拼接后应用，它可以分解为两个独立分量：$\mathbf{a}^T [W\mathbf{h}_i \| W\mathbf{h}_j] = \mathbf{a}_1^T W\mathbf{h}_i + \mathbf{a}_2^T W\mathbf{h}_j$。

- 这意味着对于给定节点 $i$，邻居的排名完全由邻居的特征 $\mathbf{h}_j$ 决定（项 $\mathbf{a}_1^T W\mathbf{h}_i$ 对 $i$ 的所有邻居都是常数）。注意力排名并不真正依赖于查询节点的特征。节点 $i$ 和节点 $k$ 将对同一组邻居进行相同的排名，这限制了表达能力。

- **GATv2**（Brody 等人，2022）通过在注意力向量之前应用非线性来修复这个问题：

$$e_{ij} = \mathbf{a}^T \text{LeakyReLU}\left(W \left[\mathbf{h}_i \| \mathbf{h}_j\right]\right)$$

- 将 LeakyReLU 移到计算内部意味着注意力分数是联合特征的非线性函数，不可分解为独立项。这使注意力成为**动态的**：邻居的排名现在取决于特定的查询节点。GATv2 严格比 GAT 更具表达能力，且不增加额外计算成本。

## 图 Transformer

- 标准的消息传递 GNN 受到图拓扑的限制：节点只能关注其直接邻居。经过 $k$ 层后，来自 $k$ 跳邻居的信息已经通过多个聚合步骤混合，丢失了保真度。这个局部瓶颈（结合过平滑，文件 3）限制了捕捉长程依赖的能力。

- **图 Transformer** 通过应用**全局自注意力**到所有节点对来打破这个瓶颈，无论它们是否共享边。每个节点可以在一层中关注所有其他节点，就像在标准 Transformer 中一样（第 7 章）。

- 基本思想：将所有节点视为 token 并应用 Transformer 自注意力：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

- 其中 $Q = XW_Q$、$K = XW_K$、$V = XW_V$ 是节点特征 $X$ 的查询、键和值投影（与第 7 章完全相同）。这是在全连通图（完全图 $K_n$，文件 2）上的 GNN。

- 问题：全连通图忽略了实际的图结构。边信息（谁实际连接到谁）丢失了。有两种方法可以恢复这一点：

- **Graphormer**（Ying 等人，2021）通过注意力分数中的**偏置项**将图结构注入 Transformer：

$$A_{ij} = \frac{(\mathbf{h}_i W_Q)(W_K^T \mathbf{h}_j^T)}{\sqrt{d_k}} + b_{\text{spatial}}(i, j) + b_{\text{edge}}(i, j)$$

- 空间偏置 $b_{\text{spatial}}$ 编码节点 $i$ 和 $j$ 之间的最短路径距离。边偏置 $b_{\text{edge}}$ 编码最短路径上的边特征。此外，Graphormer 使用**中心性编码**，将节点的度添加到其输入嵌入中，为模型提供关于每个节点结构角色的信息。

- **GPS**（通用、强大、可扩展图 Transformer，Rampášek 等人，2022）在每一层结合局部消息传递和全局注意力：

$$\mathbf{h}_i' = \text{MLP}\left(\mathbf{h}_i^{\text{MPNN}} + \mathbf{h}_i^{\text{Attention}}\right)$$

- 每一层同时应用标准 GNN（用于局部结构）和 Transformer（用于全局上下文），然后组合结果。这结合了两者的优点：来自消息传递的局部结构和来自注意力的长程依赖。

## 位置和结构编码

- Transformer 在序列上使用位置编码（第 7 章）来注入顺序信息。图没有规范排序，因此需要特定于图的编码。

- **拉普拉斯特征向量编码**使用图拉普拉斯（文件 2）的特征向量作为位置特征。$k$ 个最小的非平凡特征向量提供图的谱嵌入：在图中"邻近"的节点具有相似的特征向量值。这些被拼接到节点特征上。

- 一个微妙之处：拉普拉斯特征向量具有符号模糊性（如果 $\mathbf{u}$ 是特征向量，那么 $-\mathbf{u}$ 也是）。模型必须对这些符号翻转保持不变。解决方案包括在训练期间使用随机符号翻转作为数据增强，或学习符号不变变换。

- **随机游走编码**计算从节点 $i$ 开始的随机游走在 $k$ 步后返回节点 $i$ 的概率，对于 $k = 1, 2, \ldots, K$。这些概率编码局部结构信息：密集簇中的节点具有高返回概率，而稀疏区域中的节点具有低返回概率。着陆概率 $p_{ii}^{(k)} = (A_{\text{rw}}^k)_{ii}$，其中 $A_{\text{rw}} = D^{-1}A$ 是随机游走转移矩阵。

- **度编码**简单地将节点度作为特征添加。这效果惊人地好，因为度是一个强烈的结构信号：叶节点（度为 1）、桥接节点和枢纽节点表现不同。

- 这些编码提供了普通 Transformer 缺乏的结构信息，使图 Transformer 在需要长程推理的任务上优于标准消息传递 GNN。

## 可扩展性

- GNN 的基本可扩展性挑战是图可以有数百万个节点和数十亿条边。在完整图上训练 GNN 需要存储所有节点特征和整个邻接矩阵在内存中，这通常不可行。

- GNN 的**小批量训练**比图像或序列更复杂，因为节点是相互连接的。天真地采样一批节点需要它们的邻居（第 1 层）、它们邻居的邻居（第 2 层），依此类推。这种**邻居爆炸**意味着 1000 个目标节点的一批可能在计算图中需要数百万个节点。

- **邻居采样**（GraphSAGE 风格，文件 3）通过每层每个节点采样固定数量的邻居来限制爆炸。对于 2 层且每层 15 个样本，每个目标节点的子图最多有 $15^2 = 225$ 个节点，无论完整图的大小如何。

- **Cluster-GCN**（Chiang 等人，2019）使用图聚类算法（如 METIS）将图划分为簇，然后一次训练一个簇。簇内边是密集的（大多数邻居在同一个簇中），因此子图捕捉了相关结构。跨簇边通过偶尔包含簇之间的边来处理。

- **图 Transformer 可扩展性**更难，因为全局注意力是 $O(n^2)$。对于有数百万节点的图，完全注意力不可行。解决方案包括：
    - 稀疏注意力模式（只关注图中 $k$ 个最近的节点）
    - 线性注意力近似
    - 结合局部消息传递（便宜，$O(|E|)$）与粗化图上的全局注意力（更少的节点）

## 时序和动态图

- 到目前为止，我们研究的图是**静态的**：节点、边和特征是固定的。但许多现实世界的图**随时间演变**：新用户加入社交网络、金融交易创建边、交通模式在一天中变化、分子相互作用波动。

- **时序图**为每条边添加时间戳：$(i, j, t)$ 表示节点 $i$ 在时间 $t$ 与节点 $j$ 交互。挑战是学习同时捕捉图结构和时序动态的表示。

- 有两种范式：

- **离散时间动态图（DTDG）**：图表示为快照序列 $G_1, G_2, \ldots, G_T$，每个时间步一个。GNN 处理每个快照，RNN 或时序注意力机制捕捉跨快照的演变。这很简单，但丢失了细粒度的时间信息（快照之间的事件丢失），并且需要选择快照频率。

- **连续时间动态图（CTDG）**：事件被建模为带时间戳的交互流。每个事件 $(i, j, t)$ 在发生时精确更新节点 $i$ 和 $j$ 的表示。这保留了所有时间信息。

- **时序图网络（TGN）**（Rossi 等人，2020）是领先的 CTDG 架构。每个节点维护一个**记忆状态**$\mathbf{s}_i(t)$，每当节点参与交互时更新：

$$\mathbf{s}_i(t^+) = \text{GRU}\left(\mathbf{s}_i(t^-), \; \mathbf{m}_i(t)\right)$$

- 其中 $\mathbf{m}_i(t)$ 是从交互计算的消息（组合两个节点的特征、边特征和时间编码）。GRU（第 6 章）选择性地保留和忘记过去的信息，使记忆能够捕捉长期模式，同时适应最近的事件。

- **时间编码**将自上次交互以来经过的时间表示为特征向量，类似于 Transformer 中的位置编码（第 7 章）。常用方法使用可学习的傅里叶特征：

$$\Phi(t) = \left[\cos(\omega_1 t), \sin(\omega_1 t), \ldots, \cos(\omega_d t), \sin(\omega_d t)\right]$$

- 这为模型提供了丰富的时间间隔表示："该用户上次活跃在 5 分钟前"与"3 个月前"被嵌入为不同的表示。

- **时序图注意力（TGAT）** 对节点的时间邻域应用自注意力：最近交互集，每个交互都按特征相关性（如 GAT）和时间近端加权。来自遥远过去的交互自然被降权。

- 应用包括欺诈检测（金融图中的异常交易模式）、交通预测（根据历史流量模式预测拥堵）、社交网络动态（预测病毒式内容传播）以及随时间推移的药物相互作用预测。

## 编码任务（使用 CoLab 或 notebook）

1. 从头实现单个 GAT 注意力头。计算节点与其邻居之间的注意力权重，并验证它们的和为 1。
```python
import jax
import jax.numpy as jnp

rng = jax.random.PRNGKey(0)
k1, k2, k3 = jax.random.split(rng, 3)

n_nodes, d_in, d_out = 5, 4, 3

# Random node features
H = jax.random.normal(k1, (n_nodes, d_in))

# Learnable parameters
W = jax.random.normal(k2, (d_in, d_out)) * 0.5
a = jax.random.normal(k3, (2 * d_out,)) * 0.5

# Adjacency (node 0 connects to 1, 2, 3)
neighbours_of_0 = [1, 2, 3]

# Transform features
Wh = H @ W  # (n_nodes, d_out)

# Compute attention scores for node 0
h_i = Wh[0]
scores = []
for j in neighbours_of_0:
    h_j = Wh[j]
    e_ij = jnp.dot(a, jnp.concatenate([h_i, h_j]))
    e_ij = jax.nn.leaky_relu(e_ij, negative_slope=0.2)
    scores.append(float(e_ij))

scores = jnp.array(scores)
alpha = jax.nn.softmax(scores)

print(f"Raw scores: {scores}")
print(f"Attention weights: {alpha}")
print(f"Sum of weights: {alpha.sum():.4f}")

# Weighted aggregation
h_new = sum(alpha[k] * Wh[neighbours_of_0[k]] for k in range(len(neighbours_of_0)))
print(f"Updated node 0 features: {h_new}")
```

2. 比较 GCN（固定权重）与 GAT（学习权重）聚合。展示 GAT 可以为邻居分配不同权重，而 GCN 均匀对待它们。
```python
import jax
import jax.numpy as jnp

# 4 nodes: node 0 connects to 1, 2, 3
A = jnp.array([[0,1,1,1],
               [1,0,0,0],
               [1,0,0,0],
               [1,0,0,0]], dtype=float)

# Features: node 1 is very relevant, node 2 is noise, node 3 is moderate
H = jnp.array([[0.0, 0.0],   # node 0
               [1.0, 0.0],   # node 1 (signal)
               [0.0, 0.0],   # node 2 (noise)
               [0.5, 0.0]])  # node 3 (moderate)

# GCN: normalised adjacency weights
A_hat = A + jnp.eye(4)
D_inv = jnp.diag(1.0 / A_hat.sum(axis=1))
gcn_weights = (D_inv @ A_hat)[0]  # weights for node 0
print(f"GCN weights for node 0: {gcn_weights}")
print("  → All neighbours get roughly equal weight")

# GAT: learned attention (simulated)
# Suppose the attention mechanism learns to focus on node 1
gat_weights = jnp.array([0.1, 0.7, 0.05, 0.15])  # learned
print(f"\nGAT weights for node 0: {gat_weights}")
print("  → Node 1 (informative) gets most attention")

gcn_output = gcn_weights @ H
gat_output = gat_weights @ H
print(f"\nGCN output: {gcn_output}  (diluted by noise)")
print(f"GAT output: {gat_output}  (focused on signal)")
```

3. 演示位置编码的好处。计算图的拉普拉斯特征向量编码，并展示结构相似的节点获得相似的编码。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Barbell graph: two cliques connected by a bridge
n = 10
A = jnp.zeros((n, n))
# Clique 1: nodes 0-4
for i in range(5):
    for j in range(i+1, 5):
        A = A.at[i,j].set(1).at[j,i].set(1)
# Clique 2: nodes 5-9
for i in range(5, 10):
    for j in range(i+1, 10):
        A = A.at[i,j].set(1).at[j,i].set(1)
# Bridge
A = A.at[4,5].set(1).at[5,4].set(1)

D = jnp.diag(A.sum(axis=1))
L = D - A
eigenvalues, eigenvectors = jnp.linalg.eigh(L)

# Use first 3 non-trivial eigenvectors as positional encoding
pe = eigenvectors[:, 1:4]

print("Laplacian Positional Encodings:")
for i in range(n):
    group = "Clique 1" if i < 5 else "Clique 2"
    bridge = " (bridge)" if i in [4, 5] else ""
    print(f"  Node {i} ({group}{bridge}): {pe[i]}")

plt.scatter(pe[:5, 0], pe[:5, 1], c="#3498db", s=80, label="Clique 1")
plt.scatter(pe[5:, 0], pe[5:, 1], c="#e74c3c", s=80, label="Clique 2")
plt.scatter(pe[[4,5], 0], pe[[4,5], 1], c="black", s=120, marker="*",
            label="Bridge nodes", zorder=5)
plt.legend(); plt.grid(True)
plt.title("Laplacian Eigenvector Positional Encodings")
plt.xlabel("Eigenvector 1"); plt.ylabel("Eigenvector 2")
plt.show()
```
